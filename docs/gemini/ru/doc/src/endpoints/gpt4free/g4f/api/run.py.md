# Модуль для запуска API g4f
## Обзор

Этот модуль содержит точку входа для запуска API g4f (gpt4free). Он импортирует модуль `g4f.api` и запускает API с возможностью отладки.

## Подробней

Данный модуль предназначен для упрощения запуска API g4f, предоставляя удобный способ включить режим отладки при необходимости. Это позволяет разработчикам более эффективно тестировать и отлаживать API.

## Функции

### `run_api`

```python
def run_api(debug: bool = False) -> None:
    """
    Запускает API g4f.

    Args:
        debug (bool, optional): Включает режим отладки, если `True`. По умолчанию `False`.

    Returns:
        None

    """
```

**Назначение**:
Функция `run_api` запускает API g4f. Если передан параметр `debug=True`, то API запускается в режиме отладки.

**Параметры**:
- `debug` (bool, optional): Флаг, указывающий, следует ли запускать API в режиме отладки. По умолчанию `False`.

**Возвращает**:
- `None`: Функция ничего не возвращает.

**Как работает функция**:
1. Функция вызывает `g4f.api.run_api()`, передавая значение параметра `debug`.

**Примеры**:
```python
import g4f.api

# Запуск API без режима отладки
g4f.api.run_api()

# Запуск API в режиме отладки
g4f.api.run_api(debug=True)
```

## Запуск модуля как скрипта (`if __name__ == "__main__":`)

```python
if __name__ == "__main__":
    g4f.api.run_api(debug=True)
```

**Назначение**:
Данный блок кода гарантирует, что функция `g4f.api.run_api(debug=True)` будет вызвана только при непосредственном запуске скрипта, а не при его импорте в другой модуль.

**Как работает функция**:
1. Проверяется, является ли текущий модуль главным исполняемым файлом (`__name__ == "__main__"`).
2. Если условие выполняется, вызывается функция `g4f.api.run_api(debug=True)`, которая запускает API g4f в режиме отладки.
