# Модуль endpoints/advertisement/facebook

## Обзор

Данный модуль содержит функции для работы с рекламой на Facebook.  Он предоставляет инструменты для взаимодействия с API Facebook, обработки данных и создания рекламных кампаний.


## Функции

### `create_facebook_ad`

**Описание**: Создаёт новую рекламную кампанию на Facebook.

**Параметры**:
- `campaign_data` (dict): Словарь с данными для создания кампании.  Требует ключи `name`, `objective`, `budget`, `target_audience` (и др.).  Полный список ключей и ожидаемые типы данных см. в документации Facebook API.
- `access_token` (str): Токен доступа к аккаунту Facebook.

**Возвращает**:
- `dict | None`: Словарь с результатами создания кампании, или `None`, если возникла ошибка.  Ключи могут включать `ad_id`, `status`, `errors`.

**Вызывает исключения**:
- `FacebookAPIError`: Возникает при ошибках, возвращаемых API Facebook.
- `ValueError`: Возникает при некорректных данных `campaign_data`.
- `TypeError`: Возникает при неверном типе данных для параметров.


### `get_ad_performance`

**Описание**: Получает данные о производительности рекламной кампании.

**Параметры**:
- `ad_id` (int): Идентификатор рекламной кампании.
- `start_date` (str, optional): Дата начала периода для получения данных.  Формат: 'YYYY-MM-DD'. По умолчанию `None` (возвращаются данные за весь период).
- `end_date` (str, optional): Дата окончания периода для получения данных.  Формат: 'YYYY-MM-DD'. По умолчанию `None` (возвращаются данные за весь период).
- `access_token` (str): Токен доступа к аккаунту Facebook.

**Возвращает**:
- `dict | None`: Словарь с данными о производительности, или `None`, если возникла ошибка.  Словарь будет содержать данные о кликах, показах, конверсиях и др., структурированные в соответствии с API Facebook.

**Вызывает исключения**:
- `FacebookAPIError`: Возникает при ошибках, возвращаемых API Facebook.
- `ValueError`: Возникает при некорректных данных `ad_id` или датах.
- `TypeError`: Возникает при неверном типе данных для параметров.
