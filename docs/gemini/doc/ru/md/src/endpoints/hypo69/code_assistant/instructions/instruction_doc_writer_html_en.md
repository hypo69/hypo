# Инструкция по генерации документации для Python-файлов

## Обзор

Данная инструкция описывает процесс создания документации для Python-файлов в формате Markdown.  Документация должна быть структурированной, содержать оглавление, описание функций, классов и методов, а также подробные пояснения параметров, возвращаемых значений и возможных исключений.

## Формат документации

### Структура файла

Каждый файл документации должен начинаться с заголовка и краткого описания содержимого.  Использовать стандарт Markdown.  Для функций и методов должны быть приведены подробные комментарии в формате docstrings.

### Комментарии к функциям и методам

Комментарии к функциям и методам должны быть в формате:

```python
def функция(параметр: тип, параметр2: тип = значение_по_умолчанию) -> тип_возвращаемого_значения:
    """
    Args:
        параметр (тип): Описание параметра.
        параметр2 (тип, optional): Описание параметра. По умолчанию значение_по_умолчанию.

    Returns:
        тип_возвращаемого_значения: Описание возвращаемого значения.

    Raises:
        Исключение: Описание ситуации, в которой возникает исключение.
    """
    # Тело функции
```

Замените `функция`, `параметр`, `тип`, `значение_по_умолчанию`, `тип_возвращаемого_значения` и `Исключение` на соответствующие значения из Python кода.

### Обработка исключений

Вместо `e` используйте `ex`.

### Оглавление (TOC)

В начале каждого файла документации должен быть раздел оглавления (TOC), содержащий ссылки на все основные разделы (Классы, Функции и т.д.).

### Структура документации

Должны быть разделы для описания классов, функций, методов с  подробными описаниями, параметрами, возвращаемыми значениями и вызываемыми исключениями.

```markdown
## Функции

### `function_name`

**Описание**: Краткое описание функции.

**Параметры**:
- `param` (str): Описание параметра `param`.
- `param1` (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию `None`.

**Возвращает**:
- `dict | None`: Описание возвращаемого значения. Возвращает словарь или `None`.

**Вызывает исключения**:
- `SomeError`: Описание ситуации, в которой возникает исключение `SomeError`.
```

## Пример файла (Markdown)

```markdown
# Модуль `my_module`

## Обзор

Краткое описание функциональности модуля.

## Классы

### `MyClass`

**Описание**: Описание класса `MyClass`.

**Методы**:
- `method_1`:  Описание метода `method_1`.
- `method_2`: Описание метода `method_2`.

## Функции

### `my_function`

**Описание**: Описание функции `my_function`.

**Параметры**:
- `arg1` (int): Описание параметра `arg1`.
- `arg2` (str, optional): Описание параметра `arg2`. По умолчанию "default".

**Возвращает**:
- `bool`: Возвращает `True` или `False`.

**Вызывает исключения**:
- `ValueError`: Если входные данные некорректны.
```

**Важно**:  Эта инструкция предназначена для генерации документации в формате Markdown.  Для HTML-формата требуется дополнительная инструкция, учитывающая особенности HTML-разметки.
