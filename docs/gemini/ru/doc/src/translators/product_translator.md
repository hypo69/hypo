# Модуль `hypotez/src/translators/product_translator.py`

## Обзор

Этот модуль предоставляет функции для управления переводами полей товаров. Он служит связующим звеном между словарем полей товара, таблицей переводов и переводчиками, позволяя получать, сохранять и переводить информацию о товарах на различные языки.

## Функции

### `get_translations_from_presta_translations_table`

**Описание**: Функция извлекает данные о переводах полей товара из таблицы переводов PrestaShop.

**Параметры**:
- `product_reference` (str): Уникальный идентификатор товара.
- `i18n` (str, опционально): Код языка перевода (например, `en_EN`, `ru-RU`). По умолчанию `None`.

**Возвращает**:
- `list`: Список словарей, содержащих переводы полей товара. Возвращает пустой список, если нет данных для указанного товара.

**Вызывает исключения**:
- Возможны исключения, связанные с работой с базой данных (например, `sqlite3.Error`).  Обработка исключений не показана, но должна быть добавлена.


### `insert_new_translation_to_presta_translations_table`

**Описание**: Функция добавляет новую запись перевода в таблицу переводов.

**Параметры**:
- `record` (dict): Словарь с данными для новой записи перевода.

**Возвращает**:
- `None`: Функция ничего не возвращает.

**Вызывает исключения**:
- Возможны исключения, связанные с работой с базой данных (например, `sqlite3.Error`).  Обработка исключений не показана, но должна быть добавлена.


### `translate_record`

**Описание**: Функция переводит поля товара с одного языка на другой с помощью внешней функции `translate`.

**Параметры**:
- `record` (dict): Словарь с данными для перевода.
- `from_locale` (str): Код исходного языка (например, `ru_RU`).
- `to_locale` (str): Код целевого языка (например, `en_US`).

**Возвращает**:
- `dict`: Словарь с переведенными данными.

**Вызывает исключения**:
- Возможны исключения, связанные с работой внешней функции `translate`.  Обработка исключений не показана, но должна быть добавлена.
