# Модуль tinytroupe.tools

## Обзор

Этот модуль предоставляет инструменты для агентов, позволяющие им выполнять специализированные задачи. Он содержит базовый класс `TinyTool` и два конкретных инструмента: `TinyCalendar` и `TinyWordProcessor`. Класс `TinyTool` определяет общие методы для инструментов, включая обработку действий, проверку реальных последствий, и контроль прав доступа. Конкретные инструменты наследуют от `TinyTool` и переопределяют необходимые методы для выполнения своих специфических задач.

## Классы

### `TinyTool`

**Описание**: Базовый класс для всех инструментов. Он определяет общие атрибуты и методы, необходимые для функционирования инструментов.

**Атрибуты**:

- `name` (str): Имя инструмента.
- `description` (str): Краткое описание инструмента.
- `owner` (str, необязательно): Агент, которому принадлежит инструмент. Если `None`, инструмент может быть использован любым агентом.
- `real_world_side_effects` (bool): Флаг, указывающий, имеет ли инструмент реальные последствия в мире. `True`, если инструмент может влиять на внешний мир; `False` в противном случае.
- `exporter` (ArtifactExporter): Экспортер, который может использоваться для экспорта результатов действий инструмента.
- `enricher` (Enricher): Инструмент обогащения, который может использоваться для обогащения результатов действий инструмента.


**Методы**:

- `__init__(self, name, description, owner=None, real_world_side_effects=False, exporter=None, enricher=None)`: Инициализирует новый инструмент.
- `_process_action(self, agent, action: dict) -> bool`: Обрабатывает действие, связанное с инструментом.  Метод должен быть переопределен в производных классах.
- `_protect_real_world(self)`: Выводит предупреждение, если инструмент имеет реальные последствия в мире.
- `_enforce_ownership(self, agent)`: Проверяет, принадлежит ли инструмент агенту.
- `set_owner(self, owner)`: Устанавливает владельца инструмента.
- `actions_definitions_prompt(self) -> str`: Возвращает запрос для определения действий инструмента. Должен быть переопределен в производных классах.
- `actions_constraints_prompt(self) -> str`: Возвращает запрос для определения ограничений действий инструмента. Должен быть переопределен в производных классах.
- `process_action(self, agent, action: dict) -> bool`:  Обрабатывает действие, включая проверку реальных последствий и проверки владения.


### `TinyCalendar`

**Описание**: Инструмент календаря, позволяющий агентам отслеживать встречи и назначения.

**Атрибуты**:
- `calenar` (dict): Словарь, хранящий события по датам.


**Методы**:

- `__init__(self, owner=None)`: Инициализирует новый инструмент календаря.
- `add_event(self, date, title, description=None, owner=None, mandatory_attendees=None, optional_attendees=None, start_time=None, end_time=None)`: Добавляет событие в календарь.
- `find_events(self, year, month, day, hour=None, minute=None)`: Находит события по заданным параметрам.
- `_process_action(self, agent, action) -> bool`: Обрабатывает действие, связанное с созданием события.
- `actions_definitions_prompt(self) -> str`: Задает доступные типы действий для календаря.
- `actions_constraints_prompt(self) -> str`: Возвращает запрос с ограничениями.


### `TinyWordProcessor`

**Описание**: Инструмент текстового процессора, позволяющий агентам создавать документы.

**Методы**:

- `__init__(self, owner=None, exporter=None, enricher=None)`: Инициализирует новый инструмент текстового процессора.
- `write_document(self, title, content, author=None)`: Создает и экспортирует документ.
- `_process_action(self, agent, action) -> bool`: Обрабатывает действие, связанное с созданием документа, включая валидацию и обработку исключений JSON.
- `actions_definitions_prompt(self) -> str`: Задает доступные типы действий для процессора.
- `actions_constraints_prompt(self) -> str`: Возвращает запрос с ограничениями.
