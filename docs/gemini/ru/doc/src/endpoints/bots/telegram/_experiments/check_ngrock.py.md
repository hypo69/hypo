# Модуль для проверки ngrok

## Обзор

Этот модуль предназначен для отправки POST-запросов к API, расположенному по адресу `127.0.0.1:8443`. Он использует библиотеку `requests` для выполнения HTTP-запросов. В модуле задаются URL, заголовки и данные, необходимые для отправки запроса, а также обрабатывается ответ от сервера.

## Подробней

Модуль используется для тестирования или взаимодействия с локальным сервером, доступным через ngrok. Он отправляет данные в формате JSON и проверяет статус ответа сервера.

## Функции

### Отправка POST-запроса

**Назначение**: Отправляет POST-запрос к указанному URL с заданными заголовками и данными.

**Как работает функция**:

1.  **Инициализация параметров**:
    *   `url` - URL API (`127.0.0.1:8443`).
    *   `headers` - Заголовки запроса, включающие токен авторизации и тип контента (`application/json`).
    *   `data` - Данные для отправки в формате JSON.

2.  **Выполнение POST-запроса**:
    *   Используется `requests.post` для отправки запроса.

3.  **Обработка ответа**:
    *   Проверяется `response.status_code`.
    *   Если код 200, выводится сообщение об успехе и JSON-ответ.
    *   Если код отличается от 200, выводится сообщение об ошибке, код статуса и текст ответа.

```python
import requests

# URL API
url = "127.0.0.1:8443"

# Заголовки
headers = {
    "Authorization": "Bearer YOUR_API_TOKEN",
    "Content-Type": "application/json"
}

# Данные для отправки
data = {
    "key1": "value1",
    "key2": "value2"
}

# Отправка POST-запроса
response = requests.post(url, headers=headers, json=data)

# Обработка ответа
if response.status_code == 200:
    print("Успешно:", response.json())
else:
    print("Ошибка:", response.status_code, response.text)
```

**Переменные**:

*   `url` (str): URL API для отправки запроса.
*   `headers` (dict): Заголовки запроса, содержащие авторизационный токен и тип контента.
*   `data` (dict): Данные, отправляемые в теле запроса в формате JSON.
*   `response` (requests.Response): Объект ответа, полученный от сервера.

**Примеры**:

Пример успешного ответа:

```
Успешно: {'message': 'Request processed successfully'}
```

Пример ответа с ошибкой:

```
Ошибка: 400 Bad Request
```
