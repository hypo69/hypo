# Модуль `hypotez/src/suppliers/aliexpress/header.py`

## Обзор

Этот модуль содержит функцию `set_project_root`, которая находит корневую директорию проекта, начиная с текущего файла.  Модуль также инициализирует переменную `settings` с настройками из файла `settings.json` в корне проекта.

## Функции

### `set_project_root`

**Описание**:  Находит корневую директорию проекта, начиная с текущего файла. Ищет вверх по дереву директорий, пока не найдёт директорию, содержащую один из файлов из набора маркеров.

**Параметры**:

- `marker_files` (tuple): Кортеж из строк, представляющих имена файлов или директорий, которые используются для определения корневой директории проекта. По умолчанию содержит `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:

- `Path`: Путь к корневой директории проекта, если она найдена. В противном случае возвращает директорию, в которой находится скрипт.

**Вызывает исключения**:

- Нет


### `__init__`

**Описание**: Этот метод не определен в данном модуле и не является частью API.


## Переменные

### `MODE`

**Описание**:  Строковая переменная, хранящая текущий режим работы (например, 'dev', 'prod'). По умолчанию установлена в 'dev'.


### `settings`

**Описание**: Словарь, содержащий настройки проекта. Инициализируется из файла `settings.json` в корне проекта.


## Использование

Для использования модуля, необходимо импортировать функцию `set_project_root` и вызвать ее, передавая в качестве аргумента кортеж файлов или папок, используемых для определения корневой директории проекта.


```python
from hypotez.src.suppliers.aliexpress.header import set_project_root

root_dir = set_project_root()
print(f"Root directory: {root_dir}")
```

## Обработка исключений

В коде используется блок `try...except` для обработки потенциальных ошибок при чтении и разборе файла `settings.json`.

- `FileNotFoundError`: Возникает, если файл `settings.json` не найден.
- `json.JSONDecodeError`: Возникает, если файл `settings.json` содержит некорректный JSON.
- В блоке `except` используется `...`, что указывает на то, что ошибки не обрабатываются, и программа продолжает работу, игнорируя проблему.  **Важно:**  В реальном проекте нужно корректно обработать эти исключения, например, вывести сообщение об ошибке или установить значение по умолчанию для `settings`.


```python
import json
try:
  data = json.loads('{"key": "value"}')
  print(data)
except json.JSONDecodeError as ex:
    print(f"Error decoding JSON: {ex}")
```
