# Модуль `main.first_version.py`

## Обзор

Этот модуль содержит код FastAPI приложения, предназначенного для обработки данных, переданных через HTML-форму.  Он использует внешнюю скрипт `script.py`, которая обрабатывает полученные данные.

## Классы

### Нет определенных классов.

## Функции

### `process_data`

**Описание**: Обрабатывает данные, переданные через POST-запрос на `/process_data`. Данные передаются в виде значений `first_name` и `last_name` в HTML-форме.  Функция выполняет внешнюю скрипт `script.py` с этими данными в качестве входных данных и возвращает результат выполнения скрипта.

**Параметры**:
- `request` (Request): Объект запроса.
- `first_name` (str): Имя.
- `last_name` (str): Фамилия.

**Возвращает**:
- `dict`: Словарь, содержащий поле `output` с результатом выполнения скрипта `script.py` или ошибку в случае возникновения проблемы.

**Вызывает исключения**:
- `HTTPException(status_code=400)`: Возникает, если не предоставлены `first_name` и/или `last_name`.
- `HTTPException(status_code=500)`: Возникает, если во время выполнения скрипта `script.py` произошла ошибка.

### `open_index`

**Описание**: Обрабатывает GET-запросы на корневой путь (`/`).  Перенаправляет запрос на `index.html` в браузере.

**Параметры**:
- Нет

**Возвращает**:
- `dict`: Словарь с сообщением о перенаправлении на `index.html`.

**Вызывает исключения**:
- Нет


##  Статические файлы

### Статические файлы

Модуль использует директорию `html` для статических файлов.

##  Взаимодействие с внешними скриптами

Этот модуль взаимодействует с внешним скриптом `script.py`.  Обработка данных выполняется в рамках `Popen`-объекта, что позволяет выполнить `script.py` асинхронно.

##  Запуск приложения

Для запуска приложения используйте следующую команду в терминале:

```bash
uvicorn main:app --reload
```


##  Примечания

- Путь к скрипту `script.py` определяется относительно пути к файлу `main.first_version.py`.
- Файл `index.html` должен быть расположен в директории `html` для корректной работы.
-  Обратите внимание, что `webbrowser.open` может не работать во всех средах (например, в терминале).
-  Используйте инструменты отладки (например, `print`) для диагностики проблем.
