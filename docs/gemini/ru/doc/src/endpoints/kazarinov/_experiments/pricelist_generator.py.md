# Модуль для экспериментов с созданием PDF отчета
## Обзор

Модуль предназначен для экспериментов с созданием PDF-отчетов. Включает в себя функциональность для генерации отчетов на основе данных в формате JSON и сохранения их в виде HTML и PDF файлов. В модуле используется класс `ReportGenerator` для создания отчетов.

## Подробней

Этот код является частью экспериментов по созданию отчетов в формате PDF. Он считывает данные из JSON-файла, генерирует HTML-файл на основе этих данных и, наконец, создает PDF-файл из HTML.

## Классы

### `ReportGenerator`

**Описание**: Класс для генерации отчетов. Подробности реализации класса содержатся в модуле `src.endpoints.kazarinov.react`.

   **Принцип работы**:
        Класс `ReportGenerator` используется для создания отчетов на основе предоставленных данных. Он принимает данные, HTML-файл и путь для сохранения PDF-файла в качестве аргументов и генерирует отчет.
        Более детальная работа класса `ReportGenerator` смотри в файле `src/endpoints/kazarinov/react.py`.

**Методы**:

   - `create_report(data, html_file, pdf_file)`: Метод для создания отчета.
   **Параметры**:
   - `data` (dict): Данные для отчета.
   - `html_file` (Path): Путь к HTML-файлу.
   - `pdf_file` (Path): Путь для сохранения PDF-файла.

## Переменные

### `base_path`

**Описание**: Путь к базовой директории, содержащей данные и файлы отчетов.
### `data`

**Описание**: Словарь, содержащий данные, загруженные из JSON-файла.
### `html_file`

**Описание**: Путь к HTML-файлу, который будет сгенерирован.
### `pdf_file`

**Описание**: Путь для сохранения сгенерированного PDF-файла.
### `r`

**Описание**: Экземпляр класса `ReportGenerator`, используемый для создания отчета.

## Как работает код

1. **Определение путей**:
   - Определяются пути к базовой директории, файлу данных JSON, HTML-файлу и PDF-файлу.

2. **Загрузка данных**:
   - Используется функция `j_loads` для загрузки данных из JSON-файла в словарь `data`.
   - `j_loads` автоматически открывает файл, читает его содержимое и преобразует JSON в структуру данных Python.

3. **Создание экземпляра ReportGenerator**:
   - Создается экземпляр класса `ReportGenerator`.

4. **Создание отчета**:
   - Вызывается метод `create_report` экземпляра `ReportGenerator` с передачей данных, пути к HTML-файлу и пути для сохранения PDF-файла.

## Примеры

### Пример создания отчета

```python
from pathlib import Path
from src.endpoints.kazarinov.react import ReportGenerator
from src.utils.jjson import j_loads

base_path = Path('/path/to/data')
data = j_loads(base_path / 'data.json')
html_file = base_path / 'report.html'
pdf_file = base_path / 'report.pdf'

r = ReportGenerator()
r.create_report(data, html_file, pdf_file)
