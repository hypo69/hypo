# Модуль header

## Обзор

Этот модуль определяет корневой путь к проекту и загружает настройки из файла `settings.json`. Он обрабатывает потенциальные ошибки при чтении файла настроек и файла README.

## Функции

### `get_project_root`

**Описание**: Находит корневой каталог проекта, начиная с текущего файла и идя вверх по дереву каталогов, пока не найдёт директорию содержащую один из файлов, указанных в параметре `marker_files`. Если корневой каталог не найден, возвращает каталог, где расположен текущий скрипт. Если корневой каталог не найден, добавляет его в путь поиска модулей Python.

**Параметры**:
- `marker_files` (tuple): Кортеж имен файлов или каталогов, используемых для определения корневого каталога проекта.

**Возвращает**:
- `Path`: Путь к корневому каталогу проекта, если он найден, иначе директорию, в которой находится скрипт.

**Вызывает исключения**:
- Нет.


## Переменные

### `__root__`

**Описание**: Путь к корневому каталогу проекта.

**Тип**: `Path`

### `settings`

**Описание**: Словарь настроек проекта, загруженный из файла `settings.json`.

**Тип**: `dict` или `None`


### `doc_str`

**Описание**: Строка с содержимым файла README, загруженного из файла `README.MD`.

**Тип**: `str` или `None`


### `__project_name__`

**Описание**: Название проекта. Значение по умолчанию 'hypotez'.

**Тип**: `str`

### `__version__`

**Описание**: Версия проекта. Значение по умолчанию пустая строка.

**Тип**: `str`

### `__doc__`

**Описание**: Документация к проекту. Значение по умолчанию пустая строка.

**Тип**: `str`

### `__details__`

**Описание**: Дополнительные подробности о проекте. Значение по умолчанию пустая строка.

**Тип**: `str`

### `__author__`

**Описание**: Автор проекта. Значение по умолчанию пустая строка.

**Тип**: `str`

### `__copyright__`

**Описание**: Авторские права на проект. Значение по умолчанию пустая строка.

**Тип**: `str`

### `__cofee__`

**Описание**: Ссылка для поддержки автора проекта. Значение по умолчанию строка со ссылкой.

**Тип**: `str`


## Обработка исключений

В модуле используются блоки `try...except` для обработки ошибок:

- `FileNotFoundError`: Возникает, если файл `settings.json` или `README.MD` не найден.
- `json.JSONDecodeError`: Возникает, если содержимое `settings.json` не является валидным JSON.

В этих случаях соответствующие переменные (settings, doc_str) устанавливаются в `None`.
