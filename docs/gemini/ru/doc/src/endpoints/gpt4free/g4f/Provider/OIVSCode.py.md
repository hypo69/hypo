# Документация для модуля OIVSCode

## Обзор

Модуль `OIVSCode.py` предназначен для определения класса `OIVSCode`, который наследуется от `OpenaiTemplate`. Этот класс предоставляет конфигурацию для взаимодействия с сервером OI VSCode, использующим OpenAI API. Он определяет параметры подключения, поддерживаемые модели и их псевдонимы.

## Подробней

Этот код определяет параметры для интеграции с сервером OI VSCode. Класс `OIVSCode` содержит информацию, такую как URL-адрес сервера, базовый URL API, флаги поддержки различных функций (стриминг, системные сообщения, история сообщений), а также список поддерживаемых моделей и их псевдонимы. Этот модуль важен для динамической настройки и выбора правильной модели при взаимодействии с сервером OI VSCode.

## Классы

### `OIVSCode`

**Описание**: Класс `OIVSCode` наследуется от `OpenaiTemplate` и предоставляет конфигурацию для взаимодействия с сервером OI VSCode.

**Принцип работы**: Класс определяет статические атрибуты, такие как URL-адрес сервера, базовый URL API, флаги поддержки различных функций (стриминг, системные сообщения, история сообщений), а также список поддерживаемых моделей и их псевдонимы.

**Наследует**:
- `OpenaiTemplate`: Класс, предоставляющий базовый шаблон для взаимодействия с OpenAI API.

**Атрибуты**:

- `label` (str): Метка, идентифицирующая провайдера как "OI VSCode Server".
- `url` (str): URL-адрес сервера OI VSCode.
- `api_base` (str): Базовый URL API для взаимодействия с сервером.
- `working` (bool): Указывает, что провайдер в настоящее время работает.
- `needs_auth` (bool): Указывает, требуется ли аутентификация для использования провайдера.
- `supports_stream` (bool): Указывает, поддерживает ли провайдер потоковую передачу данных.
- `supports_system_message` (bool): Указывает, поддерживает ли провайдер системные сообщения.
- `supports_message_history` (bool): Указывает, поддерживает ли провайдер историю сообщений.
- `default_model` (str): Модель, используемая по умолчанию ("gpt-4o-mini-2024-07-18").
- `default_vision_model` (str): Модель для обработки изображений, используемая по умолчанию (совпадает с `default_model`).
- `vision_models` (List[str]): Список моделей, поддерживающих обработку изображений.
- `models` (List[str]): Полный список поддерживаемых моделей, включая модели для обработки изображений и другие модели.
- `model_aliases` (Dict[str, str]): Словарь псевдонимов моделей, позволяющий использовать короткие имена для обращения к моделям.

## Функции
В данном классе нет функций, только статические атрибуты.
