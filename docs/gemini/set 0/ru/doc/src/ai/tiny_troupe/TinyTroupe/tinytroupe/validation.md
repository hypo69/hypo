# Модуль `tinytroupe.validation`

## Обзор

Этот модуль предоставляет инструменты для валидации инстансов `TinyPerson` с использованием LLM OpenAI.  Валидация основана на диалоге, где LLM задаёт вопросы, а `TinyPerson` отвечает.  Модуль обрабатывает обмен сообщениями, сохраняет историю взаимодействия и анализирует полученный результат для определения степени соответствия `TinyPerson` заданным ожиданиям.

## Функции

### `validate_person`

**Описание**: Проверяет инстанс `TinyPerson` с помощью OpenAI LLM, задавая вопросы и оценивая ответы. Возвращает оценку доверия (float от 0 до 1) и обоснование результата.

**Параметры**:

- `person` (TinyPerson): Инстанс класса `TinyPerson` для проверки.
- `expectations` (строка, необязательно): Ожидания, используемые при валидации. По умолчанию `None`.
- `include_agent_spec` (булево, необязательно): Включать ли описание агента в запрос. По умолчанию `True`.
- `max_content_length` (целое число, необязательно): Максимальная длина контента для отображения в ходе диалога. По умолчанию `default_max_content_display_length` (1024).

**Возвращает**:

- `float`: Оценка доверия валидации (от 0.0 до 1.0) или `None`, если валидация не удалась.
- `строка`: Обоснование оценки валидации или `None`, если валидация не удалась.

**Возможные исключения**:

- Не указано.
