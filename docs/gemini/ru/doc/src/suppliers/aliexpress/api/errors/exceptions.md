# Модуль `hypotez/src/suppliers/aliexpress/api/errors/exceptions.py`

## Обзор

Этот модуль содержит пользовательские исключения, используемые для обработки ошибок, возникающих при взаимодействии с API AliExpress.  Он определяет базовый класс `AliexpressException` и производные классы для конкретных типов ошибок.


## Классы

### `AliexpressException`

**Описание**: Базовый класс для всех исключений, связанных с API AliExpress.

**Методы**:

- `__init__(self, reason: str)`:
    **Описание**: Инициализирует экземпляр исключения.
    **Параметры**:
      - `reason` (str): Стрoка с описанием причины ошибки.
    **Возвращает**:
       -  None.

- `__str__(self) -> str`:
    **Описание**: Возвращает строковое представление исключения.
    **Параметры**:
       -  None
    **Возвращает**:
       - str: Строковое представление причины ошибки.


### `InvalidArgumentException`

**Описание**:  Исключение, выбрасываемое, когда переданные аргументы некорректны.

**Наследование**: `AliexpressException`


### `ProductIdNotFoundException`

**Описание**: Исключение, выбрасываемое, если идентификатор продукта не найден.

**Наследование**: `AliexpressException`


### `ApiRequestException`

**Описание**: Исключение, выбрасываемое, если запрос к API AliExpress завершился ошибкой.

**Наследование**: `AliexpressException`


### `ApiRequestResponseException`

**Описание**: Исключение, выбрасываемое, если ответ от API AliExpress не является валидным.

**Наследование**: `AliexpressException`


### `ProductsNotFoudException`

**Описание**: Исключение, выбрасываемое, если продукты не найдены.

**Наследование**: `AliexpressException`


### `CategoriesNotFoudException`

**Описание**: Исключение, выбрасываемое, если категории не найдены.

**Наследование**: `AliexpressException`


### `InvalidTrackingIdException`

**Описание**: Исключение, выбрасываемое, если идентификатор отслеживания не предоставлен или некорректен.

**Наследование**: `AliexpressException`
