# Модуль `post_event`

## Обзор

Модуль `post_event` предназначен для автоматизации процесса публикации календарных событий в группах Facebook с использованием Selenium WebDriver. Он содержит функции для ввода заголовка события, даты, времени и описания, а также для отправки события.

## Подробней

Этот модуль является частью проекта `hypotez` и используется для автоматизации маркетинговых кампаний в Facebook. Он предоставляет набор функций, которые позволяют программно заполнять и отправлять форму создания события в Facebook, что экономит время и уменьшает вероятность ошибок при ручном вводе данных.

## Функции

### `post_title`

```python
def post_title(d: Driver, title: str) -> bool:
    """
    Отправляет заголовок события.

    Args:
        d (Driver): Экземпляр драйвера, используемый для взаимодействия с веб-страницей.
        title (str): Заголовок события, который необходимо отправить.

    Returns:
        bool: `True`, если заголовок успешно отправлен, иначе `None`.

    Example:
        >>> driver = Driver(...)
        >>> post_title(driver, 'Заголовок мероприятия')
        True
    """
```

**Назначение**: Отправляет заголовок события в соответствующее поле на веб-странице.

**Как работает функция**:
1. Функция `post_title` принимает экземпляр драйвера `d` и заголовок события `title` в качестве аргументов.
2. Использует метод `execute_locator` объекта `d` для отправки заголовка события в поле, определенное локатором `locator.event_title`.
3. Если отправка не удалась, функция логирует ошибку с помощью `logger.error` и возвращает `None`.
4. В случае успешной отправки функция возвращает `True`.

Внутри функции происходят следующие действия и преобразования:
A. Получение драйвера и заголовка события
|
B. Отправка заголовка события с использованием локатора
|
C. Проверка успешности отправки
|
D. Логирование ошибки, если отправка не удалась
|
E. Возврат значения в зависимости от успешности отправки

### `post_event`

```python
def post_event(d: Driver, event: SimpleNamespace) -> bool:
    """
    Управляет процессом публикации события с заголовком, датой, временем и описанием.

    Args:
        d (Driver): Экземпляр драйвера, используемый для взаимодействия с веб-страницей.
        event (SimpleNamespace): Объект SimpleNamespace, содержащий данные события (заголовок, дата, время, описание и промо-ссылка).

    Returns:
        bool: `True`, если событие успешно опубликовано, иначе `None`.

    Example:
        >>> driver = Driver(...)
        >>> event = SimpleNamespace(title="Заголовок", start="2024-01-01 10:00", description="Описание", promotional_link="https://example.com")
        >>> post_event(driver, event)
    """
```

**Назначение**: Координирует процесс публикации события, вызывая функции для отправки заголовка, даты, времени и описания, а также для отправки самого события.

**Как работает функция**:
1. Функция `post_event` принимает экземпляр драйвера `d` и объект `event` типа `SimpleNamespace`, содержащий данные события.
2. Вызывает функцию `post_title` для отправки заголовка события. Если отправка не удалась, функция возвращает `None`.
3. Разделяет строку `event.start` на дату и время.
4. Вызывает функцию `post_date` для отправки даты события. Если отправка не удалась, функция возвращает `None`.
5. Вызывает функцию `post_time` для отправки времени события. Если отправка не удалась, функция возвращает `None`.
6. Вызывает функцию `post_description` для отправки описания события, включающего текст `event.description` и `event.promotional_link`. Если отправка не удалась, функция возвращает `None`.
7. Использует метод `execute_locator` объекта `d` для отправки события. Если отправка не удалась, функция возвращает `None`.
8. Ожидает 30 секунд.
9. Возвращает `True`.

|
C. Разделение даты и времени события
|
D. Отправка даты события
|
E. Отправка времени события
|
F. Отправка описания события
|
G. Отправка события
|
H. Ожидание
|
I. Возврат `True`
