# Модуль errors

## Обзор

Модуль `errors` содержит определения пользовательских исключений, используемых в проекте `hypotez` для обработки различных ошибок, возникающих при взаимодействии с GPT4Free. Эти исключения позволяют более точно определять и обрабатывать специфические ситуации, такие как отсутствие провайдера, проблемы с сетью, ошибки аутентификации и т.д.

## Подробней

Этот модуль предоставляет набор исключений, которые могут быть вызваны в различных частях проекта `hypotez`, особенно при работе с внешними API и сервисами. Использование специализированных исключений позволяет улучшить обработку ошибок и повысить надежность приложения.

## Классы

### `ProviderNotFoundError`

**Описание**: Исключение, возникающее, когда провайдер не найден.

**Принцип работы**:

Данное исключение сигнализирует о том, что запрошенный провайдер не существует или не может быть найден в текущей конфигурации.

### `ProviderNotWorkingError`

**Описание**: Исключение, возникающее, когда провайдер не работает.

**Принцип работы**:

Это исключение указывает на то, что провайдер, который должен быть доступен, в данный момент не функционирует должным образом (например, из-за проблем с сетью или внутренней ошибки провайдера).

### `StreamNotSupportedError`

**Описание**: Исключение, возникающее, когда потоковая передача не поддерживается.

**Принцип работы**:

Данное исключение вызывается, когда запрашивается потоковая передача данных, но текущий провайдер или модель не поддерживают эту функциональность.

### `ModelNotFoundError`

**Описание**: Исключение, возникающее, когда модель не найдена.

**Принцип работы**:

Это исключение сигнализирует о том, что запрошенная модель не существует или недоступна.

### `ModelNotAllowedError`

**Описание**: Исключение, возникающее, когда модель не разрешена для использования.

**Принцип работы**:

Данное исключение вызывается, когда у пользователя нет прав на использование запрошенной модели.

### `RetryProviderError`

**Описание**: Исключение, предлагающее повторить попытку с другим провайдером.

**Принцип работы**:

Это исключение указывает на то, что текущий провайдер временно недоступен, и рекомендуется попробовать использовать другой провайдер.

### `RetryNoProviderError`

**Описание**: Исключение, предлагающее повторить попытку без провайдера.

**Принцип работы**:

Данное исключение вызывается, когда не удается найти подходящего провайдера, и рекомендуется повторить попытку без указания конкретного провайдера.

### `VersionNotFoundError`

**Описание**: Исключение, возникающее, когда версия не найдена.

**Принцип работы**:

Это исключение сигнализирует о том, что запрошенная версия чего-либо (например, модели или API) не существует.

### `ModelNotSupportedError`

**Описание**: Исключение, возникающее, когда модель не поддерживается.

**Принцип работы**:

Данное исключение вызывается, когда текущая версия программного обеспечения не поддерживает запрошенную модель.

### `MissingRequirementsError`

**Описание**: Исключение, возникающее, когда отсутствуют необходимые зависимости.

**Принцип работы**:

Это исключение указывает на то, что для работы приложения необходимо установить дополнительные библиотеки или компоненты.

### `NestAsyncioError`

**Описание**: Исключение, возникающее при проблемах с асинхронностью. Наследуется от `MissingRequirementsError`.

**Принцип работы**:

Данное исключение вызывается, когда возникают проблемы с асинхронным кодом, например, при неправильной конфигурации `nest_asyncio`.

### `MissingAuthError`

**Описание**: Исключение, возникающее, когда отсутствует аутентификация.

**Принцип работы**:

Это исключение сигнализирует о том, что для доступа к ресурсу требуется аутентификация, которая не была предоставлена.

### `PaymentRequiredError`

**Описание**: Исключение, возникающее, когда требуется оплата.

**Принцип работы**:

Данное исключение вызывается, когда для доступа к функциональности необходимо произвести оплату.

### `NoMediaResponseError`

**Описание**: Исключение, возникающее, когда отсутствует ответ с медиа-контентом.

**Принцип работы**:

Это исключение указывает на то, что ожидаемый медиа-контент не был получен в ответе.

### `ResponseError`

**Описание**: Исключение, возникающее при общей ошибке ответа.

**Принцип работы**:

Данное исключение является базовым для всех ошибок, связанных с ответами от внешних сервисов.

### `ResponseStatusError`

**Описание**: Исключение, возникающее при ошибке в статусе ответа. Наследуется от `ResponseError`.

**Принцип работы**:

Это исключение сигнализирует о том, что статус ответа указывает на ошибку (например, код 500).

### `RateLimitError`

**Описание**: Исключение, возникающее при превышении лимита запросов. Наследуется от `ResponseStatusError`.

**Принцип работы**:

Данное исключение вызывается, когда количество запросов превышает установленный лимит.

### `NoValidHarFileError`

**Описание**: Исключение, возникающее, когда отсутствует валидный HAR-файл.

**Принцип работы**:

Это исключение сигнализирует о том, что предоставленный HAR-файл не является валидным или не может быть обработан.

### `TimeoutError`

**Описание**: Исключение, возникающее при превышении времени ожидания API-запроса.

**Принцип работы**:

Данное исключение вызывается, когда API-запрос занимает слишком много времени и превышает установленный лимит ожидания.

### `ConversationLimitError`

**Описание**: Исключение, возникающее при достижении лимита на количество разговоров с AI.

**Принцип работы**:

Это исключение указывает на то, что достигнут лимит на количество взаимодействий (разговоров) с AI-сервисом.
