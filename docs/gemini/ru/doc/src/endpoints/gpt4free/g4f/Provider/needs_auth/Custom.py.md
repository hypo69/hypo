# Модуль для определения пользовательских провайдеров

## Обзор

Модуль содержит классы для определения пользовательских провайдеров, наследуемых от `OpenaiTemplate`. Это позволяет гибко настраивать взаимодействие с различными API, в частности, с локальными серверами, эмулирующими OpenAI API.

## Подробнее

Этот модуль предоставляет базовые классы для создания пользовательских провайдеров, совместимых с шаблоном OpenAI. Он определяет основные атрибуты и флаги, необходимые для работы с провайдерами, такие как признак работоспособности (`working`), необходимость аутентификации (`needs_auth`) и базовый URL API (`api_base`).

## Классы

### `Custom`

**Описание**: Базовый класс для пользовательских провайдеров, наследуемый от `OpenaiTemplate`.

**Наследует**:
- `OpenaiTemplate`: Предоставляет общую структуру для провайдеров, совместимых с OpenAI API.

**Атрибуты**:
- `label` (str): Метка для идентификации провайдера ("Custom Provider").
- `working` (bool): Флаг, указывающий на работоспособность провайдера (True).
- `needs_auth` (bool): Флаг, указывающий на необходимость аутентификации (False).
- `api_base` (str): Базовый URL API ("http://localhost:8080/v1").
- `sort_models` (bool): Флаг, указывающий на необходимость сортировки моделей (False).

### `Feature`

**Описание**: Класс для провайдеров с дополнительными функциями, наследуемый от `Custom`.

**Наследует**:
- `Custom`: Предоставляет базовую структуру пользовательского провайдера.

**Атрибуты**:
- `label` (str): Метка для идентификации провайдера ("Feature Provider").
- `working` (bool): Флаг, указывающий на работоспособность провайдера (False).
