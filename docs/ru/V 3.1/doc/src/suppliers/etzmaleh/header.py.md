# Модуль header

## Обзор

Модуль `header` предназначен для определения основных настроек и параметров проекта `hypotez`. Он отвечает за установку корневой директории проекта, загрузку настроек из файла `settings.json` и чтение документации из файла `README.MD`. Также, модуль определяет глобальные переменные, такие как имя проекта, версия, автор и т.д.

## Подорбней

Модуль `header` играет важную роль в инициализации проекта `hypotez`. Он определяет корневую директорию проекта, что позволяет корректно находить другие модули и файлы. Загрузка настроек из `settings.json` позволяет гибко конфигурировать проект, а чтение `README.MD` предоставляет информацию о проекте. Этот модуль гарантирует, что основные параметры проекта доступны для использования в других частях кодовой базы.

## Функции

### `set_project_root`

```python
def set_project_root(marker_files: tuple = ('__root__', '.git')) -> Path:
    """
    Args:
        marker_files (tuple): Filenames or directory names to identify the project root.

    Returns:
        Path: Path to the root directory if found, otherwise the directory where the script is located.
    """
```

**Описание**: Находит корневую директорию проекта, начиная с директории текущего файла и двигаясь вверх по иерархии каталогов. Поиск прекращается при обнаружении каталога, содержащего один из файлов-маркеров.

**Параметры**:
- `marker_files` (tuple): Кортеж имен файлов или директорий, используемых для идентификации корневой директории проекта. По умолчанию `('__root__', '.git')`.

**Возвращает**:
- `Path`: Путь к корневой директории, если она найдена. В противном случае возвращает директорию, в которой расположен скрипт.

**Примеры**:
```python
from pathlib import Path
# Допустим, что текущий файл находится в поддиректории проекта, а корневая директория содержит файл '.git'
root_path = set_project_root()
print(root_path)
# Вывод: Path('/путь/к/корневой/директории/проекта')
