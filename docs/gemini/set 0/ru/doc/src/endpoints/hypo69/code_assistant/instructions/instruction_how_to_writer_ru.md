# Инструкции по генерации документации к коду

## Обзор

Данные инструкции предназначены для создания документации к Python-коду в формате Markdown.  Они описывают шаги и структуру для создания понятной и полезной документации, которая поможет разработчикам использовать и понимать код.


## Инструкции

### Анализ кода

Первым шагом является тщательный анализ предоставленного Python-кода. Понимание логики и действий, которые выполняет данный фрагмент кода, является фундаментальным для создания эффективной документации.


### Создание пошаговой инструкции

После анализа кода следует разработать пошаговую инструкцию, описывающую его функциональность. Эта инструкция должна содержать следующие разделы:


#### Описание

В этом разделе подробно объясняется, что делает данный блок кода.  Необходимо описывать, какие задачи он выполняет, какие данные обрабатывает и как они используются.


#### Шаги выполнения

Этот раздел содержит пошаговое описание последовательности действий в коде.  Каждый шаг должен быть чётко определён и объяснён, чтобы пользователь мог понять, как код достигает результата.  Избегайте общих фраз. Используйте конкретные глаголы (например, "проверяет", "вычисляет", "сохраняет").


#### Пример использования

В этом разделе должен быть предоставлен пример того, как использовать данный фрагмент кода в реальном проекте. Пример кода должен быть наглядным и демонстрировать правильное использование функций и методов.


### Форматирование документации

Документация должна быть оформлена в формате reStructuredText (RST) с использованием соответствующих тегов и структур.  Например:

```rst
Как использовать этот блок кода
=========================================================================================

Описание
-------------------------
[Объяснение, что делает код. Например, "Эта функция проверяет валидность введённых данных."]

Шаги выполнения
-------------------------
1. [Описание первого шага. Например, "Получает данные от пользователя."]
2. [Описание второго шага. Например, "Проверяет, что данные соответствуют заданным правилам."]
3. [Продолжай по необходимости...]

Пример использования
-------------------------
.. code-block:: python

    # Пример кода, демонстрирующий использование данной функции
    result = validate_input(user_input)
    if result:
        print("Введённые данные валидны.")
    else:
        print("Введённые данные невалидны.")

```


### Избегайте расплывчатых терминов

Документация должна избегать расплывчатых терминов и быть максимально конкретной.  Вместо общих фраз типа "получаем" или "делаем", используйте точные глаголы и описания, например: "проверяет", "валидирует", "отправляет".

## Пример

```rst
Как использовать функцию validate_input
========================================================================================

Описание
-------------------------
Функция validate_input проверяет валидность введённых пользователем данных.

Шаги выполнения
-------------------------
1. Функция получает данные от пользователя.
2. Функция проверяет, что данные соответствуют заданным правилам (например, длина, формат).
3. Если данные валидны, функция возвращает True, в противном случае - False.

Пример использования
-------------------------
.. code-block:: python

    # Пример кода, демонстрирующий использование данной функции
    user_input = input("Введите данные: ")
    is_valid = validate_input(user_input)
    if is_valid:
        print("Введённые данные валидны.")
    else:
        print("Введённые данные невалидны.")

```
