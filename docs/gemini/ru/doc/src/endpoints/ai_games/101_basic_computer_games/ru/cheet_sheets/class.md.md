## Структура класса

### Описание

Класс — это шаблон для создания объектов, которые имеют атрибуты (данные) и методы (функции). Объекты, созданные на основе класса, называются экземплярами класса.

### Пример

```python
class ClassName:
    # Атрибуты класса
    def __init__(self, param1, param2):
        # Конструктор (инициализатор) класса
        self.param1 = param1
        self.param2 = param2

    # Методы класса
    def method(self):
        return f'{self.param1} и {self.param2}'
```

### Конструктор (`__init__`)

**Описание**: Специальный метод, который автоматически вызывается при создании нового объекта. Используется для инициализации атрибутов объекта.

**Параметры**:

-   `self`: Ссылка на текущий экземпляр класса.
-   `param1`: Первый параметр для инициализации объекта.
-   `param2`: Второй параметр для инициализации объекта.

**Возвращает**:
    - None

### Атрибуты класса

**Описание**: Переменные, которые принадлежат объектам этого класса. Они определяются внутри конструктора (`__init__`) и доступны с помощью ссылки на объект.

### Методы класса

**Описание**: Функции, которые могут манипулировать атрибутами объекта.

## Создание объекта класса

### Описание

Объекты создаются как экземпляры класса. При создании объекта передаются значения для атрибутов, которые сохраняются в объекте.

### Пример

```python
class Car:
    def __init__(self, make, model, year):
        self.make = make
        self.model = model
        self.year = year

    def description(self):
        return f'{self.year} {self.make} {self.model}'

# Создание объекта
my_car = Car('Toyota', 'Corolla', 2020)
print(my_car.description())  # Вывод: 2020 Toyota Corolla
```

## Типы методов

### Методы экземпляра

**Описание**: Обычные методы, которые действуют на экземпляры класса. Они принимают ссылку на объект (`self`) как первый параметр.

**Пример**:

```python
def method(self):
   pass
```

### Методы класса

**Описание**: Методы, которые принимают сам класс (`cls`) как первый параметр. Для определения таких методов используется декоратор `@classmethod`.

**Пример**:

```python
class MyClass:
    @classmethod
    def class_method(cls):
        pass
```

### Статические методы

**Описание**: Методы, которые не используют `self` или `cls`. Они объявляются с помощью декоратора `@staticmethod`.

**Пример**:

```python
class MyClass:
    @staticmethod
    def static_method():
        pass
```

## Наследование

### Описание

Класс может наследовать поведение другого класса, расширяя или изменяя его функциональность.

### Пример

```python
class Animal:
    def speak(self):
        return 'Голос животного'

class Dog(Animal):  # Класс Dog наследует от класса Animal
    def speak(self):
        return 'Гав'

# Создание объектов
dog = Dog()
print(dog.speak())  # Вывод: Гав
```

## Полиморфизм

### Описание

Полиморфизм означает способность объектов разных классов использовать одинаковые методы с разной реализацией.

### Пример

```python
class Cat(Animal):
    def speak(self):
        return 'Мяу'

# Создание объектов
cat = Cat()
print(cat.speak())  # Вывод: Мяу
```

## Инкапсуляция

### Описание

Инкапсуляция позволяет скрывать внутренние детали реализации и предоставлять доступ к данным через публичные методы.

### Пример

```python
class Car:
    def __init__(self, make, model):
        self._make = make  # Защищённый атрибут
        self._model = model

    def get_make(self):
        return self._make

    def set_make(self, make):
        self._make = make

# Создание объекта
my_car = Car('Toyota', 'Corolla')
print(my_car.get_make())  # Вывод: Toyota
my_car.set_make('Honda')
print(my_car.get_make())  # Вывод: Honda
```

## Прочие особенности классов

### Деструктор (`__del__`)

**Описание**: Специальный метод, который вызывается, когда объект уничтожается.

**Пример**:

```python
class MyClass:
   def __del__(self):
        print("Объект уничтожен")

obj = MyClass()
del obj  # Объект будет уничтожен и вызовется метод __del__
```

### Магические методы

**Описание**: Специальные методы с двумя подчеркиваниями (например, `__init__`, `__str__`, `__repr__`, `__eq__`). Они позволяют переопределять стандартное поведение операций.

**Пример**:

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __repr__(self):
        return f'Point({self.x}, {self.y})'

p = Point(3, 4)
print(p)  # Вывод: Point(3, 4)
```
