# Модуль src.scenario

## Обзор

Модуль `src.scenario` предназначен для автоматизации взаимодействия с поставщиками, используя сценарии, описанные в JSON-файлах. Он адаптирует процесс извлечения и обработки данных о продуктах с веб-сайтов поставщиков и синхронизирует эту информацию с базой данных (например, PrestaShop).

## Функции

### `run_scenario_files`

**Описание**: Выполняет сценарии, заданные в списке файлов.

**Параметры**:
- `s (object)`:  Объект, предоставляющий доступ к функциям для работы с файлами и сценариями.
- `scenario_files_list (list[str])`: Список путей к файлам сценариев.

**Возвращает**:
- `None`: Функция не возвращает явного значения.

**Вызывает исключения**:
- `FileNotFoundError`: Если файл сценария не найден.
- `JSONDecodeError`: Если содержимое файла сценария не соответствует формату JSON.
- `Exception`: Другие возможные ошибки при выполнении.


### `run_scenario_file`

**Описание**: Загружает сценарии из файла и выполняет их.

**Параметры**:
- `s (object)`:  Объект, предоставляющий доступ к функциям для работы с файлами и сценариями.
- `scenario_file (str)`: Путь к файлу сценария.

**Возвращает**:
- `None`: Функция не возвращает явного значения.

**Вызывает исключения**:
- `FileNotFoundError`: Если файл сценария не найден.
- `JSONDecodeError`: Если содержимое файла сценария не соответствует формату JSON.
- `Exception`: Другие возможные ошибки при выполнении.



### `run_scenario`

**Описание**: Обрабатывает отдельный сценарий.

**Параметры**:
- `s (object)`: Объект, предоставляющий доступ к функциям для работы с веб-сайтами и базой данных.
- `scenario (dict)`: Словарь, представляющий сценарий (из JSON).

**Возвращает**:
- `None`: Функция не возвращает явного значения.

**Вызывает исключения**:
- `Exception`: Любые ошибки при работе с веб-сайтом или базой данных.


### `dump_journal`

**Описание**: Сохраняет журнал выполнения сценариев в файл.

**Параметры**:
- `s (object)`: Объект, предоставляющий доступ к функциям для работы с файлами.
- `journal (list)`: Список записей журнала.

**Возвращает**:
- `None`: Функция не возвращает явного значения.

**Вызывает исключения**:
- `Exception`: Любые ошибки при записи в файл.

            "presta_categories": {

            }
        }

## Как это работает

Модуль загружает сценарии из JSON-файлов, содержащих информацию о URL-адресах веб-страниц и необходимых категориях.  Затем, по каждому сценарию, он выполняет запрос на указанный URL, извлекает данные о продуктах, и сохраняет их в базе данных PrestaShop, учитывая указанные категории.  Журналирование позволяет отслеживать процесс и выявлять ошибки.

**Важно:** Данный код требует реализацию функций работы с веб-сайтами и базой данных (`s` объект).  В данном описании это опущенно для фокусировки на структуре документации.
