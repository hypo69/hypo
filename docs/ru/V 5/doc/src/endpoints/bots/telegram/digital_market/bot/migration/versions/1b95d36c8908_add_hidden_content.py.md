# `add_hidden_content`

## Обзор

Данный скрипт миграции Alembic добавляет новый столбец `hidden_content` в таблицу `products` и предоставляет возможность отката этого изменения.

## Подробней

Этот скрипт используется для обновления схемы базы данных в проекте `hypotez`. Он добавляет столбец `hidden_content`, предназначенный для хранения скрытого контента продукта, в таблицу `products`. Миграция выполняет это добавление при вызове функции `upgrade` и удаляет столбец при вызове функции `downgrade`, обеспечивая возможность отката изменений. Alembic используется для управления версиями схемы базы данных, чтобы изменения можно было применять и отменять контролируемым образом.

## Функции

### `upgrade`

**Описание**: Добавляет столбец `hidden_content` в таблицу `products`.

**Как работает функция**:
Функция `upgrade` использует `op.add_column` из библиотеки `alembic` для добавления нового столбца `hidden_content` типа `Text` в таблицу `products`. Параметр `nullable=False` указывает, что столбец не может содержать `NULL` значения.

**Параметры**:

-   Нет параметров.

**Возвращает**:

-   `None`

**Вызывает исключения**:

-   Нет информации об исключениях.

### `downgrade`

**Описание**: Удаляет столбец `hidden_content` из таблицы `products`.

**Как работает функция**:

Функция `downgrade` использует `op.drop_column` из библиотеки `alembic` для удаления столбца `hidden_content` из таблицы `products`. Это позволяет откатить изменения, внесенные функцией `upgrade`.

**Параметры**:

-   Нет параметров.

**Возвращает**:

-   `None`

**Вызывает исключения**:

-   Нет информации об исключениях.
