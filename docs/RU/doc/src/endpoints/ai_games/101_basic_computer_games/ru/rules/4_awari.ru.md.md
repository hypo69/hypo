# Awari (Удаление бобов из ям)

## Обзор

Документ описывает правила и логику реализации игры Awari, традиционной настольной игры, где два игрока соревнуются за накопление большего количества бобов в своих "домашних ямах". Игра включает в себя выбор ям, "посев" бобов по доске и захват бобов противника.

## Правила игры

Игра *Awari* (или *Калах*) — это стратегическая игра для двух игроков, где каждый стремится собрать наибольшее количество бобов в своем "доме". Игровое поле состоит из двух рядов ям и двух "домов" для каждого игрока.

- **Особенности:**
  - Доска состоит из двух рядов по 6 ям и двух "домов" (по одному на игрока).
  - Игрок выбирает яму на своей стороне и перемещает бобы по часовой стрелке.
  - Если последний боб попадает в пустую яму, напротив которой есть бобы, игрок забирает все эти бобы в свой "дом".
  - Игра заканчивается, когда на одной из сторон доски заканчиваются бобы.
  - Победителем становится тот, у кого в "доме" больше бобов.

## Пошаговая инструкция для реализации

### 1. Инициализация игры

- Создается массив, представляющий игровую доску:
  ```
  [0, 4, 4, 4, 4, 4, 4, 0, 4, 4, 4, 4, 4, 4]
  ```
  - Индексы 1–6: ямы игрока.
  - Индексы 8–13: ямы компьютера.
  - Индексы 0 и 7: дома игроков.
- Устанавливается начальное количество бобов в каждой яме (например, 4).

### 2. Основной цикл игры

- **Ход игрока**:
    1. Запрашивается номер ямы, из которой игрок хочет взять бобы.
    2. Проверяется валидность выбора:
        - Яма не пуста.
        - Яма принадлежит игроку.
    3. Бобы из выбранной ямы перемещаются, "сея" их по кругу:
        - Добавляется по одному бобу в каждую следующую яму.
        - Дом противника пропускается.
    4. Проверяются условия:
        - Если последний боб попал в пустую яму игрока, а напротив есть бобы, они перемещаются в "дом".
        - Если последний боб попал в "дом", игрок получает дополнительный ход.
- **Ход компьютера**:
    - Используется простой алгоритм выбора ямы (например, первая непустая) или более сложный для "обучающегося" компьютера.
    - Повторяются шаги, как для игрока.
- **Завершение игры**:
    - Игра заканчивается, если на одной из сторон (игрока или компьютера) закончились бобы.
    - Все оставшиеся бобы перемещаются в "дом" игрока или компьютера на их стороне.

### 3. Подсчет очков

- Подсчитываются бобы в "домах".
- Объявляется победитель:
  ```
  Игра окончена. Вы выиграли с [X] бобами!
  ```

### 4. Дополнительные функции

- Отображение текущего состояния доски после каждого хода.
- Возможность начать новую игру или выйти.

## Пример работы программы

1. **Начало игры:**
   ```
   Добро пожаловать в игру Awari!
   Текущая доска:
   [0, 4, 4, 4, 4, 4, 4, 0, 4, 4, 4, 4, 4, 4]
   Ваш ход. Выберите яму (1-6):
   > 3
   ```

2. **После хода игрока:**
   ```
   Вы взяли 4 боба из ямы 3.
   Текущая доска:
   [0, 4, 4, 0, 5, 5, 5, 0, 4, 4, 4, 4, 4, 4]
   Ход компьютера...
   Компьютер выбрал яму 10.
   ```

3. **Завершение игры:**
   ```
   Игра окончена!
   Вы набрали 26 бобов. Компьютер набрал 22 боба.
   Вы победили!
   ```

## Возможные ограничения

- Необходимость контроля корректности ходов компьютера.
- Потенциальная потребность в оптимизации при увеличении размера игрового поля.
