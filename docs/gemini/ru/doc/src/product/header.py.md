# Модуль `header`

## Обзор

Модуль `header.py` предназначен для определения корневого пути проекта, загрузки настроек из `settings.json`, чтения документации из `README.MD`, а также для определения основных метаданных проекта, таких как имя, версия и автор.

## Функции

### `set_project_root`

**Описание**:
Находит корневую директорию проекта, начиная с директории текущего файла, ища вверх по дереву директорий до тех пор, пока не встретит одну из заданных маркерных файлов.

**Параметры**:
- `marker_files` (tuple, optional): Кортеж имен файлов или директорий, используемых для определения корневой директории проекта. По умолчанию `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:
- `Path`: Путь к корневой директории проекта, если она найдена. Если ни один маркерный файл не найден, возвращает директорию, в которой находится скрипт.

## Переменные
### `MODE`

**Описание**:
Указывает режим работы приложения. В данном случае установлено значение `'dev'`.

### `__root__`
**Описание**:
Содержит путь к корневой директории проекта, определенный функцией `set_project_root`.

### `settings`
**Описание**:
Содержит загруженные настройки проекта из файла `settings.json`. Если файл не найден или произошла ошибка при чтении, значение будет `None`.

### `doc_str`
**Описание**:
Содержит строку с содержимым файла `README.MD`, если он найден и успешно прочитан. Если файл не найден или произошла ошибка при чтении, значение будет `None`.

### `__project_name__`
**Описание**:
Имя проекта, полученное из настроек или по умолчанию `'hypotez'`.

### `__version__`
**Описание**:
Версия проекта, полученная из настроек. Если настройки не содержат версии, то будет пустая строка `''`.

### `__doc__`
**Описание**:
Содержит строку документации проекта, полученную из файла `README.MD`. Если чтение не удалось, то будет пустая строка `''`.

### `__details__`
**Описание**:
Содержит дополнительные детали проекта. В текущей версии пустая строка `''`.

### `__author__`
**Описание**:
Автор проекта, полученный из настроек. Если настройки не содержат автора, то будет пустая строка `''`.

### `__copyright__`
**Описание**:
Информация об авторских правах, полученная из настроек. Если настройки не содержат информации, то будет пустая строка `''`.

### `__cofee__`
**Описание**:
Сообщение с предложением угостить разработчика кофе, полученное из настроек. Если настройки не содержат данной информации, то используется сообщение по умолчанию: `"Treat the developer to a cup of coffee for boosting enthusiasm in development: https://boosty.to/hypo69"`.
