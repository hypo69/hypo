# Модуль hypotez/src/endpoints/prestashop/api/header.py

## Обзор

Этот модуль содержит функции для определения корневой директории проекта, загрузки настроек из файла `settings.json` и получения документации из файла `README.MD`.  Он также предоставляет доступ к переменным, описывающим проект (название, версия, автор, копирайт, и т.д.).


## Функции

### `set_project_root`

**Описание**: Определяет корневую директорию проекта, начиная с директории текущего файла и идя вверх по дереву директорий.  Ищет файлы или директории, указанные в `marker_files`, чтобы определить, где находится корень проекта.

**Параметры**:

- `marker_files` (tuple): Кортеж имен файлов или директорий, по которым определяется корень проекта.  По умолчанию: `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:

- `Path`: Путь к корневой директории проекта, если найдена. В противном случае возвращает путь к текущей директории.


### (Замещение __root__):

**Описание**: Обновляет глобальную переменную `__root__` значением, полученным из функции `set_project_root()`.

**Параметры**:

*Нет параметров.*

**Возвращает**:

*Нет возвращаемого значения.*


## Глобальные переменные

**Описание**: Эти переменные содержат информацию о проекте, загруженную из `settings.json` и `README.MD`.

### `__root__` (Path)

Путь к корневой директории проекта, полученный с помощью `set_project_root()`.


### `settings` (dict)

Словарь, содержащий настройки проекта из `settings.json`. Возвращает `None` если файл не найден или некорректен.


### `doc_str` (str)

Строка документации, полученная из файла `README.MD`. Возвращает `None`, если файл не найден или некорректен.


### `__project_name__` (str)

Имя проекта. Значение по умолчанию 'hypotez'.


### `__version__` (str)

Версия проекта.


### `__doc__` (str)

Документация проекта.


### `__details__` (str)

Подробная информация о проекте.


### `__author__` (str)

Автор проекта.


### `__copyright__` (str)

Копирайт проекта.


### `__cofee__` (str)

Ссылка на поддержку разработчика.


**Обработка исключений**:

В коде используются блоки `try...except`, чтобы обработать возможные ошибки при чтении файлов (`FileNotFoundError`, `json.JSONDecodeError`).  В случае ошибки эти переменные (settings, doc_str) остаются со значением по умолчанию (None или пустая строка).
