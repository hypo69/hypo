# Модуль header

## Обзор

Модуль `header.py` предназначен для определения метаданных проекта, таких как имя, версия, авторские права и т. д. Он также включает функцию для определения корневой директории проекта.

## Подорбней

Этот модуль выполняет следующие задачи:

1.  Определение корневой директории проекта с помощью функции `set_project_root`.
2.  Чтение настроек из файла `settings.json` и документации из `README.MD`.
3.  Установка метаданных проекта, таких как имя, версия, автор, и т. д.

Этот модуль важен, так как он предоставляет централизованное место для хранения метаданных проекта, которые могут использоваться другими модулями. Функция `set_project_root` используется для определения абсолютных путей к файлам и директориям проекта, что упрощает работу с файловой системой.

## Функции

### `set_project_root`

```python
def set_project_root(marker_files: tuple[str] = ('__root__', '.git')) -> Path:
    """
    Args:
        marker_files (tuple): Filenames or directory names to identify the project root.
    
    Returns:
        Path: Path to the root directory if found, otherwise the directory where the script is located.
    """
```

**Описание**: Находит корневой каталог проекта, начиная с каталога текущего файла, и останавливается на первом каталоге, содержащем любой из указанных маркерных файлов.

**Параметры**:

*   `marker_files` (tuple[str], optional): Кортеж имен файлов или каталогов для идентификации корня проекта. По умолчанию `('__root__', '.git')`.

**Возвращает**:

*   `Path`: Путь к корневому каталогу, если он найден, в противном случае - каталог, где расположен скрипт.

**Примеры**:

```python
from pathlib import Path
root_path = set_project_root()
print(f"Root path: {root_path}")
