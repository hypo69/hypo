# Модуль `hypotez/src/utils/file.py`

Данный модуль предоставляет функции для работы с файлами, включая чтение, запись, поиск и обработку файлов в директориях. Он предназначен для использования в проекте `hypotez`.

## Функции

### `save_text_file(data, file_path, mode='w', exc_info=True)`

Сохраняет данные в текстовый файл.

**Аргументы:**

* `data`: Данные для записи (строка, список строк или словарь).
* `file_path`: Путь к файлу для сохранения.
* `mode`: Режим записи (`'w'` - запись, `'a'` - добавление). По умолчанию `'w'`.
* `exc_info`: Если `True`, выводит трассировку стека при ошибке. По умолчанию `True`.

**Возвращаемое значение:**

* `bool`: `True`, если файл успешно сохранен, `False` в противном случае.

**Примеры использования:**

```python
# Сохранение строки в файл
save_text_file("Hello, world!", "my_file.txt")

# Сохранение списка строк в файл
save_text_file(["Line 1", "Line 2"], "my_list.txt", mode="a")

# Сохранение словаря в файл
save_text_file({"key1": "value1", "key2": "value2"}, "my_dict.json")
```


### `read_text_file(file_path, as_list=False, extensions=None, exc_info=True)`

Читает содержимое файла.

**Аргументы:**

* `file_path`: Путь к файлу или директории.
* `as_list`: Если `True`, возвращает содержимое файла как список строк. По умолчанию `False`.
* `extensions`: Список расширений файлов, которые нужно читать, если `file_path` - это директория. По умолчанию `None`.
* `exc_info`: Если `True`, выводит трассировку стека при ошибке. По умолчанию `True`.

**Возвращаемое значение:**

* `str | list[str] | None`: Содержимое файла в виде строки или списка строк, или `None` при ошибке.

**Примеры использования:**

```python
# Чтение содержимого файла в виде строки
content = read_text_file("my_file.txt")

# Чтение содержимого файла в виде списка строк
lines = read_text_file("my_file.txt", as_list=True)

# Чтение содержимого файлов в директории с расширением .txt
contents_from_dir = read_text_file("my_dir", extensions=[".txt"])

```

### `get_filenames(directory, extensions='*', exc_info=True)`

Возвращает список имен файлов в указанной директории, опционально фильтруя по расширениям.

### `recursively_yield_file_path(root_dir, patterns='*', exc_info=True)`

Рекурсивно генерирует пути к файлам, соответствующим заданным шаблонам.

### `recursively_get_filenames(root_dir, patterns='*', exc_info=True)`

Рекурсивно получает пути к файлам, соответствующим заданным шаблонам.

### `recursively_read_text_files(root_dir, patterns, as_list=False, exc_info=True)`

Рекурсивно считывает содержимое текстовых файлов из указанной директории, которые соответствуют шаблонам.

### `get_directory_names(directory, exc_info=True)`

Возвращает список имен поддиректорий в указанной директории.


### `read_files_content(root_dir, patterns, as_list=False, exc_info=True)`

Считывает содержимое файлов, соответствующих шаблонам, из указанной директории, рекурсивно.

### `remove_bom(file_path)`

Удаляет BOM (Byte Order Mark) из текстового файла.

### `traverse_and_clean(directory)`

Рекурсивно обходит директорию и удаляет BOM из файлов `.py`.

### `main()`

Точка входа для удаления BOM из файлов `.py` в указанной директории.


## Примечания

* Функции используют `pathlib` для работы с путями, что делает код более ясным и безопасным.
* Функции обрабатывают исключения и логируют ошибки с помощью логгера.
* Функции `recursively_*` предназначены для обработки больших директорий.
* Функции используют `fnmatch` для поиска файлов по шаблонам.
* Код содержит примеры использования.
* Документация расширена для лучшего понимания каждой функции.
* Добавлены пояснения к возвращаемым значениям и аргументам.
* Добавлен пример использования `recursively_read_text_files` с итерацией по результатам.
* Добавлен пример использования `get_directory_names`.


Этот модуль предоставляет удобный и надежный инструмент для работы с файлами в проекте `hypotez`.
