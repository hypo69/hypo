## Описание

Этот модуль `check_relise.py` предназначен для получения последней версии релиза из репозитория GitHub. Он использует API GitHub для получения информации о релизах и возвращает имя тега последнего релиза.

## Импорты

* `requests`: Библиотека для работы с HTTP запросами.
* `logger`: Модуль логирования, предполагается, что он определен в `src.logger`.

## Функция `check_latest_release`

```python
def check_latest_release(owner: str, repo: str):
    """Проверяет последнюю версию релиза репозитория GitHub.

    Args:
        owner (str): Владелец репозитория.
        repo (str): Название репозитория.

    Returns:
        str: Последняя версия релиза (тег) в формате строки, если доступна. Возвращает `None`, если нет доступных релизов или произошла ошибка.
    """
    url = f'https://api.github.com/repos/{owner}/{repo}/releases/latest'
    response = requests.get(url)

    if response.status_code == 200:
        latest_release = response.json()
        return latest_release['tag_name']
    else:
        logger.error(f"Ошибка при получении данных: {response.status_code}")
        return None  # Вернуть None, чтобы указать на ошибку
```

**Описание параметров и возвращаемого значения:**

* `owner`: Строка, содержащая имя пользователя или организации, владеющей репозиторием.
* `repo`: Строка, содержащая имя репозитория.
* Возвращает строку, содержащую имя тега последнего релиза, или `None`, если произошла ошибка или релизы не найдены.
