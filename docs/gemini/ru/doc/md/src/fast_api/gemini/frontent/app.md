# Модуль `app.js`

## Обзор

Данный модуль реализует интерфейс чат-бота с использованием React и FastAPI.  Пользователь вводит сообщение, которое отправляется на сервер FastAPI.  Полученный ответ от сервера отображается в чате.

## Компоненты

### `App`

**Описание**:  Главный компонент приложения, управляющий отображением чата и логикой отправки сообщений.

**Состояние**:
- `input` (строка): текущее введённое сообщение пользователя.
- `messages` (массив): массив сообщений, содержащих `role` (пользователь/бот) и `content` (содержимое сообщения).

**Методы**:

#### `sendMessage`

**Описание**: Обрабатывает отправку сообщения от пользователя.

**Логика**:
1. Проверяет, пустое ли введённое сообщение. Если да, то функция возвращает управление.
2. Добавляет сообщение пользователя в массив `messages`.
3. Выполняет `fetch` запрос к API сервера FastAPI.
4. Обрабатывает ответ сервера: парсит `JSON` и добавляет ответ бота в массив `messages`.
5. Очищает поле ввода.
6. Обрабатывает возможные ошибки при запросе к API.

**Возможные ошибки**:
- `Error`:  Возникает при ошибке запроса к API сервера.

**Параметры**:
- Нет входных параметров.

**Возвращает**:
- Нет возвращаемого значения.

#### `return`

**Описание**: Определяет возвращаемый JSX элемент.

**JSX элемент**:
- Содержит чат-бокc для отображения сообщений.
- Форма ввода сообщения с кнопкой отправки.

**Структура возвращаемого значения**:
- `chat-box`: блок для отображения истории сообщений, использующий `map` для отрисовки каждого сообщения.
- `input-group`: содержит поле ввода сообщения и кнопку `Send`.


## Внешние зависимости

- `React`: Библиотека для создания пользовательских интерфейсов.
- `ReactDOM`: Библиотека для рендеринга React компонентов.
- `fetch`: API для выполнения HTTP запросов.
