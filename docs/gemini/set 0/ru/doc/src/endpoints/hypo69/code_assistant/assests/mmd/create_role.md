# Порядок действий для создания новой роли

## Обзор

Данный документ описывает порядок действий для создания новой роли в системе.  Процесс включает добавление информации о роли в различные файлы конфигурации и создание необходимых инструкций.

## Этапы

### 1. Файл `code_assistant.json`

**Описание**: В этом файле хранится информация о списках доступных ролей.

**Действия**:
- Добавить новую роль в соответствующий раздел файла `code_assistant.json`.


### 2. Файл переводов `translations/translations.json`

**Описание**: Файл содержит переводы для отображения сообщений о ролях.

**Действия**:
- Добавить новые переводы для новой роли в раздел `translations/translations.json`.  Включает переводы для названий и описаний ролей.


### 3. Создание системного промпта в `ai/prompts/develpoper`

**Описание**:  Протокол взаимодействия с моделью (LLM) в отношении новой роли.

**Действия**:
- Создать новый системный промпт в файле `ai/prompts/develpoper`, который настраивает модель на взаимодействие с пользователем в контексте данной роли.


### 4. Создание командной инструкции в `instructions/...`

**Описание**:  Документация для пользователя о новой роли, как использовать её.

**Действия**:
- Создать текстовую инструкцию, которая описывает, как пользователь может воспользоваться новой ролью.  Инструкция должна размещаться в директории `instructions/...`
