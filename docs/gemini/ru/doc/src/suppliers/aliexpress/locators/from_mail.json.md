# Описание файла `from_mail.json`

## Обзор

Файл `from_mail.json` содержит JSON-конфигурацию для определения локаторов элементов на странице, получаемой из почтовой рассылки AliExpress. В частности, он описывает локатор для блока товаров на странице с акциями.

## Структура JSON

### Ключ `product_blocks`

Ключ `product_blocks` содержит настройки для определения локатора блока товаров.

#### Атрибут `attribute`

**Описание**: Атрибут, по которому нужно извлекать информацию из элемента. В данном случае, значение `null`, что означает, что извлекать атрибут не нужно.

#### Атрибут `by`

**Описание**: Метод поиска элемента. В данном случае используется `XPATH`.

#### Атрибут `selector`

**Описание**: XPATH-селектор для поиска элемента.
```
//td[div[contains(@class, '-item-card') and not(contains(@class, 'item-card-text'))]]
```
Данный селектор ищет `<td>` элементы, у которых есть дочерний `<div>` с классом, содержащим `-item-card`, и не содержащим `item-card-text`.
Предположительно, это  ячейки таблицы с карточками товаров, которые  расположены на странице deals из почтовой рассылки.

#### Атрибут `if_list`

**Описание**: Определяет, как обрабатывать список элементов. Значение `first` указывает, что нужно выбирать первый найденный элемент.

#### Атрибут `use_mouse`

**Описание**: Флаг, указывающий, нужно ли использовать мышь для взаимодействия с элементом. В данном случае `false`.

#### Атрибут `mandatory`

**Описание**: Флаг, указывающий, является ли элемент обязательным. В данном случае `false`.

#### Атрибут `timeout`

**Описание**: Тайм-аут ожидания элемента. В данном случае `0`.

#### Атрибут `timeout_for_event`

**Описание**: Событие, которое нужно ожидать перед поиском элемента. В данном случае `presence_of_element_located`.

#### Атрибут `event`

**Описание**: Событие, связанное с тайм-аутом. В данном случае пустая строка `""`, что означает отсутствие события.

#### Атрибут `locator_description`

**Описание**: Описание локатора.
```
Товары со страницы deals, котору я делаю из почтовой рассылки.
```
