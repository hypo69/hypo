# Модуль для сбора данных о товарах с Gearbest
## Обзор

Модуль `graber.py` предназначен для сбора информации о товарах с сайта Gearbest. Он наследует функциональность основного класса `Graber` и переопределяет ее при необходимости для учета особенностей структуры сайта Gearbest. Модуль использует веб-драйвер для взаимодействия с сайтом и извлечения данных.

## Подробней

Этот модуль является частью системы сбора данных о товарах из различных интернет-магазинов. Он специализируется на сайте Gearbest и использует общие механизмы, реализованные в родительском классе `Graber`, для извлечения информации о товарах. Если структура страницы товара на Gearbest отличается от стандартной, в этом классе переопределяются соответствующие методы для корректной обработки данных.

## Классы

### `Graber`

**Описание**: Класс `Graber` предназначен для сбора данных о товарах с сайта Gearbest.

**Принцип работы**:
Класс наследуется от базового класса `Graber` (`Grbr`) из модуля `src.suppliers.graber`. При инициализации класса устанавливается префикс поставщика (`supplier_prefix`) равным 'etzmaleh'. Также класс инициализирует родительский класс с указанием префикса поставщика, драйвера веб-браузера и индекса языка. Кроме того, здесь устанавливаются глобальные настройки через `Context`, в частности, `Context.locator_for_decorator` устанавливается в `None`, что отключает выполнение декоратора `@close_pop_up`.

**Наследует**:
- `Graber` (`Grbr`) из модуля `src.suppliers.graber`.

**Аттрибуты**:
- `supplier_prefix` (str): Префикс поставщика, устанавливается равным `'etzmaleh'`.

**Методы**:
- `__init__(self, driver: Driver, lang_index)`: Инициализирует класс `Graber`.

### `__init__`

```python
def __init__(self, driver: Driver, lang_index):
    """Инициализация класса сбора полей товара."""
    self.supplier_prefix = 'etzmaleh'
    super().__init__(supplier_prefix=self.supplier_prefix, driver=driver, lang_index=lang_index)
    # Устанавливаем глобальные настройки через Context
    Context.locator_for_decorator = None  # <- если будет уастановлено значение - то оно выполнится в декораторе `@close_pop_up`
```

**Назначение**: Инициализирует экземпляр класса `Graber`, устанавливая префикс поставщика и вызывая конструктор родительского класса.

**Параметры**:
- `driver` (Driver): Экземпляр веб-драйвера для взаимодействия с сайтом.
- `lang_index` (int): Индекс языка, используемый при сборе данных.

**Возвращает**:
- `None`

**Как работает функция**:
1. Устанавливает атрибут `supplier_prefix` экземпляра класса равным `'etzmaleh'`.
2. Вызывает конструктор родительского класса (`Grbr`) с переданными параметрами, включая префикс поставщика, драйвер и индекс языка.
3. Устанавливает значение `Context.locator_for_decorator` в `None`. Это делается для отключения выполнения декоратора `@close_pop_up`, который, вероятно, предназначен для закрытия всплывающих окон. Если `Context.locator_for_decorator` будет установлено в какое-либо значение, этот локатор будет выполнен в декораторе.

**Примеры**:

```python
from src.webdriver.driver import Driver, Chrome
# Пример создания экземпляра класса Graber
driver = Driver(Chrome)
graber = Graber(driver=driver, lang_index=0)
```
