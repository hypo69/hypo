# Модель `gemini`

## Обзор

Данный модуль описывает модель `gemini`, предоставляя API для взаимодействия с ней.  Он включает в себя настройки для системных и командных инструкций, а также механизм для управления запросами.

## Классы

### `__init__`

**Описание**: Конструктор класса, инициализирующий модель.

**Параметры**:

- `api_key` (str): Ключ доступа к API модели.
- `model_name` (Optional[str], optional): Имя модели. По умолчанию `None`.
- `generation_config` (Optional[Dict], optional): Настройки генерации. По умолчанию `None`.
- `system_instruction` (Optional[str], optional): Системная инструкция для модели. По умолчанию `None`.
- `**kwargs`: Дополнительные параметры.

**Возвращает**:
- `None`: Конструктор не возвращает значение.


## Функции

Не найдено функций в представленном коде.


## Системные инструкции (`system_instruction`)

**Описание**: Системная инструкция, задающая общее поведение модели. Она устанавливается при инициализации и применяется ко всем запросам. Изменить ее после инициализации невозможно.

## Командные инструкции (`command_instruction`)

**Описание**: Командная инструкция, которая добавляется к системной инструкции перед каждым запросом.  Ее можно изменять для каждого отдельного запроса.

**Дополнительная информация**:

- Глобальные инструкции хранятся в папке `src/ai/prompts`.
- Инструкции для запросов, как правило, находятся в папке `instruction` у клиента.
- Клиенты прописаны в `src.endpoints`.
- Также глобальные инструкции могут храниться на стороне клиента.


**Примеры названий файлов с инструкциями**:

```
instruction_doc_writer_html_en.md
instruction_code_checker_en.md
instruction_code_checker_he.md
instruction_code_checker_ru.md
instruction_code_explainer_en.md
instruction_code_explainer_ru.md
```
