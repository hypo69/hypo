# Порядок действий для создания новой роли

## Обзор

Данный документ описывает пошаговый процесс создания новой роли в системе.  Он включает в себя добавление роли в список ролей, файлы переводов, системные промты и соответствующие инструкции.

## Действия

### Шаг 1: Файл `code_assistant.json`

**Описание**: В файле `code_assistant.json` хранится информация о доступных ролях.  Необходимо добавить новую роль в этот список.

### Шаг 2: Добавление роли в список ролей

**Описание**:  Добавьте запись о новой роли в `code_assistant.json`. Запись должна содержать уникальное имя роли и, возможно, другие характеристики.


### Шаг 3: Добавление роли в файл переводов `translations/translations.json`

**Описание**: В файле `translations/translations.json` хранятся переводы для различных элементов интерфейса.  Необходимо добавить переводы для новой роли на все поддерживаемые языки.


### Шаг 4: Создание системного промпта в `ai/prompts/develpoper`

**Описание**: Создайте новый системный промпт для новой роли в директории `ai/prompts/develpoper`. Этот промпт будет использоваться моделью для понимания и выполнения запросов от пользователя, действующего от имени этой роли.


### Шаг 5: Создание командной инструкции в `instructions/...`

**Описание**: Создайте файл с инструкцией для новой роли в директории `instructions/...`.  Этот файл будет содержать подробную информацию о том, как использовать новую роль в системе.  Например, возможные запросы и ожидаемые результаты.
