# Модуль schemas.py

## Обзор

Модуль `schemas.py` содержит Pydantic-схемы, используемые для валидации и представления данных, связанных с продуктами в цифровом магазине Telegram-бота. Он определяет структуры данных для идентификации продукта, создания и обновления информации о продукте.

## Подробней

Этот модуль предоставляет модели данных, используемые для определения структуры запросов и ответов при взаимодействии с API, управляющим продуктами в Telegram-боте. Схемы Pydantic обеспечивают проверку типов данных, минимальной длины строк и других ограничений, что помогает гарантировать корректность данных, передаваемых между клиентом и сервером.

## Классы

### `ProductIDModel`

**Описание**: Модель для представления идентификатора продукта.

**Как работает класс**:

Класс `ProductIDModel` используется для представления идентификатора продукта. Он содержит одно поле `id`, которое является целым числом. Pydantic автоматически проверяет, что переданное значение является целым числом.

**Параметры**:
- `id` (int): Идентификатор продукта.

**Примеры**:

```python
from pydantic import ValidationError

try:
    product_id = ProductIDModel(id=123)
    print(product_id)
except ValidationError as ex:
    print(ex)
```

### `ProductModel`

**Описание**: Модель для представления информации о продукте.

**Как работает класс**:

Класс `ProductModel` используется для представления информации о продукте, включая его имя, описание, цену, идентификатор категории, идентификатор файла и скрытый контент. Pydantic автоматически проверяет, что значения соответствуют указанным типам и ограничениям. Например, `name` и `description` должны иметь минимальную длину 5 символов, а `price` и `category_id` должны быть больше 0.

**Параметры**:
- `name` (str): Название продукта (минимум 5 символов).
- `description` (str): Описание продукта (минимум 5 символов).
- `price` (int): Цена продукта (должна быть больше 0).
- `category_id` (int): Идентификатор категории продукта (должен быть больше 0).
- `file_id` (str | None): Идентификатор файла продукта (может быть `None`).
- `hidden_content` (str): Скрытый контент продукта (минимум 5 символов).

**Примеры**:

```python
from pydantic import ValidationError

try:
    product = ProductModel(
        name='Test Product',
        description='This is a test product',
        price=100,
        category_id=1,
        file_id='file123',
        hidden_content='Hidden content'
    )
    print(product)
except ValidationError as ex:
    print(ex)

try:
    product = ProductModel(
        name='Test',
        description='This',
        price=-100,
        category_id=-1,
        file_id='file123',
        hidden_content='Hide'
    )
except ValidationError as ex:
    print(ex)
```
