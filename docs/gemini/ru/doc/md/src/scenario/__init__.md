# Модуль `hypotez/src/scenario/__init__.py`

## Обзор

Данный модуль предоставляет функции для выполнения сценариев, предназначенных для поставщиков. Он реализует логику взаимодействия между сценариями, исполнителем и поставщиком данных (например, PrestaShop).  Модуль включает функции для работы с отдельными сценариями (`run_scenario`), сценариями из файлов (`run_scenario_files`, `run_scenario_file`), а также для работы с списком сценариев (`run_scenarios`).

## Функции

### `run_scenario_files`

**Описание**: Выполняет сценарии, заданные в переданных файлах.

**Параметры**:
- `supplier` (объект): Объект поставщика, для которого выполняются сценарии.
- `scenario_files` (список строк): Список путей к файлам со сценариями.

**Возвращает**:
- `None`: Функция не возвращает значения.

**Вызывает исключения**:
- `TypeError`: Если тип параметра `scenario_files` не соответствует ожидаемому.
- `FileNotFoundError`: Если один или более файлов в `scenario_files` не найдены.
- `ValueError`: Если в файлах сценариев обнаружены ошибки валидации.
- `OtherExceptions`: Другие возможные исключения при выполнении сценариев.


### `run_scenarios`

**Описание**: Выполняет сценарии, переданные в виде списка словарей.

**Параметры**:
- `supplier` (объект): Объект поставщика, для которого выполняются сценарии.
- `scenarios` (список словарей): Список словарей, содержащих данные сценариев.

**Возвращает**:
- `None`: Функция не возвращает значения.

**Вызывает исключения**:
- `TypeError`: Если тип параметра `scenarios` не соответствует ожидаемому.
- `ValueError`: Если в словарях сценариев обнаружены ошибки валидации.
- `OtherExceptions`: Другие возможные исключения при выполнении сценариев.


### `run_scenario_file`

**Описание**: Выполняет сценарий, заданный в переданном файле.

**Параметры**:
- `supplier` (объект): Объект поставщика, для которого выполняются сценарии.
- `scenario_file` (строка): Путь к файлу со сценарием.

**Возвращает**:
- `None`: Функция не возвращает значения.

**Вызывает исключения**:
- `TypeError`: Если тип параметра `scenario_file` не соответствует ожидаемому.
- `FileNotFoundError`: Если файл не найден.
- `ValueError`: Если в файле сценария обнаружены ошибки валидации.
- `OtherExceptions`: Другие возможные исключения при выполнении сценариев.


### `run_scenario`

**Описание**: Выполняет один сценарий, переданный в виде словаря.

**Параметры**:
- `supplier` (объект): Объект поставщика, для которого выполняются сценарии.
- `scenario` (словарь): Словарь, содержащий данные сценария.

**Возвращает**:
- `None`: Функция не возвращает значения.

**Вызывает исключения**:
- `TypeError`: Если тип параметра `scenario` не соответствует ожидаемому.
- `ValueError`: Если в словаре сценария обнаружены ошибки валидации.
- `OtherExceptions`: Другие возможные исключения при выполнении сценариев.


### `execute_PrestaShop_insert`

**Описание**: Выполняет вставку данных в базу данных PrestaShop.

**Параметры**:
- `data` (данные): Данные для вставки.

**Возвращает**:
- `None`: Функция не возвращает значения.

**Вызывает исключения**:
- `PrestaShopError`: При возникновении ошибок при взаимодействии с базой данных PrestaShop.
- `OtherExceptions`: Другие возможные исключения.


### `execute_PrestaShop_insert_async`

**Описание**: Выполняет асинхронную вставку данных в базу данных PrestaShop.

**Вызывает исключения**:
- `PrestaShopError`: При возникновении ошибок при асинхронном взаимодействии с базой данных PrestaShop.
- `OtherExceptions`: Другие возможные исключения.
