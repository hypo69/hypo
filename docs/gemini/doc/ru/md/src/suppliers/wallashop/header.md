# Модуль hypotez/src/suppliers/wallashop/header.py

## Обзор

Этот модуль содержит функции для определения корневой директории проекта и загрузки настроек из файла `settings.json`. Он также предоставляет доступ к информации о проекте, такой как имя, версия, описание и контактные данные.

## Функции

### `set_project_root`

**Описание**: Определяет корневую директорию проекта, начиная с текущей директории и идя вверх по дереву каталогов. Поиск останавливается на первой директории, содержащей файлы или папки, указанные в списке `marker_files`.

**Параметры**:

- `marker_files` (tuple): Кортеж строк, представляющих имена файлов или директорий, по которым определяется корневая директория проекта. По умолчанию: `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:

- `Path`: Объект `Path` представляющий путь к корневой директории проекта, если она найдена. В противном случае, возвращается путь к директории, в которой расположен текущий скрипт.

**Вызывает исключения**:

- Нет.


## Переменные

### `__root__`

**Описание**: Путь к корневой директории проекта.


### `settings`

**Описание**: Словарь, содержащий настройки проекта, загруженные из файла `settings.json`.


### `doc_str`

**Описание**: Строка, содержащая содержимое файла `README.MD`.


### `__project_name__`

**Описание**: Имя проекта. Получает значение из `settings`, по умолчанию "hypotez".

### `__version__`

**Описание**: Версия проекта. Получает значение из `settings`, по умолчанию "".

### `__doc__`

**Описание**: Описание проекта. Получает значение из `doc_str`, по умолчанию "".

### `__details__`

**Описание**: Дополнительные детали о проекте. По умолчанию "".

### `__author__`

**Описание**: Автор проекта. Получает значение из `settings`, по умолчанию "".

### `__copyright__`

**Описание**: Авторские права. Получает значение из `settings`, по умолчанию "".

### `__cofee__`

**Описание**: Ссылка на спонсорскую платформу для поддержки разработчика. Получает значение из `settings`, по умолчанию ссылка на boosty.to.


## Обработка исключений

В модуле используются блоки `try...except` для обработки потенциальных ошибок:

- `FileNotFoundError`: Возникает, если файл `settings.json` или `README.MD` не найден.
- `json.JSONDecodeError`: Возникает, если содержимое файла `settings.json` не является корректным JSON.

В блоках `except` используется `...` для обозначения того, что код не должен предпринимать никаких действий при возникновении указанных ошибок.
