# Модуль вопросов

## Обзор

Этот модуль содержит вопросы, ответы на которые могут быть полезны для понимания логики и функциональности других модулей.  В данном случае, рассматриваются вопросы о наименовании конфигурационных файлов.

## Вопросы

### `Почему конфигурационные файлы называются по имени модуля (например: suppliers.json)?`

**Описание**: Вопрос о логике именования конфигурационных файлов.

**Ответ**:

Имена файлов `suppliers.json`, `products.json` и т.д. отражают связь конфигурационного файла с конкретным модулем.  Это делает информацию более структурированной и облегчает поиск нужных данных для моделей машинного обучения, которые работают с данными.

### `Не лучше ли было их именовать config.json?`

**Описание**: Вопрос о возможной альтернативе именованию конфигурационных файлов.

**Ответ**:

Использование имен файлов `config.json` для всех конфигурационных данных, хотя и упрощает структуру файлов, не дает информации о том, какой именно раздел конфигурации хранится внутри.  Например, `suppliers.json` чётко указывает, что данный файл содержит конфигурацию для поставщиков.
