# Модуль hypotez/src/endpoints/hypo69/header.py

## Обзор

Данный модуль содержит функции для определения корневой директории проекта, загрузки настроек из файла `settings.json` и получения информации о проекте (имя, версия, описание, автор и т.д.).  Он также обрабатывает потенциальные ошибки при чтении файлов настроек и документации.

## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная с директории текущего файла. Поиск происходит вверх по директориям, пока не будет найдена директория, содержащая указанные файлы-маркеры (например, `pyproject.toml`, `requirements.txt`, `.git`).

**Параметры**:

- `marker_files` (tuple): Кортеж строк, представляющих имена файлов или директорий, которые используются для определения корневой директории проекта. По умолчанию - (`'pyproject.toml'`, `'requirements.txt'`, `.git`).


**Возвращает**:

- `Path`: Путь к корневой директории проекта, если найдена. В противном случае возвращает путь к директории, где расположен текущий скрипт.

**Пример использования**:

```python
root_path = set_project_root()
print(root_path)
```

**Обрабатываемые исключения**:

- Нет обработанных исключений.


###  (Загрузка настроек и документации)

**Описание**: Загружает настройки из файла `settings.json` и документацию из файла `README.MD` в корневой директории проекта. Обрабатывает возможные ошибки (например, если файл не найден или содержит некорректный JSON).


**Параметры**:

- Нет входных параметров.

**Возвращает**:

- `None`: Возвращает None, так как функции загружают данные глобально.


**Обрабатываемые исключения**:

- `FileNotFoundError`: Если файл настроек `settings.json` или файл документации `README.MD` не найдены.
- `json.JSONDecodeError`: Если файл `settings.json` содержит некорректный JSON.

## Глобальные переменные

### `__root__`

**Описание**: Переменная, содержащая путь к корневой директории проекта. Инициализируется вызовом функции `set_project_root()`. Тип - `Path`.


### `settings`

**Описание**: Словарь с настройками проекта, загруженный из файла `settings.json`. Тип - `dict`. Возможные значения - `None`.

### `doc_str`

**Описание**: Строка с текстовой документацией проекта из файла `README.MD`. Тип - `str`. Возможные значения - `None`.


### `__project_name__`, `__version__`, `__doc__`, `__details__`, `__author__`, `__copyright__`, `__cofee__`


**Описание**:  Глобальные переменные, содержащие соответствующие данные из файла настроек `settings.json`. Имеют тип `str`. Если ключ не найден, или `settings` равен `None`, значение устанавливается по умолчанию.
