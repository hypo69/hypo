# Анализ кода модуля `header.py`

- **Соответствие стандартам**: 7
- **Плюсы**:
    - Код выполняет свою основную функцию - определение корневой директории проекта.
    - Используется `pathlib` для работы с путями, что является хорошей практикой.
    - Присутствует docstring для функции `set_project_root`, хотя и нуждается в доработке.

    - Используется стандартный `json`, вместо `j_loads` или `j_loads_ns` из `src.utils.jjson`.
    - docstring не соответствует стандарту RST.
    - Не хватает подробных комментариев к коду.
    - Не хватает выравнивания импортов.
    - Не все переменные имеют аннотации типов.
    - Не обработана ситуация когда не находит корневую директорию.
    - Не все переменные и константы имеют аннотацию типов.

## Оптимизированный код:

```python
# -*- coding: utf-8 -*-

#! venv/bin/python/python3.12
"""
Модуль для определения корневой директории проекта.
===================================================

Этот модуль содержит функцию :func:`set_project_root`, которая используется для определения корневой директории проекта.
Функция ищет маркерные файлы в родительских директориях и возвращает путь к корневой директории.

Пример использования:
----------------------
.. code-block:: python

    from pathlib import Path
    from src.templates.header import set_project_root

    root_dir = set_project_root()
    print(f"Корневая директория проекта: {root_dir}")
"""
import sys
from pathlib import Path
from typing import Tuple
from packaging.version import Version # сохраняем импорт
from src.logger import logger # исправляем импорт
from src import gs # сохраняем импорт


def set_project_root(marker_files: Tuple[str, ...] = ('__root__', '.git')) -> Path:
    """
    Ищет корневую директорию проекта, начиная с директории текущего файла.
    
    Поиск ведется вверх по дереву директорий до тех пор, пока не будет найдена директория,
    содержащая один из маркерных файлов.

    :param marker_files: Кортеж имен файлов или директорий, которые используются для определения корневой директории.
    :type marker_files: Tuple[str, ...], optional
    :return: Путь к корневой директории проекта.
    :rtype: Path
    :raises FileNotFoundError: Если корневая директория не найдена.

    Пример:
        >>> from pathlib import Path
        >>> root_dir = set_project_root(marker_files=('__root__', '.git'))
        >>> print(root_dir)
        ...
    """
    current_path: Path = Path(__file__).resolve().parent # получаем абсолютный путь к директории текущего файла
    
    for parent in [current_path] + list(current_path.parents): # проходим по текущей директории и всем ее родительским
        if any((parent / marker).exists() for marker in marker_files): # если существует хотя бы один маркерный файл
            if parent not in sys.path: # если путь еще не добавлен в sys.path
                sys.path.insert(0, str(parent)) # добавляем путь в начало sys.path
            return parent # возвращаем найденную корневую директорию
    
    logger.error(f"Не удалось найти корневую директорию, маркеры: {marker_files} в: {current_path}")# логгируем ошибку если не удалось найти корневую директорию
    raise FileNotFoundError(f"Не удалось найти корневую директорию, маркеры: {marker_files} в: {current_path}")# выбрасываем исключение если не удалось найти корневую директорию


# получаем корневую директорию проекта
__root__: Path = set_project_root()
"""__root__ (Path): Путь к корневой директории проекта."""
```
