# Модуль gs.py

## Обзор

Модуль `gs.py` предназначен для загрузки параметров конфигурации приложения. Он использует функцию `j_loads_ns` для загрузки данных из файла `config.json`, расположенного в директории `src`.

## Подробней

Основная задача этого модуля - предоставить глобальный доступ к настройкам приложения, загруженным из файла конфигурации. Это особенно полезно, когда необходимо избежать жестко закодированных значений и обеспечить гибкость конфигурации. Функция `j_loads_ns` используется для загрузки JSON-файла и преобразования его в пространство имен, что позволяет обращаться к параметрам конфигурации как к атрибутам объекта.

## Функции

### `j_loads_ns`

```python
def j_loads_ns(file_path: str | Path) -> dict:
    """
    Загружает JSON-файл и преобразует его в пространство имен.

    Args:
        file_path (str | Path): Путь к JSON-файлу.

    Returns:
        dict: Словарь, содержащий загруженные данные.

    Raises:
        FileNotFoundError: Если файл не найден.
        JSONDecodeError: Если файл содержит некорректный JSON.

    Example:
        >>> config = j_loads_ns('config.json')
        >>> print(config['ключ'])
        'значение'

     **Как работает функция**:
    1. Принимает путь к файлу.
    2. Открывает файл по указанному пути.
    3. Загружает JSON-данные из файла.
    4. Возвращает словарь с загруженными данными.
    """
    ...
```

**Описание**: Загружает JSON-файл и преобразует его содержимое в словарь.

**Параметры**:
- `file_path` (str | Path): Путь к JSON-файлу.

**Возвращает**:
- `dict`: Словарь, содержащий загруженные данные из JSON-файла.

**Вызывает исключения**:
- `FileNotFoundError`: Если файл не найден.
- `JSONDecodeError`: Если файл содержит некорректный JSON.

**Примеры**:
```python
config = j_loads_ns('config.json')
print(config['ключ'])
```

## Переменные

### `gs`

```python
gs = j_loads_ns(__root__ / 'src' / 'config.json')
"""Загружаю конфигурацию из файла.
Применяется тогда, когда нет файла credentials.kdbx
"""
```

**Описание**: Глобальная переменная, содержащая конфигурацию, загруженную из файла `config.json`.

**Как работает**: Переменная `gs` инициализируется путем вызова функции `j_loads_ns` с указанием пути к файлу конфигурации. Это позволяет получить доступ к параметрам конфигурации в любой части приложения.
