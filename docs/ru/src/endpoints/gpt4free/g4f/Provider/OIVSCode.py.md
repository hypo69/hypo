# Документация модуля `OIVSCode`

## Обзор

Модуль `OIVSCode` предоставляет класс `OIVSCode`, который является шаблоном для взаимодействия с сервером OI VSCode. Он наследует функциональность от класса `OpenaiTemplate` и определяет специфические параметры, такие как URL, базовый URL API, поддерживаемые модели и их псевдонимы.

## Подробней

Этот модуль используется для настройки и интеграции с сервером OI VSCode, предоставляя удобный интерфейс для взаимодействия с различными моделями, поддерживаемыми этим сервером. Он определяет, какие модели поддерживаются, требует ли сервер аутентификацию, поддерживает ли потоковую передачу и историю сообщений.

## Классы

### `OIVSCode`

**Описание**: Класс `OIVSCode` предоставляет конфигурацию для взаимодействия с сервером OI VSCode.

**Наследует**:

- `OpenaiTemplate`: Класс `OIVSCode` наследует атрибуты и методы от класса `OpenaiTemplate`, который, вероятно, предоставляет общую логику для взаимодействия с API OpenAI-подобных сервисов.

**Атрибуты**:

- `label` (str): Метка для данного провайдера, `"OI VSCode Server"`.
- `url` (str): URL сервера OI VSCode, `"https://oi-vscode-server.onrender.com"`.
- `api_base` (str): Базовый URL API сервера OI VSCode, `"https://oi-vscode-server-2.onrender.com/v1"`.
- `working` (bool): Указывает, работает ли данный провайдер, `True`.
- `needs_auth` (bool): Указывает, требуется ли аутентификация для данного провайдера, `False`.
- `supports_stream` (bool): Указывает, поддерживает ли провайдер потоковую передачу, `True`.
- `supports_system_message` (bool): Указывает, поддерживает ли провайдер системные сообщения, `True`.
- `supports_message_history` (bool): Указывает, поддерживает ли провайдер историю сообщений, `True`.
- `default_model` (str): Модель, используемая по умолчанию, `"gpt-4o-mini-2024-07-18"`.
- `default_vision_model` (str): Модель для работы с изображениями, используемая по умолчанию, совпадает с `default_model`.
- `vision_models` (list[str]): Список моделей, поддерживающих работу с изображениями, `[default_model, "gpt-4o-mini"]`.
- `models` (list[str]): Список всех поддерживаемых моделей, включая модели для работы с изображениями и другие модели, `vision_models + ["deepseek-ai/DeepSeek-V3"]`.
- `model_aliases` (dict[str, str]): Словарь псевдонимов моделей, позволяющий использовать короткие имена для обращения к моделям, `{"gpt-4o-mini": "gpt-4o-mini-2024-07-18", "deepseek-v3": "deepseek-ai/DeepSeek-V3"}`.

## Функции

В данном модуле нет отдельных функций, только атрибуты класса `OIVSCode`.
