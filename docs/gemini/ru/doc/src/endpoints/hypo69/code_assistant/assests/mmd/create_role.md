# Порядок действий для создания новой роли

## Обзор

Данный документ описывает шаги, необходимые для создания новой роли в системе.  Процесс включает добавление записи о роли в конфигурационный файл, файлы переводов, создание системного промпта и командной инструкции.

## Действия

### Шаг 1: Файл `code_assistant.json`

**Описание**:  Этот JSON-файл содержит список доступных ролей.  В нем необходимо добавить запись о новой роли.

**Пример записи**:

```json
{
  "role_name": "НовоеИмяРоли",
  "description": "Описание новой роли",
  "permissions": [
    "permission1",
    "permission2"
  ]
}
```

### Шаг 2: Файл переводов `translations/translations.json`

**Описание**:  Файл для локализации. Необходимо добавить перевод названия новой роли на все поддерживаемые языки.

**Пример добавления перевода**:

```json
{
  "role_name": "НоваяРоль": {
    "ru": "НовоеИмяРоли",
    "en": "NewRoleName"
  }
}
```


### Шаг 3: Системный промпт в `ai/prompts/develpoper`

**Описание**:  Создание системного промпту (инструкций) для модели, определяющего поведение агента с новой ролью.

**Пример**:

```
В качестве <НоваяРоль> вы... (Здесь добавляется описание ожидаемого поведения агента)
```

### Шаг 4: Командная инструкция в `instructions/...`

**Описание**:  Создание подробной инструкции для разработчиков о том, как использовать новую роль.

**Пример**:

```markdown
# Использование новой роли <НовоеИмяРоли>

Эта инструкция описывает использование новой роли <НовоеИмяРоли> в системе.

## Описание роли

<Здесь описывается предназначение и функциональность новой роли>

## Примеры использования

<Примеры использования новой роли>

## Подробная информация

<Ссылки на дополнительные ресурсы>
```
