# Модуль check_release

## Обзор

Модуль `check_release` предназначен для получения версии последнего релиза репозитория на GitHub.  Он использует API GitHub для получения информации о релизе и возвращает имя тега последнего релиза.

## Функции

### `check_latest_release`

**Описание**: Функция `check_latest_release` запрашивает информацию о последнем релизе репозитория на GitHub и возвращает имя тега последнего релиза.

**Параметры**:

- `owner` (str): Имя владельца репозитория на GitHub.
- `repo` (str): Имя репозитория на GitHub.

**Возвращает**:

- `str`: Имя тега последнего релиза, если он доступен. Возвращает `None`, если запрос не удался.

**Вызывает исключения**:

- Возможны исключения, связанные с запросом к API GitHub (например, `requests.exceptions.RequestException`).  Эти исключения не обрабатываются явно в текущем коде, но в случае ошибки функция возвращает `None`.


## Константы

### `MODE`

**Описание**:  Переменная `MODE` содержит значение `'dev'`.  Предположительно, используется для определения режима работы (например, 'dev', 'prod'), но в данной реализации это не используется.

## Использование

Для использования модуля необходимо импортировать его:

```python
from hypotez.src.check_release import check_latest_release
```

Затем вызвать функцию `check_latest_release`, передав имя владельца и имя репозитория:

```python
latest_release = check_latest_release('owner_name', 'repo_name')
if latest_release:
    print(f"Последняя версия релиза: {latest_release}")
else:
    print("Не удалось получить информацию о последнем релизе.")
```

**Обработка ошибок:**

Обратите внимание, что текущая реализация не содержит полной обработки ошибок.  При проблемах с запросом к API GitHub функция возвращает `None`. Необходимо добавить обработку исключений `requests.exceptions.RequestException` для создания более надежного кода.
