# Инструкция по написанию документации для кода

## Обзор

Данная инструкция описывает формат и структуру документации для Python-кода, создаваемой в формате Markdown. Документация должна быть подробной и обеспечивать полное понимание функциональности кода.

## Структура документации

Каждый файл документации должен содержать:

* **Заголовок**:  Краткое описание файла.
* **Оглавление**: Список ссылок на разделы документации.
* **Раздел "Описание"**: Краткое описание функциональности кода.
* **Раздел "Шаги выполнения"**: Подробное описание шагов, выполняемых кодом (если применимо).
* **Раздел "Примеры использования"**: Примеры использования кода.
* **Раздел "Функции"**:  Документация функций в формате, описанном в инструкции.
* **Раздел "Классы"**:  Документация классов в формате, описанном в инструкции.


## Формат комментариев для функций и методов (Python)

Используйте следующий шаблон для создания документации к функциям и методам:

```python
def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:
    """
    Args:
        param (str): Описание параметра `param`.
        param1 (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию `None`.

    Returns:
        dict | None: Описание возвращаемого значения. Возвращает словарь или `None`.

    Raises:
        ValueError: Подробное описание ситуации, в которой возникает исключение `ValueError`.
    """
    # Тело функции
    pass
```

##  Правила оформления Markdown

* Используйте заголовки разного уровня (#, ##, ###, ####).
* Используйте списки для структурирования информации.
* Используйте отступы и форматирование для улучшения читаемости.
* При использовании `code-block` указывайте язык (`python`, `javascript`, etc.)

## Функции

### `process_data`

**Описание**: Функция обрабатывает входные данные и возвращает результат.

**Параметры**:
* `data` (list): Список входных данных.
* `filter_criteria` (str, optional): Критерии фильтрации данных. По умолчанию `None`.

**Возвращает**:
* `list`: Обработанный список данных.

**Вызывает исключения**:
* `ValueError`: Если входные данные имеют неправильный формат.


## Примеры использования

```python
# Пример использования функции process_data
data = [1, 2, 3, 4, 5]
result = process_data(data, filter_criteria="even")
print(result)  # Должно вывести [2, 4]
```
