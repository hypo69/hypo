## Введение

Класс `ExecuteLocator` предназначен для работы с веб-элементами через Selenium WebDriver. Он предоставляет методы для взаимодействия с веб-страницей, например, для получения атрибутов элементов и отправки сообщений.  Это руководство расскажет, как настроить окружение, написать тесты для `ExecuteLocator` и запустить их.

## Подготовка окружения

### Установка зависимостей

Убедитесь, что у вас установлены необходимые библиотеки:

```bash
pip install -r requirements.txt
```

Файл `requirements.txt` должен содержать:

```
pytest==7.4.0
selenium==4.16.1
```

### Настройка WebDriver

Установите WebDriver для вашего браузера (например, ChromeDriver для Chrome).


## Написание тестов

## Запуск тестов

Для запуска тестов используйте команду:

```bash
pytest tests/test_executor.py
```

## Проверка результатов тестирования

`pytest` выведет результаты в терминале.  Убедитесь, что все тесты прошли успешно. В случае ошибок, `pytest` укажет подробности.

## Обновление тестов

При изменениях в коде `ExecuteLocator` обновляйте тесты, чтобы они проверяли новые функции.

При добавлении или изменении тестов обновите соответствующую документацию, чтобы другие разработчики и тестировщики понимали, как тестируются методы `ExecuteLocator`.

## Пример документации для тестов

```markdown
### Описание тестов

- **Тестирование метода `get_webelement_by_locator`:** Проверяет случаи, когда найден один элемент, несколько элементов и когда элемент не найден.

- **Тестирование метода `get_attribute_by_locator`:** Проверяет получение атрибута у элемента.

- **Тестирование метода `send_message`:** Проверяет отправку сообщения элементу и работу с задержкой между символами.

### Используемые библиотеки

- `pytest`: для написания и запуска тестов.
- `unittest.mock`: для создания мок-объектов и имитации поведения WebDriver.
- `selenium`: для работы с веб-драйвером.

### Как запустить тесты

```bash
pytest tests/test_executor.py
```

### Как читать результаты тестов

- **Passed**: Тест успешно пройден.
- **Failed**: Тест не пройден.  Проверьте вывод `pytest` для деталей ошибки.
