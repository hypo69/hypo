# Модуль `hypotez/src/ai/gemini/header.py`

## Обзор

Данный модуль содержит функции для определения корневого пути к проекту `hypotez`, а также для загрузки настроек из файла `settings.json` и документации из `README.MD`.

## Функции

### `get_project_root`

**Описание**: Находит корневую директорию проекта, начиная с текущей директории файла, и восходя по дереву директорий, пока не найдёт директорию, содержащую файлы-маркеры (`pyproject.toml`, `requirements.txt`, `.git`).

**Параметры**:

- `marker_files` (tuple): Список файлов или директорий, которые используются для определения корневой директории проекта. По умолчанию `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:

- `Path`: Путь к корневой директории проекта, если найдена, в противном случае текущая директория.


**Вызывает исключения**:

- Нет


### `get_settings`

**Описание**: Загружает настройки из файла `settings.json`.

**Возвращает**:
- `dict`: Словарь со значениями настроек из файла. Возвращает `None`, если файл не найден или содержит невалидные данные.

**Вызывает исключения**:
- `FileNotFoundError`: Если файл `settings.json` не найден.
- `json.JSONDecodeError`: Если файл `settings.json` не содержит валидный JSON.


###  `get_doc_string`

**Описание**: Загружает строку документации из файла `README.MD`.


**Возвращает**:
- `str`: Строка документации. Возвращает `None` если файл не найден или содержит невалидные данные.

**Вызывает исключения**:
- `FileNotFoundError`: Если файл `README.MD` не найден.
- `json.JSONDecodeError`: Если файл `README.MD` содержит невалидный JSON.

## Переменные

### `__root__`

**Описание**: Путь к корневой директории проекта.
- `Path`: Тип данных `Path`.

### `settings`

**Описание**: Словарь настроек проекта, загруженный из файла `settings.json`.
- `dict`: Тип данных `dict`. Может быть `None`.

### `doc_str`

**Описание**: Строка документации проекта, загруженная из файла `README.MD`.
- `str`: Тип данных `str`. Может быть `None`.

### `__project_name__`

**Описание**: Название проекта.
- `str`: Тип данных `str`. Значение берется из файла `settings.json`, по умолчанию `hypotez`.

### `__version__`

**Описание**: Версия проекта.
- `str`: Тип данных `str`. Значение берется из файла `settings.json`, по умолчанию `''`.

### `__doc__`

**Описание**: Текст документации проекта.
- `str`: Тип данных `str`. Значение берется из файла `README.MD`, по умолчанию `''`.

### `__details__`

**Описание**: Дополнительные детали о проекте.
- `str`: Тип данных `str`. Значение берется из файла `settings.json`, по умолчанию `''`.

### `__author__`

**Описание**: Автор проекта.
- `str`: Тип данных `str`. Значение берется из файла `settings.json`, по умолчанию `''`.

### `__copyright__`

**Описание**: Авторские права на проект.
- `str`: Тип данных `str`. Значение берется из файла `settings.json`, по умолчанию `''`.

### `__cofee__`

**Описание**: Ссылка на способ пожертвования разработчику.
- `str`: Тип данных `str`. Значение берется из файла `settings.json`, по умолчанию ссылка на Boosty.
