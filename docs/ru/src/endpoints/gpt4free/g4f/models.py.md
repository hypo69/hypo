# Модуль `models`

## Обзор

Модуль `models` содержит определения классов и структур данных для представления различных моделей машинного обучения, используемых в проекте `hypotez`. Он включает в себя классы для текстовых, аудио и визуальных моделей, а также утилиты для управления и доступа к этим моделям. Модуль предоставляет удобный способ организации и использования различных AI-моделей.

## Подробнее

Этот модуль является ключевым компонентом системы, поскольку он определяет модели, доступные для использования через API. Он включает классы `Model`, `ImageModel`, `AudioModel` и `VisionModel`, а также утилиту `ModelUtils` для преобразования строковых идентификаторов в экземпляры моделей. Модуль также содержит определения моделей по умолчанию и списки моделей, предоставляемых различными провайдерами, такими как OpenAI, Meta, Google и другими.

## Классы

### `Model`

**Описание**:
Базовый класс для представления конфигурации модели машинного обучения.

**Принцип работы**:
Класс `Model` используется для хранения информации о конкретной модели машинного обучения, включая её имя, базового провайдера и предпочтительного провайдера. Этот класс служит основой для других классов моделей, таких как `ImageModel`, `AudioModel` и `VisionModel`.

**Методы**:

- `__all__() -> list[str]`:
    - **Назначение**: Возвращает список всех имен моделей.
    - **Параметры**: Нет.
    - **Возвращает**: `list[str]` - список строк, содержащих имена всех моделей.
    - **Как работает функция**: Функция `__all__()` является статическим методом, который возвращает список всех доступных моделей, хранящихся в переменной `_all_models`. Этот метод предоставляет удобный способ получения списка всех моделей, определенных в модуле.

### `ImageModel(Model)`

**Описание**:
Класс для представления конфигурации модели обработки изображений.

### `AudioModel(Model)`

**Описание**:
Класс для представления конфигурации модели обработки аудио.

### `VisionModel(Model)`

**Описание**:
Класс для представления конфигурации модели обработки визуальных данных.

### `ModelUtils`

**Описание**:
Утилитный класс для сопоставления строковых идентификаторов с экземплярами моделей.

**Принцип работы**:
Класс `ModelUtils` содержит словарь `convert`, который отображает строковые идентификаторы моделей на соответствующие экземпляры класса `Model`. Это позволяет легко получать доступ к моделям по их именам.

**Атрибуты**:

- `convert (dict[str, Model])`:
    - **Описание**: Словарь, сопоставляющий строковые идентификаторы моделей с экземплярами класса `Model`.
    - **Назначение**: Обеспечивает быстрый доступ к моделям по их именам.

## Функции

### `default`

**Описание**:
Определяет модель по умолчанию.

### `default_vision`

**Описание**:
Определяет модель обработки визуальных данных по умолчанию.
Определяет модель для работы с визуальными данными по умолчанию, используя указанный список провайдеров.
Внутри происходит следующее:
- Инициализируется объект `Model` с именем и базовым провайдером, установленными в пустые строки.
- В качестве `best_provider` устанавливается объект `IterListProvider`, который получает список провайдеров: `Blackbox`, `OIVSCode`, `TypeGPT`, `DeepInfraChat`, `PollinationsAI`, `Dynaspark`, `HuggingSpace`, `GeminiPro`, `HuggingFaceAPI`, `CopilotAccount`, `OpenaiAccount`, `Gemini`.
- Параметр `shuffle` для `IterListProvider` установлен в `False`, что означает, что провайдеры будут использоваться в том порядке, в котором они указаны.
- Созданный объект `Model` присваивается переменной `default_vision`.

### `demo_models`

**Описание**:
Словарь, содержащий демонстрационные модели и их провайдеров.
Содержит словарь с демонстрационными моделями и их провайдерами.
Внутри происходят следующие действия:
- Определяется словарь `demo_models`, который содержит пары ключ-значение, где ключом является имя модели (например, `llama_3_2_11b.name`), а значением — список, содержащий саму модель и список провайдеров, связанных с этой моделью.
- Например, для модели `llama_3_2_11b` указывается список `[HuggingChat]` в качестве провайдера.
- Аналогично, для каждой модели указывается список провайдеров, которые могут быть использованы для этой модели.
