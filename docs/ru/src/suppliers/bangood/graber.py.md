# Модуль `src.suppliers.bangood.graber`

## Обзор

Модуль `src.suppliers.bangood.graber` предназначен для сбора данных о товарах с сайта `bangood.com`. Он содержит класс `Graber`, который наследует функциональность от родительского класса `Graber` (из модуля `src.suppliers.graber`). Основная задача класса `Graber` - извлечение и обработка различных полей со страницы товара.

## Подробней

Этот модуль является частью системы, предназначенной для сбора информации о товарах из различных онлайн-магазинов. Он специализируется на работе с сайтом `bangood.com`.
Модуль использует веб-драйвер для взаимодействия с веб-страницами и извлечения данных. Для обработки полей товаров, требующих нестандартного подхода, класс `Graber` переопределяет функции обработки полей, определенные в родительском классе.

## Классы

### `Graber`

**Описание**: Класс `Graber` предназначен для захвата данных о товарах с сайта `bangood.com`. Он наследует функциональность от класса `Graber` из модуля `src.suppliers.graber` и специализируется на обработке данных именно с этого сайта.

**Как работает класс**:
1.  Класс инициализируется с использованием веб-драйвера и индекса языка.
2.  Устанавливает префикс поставщика (`supplier_prefix`) как `'bangood'`. Этот префикс используется для идентификации поставщика в системе.
3.  Вызывает конструктор родительского класса `Graber` (`super().__init__(...)`) для инициализации общих параметров.
4.  Использует `Context.locator_for_decorator`, чтобы установить значение для декоратора `@close_pop_up`, если необходимо выполнить предварительные действия перед отправкой запроса к веб-драйверу.

**Методы**:

*   `__init__`: Инициализация класса `Graber`.

    **Параметры**:

    *   `driver` (Driver): Экземпляр веб-драйвера для взаимодействия с сайтом.
    *   `lang_index` (int): Индекс языка, используемый на сайте.

    **Как работает метод**:

    1.  Устанавливает атрибут `supplier_prefix` равным `'bangood'`.
    2.  Вызывает конструктор родительского класса `Graber` с соответствующими параметрами.
    3.  Устанавливает `Context.locator_for_decorator` в `None`.

```python
    def __init__(self, driver: Driver, lang_index:int):
        """Инициализация класса сбора полей товара."""
        self.supplier_prefix = 'bangood'
        super().__init__(supplier_prefix=self.supplier_prefix, driver=driver, lang_index=lang_index)
        # Устанавливаем глобальные настройки через Context
        
        Context.locator_for_decorator = None # <- если будет уастановлено значение - то оно выполнится в декораторе `@close_pop_up`
