# Модуль OIVSCode

## Обзор

Модуль `OIVSCode` представляет собой класс, который наследует от класса `OpenaiTemplate` и предназначен для взаимодействия с сервером OI VSCode. Он определяет параметры подключения и настройки для использования моделей OpenAI и DeepSeek-V3 через этот сервер. Модуль содержит информацию о поддерживаемых моделях, их псевдонимах и возможностях, таких как потоковая передача и поддержка системных сообщений.

## Подробней

Этот модуль используется для настройки и интеграции с сервером OI VSCode, позволяя использовать различные модели OpenAI и DeepSeek-V3 для обработки запросов. Он определяет URL-адреса API, поддерживаемые модели и их псевдонимы, а также указывает, какие функции, такие как потоковая передача и системные сообщения, поддерживаются. Это позволяет упростить процесс настройки и использования этих моделей через сервер OI VSCode.

## Классы

### `OIVSCode`

**Описание**: Класс `OIVSCode` наследует от `OpenaiTemplate` и предоставляет конфигурацию для использования сервера OI VSCode с моделями OpenAI и DeepSeek-V3.

**Принцип работы**:
1.  **Наследование**: Класс наследуется от `OpenaiTemplate`, что позволяет повторно использовать общую логику и интерфейсы.
2.  **Конфигурация**: Определяет URL-адреса API, поддерживает потоковую передачу, системные сообщения и историю сообщений.
3.  **Модели**: Указывает поддерживаемые модели и их псевдонимы.

**Методы**:
-   `__init__`: Конструктор класса, который инициализирует параметры подключения и настройки.

**Параметры**:
-   `label` (str): Метка для сервера OI VSCode.
-   `url` (str): URL-адрес сервера OI VSCode.
-   `api_base` (str): Базовый URL-адрес API сервера OI VSCode.
-   `working` (bool): Указывает, работает ли сервер.
-   `needs_auth` (bool): Указывает, требуется ли аутентификация.
-   `supports_stream` (bool): Указывает, поддерживает ли сервер потоковую передачу.
-   `supports_system_message` (bool): Указывает, поддерживает ли сервер системные сообщения.
-   `supports_message_history` (bool): Указывает, поддерживает ли сервер историю сообщений.
-   `default_model` (str): Модель, используемая по умолчанию.
-   `default_vision_model` (str): Модель для работы с изображениями, используемая по умолчанию.
-   `vision_models` (list[str]): Список моделей для работы с изображениями.
-   `models` (list[str]): Список всех поддерживаемых моделей.
-   `model_aliases` (dict[str, str]): Словарь псевдонимов моделей.

## Примеры

### Пример создания экземпляра класса `OIVSCode`

```python
from .template import OpenaiTemplate

class OIVSCode(OpenaiTemplate):
    label = "OI VSCode Server"
    url = "https://oi-vscode-server.onrender.com"
    api_base = "https://oi-vscode-server-2.onrender.com/v1"
    
    working = True
    needs_auth = False
    supports_stream = True
    supports_system_message = True
    supports_message_history = True
    
    default_model = "gpt-4o-mini-2024-07-18"
    default_vision_model = default_model
    vision_models = [default_model, "gpt-4o-mini"]
    models = vision_models + ["deepseek-ai/DeepSeek-V3"]
    
    model_aliases = {
        "gpt-4o-mini": "gpt-4o-mini-2024-07-18",
        "deepseek-v3": "deepseek-ai/DeepSeek-V3"
    }
