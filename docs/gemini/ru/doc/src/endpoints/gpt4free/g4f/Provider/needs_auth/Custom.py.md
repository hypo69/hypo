# Модуль Custom

## Обзор

Модуль `Custom` предоставляет классы для работы с пользовательскими провайдерами, использующими шаблоны OpenAI. Он определяет базовый класс `Custom`, который наследуется от `OpenaiTemplate`, и класс `Feature`, представляющий собой специфический провайдер.

## Подробней

Модуль предназначен для упрощения интеграции с пользовательскими API, совместимыми с форматом OpenAI. Класс `Custom` предоставляет общие атрибуты и настройки, необходимые для работы с такими API, а класс `Feature` может использоваться для определения конкретных пользовательских провайдеров.

## Классы

### `Custom`

**Описание**: Базовый класс для пользовательских провайдеров, использующих шаблоны OpenAI.

**Наследует**:
- `OpenaiTemplate`: Предоставляет базовую функциональность для работы с OpenAI-совместимыми API.

**Атрибуты**:
- `label` (str): Название провайдера ("Custom Provider").
- `working` (bool): Указывает, работает ли провайдер (True).
- `needs_auth` (bool): Указывает, требуется ли аутентификация (False).
- `api_base` (str): Базовый URL API ("http://localhost:8080/v1").
- `sort_models` (bool): Указывает, нужно ли сортировать модели (False).

**Принцип работы**:
Класс `Custom` предоставляет основные атрибуты, необходимые для работы с пользовательским OpenAI-совместимым API. Он наследует `OpenaiTemplate`, который, вероятно, содержит методы для взаимодействия с API и обработки ответов. Атрибуты класса определяют название провайдера, его текущий статус, необходимость аутентификации, базовый URL API и необходимость сортировки моделей.

### `Feature`

**Описание**: Класс, представляющий собой специфического пользовательского провайдера.

**Наследует**:
- `Custom`: Наследует атрибуты и методы базового класса `Custom`.

**Атрибуты**:
- `label` (str): Название провайдера ("Feature Provider").
- `working` (bool): Указывает, работает ли провайдер (False).

**Принцип работы**:
Класс `Feature` наследует все атрибуты от `Custom` и переопределяет `label` и `working`.  `working = False` может означать, что этот провайдер находится в разработке или не готов к использованию.

## Функции

В данном модуле нет явных функций, кроме методов, унаследованных от `OpenaiTemplate`. Однако, если в `OpenaiTemplate` есть методы, они будут доступны и в классах `Custom` и `Feature`.

```python
from __future__ import annotations

from ..template import OpenaiTemplate

class Custom(OpenaiTemplate):
    label = "Custom Provider"
    working = True
    needs_auth = False
    api_base = "http://localhost:8080/v1"
    sort_models = False

class Feature(Custom):
    label = "Feature Provider"
    working = False
