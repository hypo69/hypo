# ИНСТРУКЦИЯ ПОЛЬЗОВАТЕЛЯ ДЛЯ БOTA CHAT_GPT_NODEJS

## Обзор

Этот бот предоставляет набор команд для взаимодействия с моделью ChatGPT.  Он позволяет пользователям приветствовать бота, обучать модель, тестировать ее, архивировать файлы и выбирать наборы данных для обучения.


## Команды

### `!hi`

**Описание**: Приветствует пользователя.

**Использование**: `!hi`

**Возвращает**:  Приветствие от бота.


### `!train`

**Описание**: Обучает модель с помощью предоставленных данных.

**Использование**: `!train <data> <data_dir> <positive> <attachment>`

**Параметры**:

- `<data>` (строка): Путь к файлу данных для обучения.
- `<data_dir>` (строка): Путь к папке данных для обучения.
- `<positive>` (строка): Значение для положительной метки данных.
- `<attachment>` (строка): Путь к прикрепленному файлу данных (если используется).

**Возвращает**:  Сообщение об успешном обучении или ошибку.

**Вызывает исключения**:
- `FileNotFoundError`: Если указанный файл или папка не найдены.
- `TypeError`: При неверном формате входных данных.


### `!test`

**Описание**: Тестирует модель с помощью предоставленных JSON данных.

**Использование**: `!test <test_data>`

**Параметры**:
- `<test_data>` (строка): Путь к файлу с JSON данными для тестирования.

**Возвращает**:  Результаты тестирования модели или сообщение об ошибке.

**Вызывает исключения**:
- `FileNotFoundError`: Если указанный файл не найден.
- `JSONDecodeError`: Если формат входных JSON данных некорректен.


### `!archive`

**Описание**: Архивирует файлы в указанной директории.

**Использование**: `!archive <directory>`

**Параметры**:
- `<directory>` (строка): Путь к директории для архивирования.

**Возвращает**:  Сообщение об успешном архивировании или ошибку.

**Вызывает исключения**:
- `FileNotFoundError`: Если указанная директория не найдена.
- `ValueError`: Если не удаётся обработать путь к директории.


### `!select_dataset`

**Описание**: Выбирает набор данных для обучения из указанной директории.

**Использование**: `!select_dataset <path_to_dir_positive> <positive>`

**Параметры**:

- `<path_to_dir_positive>` (строка): Путь к директории с положительными примерами данных.
- `<positive>` (строка): Значение для положительной метки данных.

**Возвращает**:  Успех/неуспех выбора набора данных или сообщение об ошибке.

**Вызывает исключения**:
- `FileNotFoundError`: Если директория не найдена.
- `ValueError`: Если не удалось обработать путь к директории.


### `!instruction`

**Описание**: Выводит эту страницу инструкции.

**Использование**: `!instruction`

**Возвращает**:  Текст этой страницы инструкции.
