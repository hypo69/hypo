# Модуль `hypotez/src/endpoints/advertisement/facebook/start_event.py`

## Обзор

Данный модуль отвечает за отправку рекламных мероприятий (событий) в группы Facebook.  Он использует `FacebookPromoter` для автоматической публикации заданных событий.

## Переменные

### `MODE`

**Описание**:  Строковая константа, хранящая режим работы (например, 'dev', 'prod').

### `filenames`

**Описание**: Список путей к файлам с данными о группах Facebook.

### `excluded_filenames`

**Описание**: Список путей к файлам с данными о группах, которые нужно исключить из обработки.

### `events_names`

**Описание**: Список названий событий, которые нужно опубликовать.

## Классы

### `FacebookPromoter`

**Описание**:  Класс для работы с публикацией событий в Facebook группы.  Подробная документация отсутствует в представленном коде.


## Функции

### `run_events`

**Описание**: Запускает процесс публикации событий в указанных группах.

**Параметры**:

- `events_names` (list): Список названий событий для публикации.
- `group_file_paths` (list): Список путей к файлам с данными о группах Facebook.

**Возвращает**:
- Нет возвращаемого значения (None).

**Обрабатываемые исключения**:
- Нет указанных обработчиков исключений.


## Использование

Модуль запускается с использованием цикла `while True`.  В цикле, с определенным интервалом времени, `FacebookPromoter` выполняет публикацию событий `run_events`.  Программа завершается при прерывании пользователем (Ctrl+C).

## Примечания

-  Необходимо указать полный путь к файлам с данными о группах (`filenames`).
-  Требуется объект `Driver`, инициализированный для взаимодействия с веб-драйвером.
-  В коде отсутствует обработка исключений в цикле.  Это может приводить к незапланированному завершению процесса.
-  Отсутствует логика работы с конкретными файлами и событиями.
