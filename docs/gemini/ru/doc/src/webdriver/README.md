# Модуль `webdriver.executor`

## Обзор

Модуль `webdriver.executor` предоставляет фреймворк для выполнения действий на веб-страницах с использованием WebDriver. Он обрабатывает скрипты и локейторы для автоматизации действий с веб-элементами.

## Основные возможности

- Выполнение алгоритмов навигации, указанных в скриптовых файлах.
- Выполнение алгоритмов взаимодействия с веб-страницей, указанных в файлах локейторов.

## Функциональность

### Обработка локейторов

#### Класс `ExecuteLocator`

##### **Описание:**  Класс `ExecuteLocator` предназначен для выполнения действий над веб-элементами на основе данных, переданных в словаре локейтора.

##### **Атрибуты:**

- `driver`: Экземпляр WebDriver, используемый для взаимодействия с браузером.
- `actions`: Экземпляр `ActionChains` для выполнения сложных действий с веб-элементами.
- `by_mapping`: Словарь, сопоставляющий строковые представления локейторов с объектами `By` из Selenium.


##### **Методы:**

###### `__init__(self, driver, *args, **kwargs)`

- **Описание:** Конструктор класса. Инициализирует WebDriver и `ActionChains`.
- **Аргументы:**
    - `driver`: Экземпляр WebDriver.
    - `*args`, `**kwargs`: Дополнительные аргументы и ключевые аргументы.
- **Возвращает:**  Экземпляр класса `ExecuteLocator`.

###### `execute_locator(self, locator: dict, message: str = None, typing_speed: float = 0, continue_on_error: bool = True) -> Union[str, list, dict, WebElement, bool]`

- **Описание:** Основной метод для выполнения действий на основе локейтора.
- **Аргументы:**
    - `locator` (dict): Словарь с параметрами для выполнения действий.
    - `message` (str, optional): Сообщение для отправки, если необходимо. По умолчанию `None`.
    - `typing_speed` (float, optional): Скорость ввода для отправки сообщений. По умолчанию `0`.
    - `continue_on_error` (bool, optional): Флаг, указывающий на продолжение выполнения, если произошла ошибка. По умолчанию `True`.
- **Возвращает:**  Результат выполнения действия (строка, список, словарь, WebElement или `False`, если элемент не найден или произошла ошибка).
- **Возможные исключения:**
    - `NoSuchElementException`: Элемент не найден.
    - `TimeoutException`: Ошибка по истечении времени ожидания.
    - `ExecuteLocatorException`: Общее исключение для модуля.

###### `get_webelement_by_locator(self, locator: dict | SimpleNamespace, message: str = None) -> WebElement | List[WebElement] | bool`

- **Описание:** Получает веб-элемент(ы) на основе локейтора.
- **Аргументы:**
    - `locator` (dict | SimpleNamespace): Словарь или объект SimpleNamespace с параметрами локейтора.
    - `message` (str, optional): Сообщение для отправки при ошибке. По умолчанию `None`.
- **Возвращает:** Web элемент или список элементов, или `False`, если элементы не найдены.
- **Возможные исключения:** Те же, что и в `execute_locator`.

###### `get_attribute_by_locator(self, locator: dict | SimpleNamespace, message: str = None) -> str | list | dict | bool`

- **Описание:** Возвращает атрибут элемента на основе локейтора.
- **Аргументы:**  Аналогично `get_webelement_by_locator`.
- **Возвращает:** Значение атрибута или `False`.
- **Возможные исключения:** Те же, что и в `execute_locator`.

###### `send_message(self, locator: dict | SimpleNamespace, message: str, typing_speed: float, continue_on_error: bool) -> bool`

- **Описание:** Отправляет сообщение в веб-элемент.
- **Аргументы:**  Аналогично `execute_locator`, но с дополнительным параметром `message`.
- **Возвращает:** `True`, если сообщение отправлено успешно, `False` - в противном случае.
- **Возможные исключения:** Те же, что и в `execute_locator`.
