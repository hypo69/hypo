# ИНСТРУКЦИЯ ПО ОБРАБОТКЕ JSON-ДАННЫХ ДЛЯ СОЗДАНИЯ ОПИСАНИЯ КОМПЬЮТЕРНОЙ СИСТЕМЫ

## Обзор

Данная инструкция описывает процесс обработки входных JSON-данных, содержащих информацию о компонентах компьютера, для создания описания системы на языке Markdown, с последующим преобразованием в JSON-формат.  В выходном JSON будут содержаться: заголовок, описание, тип системы (например, игровая, рабочая станция),  переведенные данные о компонентах.  Все текстовые данные в выходном формате должны быть в кодировке UTF-8, без использования Unicode Escape Sequence.

## Требования

* **Формат входных данных:** JSON, содержащий информацию о компонентах компьютера.
* **Формат выходных данных:** Структурированный JSON, соответствующий заданной схеме.
* **Язык:** Все текстовые данные должны быть на русском языке, включая названия компонентов, их описания и спецификации.  В случае непереводимых технических терминов допускается использование англоязычного текста.
* **Обработка ошибок:** Необходимо проверять все входные данные на корректность, отсутствие ошибок структуры и целостности.  В случае ошибок, генерировать соответствующее сообщение об ошибке.
* **Кодировка:** Использовать UTF-8.  Исключить Unicode Escape Sequence.
* **Определение типа системы:**  Выполнить классификацию типа системы (например, игровая, рабочая станция).  Основываясь на информации о компонентах.
* **Соответствие требованиям:**  Выходной JSON должен соответствовать предоставленной структуре, включая все требуемые поля.  Проводить проверку структуры полученных данных.


## Функции (Методы) Обработки

### `parse_json_input(input_json: str) -> dict | None`

**Описание**: Функция анализирует входной JSON, извлекая информацию о компонентах компьютера.

**Параметры**:
* `input_json` (str):  Входные данные в формате JSON.

**Возвращает**:
* `dict | None`: Словарь с данными о системе или `None` в случае ошибки валидации входных данных.

**Вызывает исключения**:
* `ValueError`: В случае некорректного формата входных данных.
* `TypeError`: Если входные данные не являются JSON-строкой.


### `translate_component_data(component_data: dict) -> dict`

**Описание**: Функция переводит данные о компоненте на русский язык.

**Параметры**:
* `component_data` (dict):  Словарь с данными о компоненте.

**Возвращает**:
* `dict`: Словарь с переведенными данными.

**Вызывает исключения**:
* `TypeError`: Если входные данные не являются словарем.
* `ValueError`: В случае проблем с переводом или недостающих данных.


### `categorize_system(components: list) -> str`

**Описание**: Функция классифицирует тип компьютера.

**Параметры**:
* `components` (list): Список компонентов.

**Возвращает**:
* `str`: Строка, описывающая тип системы (например, "Игровая", "Рабочая станция").

**Вызывает исключения**:
* `TypeError`: Если входные данные не являются списком.
* `ValueError`: В случае невозможности классификации на основании входных данных.
