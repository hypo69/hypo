# Модуль схем данных для управления продуктами в цифровом магазине Telegram-бота

## Обзор

Этот модуль содержит схемы данных, используемые для валидации и представления информации о продуктах в цифровом магазине Telegram-бота. Он включает в себя модели для идентификации продукта и для представления полной информации о продукте, включая его название, описание, цену, категорию, ID файла и скрытый контент.

## Подробнее

Модуль `schemas.py` определяет структуры данных (схемы) с использованием библиотеки `pydantic`, которые служат для стандартизации и проверки данных, передаваемых между различными компонентами системы, в частности, между Telegram-ботом и цифровым магазином. Это обеспечивает согласованность данных и упрощает их обработку.

## Классы

### `ProductIDModel`

**Описание**: Модель для представления идентификатора продукта.

**Принцип работы**: Класс `ProductIDModel` использует `pydantic.BaseModel` для определения структуры данных, содержащей только ID продукта. Это позволяет удобно передавать и проверять ID продукта в различных частях системы.

**Атрибуты**:
- `id` (int): Уникальный идентификатор продукта.

### `ProductModel`

**Описание**: Модель для представления полной информации о продукте.

**Принцип работы**: Класс `ProductModel` наследуется от `pydantic.BaseModel` и определяет структуру данных, содержащую всю необходимую информацию о продукте. `Field` используется для определения ограничений на значения атрибутов, таких как минимальная длина строк и положительное значение цены и ID категории.

**Атрибуты**:
- `name` (str): Название продукта (минимальная длина 5 символов).
- `description` (str): Описание продукта (минимальная длина 5 символов).
- `price` (int): Цена продукта (должна быть больше 0).
- `category_id` (int): ID категории продукта (должен быть больше 0).
- `file_id` (str | None): ID файла продукта (может быть `None`).
- `hidden_content` (str): Скрытый контент продукта (минимальная длина 5 символов).
