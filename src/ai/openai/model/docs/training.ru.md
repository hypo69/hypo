Класс `OpenAIModel`, предназначен для взаимодействия с OpenAI API, управления помощниками и диалогом, а также для обучения модели на основе диалогов. Ниже краткое описание ключевых частей кода:

### Основные функции:

1. **Инициализация модели и помощника**:
   - При создании объекта класса `OpenAIModel` происходит инициализация клиента OpenAI с использованием API-ключа и загружаются доступные помощники.
   - Параметры могут задаваться при создании объекта, такие как инструкции для системы и ID помощника.

2. **list_models и list_assistants**:
   - Эти свойства позволяют динамически загружать список доступных моделей и ассистентов, либо из OpenAI API, либо из JSON файла.
   - Модели и ассистенты используются для выбора подходящего алгоритма и помощника.

3. **ask**:
   - Основная функция взаимодействия с моделью. Принимает сообщение пользователя и возвращает ответ от модели.
   - Функция также анализирует тональность ответа (позитивная, негативная или нейтральная) с помощью функции `determine_sentiment`.
   - Ответы сохраняются в формате JSON для последующего анализа.

4. **train**:
   - Функция для обучения модели. Она принимает данные в виде CSV-файлов или строк и обучает модель.
   - Также можно указать, является ли тренируемый набор данных положительным или отрицательным.

5. **dynamic_train**:
   - Функция динамического обучения. Она загружает предыдущие диалоги из JSON-файла и использует их для дообучения модели в реальном времени.
   - Используется в сценариях, когда модель должна адаптироваться к текущему контексту общения.

6. **_save_dialogue**:
   - Эта вспомогательная функция сохраняет текущий диалог в JSON-файл, что позволяет хранить историю разговоров для дальнейшего анализа и дообучения.

7. **save_job_id**:
   - Функция сохраняет ID задачи обучения вместе с описанием в файл, что полезно для отслеживания запущенных тренировочных задач.

### Пример работы:
1. Пользователь отправляет сообщение в функцию `ask`.
2. Модель обрабатывает сообщение, добавляет системную инструкцию (если есть), отправляет запрос в OpenAI API и получает ответ.
3. Ответ анализируется на предмет тональности и сохраняется вместе с исходным сообщением в JSON-файл.
4. Если в процессе диалога необходимо дообучение, можно вызвать функцию `dynamic_train`, которая загрузит предыдущие диалоги и обучит модель на основе этих данных.
