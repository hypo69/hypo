# Модуль quickstart

## Обзор

Данный модуль предоставляет функции для взаимодействия с Google Apps Script API.  Он позволяет создавать новые проекты скриптов, загружать файлы (например, код JavaScript) в проект и выводить URL-адрес созданного скрипта.

## Функции

### `main`

**Описание**: Функция, которая выполняет основную логику взаимодействия с API Google Apps Script.  Создает новый проект, загружает файлы кода и манифеста, а затем выводит ссылку на созданный проект.

**Параметры**:
Нет входных параметров.

**Возвращает**:
Нет значения возврата.

**Вызывает исключения**:
- `googleapiclient.errors.HttpError`: Возникает при проблемах во время взаимодействия с API, например, при ошибке аутентификации или проблемах на стороне сервера.  Ошибка будет содержать подробности об ошибке в `error.content`.

## Импорты

### `pathlib`
Используется для работы с путями к файлам.
### `google.auth.transport.requests`
Обеспечивает средства аутентификации для запросов к Google API.
### `google.oauth2.credentials`
Предоставляет возможность работы с учетными данными для доступа к API.
### `google_auth_oauthlib.flow`
Используется для процесса авторизации пользователя с помощью Google OAuth 2.0.
### `googleapiclient`
Библиотека для взаимодействия с API Google.
### `googleapiclient.discovery`
Предоставляет инструменты для работы с API Google.
### `header`
Вероятно, содержит вспомогательные функции или константы.
### `src.gs`
Вероятно, содержит функции для работы с Google Cloud Storage.


## Константы

### `SCOPES`

Список областей доступа, необходимых для работы с API.

### `SAMPLE_CODE`

Пример кода JavaScript, который будет добавлен в создаваемый проект.

### `SAMPLE_MANIFEST`

Пример манифеста проекта Apps Script.


## Переменные

### `MODE`

Переменная, хранящая значение режима работы (например, 'development').


## Замечания

Функция `main()` выполняет следующие шаги:

1.  Получает токен доступа к API. Если токен уже существует, он загружается из файла `token.json`. В противном случае, запускается процедура авторизации, а полученный токен сохраняется в `token.json`.
2.  Строится объект сервиса для взаимодействия с API.
3.  Создает новый проект в Apps Script.
4.  Загружает в проект файлы `SAMPLE_CODE` и `SAMPLE_MANIFEST`.
5.  Выводит ссылку на созданный проект в консоль.
6.  Обрабатывает возможные исключения `googleapiclient.errors.HttpError`, которые могут возникнуть во время взаимодействия с API.

Файл `credentials.json` должен содержать данные для аутентификации вашего приложения Google.  Файл `token.json` нужен для хранения токена доступа.
