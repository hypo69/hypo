# Модуль `mocks.py`

## Обзор

Модуль содержит определения фиктивных (mock) классов провайдеров, используемых для модульного тестирования функциональности, связанной с `gpt4free`. Эти фиктивные провайдеры имитируют различные сценарии работы реальных провайдеров, такие как успешное создание завершений, асинхронные операции, генерация изображений, ошибки аутентификации и другие исключительные ситуации.

## Подробней

Этот модуль предоставляет набор мок-классов, которые наследуются от абстрактных классов провайдеров `g4f`. Они используются для изоляции тестируемого кода от внешних зависимостей и для проверки корректности обработки различных сценариев, возникающих при работе с провайдерами `gpt4free`.

## Классы

### `ProviderMock`

**Описание**: Мок-класс, имитирующий синхронного провайдера.

**Принцип работы**:
Этот класс используется для имитации простого провайдера, который всегда возвращает строку "Mock" при создании завершения.

**Методы**:
- `create_completion`: Метод класса, который имитирует создание завершения и возвращает строку "Mock".

### `AsyncProviderMock`

**Описание**: Мок-класс, имитирующий асинхронного провайдера.

**Принцип работы**:
Этот класс используется для имитации асинхронного провайдера, который всегда возвращает строку "Mock" при асинхронном создании.

**Методы**:
- `create_async`: Асинхронный метод класса, который имитирует асинхронное создание и возвращает строку "Mock".

### `AsyncGeneratorProviderMock`

**Описание**: Мок-класс, имитирующий асинхронного провайдера-генератора.

**Принцип работы**:
Этот класс используется для имитации асинхронного провайдера-генератора, который всегда генерирует строку "Mock" при асинхронном создании генератора.

**Методы**:
- `create_async_generator`: Асинхронный метод класса, который имитирует асинхронное создание генератора и генерирует строку "Mock".

### `ModelProviderMock`

**Описание**: Мок-класс, который возвращает имя модели.

**Принцип работы**:
Этот класс используется для имитации провайдера, который возвращает имя модели, переданной в метод `create_completion`.

**Методы**:
- `create_completion`: Метод класса, который имитирует создание завершения и возвращает имя модели.

### `YieldProviderMock`

**Описание**: Мок-класс, который генерирует содержимое сообщений.

**Принцип работы**:
Этот класс используется для имитации провайдера, который генерирует содержимое каждого сообщения, переданного в метод `create_async_generator`.

**Методы**:
- `create_async_generator`: Асинхронный метод класса, который имитирует асинхронное создание генератора и генерирует содержимое каждого сообщения.

### `YieldImageResponseProviderMock`

**Описание**: Мок-класс, который генерирует объект `ImageResponse`.

**Принцип работы**:
Этот класс используется для имитации провайдера, который генерирует объект `ImageResponse` с заданным запросом (prompt) и пустой строкой для URL изображения.

**Методы**:
- `create_async_generator`: Асинхронный метод класса, который имитирует асинхронное создание генератора и генерирует объект `ImageResponse`.

### `MissingAuthProviderMock`

**Описание**: Мок-класс, который вызывает исключение `MissingAuthError`.

**Принцип работы**:
Этот класс используется для имитации провайдера, который вызывает исключение `MissingAuthError` при создании завершения, что позволяет проверить обработку ошибок аутентификации.

**Методы**:

### `RaiseExceptionProviderMock`

**Описание**: Мок-класс, который вызывает исключение `RuntimeError`.

**Принцип работы**:

**Методы**:

### `AsyncRaiseExceptionProviderMock`

**Описание**: Мок-класс, который асинхронно вызывает исключение `RuntimeError`.

**Принцип работы**:
Этот класс используется для имитации асинхронного провайдера, который вызывает исключение `RuntimeError` при асинхронном создании генератора, что позволяет проверить общую обработку ошибок в асинхронном контексте.

**Методы**:
- `create_async_generator`: Асинхронный метод класса, который имитирует асинхронное создание генератора и генерирует `None`.
