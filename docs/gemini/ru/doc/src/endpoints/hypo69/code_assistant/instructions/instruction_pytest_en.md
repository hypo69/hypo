# Инструкция по написанию тестов `pytest`

## Обзор

Данная инструкция описывает шаги по написанию тестов для Python-кода с использованием фреймворка `pytest`.  Тесты должны быть написаны для функций, методов и классов, чтобы проверить их правильность, включая граничные случаи и обработку исключений.

## Требования

1. **Имена тестов:** Имена функций тестов должны быть ясными и описательными, отражая цель каждого теста.
2. **Изоляция тестов:** Тесты должны быть изолированными и независимыми друг от друга.
3. **Разнообразие сценариев:** Тесты должны охватывать различные сценарии, включая:
   - **Валидные входные данные:** Проверка работы кода с корректными входными данными.
   - **Невалидные входные данные:** Проверка обработки некорректных или неожиданных входных данных, если применимо.
   - **Граничные случаи:** Тестирование работы на границах допустимых значений.
4. **Тестирование исключений:** Использование `pytest.raises` для проверки обработки исключений.
5. **Фикстуры:** При необходимости использования фикстур, их следует определять отдельно.
6. **Комментарии:** Добавьте комментарии, объясняющие логику тестов.

## Структура тестов

Пример структуры тестов:

```python
import pytest

# Фикстуры, если нужны
@pytest.fixture
def example_data():
    """Предоставляет тестовые данные для функции."""
    return {...}

# Тесты для функции 1
def test_function1_valid_input():
    """Проверка корректного поведения с валидными входными данными."""
    # ...
    pass

def test_function1_invalid_input():
    """Проверка корректной обработки невалидных входных данных."""
    # ...
    pass

# Тесты для функции 2
def test_function2_edge_case():
    """Проверка поведения с граничными случаями."""
    # ...
    pass
```

##  Примеры использования `pytest.raises`

```python
import pytest

def my_function(x):
    if x < 0:
        raise ValueError("x должно быть неотрицательным")
    return x * 2

def test_my_function_value_error():
    with pytest.raises(ValueError, match="x должно быть неотрицательным"):
        my_function(-1)
```

##  Проверка граничных случаев

```python
def my_function(x):
  if x > 100:
    return "Значение слишком большое"
  return x * 2

def test_my_function_boundary():
  assert my_function(100) == 200
  assert my_function(101) == "Значение слишком большое"
```

**Важно:**  Этот шаблон служит руководством.  Необходимо адаптировать его к конкретному Python-коду. Вместо `...`  необходимо  вставить реальный код тестов, основанный на анализе предоставленного Python кода.
