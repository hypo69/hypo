## Требования к формату документации

Документация должна быть создана в формате Markdown (.md). Каждый файл должен содержать заголовок и краткое описание содержимого.

Для функций и методов необходимо использовать следующие комментарии:

```python
def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:
    """
    Args:
        param (str): Описание параметра `param`.
        param1 (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию `None`.

    Returns:
        dict | None: Описание возвращаемого значения. Возвращает словарь или `None`.

    Raises:
        SomeError: Описание ситуации, в которой возникает исключение `SomeError`.
    """
```

Используйте `ex` вместо `e` в блоках обработки исключений.

## Форматирование документации

Используйте правильный синтаксис Markdown для всех заголовков, списков и ссылок.  При документировании классов, функций и методов используйте структурированные разделы с описаниями, деталями параметров, возвращаемых значений и вызываемых исключений.


## Заголовки разделов

Используйте заголовки первого уровня (`#`), второго уровня (`##`), третьего уровня (`###`) и четвёртого уровня (`####`) последовательно на протяжении всего файла.
