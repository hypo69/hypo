## Введение

В данном руководстве описаны шаги для запуска и выполнения тестов, которые проверяют функциональность классов `Driver` и `ExecuteLocator` в файле `test_driver_executor.py`.  Тесты охватывают различные сценарии использования, включая взаимодействие между классами.


## Структура тестов

Файл `test_driver_executor.py` содержит тесты для двух классов: `Driver` и `ExecuteLocator`.  Тесты проверяют корректность работы отдельных методов, взаимодействие между классами и разные сценарии использования.


## Тестируемые методы и функции

- `test_navigate_to_page`: Проверяет навигацию WebDriver к указанной странице.
- `test_get_webelement_by_locator_single_element`: Проверяет получение элемента по локатору, когда элемент найден.
- `test_get_webelement_by_locator_no_element`: Проверяет получение элемента по локатору, когда элемент не найден.
- `test_send_message`: Проверяет отправку сообщения элементу.
- `test_get_attribute_by_locator`: Проверяет получение атрибута элемента по локатору.
- `test_execute_locator_event`: Проверяет выполнение события на локаторе.
- `test_get_locator_keys`: Проверяет получение ключей локатора.
- `test_navigate_and_interact`: Проверяет последовательность навигации и взаимодействия с элементами на другой странице.
- `test_invalid_locator`: Проверяет обработку некорректного локатора и соответствующее исключение.


## Запуск тестов

### Установка зависимостей

Перед запуском тестов убедитесь, что установлены все необходимые зависимости.  Выполните следующую команду:

```bash
pip install -r requirements.txt
```

Убедитесь, что в `requirements.txt` перечислены все необходимые библиотеки, включая `pytest`, `selenium` и `chromedriver`.


### Настройка WebDriver

Тесты используют Chrome WebDriver. Убедитесь, что у вас установлен ChromeDriver и укажите путь к нему в коде:

```python
from selenium import webdriver
from selenium.webdriver.chrome.service import Service

# ... (ваш код)

service = Service(executable_path="/путь/к/chromedriver")  # Замените на ваш путь
driver = webdriver.Chrome(service=service)
```

**Важно:** Замените `/путь/к/chromedriver` на фактический путь к вашему исполняемому файлу ChromeDriver на вашем компьютере.


### Запуск тестов

Для запуска тестов используйте команду:

```bash
pytest src/webdriver/_pytest/test_driver_executor.py
```

Эта команда выполнит все тесты, содержащиеся в файле `test_driver_executor.py`.


## Описание тестов

### `test_navigate_to_page`

- **Цель:** Проверить, что WebDriver корректно загружает заданную страницу.
- **Ожидаемый результат:** Текущий URL страницы должен быть "http://example.com".


### `test_get_webelement_by_locator_single_element`

- **Цель:** Проверить, что метод `get_webelement_by_locator` возвращает элемент, когда он найден.
- **Ожидаемый результат:** Возвращённый элемент должен быть экземпляром `WebElement` и содержать текст "Example Domain".


###  `test_get_webelement_by_locator_no_element`

- **Цель:** Проверить, что метод `get_webelement_by_locator` возвращает `False`, если элемент не найден.
- **Ожидаемый результат:** Возвращаемое значение должно быть `False`.


**(Другие тесты описываются аналогично, указывая цель, ожидаемый результат и подробности)**


## Отчеты о тестах

После выполнения тестов `pytest` создаст отчет с результатами тестирования.  Вы можете использовать различные флаги `pytest` для получения более подробной информации, например, для создания HTML-отчета:

```bash
pip install pytest-html
pytest src/webdriver/_pytest/test_driver_executor.py --html=report.html
```

Это сохранит отчет в файле `report.html`.
