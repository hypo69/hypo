# Руководство для Тестера

## Обзор

Данный документ предназначен для тестеров, которые будут проверять модуль, отвечающий за подготовку материалов для рекламных кампаний на платформе AliExpress. Модуль включает в себя три основных файла: `edit_campaign.py`, `prepare_campaigns.py` и `test_campaign_integration.py`.

## Файлы

### `edit_campaign.py`

**Описание**: Этот файл содержит класс `AliCampaignEditor`, который наследует от `AliPromoCampaign`. Основная задача этого класса — управление рекламной кампанией.

**Классы**:

#### `AliCampaignEditor`

**Описание**: Класс для управления рекламной кампанией.

**Методы**:

- Пока не предоставлен код класса, методы не могут быть описаны.


### `prepare_campaigns.py`

**Описание**: Этот файл содержит функции для подготовки материалов кампании, включая обновление категорий и обработку кампаний по категориям.

**Функции**:

#### `update_category`

**Описание**: Обновляет категорию в JSON файле.

**Параметры**:
- `category_data` (dict): Данные категории для обновления.

**Возвращает**:
- `bool`: `True`, если обновление успешно, `False` иначе.

**Вызывает исключения**:
- `FileNotFoundError`: Если файл JSON не найден.
- `ValueError`: При неверном формате данных.


#### `process_campaign_category`

**Описание**: Обрабатывает конкретную категорию в рамках кампании.

**Параметры**:
- `campaign_id` (str): ID кампании.
- `category_id` (str): ID категории.

**Возвращает**:
- `dict | None`: Результат обработки категории или `None` при ошибке.

**Вызывает исключения**:
- `ValueError`: Если введены некорректные данные.
- `Exception`: Другие ошибки во время обработки.


#### `process_campaign`

**Описание**: Обрабатывает всю кампанию по всем категориям.

**Параметры**:
- `campaign_id` (str): ID кампании.

**Возвращает**:
- `list | None`: Список результатов обработки каждой категории или `None` при ошибке.

**Вызывает исключения**:
- `ValueError`: При неверных входных данных.
- `Exception`: Другие ошибки во время обработки.


#### `main`

**Описание**: Асинхронная основная функция для обработки кампании.

**Параметры**:
- `campaign_id` (str): ID кампании.

**Возвращает**:
- `None`: Функция не возвращает значение.

**Вызывает исключения**:
- `Exception`: В случае возникновения ошибки при асинхронной обработке.


### `test_campaign_integration.py`

**Описание**: Этот файл содержит тесты, проверяющие взаимодействие всех компонентов модуля.

**Тесты**:

#### `test_update_category_success`

**Описание**: Проверка успешного обновления категории.

**Вызывает**: `update_category` с корректными данными.

**Ожидаемый результат**: Функция `update_category` возвращает `True`, и JSON-файл обновлен.


#### `test_update_category_failure`

**Описание**: Проверка обработки ошибки при обновлении категории.

**Вызывает**: `update_category` с некорректными данными или отсутствующим файлом.

**Ожидаемый результат**: Функция `update_category` возвращает `False`, и логгирует сообщение об ошибке.
