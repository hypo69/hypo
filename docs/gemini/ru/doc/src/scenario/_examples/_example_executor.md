# Модуль hypotez/src/scenario/_examples/_example_executor.py

## Обзор

Этот модуль содержит примеры использования функций из модуля `src.scenario.executor` для выполнения сценариев, обработки файлов сценариев, взаимодействия с API PrestaShop.  Примеры демонстрируют запуск списка файлов сценариев, одиночного файла сценария, одиночного сценария, сценария страницы продукта, добавления купона через API PrestaShop, а также асинхронные и синхронные запросы к PrestaShop.

## Функции

### `example_run_scenario_files`

**Описание**: Функция выполняет все сценарии, перечисленные в списке `scenario_files`.

**Аргументы**:
- `supplier` (объект `MockSupplier`): Объект, предоставляющий информацию о поставщике и сценариях.
- `scenario_files` (list): Список объектов `Path` с путями к файлам сценариев.

**Возвращает**:
- `bool`: `True`, если все сценарии выполнены успешно; `False`, если произошли ошибки.


### `example_run_scenario_file`

**Описание**: Функция выполняет сценарий из указанного файла.

**Аргументы**:
- `supplier` (объект `MockSupplier`): Объект, предоставляющий информацию о поставщике и сценариях.
- `scenario_file` (объект `Path`): Путь к файлу сценария.

**Возвращает**:
- `bool`: `True`, если сценарий выполнен успешно; `False`, если произошла ошибка.


### `example_run_scenario`

**Описание**: Функция выполняет указанный сценарий.

**Аргументы**:
- `supplier` (объект `MockSupplier`): Объект, предоставляющий информацию о поставщике и сценариях.
- `scenario` (dict): Словарь, представляющий сценарий.

**Возвращает**:
- `bool`: `True`, если сценарий выполнен успешно; `False`, если произошла ошибка.


### `example_insert_grabbed_data`

**Описание**: Функция вставляет данные, полученные из сценария, в PrestaShop.

**Аргументы**:
- `product_fields` (объект `ProductFields`): Объект, содержащий данные о продукте.

**Возвращает**:
-  `None`. Выводит сообщение об успешном выполнении.


### `example_add_coupon`

**Описание**: Функция добавляет купон в PrestaShop используя API.

**Аргументы**:
- `credentials` (dict): Словарь с учетными данными API (api_domain, api_key).
- `reference` (str): ID товара.
- `coupon_code` (str): Код купона.
- `start_date` (str): Дата начала действия купона (формат YYYY-MM-DD).
- `end_date` (str): Дата окончания действия купона (формат YYYY-MM-DD).

**Возвращает**:
- `None`. Выводит сообщение об успешном выполнении.


### `example_execute_PrestaShop_insert_async`

**Описание**: Функция асинхронно вставляет данные продукта в PrestaShop.

**Аргументы**:
- `product_fields` (объект `ProductFields`): Объект, содержащий данные продукта.


**Возвращает**:
- `None`. Выводит сообщение об успешном выполнении.


### `example_execute_PrestaShop_insert`

**Описание**: Функция синхронно вставляет данные продукта в PrestaShop.

**Аргументы**:
- `product_fields` (объект `ProductFields`): Объект, содержащий данные продукта.

**Возвращает**:
- `bool`: `True`, если вставка выполнена успешно; `False`, если произошла ошибка.


## Классы

### `MockSupplier`

**Описание**:  Класс имитирует поставщик данных для сценариев.  Содержит фиктивные методы и атрибуты, необходимые для выполнения примеров.

### `MockRelatedModules`

**Описание**: Имитирует вспомогательный модуль, предоставляющий данные о продуктах и страницах.


### `MockDriver`

**Описание**: Имитирует драйвер для работы с веб-страницами.


## Модули

### `src.scenario.executor`

**Описание**:  Модуль, предоставляющий функции для выполнения сценариев.


### `src.utils`

**Описание**:  Вспомогательный модуль, содержащий общие функции.


### `src.product`

**Описание**:  Модуль, связанный с данными о продуктах.


### `src.endpoints.PrestaShop`

**Описание**:  Модуль для работы с API PrestaShop.
