# Модуль `hypotez/src/suppliers/aliexpress/api/errors/exceptions.py`

## Обзор

Этот модуль определяет пользовательские исключения для обработки ошибок, возникающих при взаимодействии с API AliExpress. Он предоставляет базовый класс `AliexpressException` и ряд специфичных исключений для различных ситуаций.

## Классы

### `AliexpressException`

**Описание**: Базовый класс для всех исключений, связанных с API AliExpress. Содержит причину ошибки.

**Методы**:

- `__init__(self, reason: str)`: Инициализирует исключение с указанием причины.
- `__str__(self) -> str`: Возвращает строковое представление исключения, содержащее причину.

### `InvalidArgumentException`

**Описание**: Исключение, которое возникает, когда переданные аргументы некорректны.

**Наследование**: `AliexpressException`

### `ProductIdNotFoundException`

**Описание**: Исключение, которое возникает, если не найден указанный идентификатор продукта.

**Наследование**: `AliexpressException`

### `ApiRequestException`

**Описание**: Исключение, возникающее при ошибке запроса к API AliExpress.

**Наследование**: `AliexpressException`

### `ApiRequestResponseException`

**Описание**: Исключение, возникающее, если ответ на запрос к API AliExpress не является валидным.

**Наследование**: `AliexpressException`


### `ProductsNotFoudException`

**Описание**: Исключение, которое возникает, если не найдено ни одного продукта.

**Наследование**: `AliexpressException`

### `CategoriesNotFoudException`

**Описание**: Исключение, которое возникает, если не найдено ни одной категории.

**Наследование**: `AliexpressException`


### `InvalidTrackingIdException`

**Описание**: Исключение, которое возникает, если идентификатор отслеживания отсутствует или невалиден.

**Наследование**: `AliexpressException`
