# Модуль defaults.py

## Обзор

Модуль `defaults.py` содержит определения заголовков по умолчанию, используемых в HTTP-запросах в проекте `hypotez`. Он также определяет, доступна ли библиотека `brotli` для сжатия данных. Эти заголовки используются для имитации поведения браузера при отправке запросов к серверам, а также для указания поддерживаемых методов сжатия.

## Подробней

Этот модуль важен для обеспечения совместимости и корректной обработки данных при взаимодействии с различными веб-серверами. Он позволяет настроить заголовки запросов таким образом, чтобы они соответствовали ожиданиям сервера, а также использовать сжатие `brotli`, если оно доступно, для уменьшения объема передаваемых данных.

## Функции

### `has_brotli`

```python
has_brotli = False
```

**Назначение**: 
Флаг, указывающий, доступна ли библиотека `brotli`.

**Как работает функция**:
1. Пытается импортировать библиотеку `brotli`.
2. Если импорт успешен, устанавливает `has_brotli` в `True`.
3. Если происходит ошибка `ImportError`, устанавливает `has_brotli` в `False`.

**Примеры**:

```python
# После выполнения кода:
# Если brotli установлена:
# has_brotli == True
# Если brotli не установлена:
# has_brotli == False
```

## Переменные

### `DEFAULT_HEADERS`

```python
DEFAULT_HEADERS = {
    "accept": "*/*",
    "accept-encoding": "gzip, deflate" + (", br" if has_brotli else ""),
    "accept-language": "en-US",
    "referer": "",
    "sec-ch-ua": "\"Not(A:Brand\";v=\"99\", \"Google Chrome\";v=\"133\", \"Chromium\";v=\"133\"",
    "sec-ch-ua-mobile": "?0",
    "sec-ch-ua-platform": "\"Windows\"",
    "sec-fetch-dest": "empty",
    "sec-fetch-mode": "cors",
    "sec-fetch-site": "same-origin",
    "user-agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/123.0.0.0 Safari/537.36",
}
```

**Назначение**: 
Определяет заголовки HTTP-запроса по умолчанию.

**Описание**:
- `"accept"`: указывает типы контента, которые клиент может обработать (`*/*` означает все типы).
- `"accept-encoding"`: указывает методы сжатия, которые клиент поддерживает (`gzip`, `deflate`, и `br` если доступен `brotli`).
- `"accept-language"`: указывает предпочтительный язык (`en-US`).
- `"referer"`: указывает URL страницы, с которой был сделан запрос (в данном случае пустая строка).
- `"sec-ch-ua"`: содержит информацию о браузере клиента.
- `"sec-ch-ua-mobile"`: указывает, является ли устройство мобильным.
- `"sec-ch-ua-platform"`: указывает платформу клиента (`"Windows"`).
- `"sec-fetch-dest"`: указывает назначение запроса (`"empty"`).
- `"sec-fetch-mode"`: указывает режим запроса (`"cors"`).
- `"sec-fetch-site"`: указывает сайт, с которого был сделан запрос (`"same-origin"`).
- `"user-agent"`: содержит информацию о браузере и операционной системе клиента.

