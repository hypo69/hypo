# ИНСТРУКЦИЯ ПО СОЗДАНИЮ ДОКУМЕНТАЦИИ ДЛЯ PYTHON-ФАЙЛОВ

## Обзор

Этот документ описывает формат Markdown для создания документации Python-файлов, предназначенных для дальнейшего использования.  Он охватывает заголовки, описание содержимого, комментарии к функциям и классам, структуру оглавления, форматирование и ключевые инструкции.

## Требования к Формату

*   **Формат Markdown (.md):** Все файлы должны быть в формате Markdown.
*   **Заголовок и Краткое Описание:** Каждый файл должен начинаться с заголовка и краткого описания его содержимого.
*   **Комментарии для Функций и Классов:** Функции и классы должны быть документированы с использованием формата:
    ```python
    def функция(параметр: str, параметр1: Optional[str | dict | str] = None) -> dict | None:
        """
        Args:
            параметр (str): Описание параметра `параметр`.
            параметр1 (Optional[str | dict | str], optional): Описание параметра `параметр1`. По умолчанию `None`.

        Returns:
            dict | None: Описание возвращаемого значения. Возвращает словарь или `None`.

        Raises:
            SomeError: Описание ситуации, в которой возникает исключение `SomeError`.
        """
    ```
*   **Обработка Исключений:** Используйте `ex` вместо `e` в блоках обработки исключений.
*   **Оглавление (TOC):** В начале каждого файла должен быть раздел с оглавлением, содержащий ссылки на все основные разделы документации.
*   **Форматирование:** Используйте правильный синтаксис Markdown для заголовков, списков и ссылок.  Структурируйте описание функций и классов, включая детали параметров, возвращаемых значений и исключений.

## Описание Файтеля Python


### Prompt for Gemini AI: Assembling a Computer

**Описание:** Этот файл описывает prompt для модели Gemini AI, предназначенный для создания документации по сборке компьютера.  Ввод предоставляется в формате JSON на иврите, и модель должна выполнить перевод, определение типа сборки (гейминг, офисная, рабочая станция и т.д.), сгенерировать заголовок и описание сборки, перевести все имена компонентов и описания, а также вернуть ответ в формате JSON.

## Структура

### Входные Данные (JSON)

Входные данные представляют собой список JSON объектов, каждый из которых описывает компонент компьютера.

```json
[
  {
    "product_id": "<leave as is>",
    "product_title": "<имя компонента>",
    "product_description": "<описание или характеристики>",
    "specification": "<характеристики>",
    "image_local_saved_path": "<leave as is>"
  },
  ...
]
```

### Выходные Данные (JSON)

Выходные данные представляют собой JSON объект, содержащий переведенный заголовок, описание сборки, вероятности типов сборок и список переведенных компонентов.


```json
{
  "language_code": {
    "build_types": {
      "gaming": 0.9,
      "workstation": 0.1
    },
    "title": "Заголовок сгенерированной сборки",
    "description": "Описание сгенерированной сборки",
    "products": [
      {
        "product_id": "<leave as is>",
        "product_title": "Переведенное имя компонента",
        "product_description": "Переведенное описание компонента",
        "specification": "Переведенные характеристики",
        "image_local_saved_path": "<leave as is>"
      },
      ...
    ]
  }
}
```

## Инструкции и Разъяснения

* **Перевод:** Необходимо перевести все данные с иврита на целевой язык.
* **Классификация по типу сборки:** Необходимо определить тип сборки (гейминг, офисная, рабочая станция) на основе предоставленных компонентов.
* **Подробные описания:** Необходимо сгенерировать подробные описания сборки, выделяя особенности компонентов и возможностей системы.
* **Форматирование:**  Выходные данные должны точно соответствовать указанному формату JSON.
* **Категоризация компонентов:** Объединяйте компоненты одинаковой категории (мониторы, видеокарты) для создания перечня цен и выделения уникальных особенностей.
* **Точность терминологии:** Избегайте неформальных выражений («дешевый», «средний»). Используйте альтернативы, например, «бюджетный» или «эффективный».
* **Неполные данные:** В случае неполной информации, заполняйте поля по возможности или оставляйте их пустыми с надлежащими заглушками.
