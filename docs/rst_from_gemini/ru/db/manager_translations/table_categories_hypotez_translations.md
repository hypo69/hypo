## Класс `CategoriesHypotezFullListManager`

Класс предназначен для управления данными о категориях товаров в базе данных.

**Инициализация (`__init__`):**

* Принимает аргумент `credentials`, представляющий словарь с данными для подключения к базе данных (логин, пароль, хост, порт, имя базы данных). Данные берутся из переменной `gs.db_translations_credentials` из модуля `__init__.py`.
* Создает соединение с базой данных используя `create_engine`.
* Создает базу данных (метаданные) и сессию для работы с базой данных.
* Определяет модель таблицы (`define_model`).
* Создает таблицу в базе данных, если она не существует (`create_table`).

**Методы:**

* **`insert_record`:** Добавляет новую запись в таблицу.
    * Принимает на вход значения для всех полей таблицы.
    * Возвращает `None`.
    * Выводит сообщение о успешном добавлении записи.
* **`select_record`:** Извлекает записи из таблицы на основе заданных условий.
    * Принимает произвольное количество аргументов в формате `kwargs` (например, `id_category=1, id_parent=2`).
    * Возвращает все подходящие записи в виде списка объектов `CategoryManager`.
    * Выводит информацию о каждой найденной записи в консоль.
    * **Важно:**  Улучшенная обработка `None` значений в запросе. Теперь если `kwargs` содержат `None` значение, то это значение будет пропущено в фильтре.
* **`update_record`:** Обновляет существующую запись в таблице.
    * Принимает `id_category`, `id_parent`, `id_shop_default`, `date_upd` и `new_description` для обновления.
