## Обзор

Этот документ представляет собой инструкцию для создания документации к коду на языке Python в формате Markdown. Инструкция подробно описывает требования к форматированию, содержанию и структуре документации, а также предоставляет примеры для наглядности.

## Требования

### Ключевые требования

1. **Анализ кода**:
   - Предоставление исходного кода без изменений.
   - Описание алгоритма кода в виде пошаговой блок-схемы, показывающей, как данные перемещаются между функциями или классами.
   - Объяснение работы кода, включая импорты, классы, функции, переменные и их взаимосвязи с другими частями проекта.
   - Выделение потенциальных ошибок или областей для улучшения, если таковые имеются.

2. **Формат ответа**:
   - `<input code>`: Исходный код без изменений.
   - `<algorithm>`: Пошаговое описание алгоритма с примерами.
   - `<explanation>`: Детальное объяснение кода, включая:
     - Импорты: их назначение и связи с другими модулями.
     - Классы: их назначение, атрибуты, методы и отношения с другими компонентами.
     - Функции: их назначение, аргументы и возвращаемые значения.
     - Переменные: типы и использование.
     - Взаимосвязи с другими частями проекта и потенциальные ошибки или улучшения.
