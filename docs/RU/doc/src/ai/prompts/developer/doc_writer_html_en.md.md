## Инструкции по документированию кода

### Модули

1.  **Описание модуля**: В начале документации модуля должно быть указано его назначение.
2.  **Примеры использования**: По возможности, приведите примеры использования модуля. Код примеров должен быть заключен в блок кода с языковым идентификатором `python`.
3.  **Платформы и синопсис**: Укажите платформы и синопсис модуля.
4.  **Атрибуты и методы**: При необходимости, используйте заголовки для описания атрибутов и методов модуля.

**Пример документации модуля:**

```markdown
# Модуль: Ассистент программирования

Этот модуль содержит класс `CodeAssistant`, который используется для взаимодействия с различными моделями ИИ, такими как Google Gemini и OpenAI, для задач обработки кода.

## Пример использования

Пример использования класса `CodeAssistant`:

```python
assistant = CodeAssistant(role='code_checker', lang='ru', model=['gemini'])
assistant.process_files()
```
```

### Классы

1.  **Описание класса**: Каждый класс должен быть описан в соответствии со своим назначением. Включите описание класса, его атрибуты и методы.
2.  **Методы класса**: В разделе класса перечислите все методы, их назначение и примеры использования.
3.  **Параметры и возвращаемые значения**: Для каждого метода включите описания его параметров и возвращаемых значений, а также примеры.

**Пример документации класса:**

```markdown
# Класс: CodeAssistant

Класс `CodeAssistant` используется для взаимодействия с различными моделями ИИ, такими как Google Gemini, и предоставляет методы для анализа и генерации документации для кода.

## Атрибуты
- `role`: Роль ассистента (например, 'code_checker').
- `lang`: Язык, который будет использовать ассистент (например, 'ru').
- `model`: Список используемых моделей ИИ (например, `['gemini']`).

## Методы
### `process_files`

Метод для обработки файлов кода.

## Пример использования

```python
assistant = CodeAssistant(role='code_checker', lang='ru', model=['gemini'])
assistant.process_files()
```
```

### Функции и методы

1.  **Описание функции/метода**: Документируйте каждую функцию или метод, указывая параметры и возвращаемые значения.
2.  **Примеры использования**: Для каждой функции приведите описание ее назначения и примеры использования в блоках кода с языковым идентификатором `python`.

**Пример документации метода:**

```markdown
# Метод: process_files

Этот метод используется для анализа и обработки файлов кода.

## Параметры
- `files`: Список файлов для обработки.
- `options`: Дополнительные параметры для настройки обработки.

## Возвращаемое значение
- Возвращает результат обработки в виде списка проанализированных данных.

## Пример использования

```python
assistant = CodeAssistant(role='code_checker', lang='ru', model=['gemini'])
result = assistant.process_files(files=['file1.py', 'file2.py'], options={})
```
```

### Комментарии в коде

1.  **Формат комментариев**: Все комментарии в коде должны быть написаны в формате Markdown и объяснять, что делает конкретный фрагмент кода.
2.  **Блочные комментарии**: Используйте комментарии для описания логики и объяснения решений или временных решений в коде, а не в строках.

**Пример:**

```markdown
# Здесь обрабатывается исключение, чтобы продолжить выполнение, если файл не найден
try:
    process_file(file)
except FileNotFoundError as ex:
    handle_exception(ex)
```

### Исключения

1.  **Документация исключений**: Документируйте исключения для классов, методов и функций.
2.  **Условия возникновения**: Указывайте, какие исключения могут быть вызваны и при каких обстоятельствах.

**Пример документации исключения:**

```markdown
# Исключение: Файл не найден

Это исключение вызывается, когда файл не найден во время обработки.

## Параметры
- `file`: Путь к файлу, который не был найден.

## Пример использования

```python
try:
    open(file)
except FileNotFoundError as ex:
    raise FileNotFoundError("Файл не найден") from ex
```
```

Следуйте этим инструкциям при документировании вашего кода. Все комментарии должны быть четкими, информативными и соответствовать стандарту Markdown.
