## Обзор
Данный модуль предназначен для создания документации для разработчиков в формате Markdown. Документация должна соответствовать определенным требованиям к форматированию и структуре, которые обеспечивают ясность и последовательность.

## Требования к документации
### Формат
- Документация должна быть в формате Markdown (`.md`).
- Каждый документ должен начинаться с заголовка и краткого описания содержимого модуля.
- Документация должна соответствовать стандарту RST, с использованием примеров для лучшего понимания структуры.
- Использовать одинарные кавычки (`'`) в Python-коде, двойные кавычки (`"`) только для операций вывода, как в основной инструкции.
- Комментарии после `#` должны оставаться без изменений.
- Использовать `j_loads` или `j_loads_ns` из `src.utils.jjson` вместо стандартного `json.load`.
- Сохранять любые `...` в коде без изменений.
- Импортировать `logger` только из `src.logger`.
- Выравнивать названия функций, переменных и импортов.
- Использовать `from src.logger.logger import logger` для логирования ошибок.
- Избегать чрезмерного использования стандартных блоков `try-except`, отдавая предпочтение обработке ошибок через `logger.error`.
- Все изменения в коде должны сопровождаться комментариями.

### Структура
- Добавить раздел с оглавлением в начало каждого файла документации.
- Оглавление должно содержать ссылки на все основные разделы в документации модуля.
- Использовать заголовки уровня 1 (`#`), 2 (`##`), 3 (`###`) и 4 (`####`) для структурирования документации.
- Для классов, функций и методов, должны быть отдельные разделы с описаниями, параметрами, возвращаемыми значениями и исключениями.

## Структура документации

### Раздел модуля (Module)
   ```rst
   """
   Описание модуля
   =================================================

   Подробное описание модуля и его назначения.

   Пример использования
   ----------------------
   .. code-block:: python

       # Пример кода
   """
   ```

### Раздел функций (Functions)
   ```rst
   async def function_name(param: str, param1: Optional[str | dict | str] = None) -> dict | None:
       """
       Краткое описание функции.

       :param param: Описание параметра `param`.
       :type param: str
       :param param1: Описание параметра `param1`, по умолчанию `None`.
       :type param1: Optional[str | dict | str], optional
       :return: Описание возвращаемого значения, или `None`.
       :rtype: dict | None
       :raises SomeError: Описание ситуации, когда выбрасывается исключение `SomeError`.

       Пример:
       >>> function_name("test_param", "test_param1")
       {'test': 'value'}
       """
       ...
   ```

   **Описание**: Короткое описание функции.
   
   **Параметры**:
   - `param` (str): Описание параметра `param`.
   - `param1` (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию: `None`.
   
   **Возвращает**:
   - `dict | None`: Описание возвращаемого значения.
   
   **Исключения**:
   - `SomeError`: Описание случая, когда выбрасывается исключение `SomeError`.
   
### Раздел классов (Classes)
   ```rst
   class ClassName:
      """
      Описание класса

      :ivar attribute1: Описание атрибута `attribute1`.
      :vartype attribute1: str
      """

      def __init__(self, attribute1: str):
         """
         Инициализация класса.

         :param attribute1: Описание параметра `attribute1`.
         :type attribute1: str
         """
         self.attribute1 = attribute1

      async def method_name(self) -> bool:
         """
         Описание метода `method_name`.

         :return: Описание возвращаемого значения.
         :rtype: bool
         """
         ...
   ```
   **Описание**: Короткое описание класса.
    
   **Атрибуты**:
    - `attribute1` (str): Описание атрибута `attribute1`.
    
   **Методы**:
    - `method_name`: Короткое описание метода.
  
    **Описание**: Описание метода.
    
    **Возвращает**:
    - `bool`: Описание возвращаемого значения.
