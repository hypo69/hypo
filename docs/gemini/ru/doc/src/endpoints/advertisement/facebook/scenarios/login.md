# Модуль `hypotez/src/endpoints/advertisement/facebook/scenarios/login.py`

## Обзор

Данный модуль предоставляет сценарий авторизации на платформе Facebook.  Он содержит функцию `login`, которая выполняет вход в систему, используя предоставленный драйвер для взаимодействия с веб-элементами.

## Описание

Модуль реализует логику входа на Facebook. Он использует данные авторизации из `gs.facebook_credentials` и локаторы элементов из файла `login.json` для выполнения процесса входа.

## Функции

### `login`

**Описание**: Функция выполняет вход на Facebook.

**Параметры**:
- `d` (Driver): Экземпляр драйвера для взаимодействия с веб-элементами.  Ожидается, что `Driver` поддерживает методы для работы с веб-элементами (например, `send_key_to_webelement`, `execute_locator`, `wait`).


**Возвращает**:
- `bool`: `True`, если авторизация прошла успешно, иначе `False`.

**Вызывает исключения**:
- `Exception`:  Возникает при ошибках, таких как:
    - некорректный ввод логина
    - некорректный ввод пароля
    - ошибки при нажатии кнопки входа.  Подробные сообщения об ошибках записываются в `logger`.


## Локаторы

Локаторы веб-элементов, используемые для авторизации, загружаются из файла `login.json` в переменную `locators`.  Файл `login.json` должен находиться в `gs.path.src/endpoints/advertisement/facebook/locators/login.json`.

## Использование

Для использования функции `login` необходимо создать экземпляр `Driver`, передать его в функцию `login`.

```python
from src.webdriver import Driver
from .login import login

# ... (Создание экземпляра драйвера) ...
driver = Driver()

if login(driver):
    print("Вход на Facebook выполнен успешно")
else:
    print("Ошибка входа на Facebook")
```

## Зависимости

- `gs`
- `pathlib`
- `typing`
- `src.webdriver`
- `src.utils` (функции `j_loads`, `j_loads_ns`, `j_dumps`)
- `src.logger`


## Примечания

Файл `login.json` должен содержать корректные локаторы для элементов входа на Facebook.  Если файл не найден или содержит некорректные данные, функция вернет `False`.
