# Модуль исключений для API AliExpress

## Обзор

Этот модуль содержит определения пользовательских исключений, используемых при взаимодействии с API AliExpress. Эти исключения позволяют более точно обрабатывать ошибки, возникающие в процессе работы с API, и предоставляют информацию о причинах их возникновения.

## Подробнее

Модуль определяет базовый класс `AliexpressException`, от которого наследуются все остальные исключения. Это обеспечивает единообразный способ обработки ошибок, специфичных для API AliExpress. Каждое исключение предназначено для конкретной ситуации, что позволяет разработчикам точно определять и обрабатывать различные типы ошибок.

## Классы

### `AliexpressException`

**Описание**: Базовый класс для всех исключений API AliExpress.

**Принцип работы**:
   - Класс `AliexpressException` является родительским для всех пользовательских исключений, связанных с API AliExpress.
   - При инициализации принимает аргумент `reason`, содержащий описание причины возникновения исключения.
   - Метод `__str__` возвращает строковое представление причины исключения.

**Атрибуты**:
   - `reason` (str): Описание причины возникновения исключения.

**Методы**:
   - `__init__(self, reason: str)`: Конструктор класса, принимающий причину исключения.
   - `__str__(self) -> str`: Возвращает строковое представление исключения (причину).

### `InvalidArgumentException`

**Описание**: Исключение, возникающее при некорректных аргументах.

**Принцип работы**:
   - Наследуется от `AliexpressException`.
   - Используется для обозначения ситуаций, когда переданные в API методы аргументы не соответствуют ожидаемым значениям или формату.

### `ProductIdNotFoundException`

**Описание**: Исключение, возникающее, когда ID продукта не найден.

**Принцип работы**:
   - Наследуется от `AliexpressException`.
   - Указывает на то, что запрошенный ID продукта не существует в базе данных AliExpress.

### `ApiRequestException`

**Описание**: Исключение, возникающее, когда запрос к API AliExpress завершается неудачей.

**Принцип работы**:
   - Наследуется от `AliexpressException`.
   - Обозначает проблемы, связанные с самим запросом к API, такие как сетевые ошибки или недоступность сервера.

### `ApiRequestResponseException`

**Описание**: Исключение, возникающее, когда ответ от API содержит невалидные данные.

**Принцип работы**:
   - Наследуется от `AliexpressException`.
   - Указывает на то, что полученный от API ответ не соответствует ожидаемому формату или структуре.

### `ProductsNotFoudException`

**Описание**: Исключение, возникающее, когда продукты не найдены.

**Принцип работы**:
   - Наследуется от `AliexpressException`.
   - Возникает в случае, если по заданным критериям не было найдено ни одного продукта.

### `CategoriesNotFoudException`

**Описание**: Исключение, возникающее, когда категории не найдены.

**Принцип работы**:
   - Наследуется от `AliexpressException`.
   - Указывает на то, что запрошенные категории не существуют или не могут быть найдены.

### `InvalidTrackingIdException`

**Описание**: Исключение, возникающее, когда ID отслеживания недействителен.

**Принцип работы**:
   - Наследуется от `AliexpressException`.
   - Возникает, когда предоставленный ID отслеживания отсутствует или имеет неверный формат.
