# Модуль `src.suppliers.wallmart.graber`

## Обзор

Модуль `src.suppliers.wallmart.graber` предназначен для сбора значений полей на странице товара сайта `wallmart.com`. Он наследуется от родительского класса `Graber` и переопределяет его методы для нестандартной обработки полей, если это необходимо. Модуль также предоставляет возможность выполнения предварительных действий перед отправкой запроса к веб-драйверу через декоратор.

## Подробней

Этот модуль является частью системы для сбора данных о товарах с различных онлайн-платформ. Он специализируется на `wallmart.com`, адаптируя общие механизмы сбора данных, предоставляемые родительским классом `Graber`, к особенностям структуры страниц товаров на `wallmart.com`. Использование декоратора позволяет выполнять общие предварительные действия, такие как закрытие всплывающих окон, до фактического извлечения данных, обеспечивая более надежный и предсказуемый процесс сбора.

## Классы

### `Graber`

**Описание**: Класс `Graber` предназначен для захвата информации о товарах с сайта `wallmart.com`. Он наследуется от класса `Graber` (который, предположительно, является общим классом для граберов различных поставщиков) и адаптирует его функциональность для `wallmart.com`.

**Как работает класс**:
1.  **Инициализация**: При инициализации класса устанавливается префикс поставщика (`supplier_prefix`) как `'wallmart'`. Этот префикс используется для идентификации поставщика в системе.
2.  **Наследование**: Класс наследует функциональность от родительского класса `Graber`, который предоставляет общие методы для взаимодействия с веб-драйвером и сбора данных.
3.  **Контекст**: Класс использует `Context` для хранения глобальных настроек и параметров, таких как локаторы для выполнения действий в браузере. В частности, `Context.locator_for_decorator` используется для определения локатора, который будет выполнен в декораторе `@close_pop_up` перед выполнением основной логики сбора данных.

**Методы**:

*   `__init__`: Инициализирует экземпляр класса `Graber`.

    **Параметры**:

    *   `driver` (Driver): Экземпляр веб-драйвера для управления браузером.
    *   `lang_index` (int): Индекс языка, используемый для локализации (если применимо).

## Функции

### `__init__`

```python
def __init__(self, driver: Driver, lang_index: int):
    """Инициализация класса сбора полей товара."""
```

**Описание**: Инициализирует экземпляр класса `Graber`, устанавливая префикс поставщика и вызывая конструктор родительского класса.

**Как работает функция**:
1. Устанавливает атрибут `supplier_prefix` равным `'wallmart'`. Это используется для идентификации источника данных как Wallmart.
2. Вызывает метод `__init__` родительского класса `Grbr` с соответствующими аргументами, передавая префикс поставщика, драйвер и индекс языка.
3. Устанавливает `Context.locator_for_decorator` в `None`. Это гарантирует, что декоратор `@close_pop_up` не будет выполнять никаких действий по умолчанию, если не указан конкретный локатор для выполнения.

**Параметры**:

*   `driver` (Driver): Экземпляр веб-драйвера, используемый для взаимодействия с веб-страницей Wallmart.
*   `lang_index` (int): Индекс языка для выбора языка интерфейса Wallmart.

**Примеры**:

```python
driver = Driver()
graber = Graber(driver, 0)
print(graber.supplier_prefix)  # Вывод: wallmart
