=========================================================================================

Описание функциональности и правил для генерации, анализа и улучшения кода. Направлено на обеспечение последовательного и читаемого стиля кодирования, соответствующего требованиям.

### **Общие принципы и основные требования**

#### **1. Общие указания**
- Соблюдайте четкий и понятный стиль кодирования.
- Все изменения должны быть обоснованы и соответствовать установленным требованиям.
- Используйте линтеры и статические анализаторы кода, такие как `flake8`, `pylint`, `mypy`, для проверки соответствия кода стандартам.

#### **2. Комментарии и документация**
- Используйте `#` для внутренних комментариев.
- Вся документация для функций, методов и классов должна быть написана в формате **Sphinx**, основанном на **reStructuredText (reST)**.
- Комментарии и документация должны быть четкими, лаконичными и точными, и должны непосредственно предшествовать описываемому блоку кода.
- Документация должна включать в себя информацию о параметрах, возвращаемых значениях, исключениях и примерах использования.
- Избегайте расплывчатых терминов, таких как *«получить»* или *«делать»*. Вместо этого используйте точные термины, такие как *«извлечь»*, *«проверить»*, *«выполнить»*.

#### **3. Форматирование кода**
- Используйте одинарные кавычки (`'`) для строк и символов.
- Добавляйте пробелы вокруг операторов (`=`, `+`, `-` и т.д.).
- Все параметры должны быть с аннотациями типов.
- Избегайте использования `Union` там, где это возможно.
- Всегда добавляйте пробелы вокруг оператора `=`, чтобы повысить читаемость.
  - **Неправильно**: `x=5`
  - **Правильно**: `x = 5`

#### **4. Логирование**
- Используйте модуль `logger` из `src.logger.logger`.
- Ошибки должны логироваться с использованием `logger.error`.

#### **5. Работа с JSON**
- Для чтения JSON или конфигурационных файлов, используйте `j_loads` или `j_loads_ns` из `src.utils.jjson` вместо стандартного `json.load`.
- **Пример:**
```python
# Неправильно:
# with open('config.json', 'r', encoding='utf-8') as f:
#     data = json.load(f)

# Правильно:
data = j_loads('config.json')
```

#### **6. Сохранение комментариев**
- Все существующие комментарии, начинающиеся с `#`, должны быть сохранены без изменений.
- Если комментарий кажется устаревшим или неясным, не изменяйте его. Вместо этого отметьте его в разделе «Изменения».

#### **7. Обработка `...`**
- Оставляйте `...` как указатели в коде без изменений.
- Не документируйте строки с `...`.

#### **8. Формат ответов в Markdown**
- Все ответы должны быть выполнены в формате **Markdown**.
- Структура ответа должна включать:
  - **Улучшенный оптимизированный полный код**: Блок с улучшенным кодом, форматированным и дополненным комментариями в формате RST.
  - **Изменения**: Подробный список внесенных изменений с объяснениями, включая строки с изменениями (например, "изменена строка 5: ...").
- Код должен быть обернут в соответствующие теги подсветки синтаксиса (например, `python`, `markdown`, `json`).
