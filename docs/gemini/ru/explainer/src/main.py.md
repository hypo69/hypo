## <алгоритм>

**Пошаговая блок-схема:**

1.  **Начало:** Запуск скрипта `main.py`.
2.  **Разбор аргументов командной строки:** Функция `main()` использует `argparse` для разбора аргументов.
    *   **Пример:**
        *   `python main.py` - аргументы отсутствуют, переходит к шагу 4.
        *   `python main.py --help` - аргумент `--help` присутствует, переходит к шагу 3.
3.  **Вывод справки:** Если аргумент `--help` присутствует, вызывается функция `show_help()`, которая выводит на экран список доступных команд и завершает выполнение.
    *   **Пример:** Выводит текст справки: "Available commands:\n1. Run script 1 - Executes script 1.\n2. Run script 2 - Executes script 2.\n3. --help - Displays this help menu.\n4. exit - Exits the program.\n".
4.  **Запуск интерактивного меню:** Если аргумент `--help` отсутствует, вызывается функция `interactive_menu()`.
5.  **Отображение меню:**  Функция `interactive_menu()` выводит на экран меню с вариантами выбора:
    *   "1. Run script 1"
    *   "2. Run script 2"
    *   "3. --help - Show command list."
    *   "4. exit - Exit the program."
6.  **Ввод пользователя:** Ожидание ввода пользователя через функцию `input()`.
    *   **Пример:** Пользователь вводит "1".
7.  **Обработка выбора:** Функция `interactive_menu()` анализирует ввод пользователя:
    *   Если ввод "1", вызывается функция `script1()`, которая выводит "Script 1 started".
    *   Если ввод "2", вызывается функция `script2()`, которая выводит "Script 2 started".
    *   Если ввод "3" или "--help" (в любом регистре), вызывается функция `show_help()`.
    *   Если ввод "exit" (в любом регистре), выводится сообщение "Exiting the program." и цикл завершается, программа завершается.
    *   Если ввод не соответствует ни одному из вариантов, выводится сообщение об ошибке через `logger.error()`.
8.  **Повтор меню:** После выполнения выбранного действия, меню отображается снова (возвращаемся к шагу 5) до тех пор, пока пользователь не выберет "exit".
9.  **Завершение:** Программа завершает выполнение.

**Объяснение зависимостей:**

*   `argparse`: Этот модуль используется для разбора аргументов командной строки. Он позволяет скрипту принимать аргументы, такие как `--help`, и обрабатывать их.
*  `src.utils.jjson`: Импортируются функции `j_loads`, `j_loads_ns`.
*  `src.logger.logger`: Импортируется объект `logger` для регистрации ошибок.

## <объяснение>

### Импорты:

*   `import argparse`: Используется для обработки аргументов командной строки.
*   `from src.utils.jjson import j_loads, j_loads_ns`: Импортирует функции `j_loads` и `j_loads_ns` из модуля `src.utils.jjson`. Эти функции, вероятно, используются для загрузки данных JSON, но в текущем коде не используются напрямую.
*   `from src.logger.logger import logger`: Импортирует объект `logger` из модуля `src.logger.logger`. Он используется для вывода сообщений об ошибках.

### Функции:

*   **`script1()`:**
    *   **Назначение:** Запускает скрипт 1.
    *   **Аргументы:** Нет.
    *   **Возвращаемое значение:** Нет.
    *   **Пример:** Выводит на экран "Script 1 started". В реальной реализации здесь будет код скрипта 1.
*   **`script2()`:**
    *   **Назначение:** Запускает скрипт 2.
    *   **Аргументы:** Нет.
    *   **Возвращаемое значение:** Нет.
    *   **Пример:** Выводит на экран "Script 2 started". В реальной реализации здесь будет код скрипта 2.
*   **`show_help()`:**
    *   **Назначение:** Выводит на экран справку по доступным командам.
    *   **Аргументы:** Нет.
    *   **Возвращаемое значение:** Нет.
    *   **Пример:** Выводит на экран текст с описанием команд.
*   **`interactive_menu()`:**
    *   **Назначение:** Обеспечивает интерактивное меню для выбора и запуска скриптов.
    *   **Аргументы:** Нет.
    *   **Возвращаемое значение:** Нет.
    *   **Пример:** Выводит на экран меню и ожидает ввода пользователя, запускает выбранную функцию или выводит сообщение об ошибке.
*   **`main()`:**
    *   **Назначение:** Основная функция для обработки аргументов командной строки и запуска интерактивного меню.
    *   **Аргументы:** Нет.
    *   **Возвращаемое значение:** Нет.
    *   **Пример:**  Разбирает аргументы командной строки, вызывает `show_help()`, если есть аргумент `--help` или вызывает `interactive_menu()`.

### Переменные:

*   `parser`: Объект `argparse.ArgumentParser` для парсинга аргументов.
*   `args`: Объект, содержащий аргументы, переданные скрипту.
*   `choice`: Переменная, в которой хранится ввод пользователя.

### Потенциальные ошибки и области для улучшения:

*   **Отсутствие реализации скриптов:** Функции `script1` и `script2` в текущем виде просто выводят сообщение о начале работы. Их необходимо дополнить реальным кодом.
*   **Обработка ошибок ввода:** В текущей версии обрабатываются только некорректные команды. Возможно, стоит добавить обработку других типов ошибок ввода, например, пустую строку.
*   **Модульность:** Можно рассмотреть вынесение логики работы меню и скриптов в отдельные модули или классы для лучшей читаемости и масштабируемости.
*   **Тестирование:** Необходимо добавить тесты для проверки корректности работы функций и обработки различных сценариев.

### Цепочка взаимосвязей:

*   `main.py` зависит от `argparse` для обработки аргументов командной строки.
*   `main.py` вызывает `src.utils.jjson`, `src.logger.logger` для логирования ошибок, однако не использует их функционал в полной мере.
*   `main.py` вызывает функции `script1`, `script2`, `show_help` и `interactive_menu`, которые составляют основную логику работы программы.
*   `logger` из `src.logger.logger` используется для логирования ошибок.

Таким образом, `main.py` является основной точкой входа в программу, которая управляет выполнением скриптов через интерактивное меню и обрабатывает аргументы командной строки.
