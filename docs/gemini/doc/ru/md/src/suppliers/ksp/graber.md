# Модуль hypotez/src/suppliers/ksp/graber.py

## Обзор

Модуль `graber.py` содержит класс `Graber`, предназначенный для сбора данных о товарах с сайта `ksp.co.il`. Класс наследуется от базового класса `Graber` и предоставляет методы для извлечения различных полей товара.  Включает в себя обработку всплывающих окон, а также возможность переопределения стандартных методов для специфической обработки отдельных полей.

## Классы

### `Graber`

**Описание**: Класс `Graber` реализует логику сбора данных о товаре с сайта `ksp.co.il`. Наследуется от родительского класса `Grbr` и содержит функции для извлечения различных данных о товаре.

**Атрибуты**:

* `supplier_prefix`: Префикс поставщика (`str`).

**Методы**:

#### `__init__(self, driver: Driver)`

**Описание**: Инициализирует экземпляр класса.

**Параметры**:

* `driver (Driver)`: Экземпляр класса `Driver` для работы с веб-драйвером.

#### `grab_page(self, driver: Driver) -> ProductFields`

**Описание**: Асинхронная функция для извлечения данных о товаре.

**Параметры**:

* `driver (Driver)`: Экземпляр класса `Driver` для работы с веб-драйвером.

**Возвращает**:

* `ProductFields`: Объект, содержащий собранные данные о товаре.


## Функции

### `close_pop_up(value: Any = None) -> Callable`

**Описание**: Создает декоратор для закрытия всплывающих окон перед выполнением основной логики функции.

**Параметры**:

* `value (Any, optional):` Дополнительное значение для декоратора. По умолчанию `None`.

**Возвращает**:

* `Callable`: Декоратор, оборачивающий функцию.


### `fetch_all_data(**kwards)`

**Описание**: Асинхронная вспомогательная функция для вызова функций сбора отдельных полей.


**Параметры**:
Переменное количество именованных аргументов (`**kwards`):
- `id_product`, `additional_shipping_cost`, `delivery_in_stock`, `active`, `additional_delivery_times` и т.д.: идентификатор продукта и другие поля для сбора (строка).
Значение полей по умолчанию – пустая строка.

**Возвращает**:
Данные о товаре в виде объекта `ProductFields`.


**Примечания**:


В коде присутствует большое количество функций, таких как `id_product`, `description_short`, `name` и др. – это функции сбора конкретных полей товара.  Для каждой из них необходимо добавить описание в соответствии с заданным шаблоном.
