## Введение

Класс `ProductFields` — это Python-класс, который инкапсулирует структуру и поведение полей товаров в базе данных PrestaShop. Он разработан для упрощения процесса создания, обновления и управления данными товаров, предоставляя четкий и последовательный интерфейс для взаимодействия с полями товаров.

Класс особенно полезен для разработчиков, работающих с API PrestaShop, так как он гарантирует, что все обязательные поля будут правильно отформатированы и проверены перед отправкой в API.

## Инициализация класса

### Метод `__init__`

Метод `__init__` инициализирует класс `ProductFields`, загружая список полей товаров и настраивая значения по умолчанию для полей. Он также инициализирует объект `SimpleNamespace` для хранения полей товаров и вспомогательный словарь для дополнительных полей.

```python
def __init__(self):
    self.product_fields_list = self._load_product_fields_list()
    self.language = {'en': 1, 'he': 2, 'ru': 3}
    self.presta_fields = SimpleNamespace(**{key: None for key in self.product_fields_list})
    self.assist_fields_dict = {
        'default_image_url': '', 
        'images_urls': []
    }
    self._payload()
```

**Как работает метод `__init__`**:

Метод `__init__` выполняет инициализацию экземпляра класса `ProductFields`.

1.  **Загрузка списка полей товара**:
    *   Вызывает метод `_load_product_fields_list` для получения списка полей товара.
    *   Сохраняет полученный список в атрибуте `product_fields_list`.
2.  **Инициализация словаря языков**:
    *   Определяет словарь `language`, который сопоставляет строковые обозначения языков (`'en'`, `'he'`, `'ru'`) с их числовыми идентификаторами (1, 2, 3 соответственно).
    *   Этот словарь используется для указания языка при работе с многоязычными полями.
3.  **Создание `SimpleNamespace` для полей PrestaShop**:
    *   Создает объект `SimpleNamespace` с именем `presta_fields`.
    *   Инициализирует атрибуты этого объекта, используя список полей товара (`self.product_fields_list`).
    *   Каждому полю присваивается значение `None`. `SimpleNamespace` позволяет обращаться к полям как к атрибутам объекта (например, `self.presta_fields.name`).
4.  **Инициализация вспомогательного словаря**:
    *   Создает словарь `assist_fields_dict` для хранения дополнительных полей, таких как URL изображения по умолчанию (`default_image_url`) и список URL изображений (`images_urls`).
    *   Эти поля могут использоваться для хранения информации, которая не входит в стандартные поля товара PrestaShop.
5.  **Загрузка значений по умолчанию**:
    *   Вызывает метод `_payload` для загрузки значений по умолчанию для полей товара.
    *   Этот метод заполняет атрибуты экземпляра класса значениями из файла конфигурации.

### Метод `_load_product_fields_list`

Этот метод загружает список полей товаров из текстового файла. Файл должен содержать одно имя поля на строку.

```python
def _load_product_fields_list(self) -> List[str]:
    return read_text_file(Path(gs.path.src, 'product', 'product_fields', 'fields_list.txt'), as_list=True)
```

**Как работает функция `_load_product_fields_list`**:

Функция `_load_product_fields_list` предназначена для чтения списка полей товара из текстового файла.

1.  **Определение пути к файлу**:
    *   Использует `Path` для создания пути к файлу `fields_list.txt`, который находится в директории `src/product/product_fields` относительно базового пути, хранящегося в `gs.path.src`.
2.  **Чтение содержимого файла**:
    *   Вызывает функцию `read_text_file` для чтения содержимого файла.
    *   Передает путь к файлу и флаг `as_list=True`, что указывает на необходимость возврата содержимого файла в виде списка строк, где каждая строка представляет собой поле товара.
3.  **Возврат списка полей**:
    *   Возвращает список строк, содержащих имена полей товара.

### Метод `_payload`

Этот метод загружает значения по умолчанию для полей товаров из JSON-файла. Если файл не найден или не может быть загружен, выводится сообщение об ошибке.

```python
def _payload(self) -> bool:
    data = j_loads(Path(gs.path.src, 'product', 'product_fields', 'product_fields_default_values.json'))
    if not data:
        logger.debug(f"Ошибка загрузки полей из файла {gs.path.src}/product/product_fields/product_fields_default_values.json")
        return False
    for name, value in data.items():
        setattr(self, name, value)
    return True
```

**Как работает функция `_payload`**:

Функция `_payload` загружает значения по умолчанию для полей товаров из JSON-файла.

1.  **Загрузка данных из JSON-файла**:

    *   Использует `j_loads` для чтения данных из файла `product_fields_default_values.json`, расположенного в директории `src/product/product_fields` относительно базового пути, хранящегося в `gs.path.src`.

2.  **Обработка ошибок загрузки**:

    *   Проверяет, удалось ли загрузить данные из файла. Если `data` равно `None` (что указывает на ошибку при загрузке файла), то:
        *   Логирует сообщение об ошибке с использованием `logger.debug`, указывая на то, что не удалось загрузить поля из файла.
        *   Возвращает `False`, указывая на неудачное выполнение функции.

3.  **Установка значений по умолчанию**:

    *   Если данные успешно загружены, функция перебирает все пары ключ-значение в словаре `data`.
    *   Для каждой пары `name` (имя поля) и `value` (значение по умолчанию):
        *   Использует `setattr(self, name, value)` для установки значения атрибута экземпляра класса `self` с именем `name` в значение `value`.

4.  **Возврат значения**:

    *   После успешной установки всех значений по умолчанию функция возвращает `True`.

## Поля товара

### Одноязычные поля

Одноязычные поля — это поля, которые не требуют перевода и хранятся на одном языке. Примеры включают `id_product`, `id_supplier`, `id_manufacturer` и т.д.

#### Пример: `id_product`

```python
@property
def id_product(self) -> Optional[int]:
    return self.presta_fields.id_product

@id_product.setter
def id_product(self, value: int = None):
    try:
        self.presta_fields.id_product = value
    except ProductFieldException as ex:
        logger.error(f"""Ошибка заполнения поля: 'ID' данными {value}
        Ошибка: """, ex)
        return
```

**Как работает property `id_product`**:

Это свойство класса `ProductFields`, которое позволяет получать и устанавливать значение идентификатора продукта (`id_product`).

1.  **getter (@property)**:

    *   При обращении к `product.id_product` возвращает значение `self.presta_fields.id_product`.
    *   Если значение не установлено, возвращает `None`.

2.  **setter (@id_product.setter)**:

    *   При присвоении значения `product.id_product = value` выполняется следующее:
    *   Пытается установить значение `value` атрибуту `self.presta_fields.id_product`.
    *   Если происходит исключение `ProductFieldException`, то:
        *   Логирует сообщение об ошибке с использованием `logger.error`, указывая на то, что произошла ошибка при заполнении поля `'ID'` данными `value`.
        *   Возвращает `None`, чтобы прервать выполнение операции.

### Многоязычные поля

Многоязычные поля — это поля, которые требуют перевода и хранятся на нескольких языках. Примеры включают `name`, `description`, `meta_title` и т.д.

#### Пример: `name`

```python
@property
def name(self):
    return self.presta_fields.name or ''

@name.setter
def name(self, value: str, lang:str = 'en') -> bool:
    try:
        self.presta_fields.name: dict = {'language':
                                                    [
                                                        {'attrs':{'id':self.language[lang]}, 'value': value},
                                                    ]
                                                 }
        return True
    except ProductFieldException as ex:
        logger.error(f"""Ошибка заполнения поля: 'name' данными {value}
        Ошибка: """, ex)
        return
```

**Как работает property `name`**:

Это свойство класса `ProductFields`, которое позволяет получать и устанавливать значение имени продукта (`name`) с поддержкой многоязычности.

1.  **getter (@property)**:
    *   При обращении к `product.name` возвращает значение `self.presta_fields.name`.
    *   Если значение не установлено, возвращает пустую строку `''`.
2.  **setter (@name.setter)**:
    *   При присвоении значения `product.name = value, lang='en'` выполняется следующее:
    *   Пытается установить значение `value` атрибуту `self.presta_fields.name`.
        *   Создает словарь, который содержит информацию о языке и значении имени продукта.
            *   `'language'`: список, содержащий один словарь с атрибутами языка и значением.
            *   `'attrs'`: словарь, содержащий идентификатор языка, полученный из `self.language[lang]`.
            *   `'value'`: значение имени продукта `value`.
        *   Устанавливает значение `self.presta_fields.name` равным созданному словарю.
    *   Если происходит исключение `ProductFieldException`, то:
        *   Логирует сообщение об ошибке с использованием `logger.error`, указывая на то, что произошла ошибка при заполнении поля `'name'` данными `value`.
        *   Возвращает `None`, чтобы прервать выполнение операции.
    *   В случае успешного выполнения возвращает `True`.

## Ассоциации

Ассоциации используются для связывания товаров с другими сущностями, такими как категории, производители и поставщики. Свойство `associations` позволяет устанавливать и получать эти ассоциации.

#### Пример: `associations`

```python
@property
def associations(self) -> Optional[Dict]:
    return self.presta_fields.associations or None

@associations.setter
def associations(self, value: Dict[str, Optional[str]]):
    self.presta_fields.associations = value
```

**Как работает property `associations`**:

Это свойство класса `ProductFields`, которое позволяет получать и устанавливать ассоциации продукта с другими сущностями.

1.  **getter (@property)**:
    *   При обращении к `product.associations` возвращает значение `self.presta_fields.associations`.
    *   Если значение не установлено, возвращает `None`.
2.  **setter (@associations.setter)**:
    *   При присвоении значения `product.associations = value` устанавливает значение атрибута `self.presta_fields.associations` равным `value`.

## Значения по умолчанию

Значения по умолчанию для полей товаров могут быть загружены из JSON-файла с помощью метода `_payload`. Этот метод гарантирует, что все поля имеют значение по умолчанию, которое можно переопределить по необходимости.

## Обработка ошибок

Класс включает надежную обработку ошибок для перехвата и регистрации любых исключений, возникающих при установке полей товаров. Это гарантирует, что любые проблемы будут зарегистрированы и могут быть устранены разработчиком.

#### Пример: Обработка ошибок в `id_product`

```python
@id_product.setter
def id_product(self, value: int = None):
    try:
        self.presta_fields.id_product = value
    except ProductFieldException as ex:
        logger.error(f"""Ошибка заполнения поля: 'ID' данными {value}
        Ошибка: """, ex)
        return
```

## Примеры использования

### Пример 1: Установка одноязычного поля

```python
product = ProductFields()
product.id_product = 123
print(product.id_product)  # Вывод: 123
```

### Пример 2: Установка многоязычного поля

```python
product = ProductFields()
product.name = "Product Name", lang='en'
print(product.name)  # Вывод: {'language': [{'attrs': {'id': 1}, 'value': 'Product Name'}]}
```

### Пример 3: Установка ассоциаций

```python
product = ProductFields()
product.associations = {'categories': [{'id': 2}, {'id': 3}]}
print(product.associations)  # Вывод: {'categories': [{'id': 2}, {'id': 3}]}
```
