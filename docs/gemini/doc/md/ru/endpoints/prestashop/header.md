# Модуль logger.header

## Обзор

Модуль `header.py` определяет корневой путь к проекту. Все импорты строятся относительно этого пути.  В дальнейшем предполагается перенести эту функцию в системную переменную.

## Функции

### `get_project_root`

**Описание**: Находит корневой каталог проекта, начиная с текущего каталога файла.  Ищет вверх по иерархии каталогов, останавливаясь на первом каталоге, содержащем один из указанных файлов-маркеров.

**Параметры**:

- `marker_files` (tuple): Список имен файлов или каталогов, используемых для определения корневого каталога проекта.  По умолчанию `('pyproject.toml', 'requirements.txt', '.git')`.


**Возвращает**:

- `Path`: Путь к корневому каталогу проекта, если найден, в противном случае - путь к каталогу, где находится текущий файл.


**Пример использования**:

```python
root_path = get_project_root()
print(root_path)
```

## Переменные

### `__root__`

**Описание**: Корневой каталог проекта, полученный функцией `get_project_root`.

**Тип**: `Path`

**Примечание**: Для правильного использования значение устанавливается функцией `get_project_root()`.


### `settings`

**Описание**: Словарь настроек проекта.

**Тип**: `dict` | `None`

**Примечание**: Загружается из файла `settings.json`.


### `doc_str`

**Описание**: Текст документации проекта (README.MD).

**Тип**: `str` | `None`

**Примечание**: Загружается из файла `README.MD`.


### `__project_name__`

**Описание**: Название проекта.

**Тип**: `str`

**Примечание**: Получается из настроек `settings`.  Используется значение по умолчанию `'hypotez'`, если настройки не найдены или не содержат ключ `project_name`.


### `__version__`

**Описание**: Версия проекта.

**Тип**: `str`

**Примечание**: Получается из настроек `settings`. Используется значение по умолчанию `''`, если настройки не найдены или не содержат ключ `version`.


### `__doc__`

**Описание**: Текст документации проекта.

**Тип**: `str`

**Примечание**: Получается из переменной `doc_str`.


### `__details__`

**Описание**: Дополнительные детали проекта.

**Тип**: `str`


### `__author__`

**Описание**: Автор проекта.

**Тип**: `str`

**Примечание**: Получается из настроек `settings`. Используется значение по умолчанию `''`, если настройки не найдены или не содержат ключ `author`.


### `__copyright__`

**Описание**: Авторские права проекта.

**Тип**: `str`

**Примечание**: Получается из настроек `settings`. Используется значение по умолчанию `''`, если настройки не найдены или не содержат ключ `copyrihgnt`.


### `__cofee__`

**Описание**: Ссылка на площадку для поддержки автора.

**Тип**: `str`

**Примечание**: Получается из настроек `settings`. Используется значение по умолчанию `"Treat the developer to a cup of coffee for boosting enthusiasm in development: https://boosty.to/hypo69"`, если настройки не найдены или не содержат ключ `cofee`.


## Обработка исключений

При чтении файлов `settings.json` и `README.MD` возможны следующие исключения:

- `FileNotFoundError`: Файл не найден.
- `json.JSONDecodeError`: Ошибка декодирования JSON.


В таких случаях значения соответствующих переменных устанавливаются в `None`.
