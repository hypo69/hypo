# Инструкция для бота

## Обзор

Этот файл содержит инструкции для бота, определяющие доступные команды и их синтаксис. Он используется для отображения справочной информации о командах, которые бот может выполнять.

## Подробней

Файл представляет собой набор строк, каждая из которых описывает одну команду бота. Описание включает синтаксис команды и краткое пояснение ее назначения. Этот файл используется, когда пользователь запрашивает список доступных команд, например, с помощью команды `!instruction`.

## Функции

### `!hi`

**Описание**: Приветствует пользователя.

**Примеры**:
- `!hi`: Выводит приветствие для пользователя.

### `!train`

**Описание**: Обучает модель с использованием предоставленных данных.

**Параметры**:
- `data` (str): Файл с данными для обучения.
- `data_dir` (str): Каталог с данными для обучения.
- `positive` (str): Положительные примеры для обучения.
- `attachment` (attachment): Файл, прикрепленный к сообщению, содержащий данные для обучения.

**Примеры**:
- `!train data.txt data_dir positive_examples`: Обучает модель, используя `data.txt` как файл с данными, `data_dir` как каталог с данными и `positive_examples` как положительные примеры.
- `!train data.json data_dir positive_examples attachment`: Обучает модель, используя прикрепленный файл с данными в дополнение к указанным параметрам.

### `!test`

**Описание**: Тестирует модель с использованием предоставленных тестовых данных в формате JSON.

**Параметры**:
- `test_data` (str): JSON строка, содержащая данные для тестирования.

**Примеры**:
- `!test '{"feature1": "value1", "feature2": "value2"}'`: Тестирует модель с использованием предоставленных JSON данных.

### `!archive`

**Описание**: Архивирует файлы в указанном каталоге.

**Параметры**:
- `directory` (str): Каталог, файлы которого необходимо архивировать.

**Примеры**:
- `!archive /path/to/directory`: Архивирует файлы, находящиеся в каталоге `/path/to/directory`.

### `!select_dataset`

**Описание**: Выбирает набор данных для обучения из указанного каталога.

**Параметры**:
- `path_to_dir_positive` (str): Путь к каталогу с положительными примерами.
- `positive` (str):  Имя файла или идентификатор, указывающий на положительные примеры в каталоге.

**Примеры**:
- `!select_dataset /path/to/positive_dir positive_data`: Выбирает набор данных из каталога `/path/to/positive_dir`, используя `positive_data` для определения положительных примеров.

### `!instruction`

**Описание**: Отображает это справочное сообщение с инструкциями.

**Примеры**:
- `!instruction`: Выводит текущее сообщение с инструкциями.
