# Документация для модуля управления сайтами на PrestaShop

## Обзор

Этот документ описывает структуру и способ работы с сайтами на платформе PrestaShop, а также хранение и использование ключей API для взаимодействия с этими сайтами. Документ предназначен для разработчиков, работающих с проектом `hypotez` и осуществляющих взаимодействие с сайтами PrestaShop через API.

## Подробнее

Этот модуль предоставляет информацию о сайтах PrestaShop, используемых в проекте `hypotez`, а также описывает, как безопасно хранить и использовать ключи API для взаимодействия с этими сайтами. Информация включает в себя список сайтов, детали о хранении ключей API в защищенной базе данных паролей и примеры использования API для получения данных.

## Хранение ключей API

### Описание

В этом разделе описывается, как безопасно хранить ключи API для каждого сайта PrestaShop.

**Принцип работы:**

Ключи API для каждого сайта хранятся в файле `credentials.kdbx`. Этот файл является защищенной базой данных паролей и содержит следующие данные для каждого сайта:

*   URL сайта
*   Ключ API
*   Дополнительные метаданные (если необходимо)

Для работы с ключами из файла используйте менеджер паролей, поддерживающий формат `.kdbx`, например, [KeePass](https://keepass.info/) или [KeePassXC](https://keepassxc.org/).

## Пример использования API

### Описание

В этом разделе приведен пример использования API для подключения к сайту PrestaShop и запроса данных.

#### Запрос данных через API

**Шаблон API-запроса:**

```bash
curl -X GET 'https://<URL_сайта>/api/<endpoint>' \\\
-H 'Authorization: Basic <base64(API_KEY)>'
```

**Объяснение параметров:**

*   `<URL_сайта>` — адрес сайта, например, `e-cat.co.il`.
*   `<endpoint>` — конечная точка API (например, `products`, `customers`).
*   `<API_KEY>` — ключ API, закодированный в формате Base64.

#### Пример вызова API

Для получения списка продуктов на сайте `e-cat.co.il`:

```bash
curl -X GET 'https://e-cat.co.il/api/products' \\\
-H 'Authorization: Basic <base64(API_KEY)>'
```

## Рекомендации по безопасности

### Описание

В этом разделе даны рекомендации по обеспечению безопасности при работе с ключами API.

*   Никогда не передавайте файл `credentials.kdbx` третьим лицам. ❗
*   Убедитесь, что файл находится в защищенном месте, доступном только вам. (папка `secrets` в корне проекта исключена из `git`)
*   Регулярно обновляйте ключи API и пароли для базы данных.
