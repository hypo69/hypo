<h1>Инструкция по генерации документации в формате HTML</h1>

<h2>Обзор</h2>
<p>Данная инструкция описывает процесс создания документации в формате HTML для Python-файлов. Документация должна быть структурированной и содержать подробные описания классов, функций и параметров.</p>

<h2>Формат документации</h2>
<p>Документация должна быть создана в формате HTML, используя теги заголовков и списков для структурирования информации.  Каждый файл должен содержать:</p>
<ul>
<li>Заголовок и краткое описание файла.</li>
<li>Раздел с оглавлением (TOC).</li>
<li>Подробные описания классов и функций, включающие:</li>
<ul>
<li>Описание аргументов с указанием типов и значений по умолчанию.</li>
<li>Описание возвращаемых значений с указанием типов.</li>
<li>Описание возможных исключений с указанием типов.</li>
</ul>
</ul>

<h2>Пример структуры документации</h2>

<pre><code>html
<h1>Название модуля</h1>

<h2>Обзор</h2>
<p>Краткое описание назначения модуля.</p>

<h2>Классы</h2>

<h3><code>ClassName</code></h3>

<p><strong>Описание</strong>: Краткое описание класса.</p>

<p><strong>Методы</strong>:</p>
<ul>
<li><code>methodName</code>: Краткое описание метода.</li>
</ul>

<h2>Функции</h2>

<h3><code>functionName</code></h3>

<p><strong>Описание</strong>: Краткое описание функции.</p>

<p><strong>Аргументы</strong>:</p>
<ul>
<li><code>param1</code> (<code>type</code>): Описание параметра <code>param1</code>.</li>
<li><code>param2</code> (<code>type</code>, <em>опционально</em>): Описание параметра <code>param2</code>. Значение по умолчанию: <code>значение</code>.</li>
</ul>

<p><strong>Возвращаемое значение</strong>:</p>
<ul>
<li><code>type</code>: Описание возвращаемого значения.</li>
</ul>

<p><strong>Исключения</strong>:</p>
<ul>
<li><code>ExceptionType</code>: Описание ситуации, в которой возникает исключение <code>ExceptionType</code>.</li>
</ul>
</code></pre>

<h2>Рекомендации</h2>
<ul>
<li>Используйте семантические теги HTML для структурирования данных.</li>
<li>Убедитесь, что документация соответствует заданным требованиям.</li>
<li>Используйте clear-cut Markdown-структуру для обеспечения легко читаемого документа.</li>
</ul>

<h2>Инструкции по реализации</h2>
<ul>
<li>Разобрать входной Python-код.</li>
<li>Извлечь информацию о функциях, классах и методах.</li>
<li>Сгенерировать соответствующие HTML-теги с использованием Python.</li>
<li>Форматировать документацию в соответствии с инструкциями.</li>
</ul>

<h2>Пример входного Python-файла</h2>

def my_function(param: str, param2: int = 10) -> str:
    """
    Args:
        param (str): Описание параметра param.

        str: Описание возвращаемого значения.

    Raises:
        TypeError: Описание ситуации, в которой возникает исключение TypeError.
    """
    return "Hello, " + param
