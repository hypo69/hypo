# Модуль `hypotez/src/logger/header.py`

## Обзор

Модуль `hypotez/src/logger/header.py` определяет корневой путь к проекту `hypotez`. Все импорты строятся относительно этого пути.  В будущем планируется перенести этот функционал в системные переменные.


## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная от текущей директории файла. Поиск происходит вверх по дереву директорий, и завершается при нахождении директории, содержащей один из маркеров (файлов или директорий).

**Параметры**:

- `marker_files` (tuple): Кортеж имен файлов или директорий, используемых для определения корневой директории проекта. По умолчанию `('pyproject.toml', 'requirements.txt', '.git')`.


**Возвращает**:

- `Path`: Путь к корневой директории проекта, если найдена, иначе - текущая директория файла.


**Вызывает исключения**:
- Нет.


## Переменные

### `__root__`

**Описание**:  Путь к корневой директории проекта, полученный функцией `set_project_root()`.


### `settings`

**Описание**: Словарь, содержащий настройки проекта, загруженные из файла `settings.json`.


### `doc_str`

**Описание**: Строка, содержащая описание проекта, извлечённая из файла `README.MD`.


### `__project_name__`

**Описание**: Название проекта. Извлекается из `settings`. По умолчанию `hypotez`.


### `__version__`

**Описание**: Версия проекта. Извлекается из `settings`. По умолчанию пустая строка.


### `__doc__`

**Описание**: Описание проекта, извлечённое из файла `README.MD`.


### `__details__`

**Описание**: Подробная информация о проекте. По умолчанию пустая строка.


### `__author__`

**Описание**: Автор проекта. Извлекается из `settings`. По умолчанию пустая строка.


### `__copyright__`

**Описание**: Авторские права проекта. Извлекается из `settings`. По умолчанию пустая строка.


### `__cofee__`

**Описание**: Ссылка для поддержки разработчика. Извлекается из `settings`. По умолчанию ссылка на Boosty.


## Обработка ошибок

- При чтении `settings.json` (используются `try`/`except` блоки):
  - `FileNotFoundError`: Если файл `settings.json` не найден.
  - `json.JSONDecodeError`: Если содержимое `settings.json` не является валидным JSON.
- При чтении `README.MD` (используются `try`/`except` блоки):
  - `FileNotFoundError`: Если файл `README.MD` не найден.
  - `json.JSONDecodeError`: Если содержимое `README.MD` не является валидным JSON.
