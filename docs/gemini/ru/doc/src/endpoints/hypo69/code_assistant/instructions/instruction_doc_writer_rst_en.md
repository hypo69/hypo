# ИНСТРУКЦИЯ ПО СОЗДАНИЮ ДОКУМЕНТАЦИИ ДЛЯ PYTHON-ФАЙЛОВ В ФОРМАТЕ RST

Данная инструкция описывает создание документации для Python-файлов в формате reStructuredText (rst), предназначенной для последующей компиляции с помощью Sphinx.

## Требования к Формату

* **Формат:** reStructuredText (.rst)
* **Заголовок и описание:** Каждый файл должен начинаться с заголовка и краткого описания содержимого.
* **Комментарии функций/классов:** Все функции и классы должны быть снабжены документацией в формате docstrings, соответствующем требованиям Sphinx.  Используйте `Args`, `Returns`, `Raises`.
* **Обработка исключений:** Используйте `ex` вместо `e` в блоках обработки исключений.
* **Система оглавления (TOC):**  Для индексирования и навигации используйте директиву `.. toctree::` в файле `index.rst` для создания дерева оглавления.  Файлы модулей должны ссылаться на index.rst.
* **Документирование:** Используйте Sphinx директивы `.. automodule::`, `.. autofunction::`, `.. autoclass::`  для автоматической генерации документации для классов и функций.
* **Директивы Sphinx:** Добавьте директивы `:members:`, `:undoc-members:`, `:show-inheritance:` для генерации списка членов класса/модуля, недокументированных элементов и наследования.
* **Уровни заголовков:** Используйте заголовки первого уровня (`=`), второго уровня (`-`), третьего уровня (`~`), и четвёртого уровня (`^`).
* **Структура файла:** Файл index.rst  должен содержать ссылки на все файлы модулей.


## Примеры

**Файл `index.rst` (Пример):**

```rst
Module Documentation
====================

.. toctree::
   :maxdepth: 2

   module1
   module2
   module3

```

**Файл `module1.py` (Пример):**

```python
def my_function(param: str, param1: int = 0) -> str:
    """
    Args:
        param (str): Описание параметра param.
        param1 (int, optional): Описание параметра param1. По умолчанию 0.

    Returns:
        str:  Описание возвращаемого значения.

    Raises:
        ValueError: Описание ситуации, в которой возникает исключение ValueError.
    """
    return param + str(param1)
```

**Файл `module1.rst` (Пример):**

```rst
Module 1 Documentation
=======================

.. automodule:: module1
    :members:
    :undoc-members:
    :show-inheritance:
```

**Важные моменты:**

* Убедитесь, что у вас установлен Sphinx и его зависимости.
*  Используйте правильный синтаксис rst для всех элементов.
*  Для корректной генерации документации убедитесь, что в файлах `module*.rst` используются ссылки на файлы модулей `module*.py`.
*  Файлы `module*.rst` должны быть расположены в той же папке, что и соответствующие файлы `module*.py`.

## Создание документации

После подготовки файлов `module*.py` и `module*.rst`, используйте Sphinx для генерации HTML документации.

```bash
sphinx-build -b html . _build
```

Это сгенерирует HTML файлы в папке `_build/html`.  Откройте `_build/html/index.html` в браузере, чтобы просмотреть созданную документацию.
