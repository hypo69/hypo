# Инструкция по написанию тестов `pytest`

## Обзор

Данная инструкция описывает, как писать тесты для Python-кода с использованием библиотеки `pytest`.  Тесты должны покрывать основные функции, методы или классы, проверяя их корректность.  Важно включать граничные случаи и обработку исключений.

## Требования

1. **Имена тестов:** Используйте ясные и описательные имена функций тестов, отражающие их цель.
2. **Изолированность:** Убедитесь, что все тесты изолированы и не зависят друг от друга.
3. **Разнообразие сценариев:** Рассмотрите различные сценарии, включая:
    - Валидные входные данные.
    - Невалидные или неожиданные входные данные, где это применимо.
    - Граничные или пороговые случаи.
4. **Тестирование исключений:** Используйте `pytest.raises` для тестирования исключений.
5. **Фикстуры:** Если для функций необходимы фикстуры, определите их отдельно.
6. **Комментарии:** Добавьте комментарии, объясняющие логику тестовых случаев.

## Структура примера тестов

```python
import pytest

# Определения фикстур (если нужно)
@pytest.fixture
def example_data():
    """Предоставляет тестовые данные для функции."""
    return {...}

# Тесты для функции 1
def test_function1_valid_input():
    """Проверяет корректное поведение с валидными входными данными."""
    ...

def test_function1_invalid_input():
    """Проверяет корректную обработку невалидных входных данных."""
    ...

# Тесты для функции 2
def test_function2_edge_case():
    """Проверяет поведение с граничными случаями."""
    ...
```

## Создание тестовых случаев

Вставьте сюда код Python, для которого необходимо написать тесты.  Затем, создайте тестовые функции, покрывающие основные функции, классы и методы, и тестирующие разные сценарии, как указано в требованиях.

##  Примеры Тестовых Функций (для иллюстрации)


```python
import pytest
import my_module  # Замените на имя вашего модуля

# Предполагая, что у вас есть функция add в my_module
def test_add_positive_numbers():
    assert my_module.add(2, 3) == 5

def test_add_negative_numbers():
    assert my_module.add(-2, -3) == -5
    
def test_add_zero():
    assert my_module.add(0, 5) == 5
    
def test_add_one_number_is_zero():
    assert my_module.add(0, 0) == 0

def test_add_with_raises_exception():
    with pytest.raises(TypeError):
        my_module.add(2, "a")


def test_add_with_large_input():
   assert my_module.add(1000000000, 1000000000) == 2000000000
```


##  Рекомендации по написанию тестов

- **Валидные входные данные:** Тестируйте различные валидные входные данные, чтобы убедиться, что функция работает корректно.
- **Невалидные входные данные:** Если функция принимает различные типы данных, убедитесь, что она обрабатывает невалидные данные, такие как строки вместо чисел, и генерирует ожидаемые исключения.
- **Граничные случаи:**  Проверьте работу функции на граничных значениях (например, минимальное и максимальное значение для типов данных).
- **Исключения:** Используйте `pytest.raises` для проверки, что функция правильно обрабатывает ситуации, которые могут вызывать исключения.
- **Покрытие кода:** Постарайтесь добиться максимального покрытия кода тестами.


**Примечание:**  Этот шаблон документации должен быть дополнен конкретными тестами, соответствующими коду в `my_module.py`.
