# Модуль hypotez/src/endpoints/bots/discord/header.py

## Обзор

Этот модуль содержит функцию `set_project_root`, которая определяет корневую директорию проекта, начиная с директории текущего файла. Функция ищет директории, содержащие файлы-маркеры (например, `pyproject.toml`, `requirements.txt`, `.git`), и добавляет корневую директорию в `sys.path`, если она еще не там.  Модуль также загружает настройки из файла `settings.json` и, при наличии,  `README.MD`  для получения дополнительных метаданных о проекте, таких как имя проекта, версия, документация и прочие.

## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная с директории текущего файла.

**Параметры**:

- `marker_files` (tuple): Кортеж имен файлов или директорий, используемых для определения корневой директории проекта. По умолчанию содержит `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:

- `Path`: Путь к корневой директории проекта, если она найдена, в противном случае возвращает директорию, в которой расположен текущий скрипт.

**Вызывает исключения**:

- Нет.


## Переменные

### `__root__`

**Описание**: Путь к корневой директории проекта, полученный с помощью функции `set_project_root()`.

**Тип**: `Path`

**Описание**: Путь к корневой директории проекта, возвращаемый функцией `set_project_root()`.

### `settings`

**Описание**: Словарь со строковыми настройками, загруженный из файла `settings.json`.

**Тип**: `dict`

**Описание**:  Словарь, содержащий настройки из файла `settings.json`. Значение может быть `None`, если файл не найден или некорректно отформатирован.

### `doc_str`

**Описание**: Строка с текстом из файла `README.MD`.

**Тип**: `str`

**Описание**: Строка, содержащая текст из файла README.MD. Может быть `None`, если файл не найден или некорректно отформатирован.

### `__project_name__`

**Описание**: Имя проекта, полученное из настроек или установленное по умолчанию.

**Тип**: `str`

**Описание**:  Имя проекта, полученное из файла настроек. По умолчанию имеет значение `hypotez`.

### `__version__`

**Описание**: Версия проекта, полученная из настроек или установленная по умолчанию.

**Тип**: `str`

**Описание**: Версия проекта, полученная из файла настроек. По умолчанию имеет значение пустая строка.

### `__doc__`

**Описание**: Документация проекта, полученная из файла `README.MD` или установленная по умолчанию.

**Тип**: `str`

**Описание**: Документация проекта, полученная из файла `README.MD`. По умолчанию имеет значение пустая строка.

### `__details__`

**Описание**: Дополнительные детали проекта.

**Тип**: `str`

**Описание**: Дополнительные детали проекта (пустая строка по умолчанию).

### `__author__`

**Описание**: Автор проекта, полученный из настроек или установленный по умолчанию.

**Тип**: `str`

**Описание**: Автор проекта, полученный из файла настроек. По умолчанию имеет значение пустая строка.

### `__copyright__`

**Описание**: Авторские права проекта, полученные из настроек или установленные по умолчанию.

**Тип**: `str`

**Описание**: Авторские права проекта, полученные из файла настроек. По умолчанию имеет значение пустая строка.


### `__cofee__`

**Описание**: Ссылка на поддержку разработчика.

**Тип**: `str`

**Описание**: Ссылка для поддержки разработчика, полученная из настроек или установленная по умолчанию. По умолчанию имеет значение ссылки.

**Обработка исключений**:

В коде присутствуют блоки `try...except`, которые обрабатывают потенциальные `FileNotFoundError` и `json.JSONDecodeError`, возникающие при чтении файлов `settings.json` и `README.MD`.  В случае ошибки, соответствующая переменная устанавливается в `None` или пустую строку.
