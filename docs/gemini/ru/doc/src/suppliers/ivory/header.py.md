# Модуль `header.py`

## Обзор

Модуль `header.py` предназначен для установки корневой директории проекта, загрузки настроек из файла `settings.json`, а также чтения документации из файла `README.md`. Он также определяет основные метаданные проекта.

## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная с директории текущего файла, и останавливается на первой директории, содержащей один из маркерных файлов.

**Параметры**:
- `marker_files` (tuple, optional): Кортеж имен файлов или директорий для идентификации корня проекта. По умолчанию `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:
- `Path`: Путь к корневой директории, если она найдена, иначе - путь к директории, где расположен скрипт.

## Переменные

### `MODE`

**Описание**: Режим работы приложения, по умолчанию - `dev`.

### `__root__`

**Описание**: Путь к корневой директории проекта.

### `settings`

**Описание**: Словарь настроек, загруженный из файла `settings.json`.

### `doc_str`

**Описание**: Строка документации, загруженная из файла `README.MD`.

### `__project_name__`

**Описание**: Название проекта, по умолчанию `hypotez`.

### `__version__`

**Описание**: Версия проекта.

### `__doc__`

**Описание**: Документация проекта, считанная из файла `README.MD`.

### `__details__`

**Описание**: Дополнительная информация о проекте (в данный момент пустая строка).

### `__author__`

**Описание**: Автор проекта.

### `__copyright__`

**Описание**: Информация об авторских правах проекта.

### `__cofee__`

**Описание**: Сообщение с предложением угостить разработчика кофе.
