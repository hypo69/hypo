## Обзор
Данный документ описывает структуру и требования к документации, создаваемой для Python кода. Он предназначен для разработчиков и включает описание формата, структуры и содержания документации.

## Формат ответа

### `input code`
   - Предоставляет исходный код без каких-либо изменений.

### `algorithm`
   - Описывает рабочий процесс кода в пошаговом формате блок-схемы.
   - Для каждого логического блока предоставляются примеры его работы (если применимо).
   - Показывает, как данные передаются между функциями, классами или методами.

### `explanation`
   - Предоставляет подробное описание:
     - Imports: Объясняет их назначение и описывает их взаимосвязь с другими пакетами, особенно с теми, которые начинаются с `src.` (если применимо).
     - Classes: Описывает их назначение, атрибуты и методы, а также их взаимосвязи с другими компонентами проекта.
     - Functions: Объясняет их назначение, аргументы, возвращаемые значения и включает примеры.
     - Variables: Описывает их типы и использование.
   - Выстраивает цепочку связей с другими частями проекта (если таковые имеются).
   - Выделяет потенциальные ошибки или области для улучшения, если таковые имеются.

## Инструкции по созданию диаграмм потоков Mermaid с использованием HTML

1. **Тип графа:**
   - Используйте `flowchart` (например, `flowchart TD` для направленного графа сверху вниз).
   - Другие варианты: `LR` (слева направо), `BT` (снизу вверх), `RL` (справа налево).

2. **Имена узлов:**
   - Узлы должны иметь осмысленные и описательные имена, которые отражают операцию или состояние, которое они представляют.
   - Избегайте имен вроде `A`, `B`, `C`. Используйте четкие и понятные имена, такие как `Start`, `InitSupplier`, `ValidateInput`.

3. **Использование HTML:**
   - Применяйте HTML-теги для стилизации текста в узлах.
   - Поддерживаемые теги включают форматирование текста (например, `<b>`, `<i>`, `<h1>`, `<h3>`, `<code>`).
   - Используйте HTML-коды для специальных символов, когда это необходимо:
     - `(` → `&#40;`
     - `)` → `&#41;`
     - `'` → `&#39;`
     - `"` → `&quot;`
     - `:` → `&#58;`

4. **Соединения между узлами:**
   - Определите логические переходы между узлами, используя стрелки: `-->` для направленных или `---` для ассоциативных соединений.
   - Добавляйте текстовые метки к стрелкам, чтобы прояснить условия перехода, например, `-->|Success|`.

5. **Пример:**
   ```mermaid
   flowchart TD
       Start[<html>Начало процесса<br><b>Создание экземпляра</b></html>] 
           --> InitSupplier[<html>Инициализация поставщика<br><code>_payload&#40;params&#41;</code></html>]
       InitSupplier --> Validate[<html>Проверка параметров<br><i>is_valid&#40;params&#41;</i></html>]
       Validate -->|Успешная проверка| Success[<html><b>Успех</b><br>Создание завершено</html>]
       Validate -->|Ошибка| Error[<html>Ошибка<br><span style="color:red;">Неверные параметры</span></html>]
   ```

6. **Создание узлов:**
   - Генерируйте имена узлов на основе действия или состояния, которое они представляют.
   - Узлы должны быть краткими, но информативными. Используйте HTML-теги для улучшения читаемости, где это необходимо.

7. **Метки и комментарии:**
   - Добавляйте метки к стрелкам, чтобы объяснить условия перехода.
   - Используйте комментарии с `%%` для описания сложных соединений.

8. **Проверка синтаксиса:**
   - Убедитесь, что HTML внутри узлов действителен и не нарушает синтаксис Mermaid.

**Результат:** Диаграмма потока с осмысленными узлами, стилизованными с использованием HTML, четко представляющая логический процесс или архитектуру системы.
