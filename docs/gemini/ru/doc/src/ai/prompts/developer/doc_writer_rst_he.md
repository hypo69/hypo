## Классы

### `CodeAssistant`

**Описание**: Класс `CodeAssistant` используется для работы с различными моделями AI, такими как Google Gemini. Он предоставляет методы для анализа и создания документации для кода.

**Методы**:
- `process_files`: Метод для обработки файлов кода.

**Параметры**:
- `role` (str): Роль ассистента (например, 'code_checker').
- `lang` (str): Язык, на котором ассистент будет работать (например, 'ru').
- `model` (list): Список используемых моделей AI (например, ['gemini']).

**Пример использования**:
```python
assistant = CodeAssistant(role='code_checker', lang='ru', model=['gemini'])
assistant.process_files()
```

## Функции

### `process_files`

**Описание**: Метод `process_files` используется для анализа и обработки файлов кода.

**Параметры**:
- `files` (list): Список файлов для обработки.
- `options` (dict): Дополнительные параметры для настройки обработки файлов.

**Возвращает**:
- `list`: Список обработанных данных.

**Пример использования**:

```python
assistant = CodeAssistant(role='code_checker', lang='ru', model=['gemini'])
result = assistant.process_files(files=['file1.py', 'file2.py'], options={})
```
### Обработка исключений

**Описание**:

Обработка исключений необходима для того, чтобы продолжить выполнение программы, даже если при обработке файлов произошла ошибка.

**Пример использования**:
```python
try:
    process_file(file)
except FileNotFoundError as ex:
    handle_exception(ex)
```

### `FileNotFoundError`

**Описание**:

Это исключение возникает, когда файл не найден в процессе обработки.

**Параметры**:
- `file` (str): Путь к файлу, который не был найден.

**Пример использования**:
```python
try:
    open(file)
except FileNotFoundError as ex:
    raise FileNotFoundError("Файл не найден") from ex
```
