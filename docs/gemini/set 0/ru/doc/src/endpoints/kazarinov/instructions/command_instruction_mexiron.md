# ИНСТРУКЦИЯ ПО ОБРАБОТКЕ КОМПЬЮТЕРНЫХ КОМПОНЕНТОВ

## Обзор

Данная инструкция описывает обработку JSON-данных, содержащих информацию о компьютерных компонентах.  Инструкция определяет тип сборки (например, игровой, рабочая станция), генерирует заголовки и описания на иврите и русском языках, переводит подробности о компонентах и возвращает структурированный JSON-выход.  Поддерживается правильное форматирование, включаются оценки уверенности и соблюдаются детальные руководящие принципы для описаний и обработки компонентов.

## Шаблон ответа

### Формат ответа

Ответ должен соответствовать следующему JSON-шаблону:

```json
{
  "he": {
    "title": "Название компьютера на иврите",
    "description": "Описание компьютера на иврите",
    "build_types": {
      "gaming": "Вероятность игровой сборки",
      "workstation": "Вероятность рабочей станции"
    },
    "products": [
      {
        "product_id": "ID компонента",
        "product_title": "Название компонента на иврите",
        "product_description": "Описание компонента на иврите",
        "product_specification": "Характеристики компонента на иврите",
        "image_local_saved_path": "Путь к изображению"
      },
      {
        "product_id": "ID компонента",
        "product_title": "Название компонента на иврите",
        "product_description": "Описание компонента на иврите",
        "product_specification": "Характеристики компонента на иврите",
        "image_local_saved_path": "Путь к изображению"
      }
    ]
  },
  "ru": {
    "title": "Название компьютера на русском",
    "description": "Описание компьютера на русском",
    "build_types": {
      "gaming": "Вероятность игровой сборки",
      "workstation": "Вероятность рабочей станции"
    },
    "products": [
      {
        "product_id": "ID компонента",
        "product_title": "Название компонента на русском",
        "product_description": "Описание компонента на русском",
        "product_specification": "Характеристики компонента на русском",
        "image_local_saved_path": "Путь к изображению"
      },
      {
        "product_id": "ID компонента",
        "product_title": "Название компонента на русском",
        "product_description": "Описание компонента на русском",
        "product_specification": "Характеристики компонента на русском",
        "image_local_saved_path": "Путь к изображению"
      }
    ]
  }
}
```

**Примечание:**  В качестве значений для  `"Вероятность игровой сборки"` и `"Вероятность рабочей станции"` используются числовые значения от 0 до 1, отражающие степень уверенности в классификации типа сборки.  Значения  `"ID компонента"` и  `"image_local_saved_path"` должны быть взяты из входных данных.  Значения  `"product_title"`,  `"product_description"`, и `"product_specification"` должны быть сгенерированы функциями из соответствующего модуля.


## Требования к функциям

Функции, которые реализуют данную инструкцию, должны соответствовать формату Python-комментариев, указанному в исходном запросе.  Они должны  принимать на вход данные в JSON формате, обрабатывать их и возвращать JSON-ответ, соответствующий шаблону.

## Обработка исключений

При возникновении ошибок обработки данных, следует использовать `ex` вместо `e` в блоках `try-except`.
