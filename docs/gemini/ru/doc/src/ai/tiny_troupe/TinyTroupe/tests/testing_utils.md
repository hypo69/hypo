# Модуль `hypotez/src/ai/tiny_troupe/TinyTroupe/tests/testing_utils.py`

## Обзор

Этот модуль предоставляет утилиты для тестирования кода проекта TinyTroupe. Он содержит функции для проверки действий и стимулов, а также для работы с данными агентов, окружения и OpenAI.  Включает фикстуры для создания и управления тестовыми данными.

## Функции

### `contains_action_type`

**Описание**: Проверяет, содержит ли список действий действие заданного типа.

**Параметры**:
- `actions` (list): Список словарей, представляющих действия.  Каждый словарь должен содержать ключ `"action"` со значением, содержащим ключ `"type"`.
- `action_type` (str): Тип действия, который нужно найти.

**Возвращает**:
- `bool`: `True`, если действие заданного типа найдено в списке, иначе `False`.

### `contains_action_content`

**Описание**: Проверяет, содержит ли список действий действие с заданным содержанием.

**Параметры**:
- `actions` (list): Список словарей, представляющих действия. Каждый словарь должен содержать ключ `"action"` со значением, содержащим ключ `"content"`.
- `action_content` (str): Содержание действия, которое нужно найти.

**Возвращает**:
- `bool`: `True`, если действие с заданным содержанием найдено в списке, иначе `False`.

### `contains_stimulus_type`

**Описание**: Проверяет, содержит ли список стимулов стимул заданного типа.

**Параметры**:
- `stimuli` (list): Список словарей, представляющих стимулы.  Каждый словарь должен содержать ключ `"type"`.
- `stimulus_type` (str): Тип стимула, который нужно найти.

**Возвращает**:
- `bool`: `True`, если стимул заданного типа найден в списке, иначе `False`.

### `contains_stimulus_content`

**Описание**: Проверяет, содержит ли список стимулов стимул с заданным содержанием.

**Параметры**:
- `stimuli` (list): Список словарей, представляющих стимулы.  Каждый словарь должен содержать ключ `"content"`.
- `stimulus_content` (str): Содержание стимула, которое нужно найти.

**Возвращает**:
- `bool`: `True`, если стимул с заданным содержанием найден в списке, иначе `False`.


### `terminates_with_action_type`

**Описание**: Проверяет, заканчивается ли список действий действием заданного типа.

**Параметры**:
- `actions` (list): Список словарей, представляющих действия.
- `action_type` (str): Тип действия, с которым должен заканчиваться список.

**Возвращает**:
- `bool`: `True`, если список заканчивается действием заданного типа, иначе `False`.


### `proposition_holds`

**Описание**: Проверяет истинность данного утверждения с помощью вызова LLM.

**Параметры**:
- `proposition` (str): Утверждение для проверки.

**Возвращает**:
- `bool`: `True`, если утверждение истинно, иначе `False`.  Возможна ошибка `Exception`, если LLM вернул неожиданный результат.


### `only_alphanumeric`

**Описание**: Возвращает строку, содержащую только алфавитно-цифровые символы.

**Параметры**:
- `string` (str): Исходная строка.

**Возвращает**:
- `str`: Строка, содержащая только алфавитно-цифровые символы.

### `create_test_system_user_message`

**Описание**: Создаёт список, содержащий одно системное сообщение и одно пользовательское сообщение.

**Параметры**:
- `user_prompt` (str): Пользовательское сообщение. Может быть `None`.
- `system_prompt` (str, optional): Системное сообщение. По умолчанию "You are a helpful AI assistant."

**Возвращает**:
- `list`: Список, содержащий системное и пользовательское сообщение.

### `agents_configs_are_equal`

**Описание**: Проверяет, равны ли конфигурации двух агентов.

**Параметры**:
- `agent1`: Первый агент.
- `agent2`: Второй агент.
- `ignore_name` (bool, optional): Если `True`, игнорирует сравнение имени агента. По умолчанию `False`.

**Возвращает**:
- `bool`: `True`, если конфигурации равны, иначе `False`.


### `remove_file_if_exists`

**Описание**: Удаляет файл, если он существует по указанному пути.

**Параметры**:
- `file_path` (str): Путь к файлу.

**Возвращает**:
- None.


### `get_relative_to_test_path`

**Описание**: Возвращает путь к тестовому файлу с заданным суффиксом.

**Параметры**:
- `path_suffix` (str): Суффикс пути к файлу.

**Возвращает**:
- `str`: Полный путь к тестовому файлу.


## Фикстуры

### `focus_group_world`

**Описание**: Фикстура, возвращающая объект `TinyWorld` с заданными агентами. Использует примеры из модуля `tinytroupe.examples`.

**Возвращает**:
- `TinyWorld`: Объект тестового окружения.

### `setup`

**Описание**: Фикстура для очистки данных агентов и окружений перед запуском теста.
