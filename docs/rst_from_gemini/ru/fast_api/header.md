def find_project_root(marker_files=('pyproject.toml', 'requirements.txt', '.git')):
    """
    Находит корневой каталог проекта, начиная с текущей директории.
    Ищет вверх по дереву каталогов, пока не найдёт директорию, содержащую один из указанных файлов или каталогов.

    Args:
        marker_files (tuple): Имена файлов или каталогов, по которым определяется корневой каталог проекта.

    Returns:
        Path: Путь к корневому каталогу, если найден, иначе – директория, где находится текущий файл.
    """
    # Получаем директорию текущего файла (где вызвана эта функция)
    current_path = Path(__file__).resolve().parent

    # Проходим по родительским директориям, начиная с директории текущего файла
    for parent in [current_path] + list(current_path.parents):
        # Проверяем, существует ли любой из указанных маркеров в текущей директории
        if any((parent / marker).exists() for marker in marker_files):
            # Если найден, возвращаем эту директорию как корень проекта
            return parent

    # Если ни один из маркеров не найден, возвращаем текущую директорию как резервное значение
    return current_path
```

**Объяснение кода и важные моменты:**

* **`marker_files`:** Функция использует кортеж `marker_files` для определения файлов, которые указывают на корневой каталог проекта.  В данном случае это `pyproject.toml`, `requirements.txt` и `.git`.  Это хороший подход, поскольку эти файлы обычно присутствуют в проектах Python.

* **Обработка исключений:**  Код не содержит явной обработки исключений, но это могло бы быть добавлено (например, для проверки, если `Path(__file__)` валиден).

* **`__root__`:** Глобальная переменная `__root__` хранит результат вызова функции `find_project_root()`.

* **Добавление в `sys.path`:**  Важный момент: после нахождения корневого каталога, он добавляется в `sys.path`. Это позволяет импортировать модули из других подпапок проекта без указания полного пути.
