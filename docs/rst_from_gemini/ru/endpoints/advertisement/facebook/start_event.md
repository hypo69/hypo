# Модуль `start_event.py`

Файл: `C:\Users\user\Documents\repos\hypotez\src\endpoints\advertisement\facebook\start_event.py`

**Описание:**

Модуль `start_event.py` отвечает за отправку мероприятий в группы Facebook. Он использует предопределенные списки файлов с данными о группах и событиями, а также класс `FacebookPromoter` для автоматической публикации.

**Используемые модули:**

* `math`: Для использования функции `log` (хотя в коде она не используется).
* `header`: (Предполагается, что это пользовательский модуль для заголовков или других настроек).
* `time`: Для работы с временем.
* `jjson`: (Предполагается, что это модуль для работы с JSON данными).
* `webdriver`: Для работы с браузером (через класс `Driver` и `Chrome`).
* `FacebookPromoter`: Класс для работы с Facebook API.
* `logger`: (Предполагается, что это модуль для логирования).


**Описание переменных и списков:**

* `MODE`:  Переменная, содержащая режим работы (в данном случае 'debug').
* `filenames`: Список путей к файлам JSON с данными о группах Facebook, которые будут использоваться для публикации.
* `excluded_filenames`: Список файлов, которые будут исключены из обработки.
* `events_names`: Список названий мероприятий, которые будут публиковаться.
* `promoter`: Экземпляр класса `FacebookPromoter`, отвечающий за публикацию в Facebook.
* `d`: Экземпляр класса `Driver`, представляющий текущий браузер (Chrome).


**Логика работы:**

1. **Инициализация:** Создаётся экземпляр `FacebookPromoter` с загруженными данными из файлов.
2. **Цикл:** Программа зацикливается, выполняя следующие действия:
   * **Логирование:** Записывается текущее время в лог.
   * **Публикация:** Вызывается метод `promoter.run_events` для публикации мероприятий в заданные группы.
   * **Ожидание:** Программа останавливается на 7200 секунд (2 часа).
3. **Обработка прерывания:** Если пользователь прерывает выполнение программы (Ctrl+C), выводится сообщение об остановке кампании в лог.

* **Управление ошибками Facebook API:** `FacebookPromoter` должен содержать механизм обработки ошибок API Facebook (например, если произошла ошибка авторизации, публикация не состоялась или произошёл timeout).
* **Доступ к ключам:** Нужно обеспечить безопасный доступ к ключам/токену, необходимым для работы с Facebook API. Не хранить их в коде, а использовать secure хранилища.
* **Тестирование:** Необходимо разработать тесты для проверки работоспособности скрипта и корректности обработки данных.

* **Контроль времени выполнения:** Регулярное измерение продолжительности работы и потенциальная обработка ошибок, которые могут возникать во время долгого выполнения (например, перегрев системы, потеря подключения).
