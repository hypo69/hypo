# Модуль hypotez/src/suppliers/ivory/header.py

## Обзор

Этот модуль содержит функции для определения корневой директории проекта, загрузки настроек из файла `settings.json` и чтения документации из `README.MD`.  Он также предоставляет доступ к глобальным переменным, представляющим название проекта, версию, документацию, авторские права и другую информацию.

## Функции

### `set_project_root`

**Описание**:  Находит корневую директорию проекта, начиная с текущей директории и двигаясь вверх по дереву директорий.  Останавливается на первой директории, содержащей один из указанных маркеров.

**Параметры**:

- `marker_files` (tuple): Кортеж строк, представляющих имена файлов или директорий, по которым определяется корень проекта. По умолчанию содержит `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:

- `Path`: Путь к корневой директории проекта, если найдена, иначе путь к директории, в которой находится текущий скрипт.

**Вызывает исключения**:

- Нет.


## Глобальные переменные

### `__root__`

**Описание**: Путь к корневой директории проекта, полученный вызовом `set_project_root()`.


### `settings`

**Описание**: Словарь с настройками проекта, загруженный из файла `settings.json`.  Возможны значения `None` в случае ошибки при чтении файла или его отсутствия.


### `doc_str`

**Описание**: Строка с содержимым файла `README.MD`, если он найден. Возможны значения `None` в случае ошибки при чтении файла или его отсутствия.


### `__project_name__`

**Описание**: Название проекта, полученное из настроек. По умолчанию 'hypotez'.


### `__version__`

**Описание**: Версия проекта, полученная из настроек. По умолчанию пустая строка.


### `__doc__`

**Описание**: Документация проекта (строка), полученная из файла `README.MD`.  По умолчанию пустая строка.


### `__details__`

**Описание**: Подробная информация о проекте. По умолчанию пустая строка.


### `__author__`

**Описание**: Автор проекта, полученный из настроек. По умолчанию пустая строка.


### `__copyright__`

**Описание**: Авторские права на проект, полученные из настроек. По умолчанию пустая строка.


### `__cofee__`

**Описание**: Ссылка на страницу для поддержки разработчика. По умолчанию содержит ссылку.

## Обработка исключений (try...except)


В коде присутствуют блоки `try...except` для обработки ошибок при чтении файлов `settings.json` и `README.MD`. Если файлы не найдены или содержат невалидные данные, соответствующие переменные (settings, doc_str) принимают значение `None`.


## Использование

Этот модуль предназначен для инициализации глобальных переменных, необходимых для работы приложения, таких как путь к корневой директории проекта, данные из файла настроек и документация из `README.MD`.  Он автоматически добавляет корневую директорию в `sys.path`.
