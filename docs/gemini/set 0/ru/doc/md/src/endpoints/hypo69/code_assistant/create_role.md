# Модуль `create_role.mmd`

## Обзор

Данный модуль описывает порядок действий для создания новой роли в системе. Он представлен в виде графа направленного ациклического графа (DAG), показывающего последовательность операций.

## Порядок действий

### Граф зависимостей

```
graph TD
    A[Файл `code_assistant.json`] --> B[Добавить роль в список ролей]
    B --> C[Добавить роль в файл переводов `translations/translations.json`]
    C --> D[Создать системный промпт в `ai/prompts/develpoper`]
    D --> E[Создать командную инструкцию в `instructions/...`]

    %% Настройка стилей блоков
    style A width: 300px
    style B width: 300px
    style C width: 300px
    style D width: 300px
    style E width: 300px
```

**Описание этапов:**

* **A (Файл `code_assistant.json`):**  Исходный файл, содержащий данные о доступных ролях.
* **B (Добавить роль в список ролей):**  Процесс добавления новой роли в список доступных ролей в файле `code_assistant.json`.
* **C (Добавить роль в файл переводов `translations/translations.json`):**  Процесс добавления переведенных строк, связанных с новой ролью, в файл `translations/translations.json`.
* **D (Создать системный промпт в `ai/prompts/develpoper`):** Создание системного запроса (промпты) для новой роли в папке `ai/prompts/develpoper`.
* **E (Создать командную инструкцию в `instructions/...`):** Создание инструкций для работы с новой ролью в папке `instructions/...`.
