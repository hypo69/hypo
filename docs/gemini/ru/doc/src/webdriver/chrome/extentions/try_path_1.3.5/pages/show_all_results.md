# Модуль show_all_results.js

## Обзор

Данный скрипт JavaScript отвечает за отображение результатов поиска, полученных от расширения `tryxpath`. Он обрабатывает данные, полученные от другого скрипта, формирует HTML-контент для отображения информации о результатах, включая контекст поиска, основные результаты и детали.  Также скрипт предоставляет возможность загрузить информацию в текстовые файлы.

## Функции

### `showAllResults`

**Описание**: Функция отображает результаты поиска в интерфейсе.

**Параметры**:
- `results` (объект): Объект, содержащий результаты поиска.  Должен содержать поля `message`, `title`, `href`, `frameId`, `context` (опционально), `main` и другие необходимые для отображения поля.

**Возвращает**:
- Нет значения.

**Вызывает исключения**:
- Нет.  Обработка ошибок происходит через catch блока в самом вызове `fu.updateDetailsTable`.


### `makeTextDownloadUrl`

**Описание**: Функция создаёт URL для скачивания текстового файла.

**Параметры**:
- `text` (строка): Текст, который нужно сохранить в файл.

**Возвращает**:
- Строка: URL для скачивания файла.


### `makeInfoText`

**Описание**: Функция генерирует текстовое представление результатов поиска для скачивания.

**Параметры**:
- `results` (объект): Объект, содержащий результаты поиска.

**Возвращает**:
- Строка: Текстовое представление результатов.

**Вызывает исключения**:
- Нет.


### `makeConvertedInfoText`

**Описание**: Функция генерирует текстовое представление результатов поиска для скачивания, в формате JSON (для значений и текстового контента).

**Параметры**:
- `results` (объект): Объект, содержащий результаты поиска.

**Возвращает**:
- Строка: Текстовое представление результатов в формате JSON.

**Вызывает исключения**:
- Нет.


## Глобальные переменные

Перечисленные ниже переменные являются глобальными и используются в функции `showAllResults`.

* `detailKeys`: Массив строк, представляющих имена полей для деталей.
* `headerValues`: Массив строк, представляющих заголовки таблицы для деталей.
* `relatedTabId`: ID открытой вкладки.
* `relatedFrameId`: ID фрейма.
* `executionId`: ID выполнения запроса.
