# Модуль `hypotez/src/scenario/executor.py`

## Обзор

Этот модуль содержит функции для выполнения сценариев, загрузки их из файлов, обработки процесса извлечения информации о продуктах и вставки ее в PrestaShop.

## Функции

### `dump_journal`

**Описание**: Сохраняет данные журнала в JSON-файл.

**Параметры**:
- `s` (объект): экземпляр поставщика.
- `journal` (словарь): словарь, содержащий данные журнала.


**Возвращает**:
- Без возвращаемого значения.


### `run_scenario_files`

**Описание**: Выполняет список файлов сценариев.

**Параметры**:
- `s` (объект): экземпляр поставщика.
- `scenario_files_list` (список или путь): список путей к файлам сценариев или путь к единственному файлу.

**Возвращает**:
- `bool`: `True`, если все сценарии были выполнены успешно, `False` в противном случае.

**Возможные исключения**:
- `TypeError`: если `scenario_files_list` не является списком или строкой.

### `run_scenario_file`

**Описание**: Загружает и выполняет сценарии из файла.

**Параметры**:
- `s` (объект): экземпляр поставщика.
- `scenario_file` (путь): путь к файлу сценария.

**Возвращает**:
- `bool`: `True`, если сценарий был выполнен успешно, `False` в противном случае.

**Возможные исключения**:
- `FileNotFoundError`: если файл не найден.
- `json.JSONDecodeError`: если файл не удалось декодировать как JSON.


### `run_scenarios`

**Описание**: Выполнение списка сценариев (НЕ ФАЙЛОВ).

**Параметры**:
- `s` (объект): экземпляр поставщика.
- `scenarios` (список или словарь): список сценариев или одиночный сценарий в виде словаря.

**Возвращает**:
- Список или словарь результата выполнения сценариев, или `False` в случае ошибки.

**Возможные исключения**:
- Любые исключения, которые могут возникнуть при выполнении сценариев.


### `run_scenario`

**Описание**: Функция для выполнения полученного сценария.

**Параметры**:
- `supplier` (объект): экземпляр поставщика.
- `scenario` (словарь): словарь, содержащий детали сценария.
- `scenario_name` (строка): имя сценария.

**Возвращает**:
- Результат выполнения сценария.

**Возможные исключения**:
- Любые исключения, которые могут возникнуть при выполнении сценария.

### `insert_grabbed_data`

**Описание**: Вставка полученных данных о продукте в PrestaShop.

**Параметры**:
- `product_fields` (объект): экземпляр `ProductFields`, содержащий информацию о продукте.

**Возвращает**:
- Без возвращаемого значения.

**Примечания**:
- Требуется перенос логики в отдельный файл (в класс PrestaShop).

### `execute_PrestaShop_insert`

**Описание**: Вставка продукта в PrestaShop.

**Параметры**:
- `f` (объект): экземпляр `ProductFields`, содержащий информацию о продукте.
- `coupon_code` (строка, необязательно): код купона.
- `start_date` (строка, необязательно): дата начала акции.
- `end_date` (строка, необязательно): дата окончания акции.


**Возвращает**:
- `bool`: `True`, если вставка прошла успешно, `False` в противном случае.

**Возможные исключения**:
- Любые исключения, которые могут возникнуть при взаимодействии с PrestaShop.
