# Инструкция для бота

## Обзор

Этот файл содержит инструкцию для взаимодействия с ботом через командную строку. Он определяет доступные команды и их синтаксис.

## Подробнее

Файл предоставляет список команд, которые пользователь может использовать для взаимодействия с ботом. Каждая команда имеет краткое описание и синтаксис, что позволяет пользователю легко понять, как правильно использовать каждую команду.

## Команды

### `!hi`

**Описание**: Приветствует пользователя.

### `!train <data> <data_dir> <positive> <attachment>`

**Описание**: Обучает модель с предоставленными данными.

**Параметры**:

- `<data>`: Файл с данными для обучения.
- `<data_dir>`: Директория с данными для обучения.
- `<positive>`: Положительные примеры для обучения.
- `<attachment>`: Файл, прикрепленный как данные для обучения.

**Как работает команда**:

1.  Команда принимает данные для обучения модели из различных источников: файла (`<data>`), директории (`<data_dir>`) или прикрепленного файла (`<attachment>`).
2.  Обучает модель на основе предоставленных данных и положительных примеров (`<positive>`).

```
A (Прием данных для обучения)
↓
B (Обучение модели)
```

### `!test <test_data>`

**Описание**: Тестирует модель с предоставленными тестовыми данными в формате JSON.

**Параметры**:

- `<test_data>`: JSON данные для тестирования модели.

**Как работает команда**:

1.  Команда принимает тестовые данные в формате JSON.
2.  Тестирует обученную модель на основе предоставленных данных.

```
A (Прием тестовых данных в формате JSON)
↓
B (Тестирование модели)
```

### `!archive <directory>`

**Описание**: Архивирует файлы в указанной директории.

**Параметры**:

- `<directory>`: Директория для архивации файлов.

**Как работает команда**:

1.  Команда принимает путь к директории.
2.  Архивирует все файлы, находящиеся в указанной директории.

```
A (Прием пути к директории)
↓
B (Архивация файлов в директории)
```

### `!select_dataset <path_to_dir_positive> <positive>`

**Описание**: Выбирает набор данных для обучения из указанной директории.

**Параметры**:

- `<path_to_dir_positive>`: Путь к директории с положительными примерами данных.
- `<positive>`: Положительные примеры для обучения.

**Как работает команда**:

1.  Команда принимает путь к директории с положительными примерами.
2.  Выбирает набор данных из указанной директории для обучения модели.

```
A (Прием пути к директории с положительными примерами)
↓
B (Выбор набора данных для обучения)
```

**Описание**: Отображает это сообщение с инструкциями.

**Как работает команда**:

1. Команда выводит список доступных команд и их описания.

