# Модуль `header`

## Обзор

Модуль `header.py` предназначен для определения корневого пути проекта и загрузки настроек из `settings.json` и документации из `README.MD`. Он также устанавливает основные параметры проекта, такие как имя, версия, авторские права и т.д. Все импорты в проекте строятся относительно определенного корневого пути.

## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная с директории текущего файла, ищет вверх и останавливается на первой директории, содержащей любой из указанных маркерных файлов.

**Параметры**:
- `marker_files` (tuple): Кортеж с именами файлов или директорий, которые идентифицируют корень проекта.

**Возвращает**:
- `Path`: Путь к корневой директории, если она найдена. Иначе возвращает директорию, где расположен текущий файл.

## Переменные

### `__root__`

**Описание**: Путь к корневой директории проекта.

### `settings`

**Описание**: Словарь с настройками проекта, загруженными из файла `settings.json`.

### `doc_str`

**Описание**: Строка, содержащая документацию из файла `README.MD`.

### `__project_name__`

**Описание**: Имя проекта, по умолчанию 'hypotez'.

### `__version__`

**Описание**: Версия проекта.

### `__doc__`

**Описание**: Документация проекта из `README.MD`.

### `__details__`

**Описание**: Дополнительные детали проекта. В данный момент пустая строка.

### `__author__`

**Описание**: Автор проекта.

### `__copyright__`

**Описание**: Авторские права проекта.

### `__cofee__`

**Описание**: Сообщение с предложением угостить разработчика чашкой кофе.
