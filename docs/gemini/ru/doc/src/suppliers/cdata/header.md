# Модуль hypotez/src/suppliers/cdata/header.py

## Обзор

Данный модуль содержит функции для определения корневой директории проекта, загрузки настроек из файла `settings.json` и получения документации из файла `README.MD`.  Он также инициализирует переменные, описывающие проект.

## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная от текущего файла. Поиск происходит вверх по директориям, пока не будет найдена директория, содержащая один из заданных маркеров (файлы или директории).

**Параметры**:

- `marker_files` (tuple): Кортеж строк, представляющих имена файлов или директорий, которые используются для определения корневой директории. По умолчанию включает `'pyproject.toml'`, `'requirements.txt'` и `'.git'`.


**Возвращает**:

- `Path`: Путь к корневой директории проекта, если она найдена. В противном случае возвращает путь к директории, где расположен текущий файл.


**Вызывает исключения**:

- Нет.


### <ins>Дополнительные переменные</ins>

__root__ (Path): Путь к корневой директории проекта. Инициализируется функцией `set_project_root()`.


## Обработка файлов

### Загрузка настроек

**Описание**: Загружает настройки из файла `settings.json` в переменную `settings`.

**Вызывает исключения**:

- `FileNotFoundError`: Если файл `settings.json` не найден.
- `json.JSONDecodeError`: Если файл `settings.json` содержит некорректный JSON.


### Чтение документации

**Описание**: Читает содержимое файла `README.MD` в переменную `doc_str`.

**Вызывает исключения**:

- `FileNotFoundError`: Если файл `README.MD` не найден.
- `UnicodeDecodeError`: Если файл `README.MD` содержит некорректные данные.


## Инициализация переменных проекта

**Описание**:  Инициализирует переменные, содержащие информацию о проекте, такие как название, версия, автор, копирайт и т.д., на основе данных из `settings.json` или использует значения по умолчанию.

**Вызывает исключения**:

- Нет.
