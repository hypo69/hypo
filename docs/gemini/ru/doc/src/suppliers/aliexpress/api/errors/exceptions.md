# Модуль hypotez/src/suppliers/aliexpress/api/errors/exceptions.py

## Обзор

Модуль `exceptions.py` содержит пользовательские исключения, используемые для обработки ошибок, связанных с API AliExpress.  Он предоставляет иерархию исключений, унаследованных от базового класса `AliexpressException`, для более структурированной обработки ошибок.


## Классы

### `AliexpressException`

**Описание**: Базовый класс для всех исключений, связанных с API AliExpress. Содержит информацию о причине ошибки.

**Методы**:

- `__init__`: Инициализирует исключение с указанием причины.
- `__str__`: Возвращает строковое представление исключения, содержащее причину.

**Параметры**:

- `reason` (str): Текстовое описание причины возникновения исключения.

**Возвращает**:

- `str`: Строковое представление исключения.

### `InvalidArgumentException`

**Описание**: Исключение, генерируемое, когда переданные аргументы некорректны.

**Наследуется от**: `AliexpressException`


### `ProductIdNotFoundException`

**Описание**: Исключение, генерируемое, если переданный идентификатор продукта не найден.

**Наследуется от**: `AliexpressException`

### `ApiRequestException`

**Описание**: Исключение, генерируемое, если произошла ошибка при запросе к API AliExpress.

**Наследуется от**: `AliexpressException`

### `ApiRequestResponseException`

**Описание**: Исключение, генерируемое, если ответ от API AliExpress некорректен.

**Наследуется от**: `AliexpressException`

### `ProductsNotFoudException`

**Описание**: Исключение, генерируемое, если не удалось найти ни одного продукта.

**Наследуется от**: `AliexpressException`

### `CategoriesNotFoudException`

**Описание**: Исключение, генерируемое, если не удалось найти ни одной категории.

**Наследуется от**: `AliexpressException`

### `InvalidTrackingIdException`

**Описание**: Исключение, генерируемое, если идентификатор отслеживания неверен или отсутствует.

**Наследуется от**: `AliexpressException`
