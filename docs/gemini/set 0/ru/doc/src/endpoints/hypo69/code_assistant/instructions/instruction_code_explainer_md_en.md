## ИНСТРУКЦИЯ

Этот раздел описывает инструкции по созданию документации для Python-кода. Подробности можно найти в разделах ниже.

## Формат документации

Документация должна быть в формате Markdown (.md). Каждый файл должен начинаться с заголовка и краткого описания его содержимого. Комментарии для функций/методов должны быть в формате, приведенном в примере.

## Содержание (TOC)

В начале документации необходимо создать оглавление, содержащее ссылки на все основные разделы.

## Форматирование документации

Используйте правильный синтаксис Markdown для заголовков, списков, ссылок и структурных разделов описаний функций/методов.

## Заголовки разделов

Используйте заголовки первого уровня (#), второго уровня (##), третьего уровня (###) и четвертого уровня (####) последовательно на протяжении всего файла.
