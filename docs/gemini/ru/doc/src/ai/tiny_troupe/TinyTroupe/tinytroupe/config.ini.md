# Конфигурационный файл `config.ini`

## Обзор

Файл `config.ini` содержит конфигурационные параметры для проекта `TinyTroupe`, в частности, настройки для взаимодействия с OpenAI или Azure OpenAI, а также параметры для симуляций и журналирования.

- [Раздел `Simulation`](#раздел-simulation)
- [Раздел `Logging`](#раздел-logging)

## Раздел `OpenAI`

### Описание

Этот раздел содержит настройки для работы с OpenAI или Azure OpenAI Service.

### Параметры

- **`API_TYPE`**: Определяет тип используемого API. Допустимые значения: `openai`, `azure`. По умолчанию `openai`.
- **`AZURE_API_VERSION`**: Версия API Azure OpenAI. Используется только при `API_TYPE=azure`. Пример: `2023-05-15`.
- **`MODEL`**: Имя модели OpenAI, которую следует использовать. Пример: `gpt-4o`.
- **`MAX_TOKENS`**: Максимальное количество токенов, которое может быть сгенерировано в ответе. Пример: `4000`.
- **`TEMPERATURE`**: Параметр температуры для модели. Контролирует случайность генерации текста. Пример: `0.3`.
- **`FREQ_PENALTY`**: Параметр штрафа за частоту. Уменьшает вероятность повторения часто встречающихся токенов. Пример: `0.0`.
- **`PRESENCE_PENALTY`**: Параметр штрафа за присутствие. Увеличивает вероятность использования новых токенов. Пример: `0.0`.
- **`TIMEOUT`**: Время ожидания для API запроса в секундах. Пример: `60`.
- **`MAX_ATTEMPTS`**: Максимальное количество попыток повторения запроса при ошибке. Пример: `5`.
- **`WAITING_TIME`**: Начальное время ожидания перед повторным запросом в секундах. Пример: `1`.
- **`EXPONENTIAL_BACKOFF_FACTOR`**: Множитель экспоненциального отката при повторных попытках. Пример: `5`.
- **`EMBEDDING_MODEL`**: Модель для встраивания текста. Пример: `text-embedding-3-small`.
- **`CACHE_API_CALLS`**: Флаг, указывающий, нужно ли кэшировать результаты API вызовов. Допустимые значения: `True`, `False`. По умолчанию `False`.
- **`CACHE_FILE_NAME`**: Имя файла для кэширования API вызовов. Пример: `openai_api_cache.pickle`.
- **`MAX_CONTENT_DISPLAY_LENGTH`**: Максимальная длина отображаемого контента (для отладки). Пример: `1024`.

## Раздел `Simulation`

### Описание

Этот раздел содержит настройки для симуляций, включая предотвращение вредоносного контента и нарушения авторских прав.

### Параметры

- **`RAI_HARMFUL_CONTENT_PREVENTION`**: Флаг для включения/выключения предотвращения вредоносного контента. Допустимые значения: `True`, `False`. По умолчанию `True`.
- **`RAI_COPYRIGHT_INFRINGEMENT_PREVENTION`**: Флаг для включения/выключения предотвращения нарушения авторских прав. Допустимые значения: `True`, `False`. По умолчанию `True`.

## Раздел `Logging`

### Описание

Этот раздел определяет параметры журналирования для проекта.

### Параметры

- **`LOGLEVEL`**: Уровень детализации журнала. Возможные значения: `ERROR`, `WARNING`, `INFO`, `DEBUG`. По умолчанию `ERROR`.
