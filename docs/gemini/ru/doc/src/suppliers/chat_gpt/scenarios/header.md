# Модуль hypotez/src/suppliers/chat_gpt/scenarios/header.py

## Обзор

Этот модуль содержит функции для работы с настройками проекта, получения пути к корневой директории проекта и получения информации о проекте.

## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная с текущей директории файла. Поиск идет вверх по директориям до момента, когда будет найдена директория содержащая один из указанных файлов-маркеров. Если корневая директория не найдена, возвращает директорию, в которой расположен скрипт.

**Параметры**:

- `marker_files` (tuple): Кортеж имен файлов или директорий, которые используются в качестве маркеров для определения корневой директории проекта. По умолчанию - ('pyproject.toml', 'requirements.txt', '.git').

**Возвращает**:

- `Path`: Путь к корневой директории проекта, если она найдена. В противном случае, возвращается директория, в которой расположен скрипт.

**Вызывает исключения**:

- Нет.

## Переменные

### `__root__`

**Описание**: Путь к корневой директории проекта, полученный с помощью функции `set_project_root`.

**Тип**: `Path`

### `settings`

**Описание**: Словарь с настройками проекта, загруженный из файла `settings.json` в корневой директории проекта.

**Тип**: `dict` или `None`

### `doc_str`

**Описание**: Строка документации проекта, загруженная из файла `README.MD` в корневой директории проекта.

**Тип**: `str` или `None`

### `__project_name__`

**Описание**: Название проекта, взятое из настроек (`settings`). По умолчанию `'hypotez'`.

**Тип**: `str`

### `__version__`

**Описание**: Версия проекта, взятая из настроек (`settings`). По умолчанию `''`.

**Тип**: `str`

### `__doc__`

**Описание**: Документация проекта.

**Тип**: `str`

### `__details__`

**Описание**: Подробная информация о проекте.

**Тип**: `str`

### `__author__`

**Описание**: Автор проекта.

**Тип**: `str`

### `__copyright__`

**Описание**: Авторские права.

**Тип**: `str`

### `__cofee__`

**Описание**: Ссылка на поддержку автора (чашечка кофе).

**Тип**: `str`


## Обработка исключений

В модуле используется конструкция `try...excep` для обработки возможных ошибок при чтении файлов `settings.json` и `README.MD`. Если файл не найден или содержит недопустимые данные, соответствующая переменная инициализируется значением `None` или пустой строкой.
