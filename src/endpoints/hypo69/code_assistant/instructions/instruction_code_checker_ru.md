#  ИНСТРУКЦИЯ 
#### Основные требования:

1. **Формат документации**:
   - Использовать **reStructuredText (RST)** для всех комментариев и docstring'ов.
   - В Python-коде всегда использовать одинарные кавычки (`'`).
   
2. **Сохранение комментариев**:
   - Все существующие комментарии после `#` должны сохраняться без изменений.
   - Блоки кода, которые должны быть изменены, комментировать построчно символом `#`.

3. **Обработка данных**:
   - При чтении файлов использовать `j_loads` или `j_loads_ns` из `src.utils.jjson` вместо стандартного `json.load`.
   - Если код содержит `...`, оставлять их без изменений как точки остановки.

4. **Анализ структуры**:
   - Проверять и добавлять недостающие импорты в коде.
   - Согласовывать названия функций, переменных и импортов с ранее обработанными файлами.

5. **Рефакторинг и улучшения**:
   - Добавлять RST-комментарии ко всем функциям, методам и классам.
   - Использовать `from src.logger import logger` для логирования ошибок.
   - Избегать избыточного использования стандартных блоков `try-except`, отдавая предпочтение обработке ошибок с помощью `logger.error`.

6. **Шаблон ответа**:
   - Ответ должен содержать три блока:
     - **Полученный код** — исходный код без изменений.
     - **Улучшенный код** — код с добавленными комментариями и исправлениями.
     - **Изменения** — подробный список внесённых изменений.
   - Ответ не должен начинаться с ` ``` `, их следует использовать только для обрамления блоков кода.

7. **Выходной код**:
   - В конце ответа вернуть полный код (исходный с улучшениями) в виде одного блока, который можно скопировать и вставить вместо исходного кода. 
   - Код, который должен быть изменён, в этом блоке комментировать построчно символом `#`.

8. **Примеры кода**:
   - Включать примеры RST-документации и возможные улучшения в формате `TODO`.
