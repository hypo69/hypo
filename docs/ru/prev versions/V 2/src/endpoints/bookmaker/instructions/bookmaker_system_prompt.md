# Документация для Bookmaker System Prompt

## Обзор

Этот документ описывает промпт для модели машинного обучения, который используется для анализа и прогнозирования данных. Промпт предоставляет инструкции модели по анализу входных данных, определению их пригодности для прогнозирования и формированию ответа в формате JSON.

## Промпт

### Описание

Промпт предназначен для модели машинного обучения и включает в себя следующие инструкции:

1.  **Анализ входных данных:**
    - Модель должна проанализировать предоставленные входные данные в текстовом или числовом формате.
    - Необходимо определить, возможно ли использовать эти данные для прогнозирования (например, временные ряды, данные с паттернами).
2.  **Анализ прогноза (если данные подходят для прогнозирования):**
    - Определение возможных зависимостей или трендов в данных.
    - Оценка значимости данных для прогноза.
    - Предоставление примера прогноза, если это применимо.

## Формат ответа

### Описание

Формат ответа от модели машинного обучения должен быть в формате JSON и включать следующие поля:

-   `is_forecastable` (boolean): Определяет, пригодны ли входные данные для прогнозирования (`true` или `false`).
-   `analysis` (dict): Содержит анализ данных:
    -   `patterns_detected` (str): Описание выявленных паттернов, если `is_forecastable` равно `true`.
    -   `trend` (str): Описание тренда (если применимо), если `is_forecastable` равно `true`.
    -   `forecast_period` (str): Период, на который можно прогнозировать (если применимо), если `is_forecastable` равно `true`.
    -   `reason` (str): Причина, по которой данные непригодны для прогнозирования, если `is_forecastable` равно `false`.
-   `forecast` (str | null): Примерный прогноз (если возможно), иначе `null`.

Пример JSON ответа:
```json
{
  "is_forecastable": true,
  "analysis": {
    "patterns_detected": "описание выявленных паттернов",
    "trend": "описание тренда (если применимо)",
    "forecast_period": "период, на который можно прогнозировать (если применимо)"
  },
  "forecast": "примерный прогноз, если возможно, иначе null"
}
```

## Пример входных данных

### Описание

Примеры входных данных, которые могут быть предоставлены модели:

-   Временной ряд чисел: `[12, 14, 15, 20, 25, ...]`.
-   Таблица с несколькими столбцами: даты, продажи, категории.
-   Текстовые описания с метаданными.

## Пример результата (если прогноз невозможен)

### Описание

Пример JSON ответа, если входные данные непригодны для прогнозирования:

```json
{
  "is_forecastable": false,
  "analysis": {
    "reason": "Недостаточно данных или структура данных не подходит для прогнозирования"
  },
  "forecast": null
}
```

## Примечания

### Описание

Дополнительные заметки для модели:

-   Если входные данные недостаточны для прогнозирования, модель должна аргументировать это.
-   Модель должна использовать статистический и контекстуальный анализ для проверки.
-   Модель не должна делать предположений, если данные явно не содержат временной или прогностической структуры.
