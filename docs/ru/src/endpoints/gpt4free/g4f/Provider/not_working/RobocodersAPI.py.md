# Модуль `RobocodersAPI.py`

## Обзор

Модуль `RobocodersAPI.py` предназначен для взаимодействия с API Robocoders AI. Он предоставляет асинхронный интерфейс для отправки запросов к API, получения ответов и управления сессиями. Модуль поддерживает сохранение и обновление токенов доступа и идентификаторов сессий в кэше для повторного использования.

## Подробней

Модуль предоставляет класс `RobocodersAPI`, который наследуется от `AsyncGeneratorProvider` и `ProviderModelMixin`. Он использует библиотеку `aiohttp` для асинхронных HTTP-запросов и `BeautifulSoup` для парсинга HTML (при наличии). Основная функциональность модуля включает создание и управление сессиями, получение токенов доступа и отправку запросов к API Robocoders AI.

## Классы

### `RobocodersAPI`

**Описание**: Класс `RobocodersAPI` предоставляет интерфейс для взаимодействия с API Robocoders AI.

**Наследует**:
- `AsyncGeneratorProvider`: Обеспечивает асинхронную генерацию ответов.
- `ProviderModelMixin`: Предоставляет функциональность для работы с моделями.

**Атрибуты**:
- `label` (str): Метка провайдера API ("API Robocoders AI").
- `url` (str): URL документации API ("https://api.robocoders.ai/docs").
- `api_endpoint` (str): URL конечной точки API для чата ("https://api.robocoders.ai/chat").
- `working` (bool): Указывает, работает ли API (в данном случае `False`).
- `supports_message_history` (bool): Указывает, поддерживает ли API историю сообщений (`True`).
- `default_model` (str): Модель по умолчанию ("GeneralCodingAgent").
- `agent` (list): Список доступных агентов (моделей).
- `models` (list): Список поддерживаемых моделей, полученный из `agent`.
- `CACHE_DIR` (Path): Директория для кэширования данных (получена из `get_cookies_dir()`).
- `CACHE_FILE` (Path): Файл для кэширования данных (`robocoders.json` в директории `CACHE_DIR`).

**Методы**:
- `create_async_generator`: Создает асинхронный генератор для взаимодействия с API.
- `_get_or_create_access_and_session`: Получает или создает токен доступа и идентификатор сессии.
- `_fetch_and_cache_access_token`: Получает и кэширует токен доступа.
- `_create_and_cache_session`: Создает и кэширует идентификатор сессии.
- `_save_cached_data`: Сохраняет данные в кэш-файл.
- `_update_cached_data`: Обновляет данные в кэш-файле.
- `_clear_cached_data`: Удаляет кэш-файл.
- `_get_cached_data`: Получает данные из кэш-файла.

### `create_async_generator`

```python
    @classmethod
    async def create_async_generator(
        cls,
        model: str,
        messages: Messages,
        proxy: str = None,
        **kwargs
    ) -> AsyncResult:
        """Создает асинхронный генератор для взаимодействия с API Robocoders AI.

        Args:
            model (str): Модель для использования.
            messages (Messages): Список сообщений для отправки.
            proxy (str, optional): Прокси-сервер для использования. По умолчанию `None`.
            **kwargs: Дополнительные аргументы.

        Returns:
            AsyncResult: Асинхронный генератор, выдающий ответы от API.

        Raises:
            Exception: Если не удалось инициализировать взаимодействие с API, если токен недействителен,
                       если произошла ошибка валидации, если произошла ошибка на сервере или если произошла
                       непредвиденная ошибка.

        Внутренние функции:
            Отсутствуют.
        """
```

**Как работает функция**:

1.  **Установка таймаута**: Устанавливается таймаут для HTTP-клиента.
2.  **Создание сессии**: Создается асинхронная HTTP-сессия с установленным таймаутом.
3.  **Получение или создание токена и ID сессии**: Функция пытается получить токен доступа и ID сессии из кэша. Если их нет в кэше или они невалидны, она создает новые, используя методы `_get_or_create_access_and_session`.
4.  **Формирование заголовков**: Формируются заголовки запроса, включающие токен доступа.
5.  **Форматирование промпта**: Форматируется запрос на основе переданных сообщений.
6.  **Подготовка данных для запроса**: Подготавливаются данные для отправки в API, включающие ID сессии, промпт и выбранную модель.
7.  **Отправка запроса**: Отправляется POST-запрос к API с использованием `aiohttp`.
8.  **Обработка ответа**: Обрабатывается ответ от API построчно. Каждая строка декодируется из байтов в строку и парсится как JSON. Извлекается сообщение из полей `args.content` или `message`.
9.  **Обработка ошибок**: Обрабатываются различные ошибки, такие как неавторизованный доступ (401), ошибка валидации (422), ошибка сервера (500) и другие непредвиденные ошибки. В случае ошибки 401 кэш очищается.
10. **Продолжение диалога**: Если API возвращает действие `message` с указанием `wait_for_response`, отправляется запрос на продолжение диалога.
11. **Генерация сообщений**: Функция генерирует сообщения, полученные от API, с использованием `yield`.
