# hypotez/src/ai/prompts/developer/code_checker_ru.md

## Обзор

Данный файл содержит инструкцию для создания документации в формате Markdown для Python-кода.  Документация должна быть подробной, структурированной и соответствовать заданным форматам.

## Инструкция

### Формат документации

Документация должна быть написана в формате Markdown (.md). Каждый файл должен начинаться с заголовка и краткого описания его содержимого. Для всех классов и функций должны использоваться комментарии в формате reStructuredText (RST):

```python
def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:
    """
    Описание функции.

    :param param: Описание параметра `param`.
    :type param: str
    :param param1: Описание параметра `param1`. По умолчанию `None`.
    :type param1: Optional[str | dict | str]
    :returns: Описание возвращаемого значения. Возвращает словарь или `None`.
    :rtype: dict | None
    
    :raises SomeError: Описание ситуации, в которой возникает исключение `SomeError`.
    """
    ...
```

Используйте `ex` вместо `e` в блоках обработки исключений.

### Содержание (TOC)

В начале каждого файла документации должен быть раздел с оглавлением (Table of Contents).  Структура оглавления должна включать ссылки на все основные разделы документации модуля.

### Форматирование

Используйте правильный синтаксис Markdown для всех заголовков, списков и ссылок.  Для документирования классов, функций и методов включайте структурированные разделы с описаниями, деталями параметров, возвращаемых значений, вызываемых исключений.

### Заголовки разделов

Используйте заголовки первого уровня (`#`), второго (`##`), третьего (`###`) и четвёртого (`####`) уровней.
