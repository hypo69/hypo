# Модуль `hypotez/src/utils/string/__init__.py`

## Обзор

Данный модуль предоставляет инструменты для обработки строк, включая валидацию и нормализацию.  Модуль содержит классы и функции для работы с различными типами данных (строки, целые числа, числа с плавающей точкой, булевы значения и даты в формате SQL).


## Модули

### `hypotez.src.utils.string.validator`

Этот модуль содержит класс `ProductFieldsValidator`, предназначенный для валидации полей продукта. Подробнее см. документацию для `ProductFieldsValidator`.


### `hypotez.src.utils.string.normalizer`

Этот модуль предоставляет функции для нормализации различных типов данных.


## Функции

### `normalize_string`

**Описание**: Функция для нормализации входной строки.

**Параметры**:

- `value` (str): Строковое значение для нормализации.

**Возвращает**:

- `str`: Нормализованная строка.


### `normalize_int`

**Описание**: Функция для нормализации входного целого числа.

**Параметры**:

- `value` (str): Строковое представление целого числа для нормализации.

**Возвращает**:

- `int | None`: Нормализованное целое число или `None`, если преобразование невозможно.

**Вызывает исключения**:

- `ValueError`: Если строка `value` не может быть преобразована в целое число.


### `normalize_float`

**Описание**: Функция для нормализации входного числа с плавающей точкой.

**Параметры**:

- `value` (str): Строковое представление числа с плавающей точкой для нормализации.

**Возвращает**:

- `float | None`: Нормализованное число с плавающей точкой или `None`, если преобразование невозможно.

**Вызывает исключения**:

- `ValueError`: Если строка `value` не может быть преобразована в число с плавающей точкой.



### `normalize_boolean`

**Описание**: Функция для нормализации входного булевого значения.

**Параметры**:

- `value` (str): Строковое представление булевого значения для нормализации.

**Возвращает**:

- `bool | None`: Нормализованное булевое значение или `None`, если преобразование невозможно.

**Вызывает исключения**:

- `ValueError`: Если строка `value` не может быть преобразована в булевое значение.


### `normalize_sql_date`

**Описание**: Функция для нормализации входной даты в формате SQL.

**Параметры**:

- `value` (str): Строковое представление даты в формате SQL для нормализации.

**Возвращает**:

- `datetime | None`: Нормализованное значение даты или `None`, если преобразование невозможно.

**Вызывает исключения**:

- `ValueError`: Если строка `value` не может быть преобразована в дату в формате SQL.
