# Инструкции по генерации документации для кода

## Обзор

Этот документ описывает шаги по генерации документации к фрагментам Python-кода в формате Markdown.  Документация должна быть структурированной, понятной и содержать все необходимые детали для разработчика.

## Структура документации

Документация должна быть организована по следующему шаблону:

### Заголовок

В начале документации следует указать название модуля или функции.

### Описание

Краткое и ясное описание функциональности кода.  Что делает этот фрагмент кода?

### Шаги выполнения

Пошаговое описание работы кода.  Какие действия выполняет код, в каком порядке?  Важно быть точным и избегать абстрактных формулировок ("получаем данные", "выполняем действие"). Вместо этого используйте конкретные глаголы ("извлекает", "обрабатывает", "валидирует").

### Пример использования

Конкретный пример использования кода.  Приведенный пример должен демонстрировать, как запустить код и получить ожидаемый результат.  Важно указать входные данные и ожидаемые выходные данные.

### Использование `reStructuredText` (RST)

Для форматирования документации используется язык разметки `reStructuredText (RST)`.  Это обеспечивает структурированность и удобочитаемость документации.  Пример:

```rst
Как использовать этот блок кода
=========================================================================================

Описание
-------------------------
[Объяснение, что делает код.  Подробное описание логики.]

Шаги выполнения
-------------------------
1. [Описание первого шага.  Конкретное описание действия.]
2. [Описание второго шага.  Подробное описание действия.]
3. [Продолжай по необходимости...]

Пример использования
-------------------------
.. code-block:: python

    [Пример использования кода с входными и выходными данными]
```

## Важные рекомендации

- **Конкретность:** Избегайте неопределенных терминов. Используйте конкретные глаголы и существительные.
- **Понятность:** Документация должна быть легко понятна разработчику без дополнительных пояснений.
- **Точность:**  Все описания должны точно отражать функциональность кода.
- **Структурированность:**  Следуйте приведенному шаблону для каждой функции или модуля.
- **Примеры:**  Примеры должны быть полными и демонстрировать как использовать код.

Ниже приведены примеры использования разных структур документации.

### Пример 1: Функция проверки данных

```rst

=========================

Описание
-------------------------

Шаги выполнения
-------------------------
1. Получает входной словарь и шаблон.
2. Проверяет наличие всех ключей из шаблона в входном словаре.
3. Проверяет тип данных каждого значения в словаре по шаблону.
4. Возвращает True, если данные валидны, False - в противном случае.

Пример использования
-------------------------
.. code-block:: python

    import json

    def validate_data(data, schema):
        # ... [Код функции] ...
        return True  # или False


    data = {'name': 'John Doe', 'age': 30}

    is_valid = validate_data(data, schema)
    print(f"Данные валидны: {is_valid}")
```
