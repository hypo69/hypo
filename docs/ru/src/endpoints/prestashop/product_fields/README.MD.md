## Введение

Класс `ProductFields` — это класс Python, который инкапсулирует структуру и поведение полей продукта в базе данных PrestaShop. Он предназначен для упрощения процесса создания, обновления и управления данными продукта, предоставляя понятный и согласованный интерфейс для взаимодействия с полями продукта.

Класс особенно полезен для разработчиков, работающих с API PrestaShop, поскольку он гарантирует, что все необходимые поля правильно отформатированы и проверены перед отправкой в API.

## Инициализация класса

### `__init__`

```python
def __init__(self):
    self.product_fields_list = self._load_product_fields_list()
    self.language = {'en': 1, 'he': 2, 'ru': 3}
    self.presta_fields = SimpleNamespace(**{key: None for key in self.product_fields_list})
    self.assist_fields_dict = {
        'default_image_url': '', 
        'images_urls': []
    }
    self._payload()
```

**Описание**: Метод `__init__` инициализирует класс `ProductFields`, загружая список полей продукта и устанавливая значения по умолчанию для этих полей. Он также инициализирует объект `SimpleNamespace` для хранения полей продукта и вспомогательный словарь для дополнительных полей.

**Как работает класс**:

1.  Загружает список полей продукта, используя метод `_load_product_fields_list`.
2.  Инициализирует словарь `self.language`, который содержит соответствия между кодами языков (`en`, `he`, `ru`) и их идентификаторами.
3.  Создает объект `SimpleNamespace` с именем `self.presta_fields`, который будет использоваться для хранения данных полей продукта. Каждое поле инициализируется значением `None`.
4.  Инициализирует словарь `self.assist_fields_dict` для хранения дополнительных полей, таких как URL изображения по умолчанию и список URL изображений.
5.  Вызывает метод `self._payload()` для загрузки значений по умолчанию для полей продукта из JSON-файла.

**Параметры**:

*   Нет параметров.

**Возвращает**:

*   Нет возвращаемого значения.

**Вызывает исключения**:

*   Не вызывает исключений.

**Примеры**:

```python
product_fields = ProductFields()
```

### `_load_product_fields_list`

```python
def _load_product_fields_list(self) -> List[str]:
    return read_text_file(Path(gs.path.src, 'product', 'product_fields', 'fields_list.txt'), as_list=True)
```

**Описание**: Этот метод загружает список полей продукта из текстового файла. Ожидается, что файл содержит одно имя поля на строку.

**Как работает функция**:

1.  Формирует путь к файлу `fields_list.txt`, используя переменные окружения `gs.path.src`, и библиотеки `Path`.
2.  Вызывает функцию `read_text_file` для чтения содержимого файла. Параметр `as_list=True` указывает, что содержимое файла должно быть возвращено в виде списка строк.
3.  Возвращает список строк, где каждая строка представляет собой имя поля продукта.

**Параметры**:

*   Нет параметров.

**Возвращает**:

*   `List[str]`: Список полей продукта.

**Вызывает исключения**:

*   Не вызывает исключений.

**Примеры**:

```python
product_fields_list = self._load_product_fields_list()
```

### `_payload`

```python
def _payload(self) -> bool:
    data = j_loads(Path(gs.path.src, 'product', 'product_fields', 'product_fields_default_values.json'))
    if not data:
        logger.debug(f"Ошибка загрузки полей из файла {gs.path.src}/product/product_fields/product_fields_default_values.json")
        return False
    for name, value in data.items():
        setattr(self, name, value)
    return True
```

**Описание**: Этот метод загружает значения по умолчанию для полей продукта из JSON-файла. Если файл не найден или не может быть загружен, регистрируется отладочное сообщение.

**Как работает функция**:

1.  Формирует путь к файлу `product_fields_default_values.json`, используя переменные окружения `gs.path.src`, и библиотеки `Path`.
2.  Использует функцию `j_loads` для загрузки данных из JSON-файла.
3.  Проверяет, были ли данные успешно загружены. Если данные не были загружены (т.е. `data` равно `None`), то в лог записывается отладочное сообщение об ошибке, и функция возвращает `False`.
4.  Если данные были успешно загружены, метод перебирает все пары ключ-значение в словаре `data`.
5.  Для каждой пары ключ-значение устанавливает атрибут с именем `name` и значением `value` для текущего объекта `self`, используя функцию `setattr`.
6.  После установки всех значений метод возвращает `True`, указывая на успешную загрузку и установку значений по умолчанию.

**Параметры**:

*   Нет параметров.

**Возвращает**:

*   `bool`: `True`, если значения по умолчанию были успешно загружены и установлены, и `False` в противном случае.

**Вызывает исключения**:

*   Не вызывает исключений.

**Примеры**:

```python
success = self._payload()
if success:
    print("Значения по умолчанию успешно загружены.")
else:
    print("Не удалось загрузить значения по умолчанию.")
```

## Поля продукта

### Поля для одного языка

Поля для одного языка — это поля, которые не требуют перевода и хранятся на одном языке. Примеры включают `id_product`, `id_supplier`, `id_manufacturer` и т.д.

#### Пример: `id_product`

```python
@property
def id_product(self) -> Optional[int]:
    return self.presta_fields.id_product

@id_product.setter
def id_product(self, value: int = None):
    try:
        self.presta_fields.id_product = value
    except ProductFieldException as ex:
        logger.error(f"""Ошибка заполнения поля: 'ID' данными {value}
        Ошибка: """, ex)
        return
```

**Описание**:

*   `id_product` — это свойство, которое позволяет получить или установить идентификатор продукта.

**Как работает функция**:

*   `getter` возвращает значение `id_product` из `self.presta_fields`.
*   `setter` устанавливает значение `id_product` в `self.presta_fields`. Если происходит исключение `ProductFieldException`, оно регистрируется с помощью `logger.error`.

**Параметры**:

*   `value` (int, optional): Значение для установки `id_product`. По умолчанию `None`.

**Возвращает**:

*   `Optional[int]`: Возвращает значение `id_product` или `None`, если значение не установлено.

**Вызывает исключения**:

*   `ProductFieldException`: Вызывается, если происходит ошибка при установке значения `id_product`.

**Примеры**:

```python
product = ProductFields()
product.id_product = 123
print(product.id_product)
```

### Поля для нескольких языков

Поля для нескольких языков — это поля, которые требуют перевода и хранятся на нескольких языках. Примеры включают `name`, `description`, `meta_title` и т.д.

#### Пример: `name`

```python
@property
def name(self):
    return self.presta_fields.name or ''

@name.setter
def name(self, value: str, lang:str = 'en') -> bool:
    try:
        self.presta_fields.name: dict = {'language':
                                                    [
                                                        {'attrs':{'id':self.language[lang]}, 'value': value},
                                                    ]
                                                 }
        return True
    except ProductFieldException as ex:
        logger.error(f"""Ошибка заполнения поля: 'name' данными {value}
        Ошибка: """, ex)
        return
```

**Описание**:

*   `name` — это свойство, которое позволяет получить или установить имя продукта на определенном языке.

**Как работает функция**:

*   `getter` возвращает значение `name` из `self.presta_fields` или пустую строку, если значение не установлено.
*   `setter` устанавливает значение `name` в `self.presta_fields` для указанного языка. Если происходит исключение `ProductFieldException`, оно регистрируется с помощью `logger.error`.

**Параметры**:

*   `value` (str): Значение для установки `name`.
*   `lang` (str, optional): Язык, для которого устанавливается значение `name`. По умолчанию `'en'`.

**Возвращает**:

*   `bool`: Возвращает `True`, если значение `name` было успешно установлено, и `False` в противном случае.

**Вызывает исключения**:

*   `ProductFieldException`: Вызывается, если происходит ошибка при установке значения `name`.

**Примеры**:

```python
product = ProductFields()
product.name = "Product Name", lang='en'
print(product.name)
```

## Ассоциации

Ассоциации используются для связывания продуктов с другими сущностями, такими как категории, производители и поставщики. Свойство `associations` позволяет устанавливать и получать эти ассоциации.

#### Пример: `associations`

```python
@property
def associations(self) -> Optional[Dict]:
    return self.presta_fields.associations or None

@associations.setter
def associations(self, value: Dict[str, Optional[str]]):
    self.presta_fields.associations = value
```

**Описание**:

*   `associations` — это свойство, которое позволяет получить или установить ассоциации продукта.

**Как работает функция**:

*   `getter` возвращает значение `associations` из `self.presta_fields` или `None`, если значение не установлено.
*   `setter` устанавливает значение `associations` в `self.presta_fields`.

**Параметры**:

*   `value` (Dict[str, Optional[str]]): Значение для установки `associations`.

**Возвращает**:

*   `Optional[Dict]`: Возвращает значение `associations` или `None`, если значение не установлено.

**Вызывает исключения**:

*   Не вызывает исключений.

**Примеры**:

```python
product = ProductFields()
product.associations = {'categories': [{'id': 2}, {'id': 3}]}
print(product.associations)
```

## Значения по умолчанию

Значения по умолчанию для полей продукта можно загрузить из JSON-файла с помощью метода `_payload`. Этот метод гарантирует, что все поля имеют значение по умолчанию, которое можно переопределить при необходимости.

## Обработка ошибок

Класс включает надежную обработку ошибок для перехвата и регистрации любых исключений, которые происходят во время установки полей продукта. Это гарантирует, что любые проблемы регистрируются и могут быть устранены разработчиком.

#### Пример: Обработка ошибок в `id_product`

```python
@id_product.setter
def id_product(self, value: int = None):
    try:
        self.presta_fields.id_product = value
    except ProductFieldException as ex:
        logger.error(f"""Ошибка заполнения поля: 'ID' данными {value}
        Ошибка: """, ex)
        return
```

**Описание**:

*   Этот код демонстрирует обработку ошибок при установке значения `id_product`. Если во время установки значения возникает исключение `ProductFieldException`, оно регистрируется с помощью `logger.error`, и функция возвращает управление.

**Как работает функция**:

1.  Пытается установить значение `id_product` в `self.presta_fields`.
2.  Если происходит исключение `ProductFieldException`, оно перехватывается.
3.  Регистрирует сообщение об ошибке, используя `logger.error`, включая значение, которое пытались установить, и информацию об исключении.
4.  Возвращает управление, чтобы предотвратить дальнейшее выполнение кода.

**Параметры**:

*   `value` (int, optional): Значение для установки `id_product`. По умолчанию `None`.

**Возвращает**:

*   Нет возвращаемого значения.

**Вызывает исключения**:

*   `ProductFieldException`: Вызывается, если происходит ошибка при установке значения `id_product`.

**Примеры**:

```python
product = ProductFields()
product.id_product = 123
```
