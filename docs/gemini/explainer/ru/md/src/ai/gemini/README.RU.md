# Модель `gemini`

## Описание

Данный фрагмент кода описывает классы, связанные с моделью `gemini`, вероятно, являющейся инструментом для обработки естественного языка.  Ключевым элементом является концепция инструкций, которые влияют на поведение модели.

## `system_instruction`

Это системная инструкция, задаваемая при инициализации модели.  Она действует на протяжении всего времени работы экземпляра модели и не может быть изменена позже.  Её можно представить как общее руководство или "настройку по умолчанию" для модели. Она передается в модель как промт (`prompt`).

```python
    def __init__(self,
                 api_key: str,
                 model_name: Optional[str] = None,
                 generation_config: Optional[Dict] = None,
                 system_instruction: Optional[str] = None,
                 **kwargs):
                 ...
```

В приведенном фрагменте кода `system_instruction` — это необязательный параметр конструктора.  Если он не задан, модель будет работать с "пустой" системной инструкцией.

## `command_instruction`

Данный тип инструкции отличается от системной тем, что она передается перед *каждым* запросом к модели. Это позволяет добавлять контекст или изменять поведение модели для конкретного запроса.  Она, как и `system_instruction`, представляет собой промт (`prompt`).  Её можно изменять от запроса к запросу.

Важно отметить, что инструкции хранятся в структурированной файловой системе:

* **Глобальные инструкции:** находятся в директории `src/ai/prompts`.
* **Инструкции для конкретных запросов (или "команд"):** обычно расположены в папке `instruction` у клиента.
* **Список клиентов:** находится в `src.endpoints`.
* **Примеры названий файлов инструкций:**  Поддерживается многоязычный интерфейс. Названия файлов отражают тип инструкции и язык. Например, `instruction_code_checker_ru.md` содержит инструкции для проверки кода на русском языке.

## Использование

Модель `gemini`, скорее всего, использует эти инструкции для придания контекста и специфической направленности обработке данных, например, при переводе, написании кода, ответе на вопросы или выполнении других заданий.  Клиенты, взаимодействующие с моделью, могут загружать или получать инструкции из указанных выше источников, тем самым настраивая её поведение для конкретных задач.
