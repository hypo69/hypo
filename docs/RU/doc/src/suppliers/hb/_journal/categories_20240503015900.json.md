## Структура данных

### Корневой элемент: `scenario`

Корневым элементом JSON-файла является объект `scenario`, который содержит все категории в виде вложенных объектов. Каждая вложенная категория имеет уникальный ключ, соответствующий имени категории.

### Категории

Каждая категория представлена объектом с следующими атрибутами:

#### Атрибуты категорий

*   `url` (str): URL-адрес страницы категории на веб-сайте.
*   `name` (str): Название категории.
*   `presta_categories` (dict): Объект, содержащий информацию о категориях в PrestaShop.

#### Атрибут `presta_categories`

Объект `presta_categories` содержит следующие поля:

*   `default_category` (int): Идентификатор категории по умолчанию в PrestaShop.
*   `additional_categories` (list): Список дополнительных идентификаторов категорий PrestaShop (может быть пустым).

### Вложенные категории

Некоторые категории могут иметь вложенные подкатегории. Вложенные категории представлены аналогичной структурой и могут находиться на любом уровне вложенности.

## Назначение

Этот файл используется для автоматической синхронизации структуры категорий на сайте с соответствующими категориями в PrestaShop. Это позволяет поддерживать актуальность и согласованность структуры каталога товаров между сайтом и бэк-офисом.
