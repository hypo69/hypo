# Модуль mocks

## Обзор

Модуль содержит моки (заглушки) для различных провайдеров, используемых в проекте `hypotez`.
Эти моки предназначены для использования в юнит-тестах, чтобы имитировать поведение реальных провайдеров и изолировать тестируемый код от внешних зависимостей.
В частности, моки используются для имитации работы провайдеров, возвращающих текст, изображений, а также для имитации ошибок, таких как отсутствие аутентификации или возникновение исключений.

## Подробнее

Этот модуль предоставляет набор классов, каждый из которых имитирует определенный тип провайдера или сценарий его работы. Это позволяет тестировать различные аспекты функциональности, зависящие от провайдеров, без необходимости использования реальных провайдеров или доступа к внешним сервисам.

## Классы

### `ProviderMock`

**Описание**: Простейший мок провайдера, возвращающий строку "Mock" при создании завершения.

**Принцип работы**:
Этот класс имитирует синхронный провайдер. Метод `create_completion` генерирует строку "Mock".
Класс используется в тестах, где требуется простое подтверждение работы провайдера.

**Атрибуты**:
- `working` (bool): Всегда `True`, указывает на то, что провайдер "работает".

**Методы**:
- `create_completion`: Метод, имитирующий создание завершения.

### `AsyncProviderMock`

**Описание**: Мок асинхронного провайдера, возвращающий строку "Mock" при создании асинхронного завершения.

**Принцип работы**:
Этот класс имитирует асинхронный провайдер. Асинхронный метод `create_async` возвращает строку "Mock".
Класс используется в тестах, где требуется асинхронное подтверждение работы провайдера.

**Атрибуты**:
- `working` (bool): Всегда `True`, указывает на то, что провайдер "работает".

**Методы**:
- `create_async`: Асинхронный метод, имитирующий создание завершения.
