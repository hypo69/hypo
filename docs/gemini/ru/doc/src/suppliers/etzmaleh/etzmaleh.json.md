# Документация для файла `etzmaleh.json`

## Обзор

Файл `etzmaleh.json` содержит конфигурационные данные для парсера поставщика Etzmaleh. Он определяет параметры, необходимые для сбора и обработки данных о товарах с сайта поставщика.

## Структура файла

Файл представляет собой JSON-объект со следующими полями:

### `supplier`
**Описание**: URL-адрес веб-сайта поставщика.
**Тип**: `строка`
**Пример**: `"https://www.etzmaleh.co.il/"`

### `supplier_id`
**Описание**: Уникальный идентификатор поставщика.
**Тип**: `число`
**Пример**: `11234`

### `supplier_prefix`
**Описание**: Префикс для идентификации поставщика.
**Тип**: `строка`
**Пример**: `"etzmaleh"`

### `active_clients_list`
**Описание**: Список активных клиентов, использующих данного поставщика.
**Тип**: `массив строк`
**Пример**: `["emil-design.com", "e-cat.co.il"]`

### `start_url`
**Описание**: URL-адрес, с которого начинается парсинг сайта поставщика.
**Тип**: `строка`
**Пример**: `"https://www.etzmaleh.co.il/"`

### `price_rule`
**Описание**: Правило ценообразования для товаров поставщика.
**Тип**: `строка`
**Пример**: `"+0"`

### `if_login`
**Описание**: Флаг, указывающий, требуется ли авторизация для доступа к данным поставщика.
**Тип**: `логическое значение`
**Пример**: `false`

### `login_url`
**Описание**: URL-адрес для авторизации, если требуется.
**Тип**: `строка`
**Пример**: `""` (в данном случае авторизация не требуется)

### `lang`
**Описание**: Язык сайта поставщика.
**Тип**: `строка`
**Пример**: `"HE"` (иврит)

### `id_category_default`
**Описание**: Идентификатор категории по умолчанию.
**Тип**: `число`
**Пример**: `11246`

### `compare_categorie_dict`
**Описание**: Флаг, указывающий, нужно ли сравнивать категории.
**Тип**: `логическое значение`
**Пример**: `false`

### `collect_products_from_categorypage`
**Описание**: Флаг, указывающий, нужно ли собирать продукты со страниц категорий.
**Тип**: `логическое значение`
**Пример**: `false`

### `scenario_files`
**Описание**: Список файлов сценариев.
**Тип**: `массив`
**Пример**: `[]` (в данном случае нет файлов сценариев)

### `excluded`
**Описание**: Список исключенных элементов.
**Тип**: `массив`
**Пример**: `[]` (в данном случае нет исключенных элементов)
