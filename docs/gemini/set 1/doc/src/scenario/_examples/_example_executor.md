# Модуль hypotez/src/scenario/_examples/_example_executor.py

## Обзор

Данный модуль содержит примеры использования функций из модуля `src.scenario.executor`. Он демонстрирует, как запускать сценарии, обрабатывать файлы сценариев и взаимодействовать с API PrestaShop.  Примеры охватывают различные ситуации, включая работу со списком файлов сценариев, отдельными файлами и сценариями, а также вставку данных продукта из скриптов в PrestaShop.

## Классы

### `MockSupplier`

**Описание**:  Моковая реализация поставщика данных (Supplier), используемая в примерах. Она имитирует работу с файлами сценариев и взаимодействие с другими модулями.  Эта класс предоставляет заглушки для методов, необходимых для выполнения сценариев, например, `MockRelatedModules` и `MockDriver`.  Необходимо подставить реальные реализации для использования в рабочей среде.

**Атрибуты**:
- `supplier_abs_path`: Путь к папке со сценариями.
- `scenario_files`: Список путей к файлам сценариев.
- `current_scenario`: Текущий сценарий.
- `supplier_settings`: Настройки поставщика.
- `related_modules`: Мок объекта `MockRelatedModules`.
- `driver`: Мок объекта `MockDriver`.


### `MockRelatedModules`

**Описание**:  Моковая реализация модулей, используемых для получения данных (например, списков продуктов).


**Методы**:
- `get_list_products_in_category(s)`: Возвращает список URL страниц продуктов в категории.

- `grab_product_page(s)`: Возвращает объект `ProductFields` с данными о продукте.

- `grab_page(s)`: Асинхронный метод, возвращающий тот же результат что и `grab_product_page`.


### `MockDriver`

**Описание**: Мок объекта драйвера.

**Методы**:
- `get_url(url)`: Возвращает True, если запрос по URL успешно выполнен.


## Функции

### `example_run_scenario_files()`

**Описание**: Пример запуска списка файлов сценариев.

**Параметры**:
- `supplier`: Объект `MockSupplier` с настройками и данными.
- `scenario_files`: Список путей к файлам сценариев.

**Возвращает**:
- `bool`: True, если все сценарии успешно выполнены, иначе False.


### `example_run_scenario_file()`

**Описание**: Пример запуска одного файла сценария.

**Параметры**:
- `supplier`: Объект `MockSupplier`.
- `scenario_file`: Путь к файлу сценария.


**Возвращает**:
- `bool`: True, если сценарий успешно выполнен, иначе False.


### `example_run_scenario()`

**Описание**: Пример запуска одного сценария.

**Параметры**:
- `supplier`: Объект `MockSupplier`.
- `scenario`: Словарь со сценарием.

**Возвращает**:
- `bool`: True, если сценарий успешно выполнен, иначе False.


### `example_insert_grabbed_data()`

**Описание**: Пример вставки данных о продукте в PrestaShop.

**Параметры**:
- `product_fields`: Объект `ProductFields` с данными о продукте.

**Возвращает**:
-  Не имеет возвращаемого значения.


### `example_add_coupon()`

**Описание**: Пример добавления купона с использованием API PrestaShop.

**Параметры**:
- `credentials`: Словарь с данными авторизации.
- `reference`: Ссылка на продукт.
- `coupon_code`: Код купона.
- `start_date`: Дата начала действия купона.
- `end_date`: Дата окончания действия купона.


**Возвращает**:
- Не имеет возвращаемого значения.


### `example_execute_PrestaShop_insert_async()`

**Описание**: Пример асинхронной вставки данных о продукте в PrestaShop.

**Параметры**:
- `product_fields`: Объект `ProductFields`.

**Возвращает**:
- Не имеет возвращаемого значения.


### `example_execute_PrestaShop_insert()`

**Описание**: Пример синхронной вставки данных о продукте в PrestaShop.

**Параметры**:
- `product_fields`: Объект `ProductFields`.

**Возвращает**:
- `bool`: True, если вставка успешна, иначе False.
