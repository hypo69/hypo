# Модуль hypotez/src/suppliers/chat_gpt/header.py

## Обзор

Этот модуль содержит функцию `set_project_root` для определения корневой директории проекта и инициализирует глобальные переменные, содержащие информацию о проекте (имя, версия, описание и т.д.), загружая данные из файла `settings.json` и `README.MD`.  Он также устанавливает корневую директорию проекта в `sys.path`.

## Функции

### `set_project_root`

**Описание**:  Находит корневую директорию проекта, начиная с текущей директории и идя вверх по дереву директорий.

**Параметры**:
- `marker_files` (tuple): Список файлов или директорий, которые используются для определения корневой директории проекта. По умолчанию `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:
- `Path`: Путь к корневой директории проекта, если найдена, иначе директория, где находится скрипт.

**Вызывает исключения**:
-  Не вызывает исключения.


### Глобальные переменные

Этот модуль определяет несколько глобальных переменных, представляющих информацию о проекте:

* `__root__`: Путь к корневой директории проекта.
* `settings`: Словарь настроек проекта, загруженный из `settings.json`.
* `doc_str`: Текст из файла `README.MD`, хранящий описание проекта.
* `__project_name__`: Имя проекта.
* `__version__`: Версия проекта.
* `__doc__`: Документация проекта.
* `__details__`: Подробности проекта.
* `__author__`: Автор проекта.
* `__copyright__`: Авторские права проекта.
* `__cofee__`: Ссылка на сбор средств для разработчика.

**Примечания**: Значения этих переменных загружаются из файла `settings.json` и `README.MD`, если они существуют. В противном случае используется значение по умолчанию.

**Обработка ошибок**: Модуль использует обработку исключений `try...except` для обработки ошибок `FileNotFoundError` и `json.JSONDecodeError`, которые могут возникнуть при чтении файлов `settings.json` и `README.MD`. В случае ошибки соответствующая переменная устанавливается в значение по умолчанию.
