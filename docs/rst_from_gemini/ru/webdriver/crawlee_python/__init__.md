# hypotez/src/webdriver/crawlee_python/__init__.py

**Расположение:** `C:\Users\user\Documents\repos\hypotez\src\webdriver\crawlee_python\__init__.py`

**Роль:** Модуль инициализации для пакета `crawlee_python`.  Предоставляет функции для поиска корневой директории проекта и импортирования необходимых модулей.

**Описание:**

Модуль `__init__.py` отвечает за начальную настройку и импорт компонентов пакета `crawlee_python`.  Он определяет функцию `get_project_root`, которая находит корневую директорию проекта, начиная с текущего файла и идя вверх по директориям.  Это необходимо для корректного импорта модулей из других частей проекта.

**Функции:**

* **`get_project_root(marker_files=('pyproject.toml', 'requirements.txt', '.git'))`:**
    * Находит корневую директорию проекта.
    * Принимает кортеж `marker_files`  (по умолчанию `pyproject.toml`, `requirements.txt`, `.git`) для поиска.
    * Ищет директории, содержащие эти файлы.
    * Если корневая директория не найдена, возвращает директорию, где расположен текущий файл.
    * Добавляет найденную корневую директорию в `sys.path`, если она туда еще не добавлена, что позволяет Python находить модули из других частей проекта.
    * **Возвращает:** `Path` объект, представляющий путь к корневой директории.


**Использование:**

```python
# Внутри другого модуля из пакета crawlee_python
from . import __init__
root_path = __init__.get_project_root()
# Используйте root_path для доступа к ресурсам проекта.
```

**Комментарии:**

* Использование `Path` вместо `str` для работы с путями делает код более надежным и удобным.
* `__root__` используется как переменная для хранения корневого пути, что делает код более читаемым и уменьшает вероятность ошибок.
* Версия пакета (`__version__`), драйвера (`__driver_version__`) и информация о нём (`__name__`, `__doc__`, `__details__`, `__annotations__`, `__author__`)  получаются из подмодулей.

**Важные замечания:**

* `sys.path.insert(0, str(__root__))`:  Этот код важен для правильного импорта модулей из родительских директорий.  Он гарантирует, что Python будет искать необходимые модули в корневом каталоге проекта, прежде чем искать их в стандартной системе.  Это критично для работы импорта `from src import gs`.
* Комментарии в коде должны быть исчерпывающими, описывая назначение переменных и функций.
* Убедитесь, что `pyproject.toml`, `requirements.txt` и `.git` (или другие выбранные маркеры) существуют в корневой директории проекта.
