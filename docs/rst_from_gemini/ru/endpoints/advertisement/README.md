## Основные понятия

* **Объявление:**  Информация о предложении, например, о товаре, услуге или событии.
* **ID объявления:** Уникальный идентификатор каждого объявления.
* **Статус объявления:** Текущее состояние объявления (активное, неактивное, удаленное).
* **Категория объявления:**  Классификация объявления (например, "Товары", "Услуги").
* **Пользователь:** Пользователь, разместивший объявление.


## Маршруты API

Все запросы к API должны выполняться через HTTP.  Ниже приведены примеры основных маршрутов:

| Маршрут                   | Метод    | Описание                                                                                               |
|---------------------------|-----------|--------------------------------------------------------------------------------------------------------|
| `/advertisements`          | GET       | Получение списка всех объявлений.                                                                  |
| `/advertisements/{id}`    | GET       | Получение информации об объявлении по его ID.                                                         |
| `/advertisements`          | POST      | Создание нового объявления.                                                                      |
| `/advertisements/{id}`    | PUT       | Обновление информации об объявлении по его ID.                                                        |
| `/advertisements/{id}`    | DELETE    | Удаление объявления по его ID.                                                                       |
| `/advertisements/search`   | GET       | Поиск объявлений по заданным критериям (например, по ключевым словам, категории). |


## Запросы

Запросы должны соответствовать стандартам HTTP.  Необходимые параметры и их формат указаны в документации к каждому API-маршруту.


## Ответы

API возвращает ответы в формате JSON.  Структура ответа зависит от запрошенного маршрута.  В каждом ответе присутствует поле `status_code`, указывающее код ответа (например, 200 для успешного запроса, 400 для ошибки запроса).
