# Модуль hypotez/src/translators/header.py

## Обзор

Этот модуль содержит вспомогательные функции для работы с проектом, такие как определение корневой директории проекта и загрузка настроек из файла `settings.json`. Он также предоставляет доступ к переменным проекта, таким как имя, версия, описание и другие метаданные.

## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная с директории текущего файла и поднимаясь по дереву директорий, пока не найдет директорию, содержащую один из файлов-маркеров (например, `pyproject.toml`, `requirements.txt`, `.git`).

**Параметры**:

- `marker_files` (tuple): Кортеж имен файлов или директорий, используемых для определения корневой директории проекта.

**Возвращает**:

- `Path`: Путь к корневой директории проекта, если она найдена, иначе - путь к директории, в которой находится скрипт.

**Вызывает исключения**:

-  (Нет)


### `set_project_root`

**Описание**:  (см. выше, но с уточнением). Находит корневую директорию проекта.

**Параметры**:

- `marker_files` (tuple): Список файлов-маркеров для поиска корня проекта.  (начиная с текущего файла).

**Возвращает**:

- `Path`: Путь к корневой директории.

**Вызывает исключения**:

- (Нет)



## Переменные

### `__root__`

**Описание**: Путь к корневой директории проекта, полученный с помощью функции `set_project_root()`.


### `settings`

**Описание**: Словарь настроек проекта, загруженный из файла `settings.json` в корневой директории проекта.

**Тип**: `dict`

**Значение по умолчанию**: `None`


### `doc_str`

**Описание**: Строка с описанием проекта, загруженная из файла `README.MD` в корневой директории проекта.


**Тип**: `str`


**Значение по умолчанию**: `None`


### `__project_name__`

**Описание**: Имя проекта. Возвращает значение из `settings`, если оно существует, в противном случае, возвращает `'hypotez'`.


**Тип**: `str`


### `__version__`

**Описание**: Версия проекта. Возвращает значение из `settings`, если оно существует, в противном случае, возвращает пустую строку `''`.


**Тип**: `str`


### `__doc__`

**Описание**: Документация проекта. Возвращает значение из `doc_str`, если оно существует, в противном случае, возвращает пустую строку `''`.


**Тип**: `str`


### `__details__`

**Описание**:  Дополнительные детали о проекте.


**Тип**: `str`


### `__author__`

**Описание**: Автор проекта. Возвращает значение из `settings`, если оно существует, в противном случае, возвращает пустую строку `''`.

**Тип**: `str`


### `__copyright__`

**Описание**: Авторские права на проект. Возвращает значение из `settings`, если оно существует, в противном случае, возвращает пустую строку `''`.


**Тип**: `str`


### `__cofee__`

**Описание**: Ссылка для помощи разработчику. Возвращает значение из `settings`, если оно существует, в противном случае, возвращает строку с ссылкой.


**Тип**: `str`


## Обработка исключений

В модуле используется конструкция `try...except` для обработки потенциальных ошибок при чтении файла `settings.json` и `README.MD`.  В случае возникновения `FileNotFoundError` или `json.JSONDecodeError`, соответствующие переменные (settings, doc_str) остаются со значениями по умолчанию.
