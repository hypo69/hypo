# Модуль LambdaChat

## Обзор

Модуль `LambdaChat` предоставляет класс для взаимодействия с сервисом Lambda Chat, используя функциональность класса `HuggingChat`. Он определяет параметры подключения, список моделей и алиасы для моделей.

## Подробнее

Данный модуль предназначен для интеграции с платформой Lambda Chat в рамках проекта `hypotez`. Он расширяет возможности `HuggingChat`, адаптируя их под специфику `Lambda Chat`.

## Классы

### `LambdaChat`

**Описание**: Класс `LambdaChat` наследуется от `HuggingChat` и представляет собой адаптацию для работы с сервисом Lambda Chat.

**Наследует**:

- `HuggingChat`: Обеспечивает базовую функциональность для взаимодействия с чат-сервисами на основе Hugging Face.

**Атрибуты**:

- `label` (str): Название провайдера ("Lambda Chat").
- `domain` (str): Доменное имя сервиса ("lambda.chat").
- `origin` (str): Полный URL сервиса (f"https://{domain}").
- `url` (str): URL сервиса, совпадает с `origin`.
- `working` (bool): Флаг, указывающий на работоспособность сервиса (True).
- `use_nodriver` (bool): Флаг, указывающий на использование драйвера (False).
- `needs_auth` (bool): Флаг, указывающий необходимость аутентификации (False).
- `default_model` (str): Модель по умолчанию ("deepseek-llama3.3-70b").
- `reasoning_model` (str): Модель для рассуждений ("deepseek-r1").
- `image_models` (list): Список моделей для работы с изображениями (пустой список).
- `fallback_models` (list): Список запасных моделей, используемых при недоступности основной модели.
- `models` (list): Копия списка `fallback_models`, представляющая доступные модели.
- `model_aliases` (dict): Словарь псевдонимов моделей для удобства использования.

**Методы**:

- Отсутствуют явно определенные методы, поскольку класс наследует их от `HuggingChat`.

## Принцип работы класса `LambdaChat`:

Класс `LambdaChat` наследует функциональность от класса `HuggingChat`, предоставляя интерфейс для взаимодействия с моделями Lambda Chat.

```
A: Определение атрибутов класса (label, domain, url, working, и т.д.)
|
B: Инициализация списка моделей `fallback_models`
|
C: Создание копии `fallback_models` в атрибуте `models`
|
D: Определение псевдонимов моделей в `model_aliases`
```

Где:

- `A`: Устанавливает основные параметры подключения и конфигурации для Lambda Chat.
- `B`: Определяет список моделей, которые будут использоваться в качестве запасных вариантов.
- `C`: Копирует список запасных моделей в основной список моделей для использования.
- `D`: Создает словарь с псевдонимами моделей, чтобы упростить их использование.

## Функции

В данном модуле явно не определены функции, поскольку основная логика реализована через класс `LambdaChat`.
