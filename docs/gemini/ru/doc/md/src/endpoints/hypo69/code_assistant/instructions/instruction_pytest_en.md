# Инструкция по написанию тестов `pytest`

## Обзор

Данная инструкция описывает, как написать тесты для Python-кода с использованием фреймворка `pytest`.  Тесты должны охватывать все ключевые функции, методы и классы, проверяя их корректность, включая граничные случаи и обработку исключений.

## Требования

1. Использовать ясные и описательные имена функций тестов, отражающие их цель.
2. Обеспечить изоляцию и независимость каждого теста друг от друга.
3. Учитывать различные сценарии, включая:
   - Валидные входные данные.
   - Невалидные или непредсказуемые входные данные, если применимо.
   - Граничные или пороговые случаи.
4. Использовать `pytest.raises` для тестирования исключений.
5. При необходимости использовать фикстуры для функций, определяя их отдельно.
6. Добавлять комментарии, объясняющие логику тестовых случаев.

## Пример структуры тестов

```python
import pytest

# Определения фикстур, если необходимо
@pytest.fixture
def example_data():
    """Предоставляет тестовые данные для функции."""
    return {...}

# Тесты для Функции 1
def test_function1_valid_input():
    """Проверяет корректное поведение с валидными данными."""
    # ...
    pass

def test_function1_invalid_input():
    """Проверяет корректную обработку невалидных данных."""
    # ...
    pass

# Тесты для Функции 2
def test_function2_edge_case():
    """Проверяет поведение с граничными случаями."""
    # ...
    pass
```

##  Шаги для создания тестов

1. **Изучите код.** Тщательно изучите предоставленный Python-код, обращая внимание на входные данные, ожидаемые результаты и возможные исключения.
2. **Определите тестовые случаи.** Разработайте тестовые функции, покрывающие различные сценарии, включая корректные входные данные, некорректные входные данные, граничные случаи и обработку исключений.
3. **Используйте `pytest.raises`.** Для проверки обработки исключений используйте `pytest.raises`.
4. **Используйте фикстуры (при необходимости).** Если для функций нужны дополнительные данные, создайте фикстуры для их предоставления.
5. **Напишите тестовые функции.** Напишите тестовые функции с ясными и описательными именами, которые отражают цель проверки.
6. **Протестируйте код.** Запустите тесты с помощью `pytest`.
7. **Проанализируйте результаты.** Анализируйте результаты, исправьте ошибки и внесите корректировки в код по необходимости.


##  Рекомендации по написанию тестов

* Используйте описательные имена функций тестов, например `test_function_with_valid_input`, `test_function_with_invalid_input`.
* Разделите сложные тестовые случаи на более мелкие, изолированные тесты.
* Убедитесь, что каждый тест проверяет только одну функцию или небольшую часть функциональности.
* Используйте утверждения `pytest` для проверки ожидаемых результатов.


##  Примеры Тестов (Заготовка)

**Предполагая, что у вас есть функция `calculate_sum`:**

```python
import pytest

def calculate_sum(a, b):
    """Считает сумму двух чисел."""
    return a + b


def test_calculate_sum_valid_input():
  """Проверяет расчет суммы для валидных входных данных."""
  assert calculate_sum(2, 3) == 5

def test_calculate_sum_zero_input():
  """Проверяет расчет суммы для нулевых входных данных."""
  assert calculate_sum(0, 0) == 0

def test_calculate_sum_negative_input():
  """Проверяет расчет суммы для отрицательных входных данных."""
  assert calculate_sum(-2, -3) == -5

def test_calculate_sum_ex_typeerror():
   with pytest.raises(TypeError):
       calculate_sum("2", 3)

```
