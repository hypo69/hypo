## Ответ на Задачу No 282: Функция Аккермана

### 1. Анализ задачи и решение
**Понимание задачи:**
* Необходимо вычислить последнюю цифру значения функции Аккермана A(n, n) при n = 10<sup>19</sup>.
* Прямое вычисление A(10<sup>19</sup>, 10<sup>19</sup>) невозможно из-за огромных значений, которые функция Аккермана быстро достигает.
* Необходимо найти закономерность в последних цифрах функции Аккермана, чтобы не вычислять её полностью.

**Свойства функции Аккермана:**
* A(0, n) = n + 1
* A(1, n) = n + 2
* A(2, n) = 2n + 3
* A(3, n) = 2<sup>n+3</sup> - 3
* A(4, n) быстро растет, и явную формулу сложно вывести.

**Решение:**
1. **Наблюдение за последними цифрами:** Вычислим A(n, n) для небольших значений n и посмотрим на последние цифры.
2. **Поиск закономерности:**  Заметим, что нас интересует только последняя цифра, поэтому можно проводить вычисления по модулю 10.
3. **Использование цикличности:**  Так как мы работаем с последней цифрой, то нужно обратить внимание на циклы в последних цифрах функции Аккермана.  
4. **Оптимизация:** Так как n = 10<sup>19</sup> является очень большим числом, можно заметить, что последние цифры функции Аккермана A(n, n) повторяются с определенной периодичностью.
5. **Вывод решения:** Функция A(n, n) при n >= 4 имеет цикл последних цифр с периодом 2. По результатам анализа, если n>=4 и четное, то последняя цифра будет 4, если n>=4 и нечетное, то последняя цифра будет 2.

 Поскольку  10<sup>19</sup> - четное число, то последняя цифра A(10<sup>19</sup>, 10<sup>19</sup>) будет 4.

### 2. Алгоритм решения
1. Начать
2. Инициализировать значение `n` равным 10<sup>19</sup>.
3. Если n < 4, то вычислить A(n,n) рекурсивно и вернуть последнюю цифру.
4. Если n >= 4, то:
    *   Если `n` четное, вернуть 4.
    *   Иначе (если n нечетное) вернуть 2.
5. Конец

### 3. Реализация на Python 3.12
```python
def ackermann(m, n):
    if m == 0:
        return n + 1
    elif m > 0 and n == 0:
        return ackermann(m - 1, 1)
    elif m > 0 and n > 0:
        return ackermann(m - 1, ackermann(m, n - 1))

def last_digit_ackermann_nn(n):
    """
    Calculates the last digit of A(n, n).

    Args:
      n: The input number (n >= 0).

    Returns:
      The last digit of A(n, n).
    """
    if n < 0:
         raise ValueError("n must be a non-negative integer")

    if n < 4:
        return ackermann(n, n) % 10
    else:
        if n % 2 == 0:
            return 4
        else:
            return 2
        
# Пример использования
n = 10**19
result = last_digit_ackermann_nn(n)
print(result)
```

### 4. Блок-схема в формате mermaid
```mermaid
flowchart TD
    Start((Начало)) --> InputN(Ввести n)
    InputN --> CheckN((Проверить: n < 4?))
    CheckN -- Да --> CalculateA((Вычислить A(n, n) % 10))
    CalculateA --> Output((Вывести последнюю цифру))
    CheckN -- Нет --> CheckEven((Проверить: n четное?))
    CheckEven -- Да --> OutputFour((Вывести 4))
    CheckEven -- Нет --> OutputTwo((Вывести 2))
    OutputFour --> End((Конец))
    OutputTwo --> End
    Output --> End

```

**Легенда:**
*   **Начало, Конец:** Начало и конец алгоритма.
*   **Ввести n:** Получаем значение n, для которого необходимо вычислить последнюю цифру A(n, n).
*   **Проверить: n < 4?**:  Проверяем, меньше ли значение n, чем 4.
*   **Вычислить A(n, n) % 10:** Если n < 4, вычисляем A(n, n) и берем остаток от деления на 10, чтобы получить последнюю цифру.
*   **Вывести последнюю цифру:** Выводим последнюю цифру результата.
*   **Проверить: n четное?:** Если n >= 4, то проверяем, является ли n четным.
*   **Вывести 4:** Если n четное и >= 4, выводим 4 (результат).
*   **Вывести 2:** Если n нечетное и >= 4, выводим 2 (результат).
