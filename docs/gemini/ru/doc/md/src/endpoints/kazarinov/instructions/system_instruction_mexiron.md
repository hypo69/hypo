# ИНСТРУКЦИЯ ПО СОЗДАНИЮ ДОКУМЕНТАЦИИ ДЛЯ PYTHON-ФАЙЛА

## Обзор

Этот файл содержит инструкцию по созданию документации в формате Markdown для Python-файла, описывающего процесс сборки компьютера.  Документация должна включать описание входных данных, ожидаемого вывода, требований к формату, а также примеры.  Особое внимание уделяется переводу на русский и иврит, а также категоризации сборки.


## Входные данные

### Формат

Входные данные представляют собой список словарей JSON, где каждый словарь описывает компонент компьютера.  Ключи словарей включают:

- `product_id`: Уникальный идентификатор компонента.
- `product_title`: Название компонента.
- `product_description`: Описание компонента и его характеристики.
- `image_local_saved_path`: Путь к изображению компонента (не требует изменений).


### Структура Примера

```json
[
  {
    "product_id": "12345",
    "product_title": "Процессор Intel Core i5-13600K",
    "product_description": "Процессор Intel Core i5-13600K 16-ядерный, 24-потоковый, 3.5 ГГц (до 5.1 ГГц).",
    "image_local_saved_path": "/path/to/image.jpg"
  },
  {
    "product_id": "67890",
    "product_title": "Видеокарта NVIDIA GeForce RTX 4060",
    "product_description": "Видеокарта NVIDIA GeForce RTX 4060 с 8 ГБ памяти GDDR6.",
    "image_local_saved_path": "/path/to/image2.jpg"
  }
]
```

## Функции

### Функция `build_computer_documentation`

**Описание**: Функция принимает список словарей с информацией о компонентах компьютера и возвращает JSON-словарь с документацией на русском и иврите.

**Параметры**:

- `components`: Список словарей, описывающих компоненты компьютера.

**Возвращает**:

- `dict`: JSON-словарь с документацией на русском и иврите.

**Вызывает исключения**:

- `ValueError`: Если входные данные не соответствуют ожидаемому формату.
