# Модуль `utils` для административных задач Telegram-бота

## Обзор

Модуль содержит утилиты для выполнения административных задач в Telegram-боте, таких как удаление сообщений.

## Подробнее

Этот модуль предоставляет функции для удаления сообщений в чате с пользователем. Он используется для очистки интерфейса и поддержания порядка в диалоге бота с администратором.

## Функции

### `process_dell_text_msg`

```python
async def process_dell_text_msg(message: Message, state: FSMContext):
    """Обрабатывает текстовое сообщение, предназначенное для удаления предыдущего сообщения бота и текущего сообщения пользователя.

    Args:
        message (Message): Объект сообщения, полученный от пользователя.
        state (FSMContext): Объект состояния FSM (машина состояний) для хранения данных о контексте диалога.

    Returns:
        None

    Raises:
        Exception: Если происходит ошибка при удалении сообщения.
    """
```

**Как работает функция**:

1.  **Извлечение данных из FSMContext**:
    Функция извлекает данные из контекста FSM (Finite State Machine), чтобы получить идентификатор последнего сообщения, которое нужно удалить.
2.  **Удаление сообщения бота**:
    Если `last_msg_id` существует, функция пытается удалить сообщение с этим идентификатором из чата пользователя.
3.  **Удаление сообщения пользователя**:
    Функция пытается удалить сообщение пользователя, вызвавшего эту функцию.
4.  **Обработка ошибок**:
    Если `last_msg_id` не найден, функция логирует предупреждение. В случае любой другой ошибки при удалении сообщения, функция логирует ошибку.

**Параметры**:

*   `message` (Message): Объект сообщения, содержащий информацию о сообщении, отправленном пользователем.
*   `state` (FSMContext): Объект, представляющий состояние конечного автомата (FSM) для текущего пользователя. Используется для хранения и извлечения данных о состоянии пользователя.

**Возвращает**:

*   `None`: Функция ничего не возвращает.

**Вызывает исключения**:

*   `Exception`: В случае возникновения ошибки при удалении сообщения, информация об ошибке логируется.

**Примеры**:

Пример использования функции в обработчике сообщений:

```python
from aiogram import types
from aiogram.dispatcher import FSMContext

# Пример использования функции
async def some_handler(message: types.Message, state: FSMContext):
    await process_dell_text_msg(message, state)
```
