# Модуль tinytroupe.tools

## Обзор

Этот модуль предоставляет инструменты для агентов, позволяющие им выполнять специализированные задачи. Он содержит классы для создания и управления инструментами, включая обработку действий, контроль за реальным воздействием на мир и проверку прав доступа.

## Классы

### `TinyTool`

**Описание**: Базовый класс для всех инструментов.  Он определяет общие атрибуты и методы для инструментов, такие как имя, описание, владелец, наличие реального воздействия на мир, экспортер и обогатитель результатов.

**Атрибуты**:
- `name` (str): Имя инструмента.
- `description` (str): Краткое описание инструмента.
- `owner` (str): Агент, владеющий инструментом. Если `None`, инструмент может использоваться любым агентом.
- `real_world_side_effects` (bool): Флаг, указывающий, имеет ли инструмент реальное воздействие на мир.
- `exporter` (ArtifactExporter): Экспортер результатов. Если `None`, результаты не экспортируются.
- `enricher` (Enricher): Обогатитель результатов. Если `None`, результаты не обогащаются.

**Методы**:
- `__init__(name, description, owner=None, real_world_side_effects=False, exporter=None, enricher=None)`: Инициализирует новый инструмент.
- `_process_action(agent, action: dict) -> bool`: Обрабатывает действие, выполняемое инструментом. Подлежит переопределению в дочерних классах.
- `_protect_real_world()`: Выводит предупреждение, если инструмент имеет реальное воздействие на мир.
- `_enforce_ownership(agent)`: Проверяет право доступа к инструменту для заданного агента.
- `set_owner(owner)`: Устанавливает владельца инструмента.
- `actions_definitions_prompt() -> str`: Возвращает подсказку для определения действий инструмента. Подлежит переопределению в дочерних классах.
- `actions_constraints_prompt() -> str`: Возвращает подсказку для ограничений при использовании инструмента. Подлежит переопределению в дочерних классах.
- `process_action(agent, action: dict) -> bool`: Обрабатывает действие агента с проверкой реального воздействия и прав доступа.


### `TinyCalendar`

**Описание**: Инструмент для работы с календарем. Позволяет агентам отслеживать встречи и назначенные события.

**Атрибуты**:
- `calendar` (dict): Словарь, сопоставляющий даты с списками событий.

**Методы**:
- `__init__(owner=None)`: Инициализирует инструмент календаря.
- `add_event(date, title, description=None, owner=None, mandatory_attendees=None, optional_attendees=None, start_time=None, end_time=None)`: Добавляет событие в календарь.
- `find_events(year, month, day, hour=None, minute=None)`: Ищет события по заданным критериям.
- `_process_action(agent, action) -> bool`: Обрабатывает действие агента по добавлению события в календарь.
- `actions_definitions_prompt() -> str`: Возвращает подсказку для определения действий в календаре.
- `actions_constraints_prompt() -> str`: Возвращает подсказку с ограничениями для действий в календаре.


### `TinyWordProcessor`

**Описание**: Инструмент для работы с текстовыми документами. Позволяет агентам создавать и обогащать текстовые документы.

**Атрибуты**:
- `exporter` (ArtifactExporter): Экспортер результатов. Если `None`, результаты не экспортируются.
- `enricher` (Enricher): Обогатитель результатов. Если `None`, результаты не обогащаются.

**Методы**:
- `__init__(owner=None, exporter=None, enricher=None)`: Инициализирует инструмент word processor.
- `write_document(title, content, author=None)`: Создает и сохраняет документ.
- `_process_action(agent, action) -> bool`: Обрабатывает действие агента по написанию документа.  
- `actions_definitions_prompt() -> str`: Возвращает подсказку для определения действий с документами.
- `actions_constraints_prompt() -> str`: Возвращает подсказку с ограничениями для действий с документами.

В этом модуле нет функций, только классы.
