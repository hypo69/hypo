# Модуль конфигурации TinyTroupe

## Обзор

Этот файл (`config.ini`) содержит конфигурацию для TinyTroupe, определяя параметры взаимодействия с API OpenAI или Azure OpenAI, а также параметры моделирования.  Он устанавливает типы API, версии, модели, лимиты токенов и другие важные настройки.

## Параметры API

### `API_TYPE`

**Описание**: Тип API, используемый для взаимодействия с моделью.  По умолчанию `openai`.

**Возможные значения**: `openai`, `azure`.

### `AZURE_API_VERSION`

**Описание**: Версия API Azure OpenAI. Указано `2023-05-15`.  Важно следить за обновлениями документации Microsoft.

### `MODEL`

**Описание**: Выбранная модель OpenAI для использования.  На данный момент установлено `gpt-4o`.

### `MAX_TOKENS`

**Описание**: Максимальное количество токенов, которые могут быть использованы моделью для генерации ответа. Установлено `4000`.

### `TEMPERATURE`

**Описание**:  Параметр температуры модели, влияющий на вероятность выбора слова. Значение `0.3` приводит к более предсказуемому и структурированному выводу.

### `FREQ_PENALTY`

**Описание**: Наказание за частое повторение слов. Установлено `0.0`.

### `PRESENCE_PENALTY`

**Описание**: Наказание за повторение слов в ответе. Установлено `0.0`.

### `TIMEOUT`

**Описание**: Время ожидания ответа от API в секундах. Установлено `60`.

### `MAX_ATTEMPTS`

**Описание**: Максимальное количество попыток запроса к API в случае ошибки. Установлено `5`.

### `WAITING_TIME`

**Описание**: Время ожидания между попытками запроса в секундах. Установлено `1`.

### `EXPONENTIAL_BACKOFF_FACTOR`

**Описание**: Фактор экспоненциального увеличения времени ожидания между попытками. Установлено `5`.

### `EMBEDDING_MODEL`

**Описание**: Модель для встраивания текста. Установлено `text-embedding-3-small`.

### `CACHE_API_CALLS`

**Описание**: Включить кэширование вызовов API. Установлено `False`.

### `CACHE_FILE_NAME`

**Описание**: Имя файла для кэша вызовов API. Установлено `openai_api_cache.pickle`.

### `MAX_CONTENT_DISPLAY_LENGTH`

**Описание**: Максимальная длина выводимого контента. Установлено `1024`.


## Параметры моделирования

### [Simulation]

### `RAI_HARMFUL_CONTENT_PREVENTION`

**Описание**: Включить предотвращение генерации вредного контента. Установлено `True`.

### `RAI_COPYRIGHT_INFRINGEMENT_PREVENTION`

**Описание**: Включить предотвращение нарушения авторских прав. Установлено `True`.


## Параметры логирования

### [Logging]

### `LOGLEVEL`

**Описание**: Уровень детализации логирования.  Установлено `ERROR`. Возможные значения: `ERROR`, `WARNING`, `INFO`, `DEBUG`.
