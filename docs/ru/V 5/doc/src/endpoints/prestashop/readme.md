# Документация для работы с PrestaShop Websites

## Обзор

Этот файл `README` содержит информацию о структуре и использовании сайтов PrestaShop, а также о хранении и использовании API-ключей. Он предназначен для разработчиков, работающих с API PrestaShop и нуждающихся в информации о подключении и безопасности.

## Подробней

Этот файл предоставляет обзор сайтов PrestaShop, используемых в проекте, объясняет, как безопасно хранить API-ключи, и демонстрирует пример использования API. Он также содержит рекомендации по безопасности и ссылки на дополнительные ресурсы, такие как официальная документация PrestaShop API.

## Хранение API-ключей

API-ключи для каждого сайта хранятся в файле `credentials.kdbx`. Этот файл представляет собой безопасную базу данных паролей и содержит следующие данные для каждого сайта:

*   URL сайта
*   API-ключ
*   Дополнительные метаданные (если необходимо)

Для работы с ключами из файла используйте менеджер паролей, поддерживающий формат `.kdbx`, такой как [KeePass](https://keepass.info/) или [KeePassXC](https://keepassxc.org/).

## Пример использования API

Для подключения к API одного из сайтов используйте следующий шаблон:

### Пример API-запроса

**Шаблон API-запроса:**

```bash
curl -X GET 'https://<SITE_URL>/api/<endpoint>' \
-H 'Authorization: Basic <base64(API_KEY)>'
```

**Объяснение параметров:**

*   `<SITE_URL>` — адрес сайта, например, `e-cat.co.il`.
*   `<endpoint>` — API-endpoint (например, `products`, `customers`).
*   `<API_KEY>` — API-ключ, закодированный в Base64.

### Пример API-вызова

Для получения списка продуктов с `e-cat.co.il`:

```bash
curl -X GET 'https://e-cat.co.il/api/products' \
-H 'Authorization: Basic <base64(API_KEY)>'
```

## Рекомендации по безопасности

*   Никогда не передавайте файл `credentials.kdbx` другим лицам. ❗
*   Убедитесь, что файл хранится в безопасном месте, доступном только вам. (Папка `secrets` в корне проекта исключена из `git`).
*   Регулярно обновляйте свои API-ключи и пароли к базе данных.

## Дополнительные ресурсы

Если у вас возникли какие-либо проблемы или вопросы о подключении к API, обратитесь к [официальной документации PrestaShop API](https://devdocs.prestashop.com/), в которой содержится информация о доступных endpoints и способах взаимодействия с ними.
