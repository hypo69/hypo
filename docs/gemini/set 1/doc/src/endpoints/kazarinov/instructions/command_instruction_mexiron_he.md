# ИНСТРУКЦИЯ ПО ОБРАБОТКЕ КОМАНДЫ ДЛЯ МОДЕЛИ

## Обзор

Данная инструкция описывает обработку команд для модели, направленных на анализ JSON-данных о комплектующих компьютера, классификацию типа сборки (например, игровой ПК, рабочая станция), предоставление заголовков и описаний на иврите, перевод деталей компонентов, и вывод структурированного JSON-вывода.  Инструкция охватывает требования к формату вывода, обработке безопасности, и подробным описаниям компонентов.


## Требования

* **Кодировка:** `UTF-8` - все текстовые данные в ответе должны быть в кодировке UTF-8.  Запрещено использование последовательностей Unicode Escape Sequence (например, `\uXXXX`).
* **Структура ответа:**  Вывод должен быть в формате JSON, соответствующем заданной схеме.  Все поля должны быть заполнены корректными данными.
* **Проверка данных:** Перед отправкой необходимо выполнить проверку всех словарей вывода, чтобы убедиться в отсутствии недостающих полей или ошибок структуры.
* **Перевод:**  Все текстовые данные на входе и выходе должны быть переведены на иврит.
* **Отсутствие ошибок:**  В выводе не должно быть ошибок Unicode Escape Sequence.

## Функции (Примеры)

### `parse_json_and_generate_hebrew_description`

**Описание**: Функция парсит JSON-данные о компонентах компьютера, анализирует их и генерирует описание на иврите, классифицирует тип сборки и возвращает структурированный JSON-ответ.

**Параметры**:

* `input_json` (str): Входные JSON-данные.
* `image_paths`: Путь к файлам изображений.


**Возвращает**:

* dict: Структурированный JSON-ответ.

**Вызывает исключения**:

* `ValueError`: В случае некорректного входного формата JSON.
* `TypeError`: Если входные данные не соответствуют ожидаемому типу.
* `RuntimeError`: Если возникла непредвиденная ошибка при обработке данных.
