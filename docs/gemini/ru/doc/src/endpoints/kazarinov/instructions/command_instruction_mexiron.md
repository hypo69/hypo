# ИНСТРУКЦИЯ ПО ОБРАБОТКЕ КОМАНДЫ ДЛЯ МОДЕЛИ

## Обзор

Данная инструкция описывает обработку команд для модели, предназначенной для анализа компьютерных компонентов из JSON, классификации типа сборки (например, игровая, рабочая станция), генерации названий и описаний на иврите и русском языках, перевода деталей компонентов и возврата структурированного JSON-выхода.

## Шаблон ответа

### Формат ответа

Ответ должен строго соответствовать предоставленному шаблону JSON.  В нем должны быть поля `he` (для иврита) и `ru` (для русского) с подполями `title`, `description`, `build_types`, и `products`.  Внутри `products` должны быть описания компонентов с полями `product_id`, `product_title`, `product_description`, `product_specification` и `image_local_saved_path`.

### Требования к полям

* **`product_id`**:  Должен быть скопирован из входных данных.
* **`product_title`**:  Название продукта на соответствующем языке (иврит или русский).  Если информация для формирования названия недостаточна, поле должно быть пустым.
* **`product_description`**:  Описание продукта на соответствующем языке. Если информация для формирования описания недостаточна, поле должно быть пустым.
* **`product_specification`**:  Технические характеристики продукта на соответствующем языке. Если информация для формирования характеристик недостаточна, поле должно быть пустым.
* **`image_local_saved_path`**: Должен быть скопирован из входных данных.
* **`build_types`**: Словарь, содержащий доверительные оценки для типов сборки (например, `gaming`, `workstation`).
* **`title`**: Заголовок компьютера на соответствующем языке.
* **`description`**: Общее описание компьютера на соответствующем языке.

## Обработка исключений

В случае возникновения ошибок, модель должна возвращать ошибку в формате JSON с соответствующим кодом ошибки и сообщением.  Пример:

```json
{
  "error_code": 400,
  "error_message": "Недостаточно данных для обработки компонента."
}
```

## Кодировка ответа

Кодировка ответа — `UTF-8`.

## Функции

### `analyze_components`

**Описание**: Функция для анализа компьютерных компонентов из входных данных JSON.

**Параметры**:
- `input_json` (str):  JSON-строка с данными о компонентах.

**Возвращает**:
- `dict`:  Структурированный JSON-ответ, соответствующий шаблону.

**Вызывает исключения**:
- `ValueError`: В случае неверного формата входных данных.
- `TypeError`: Если тип входных данных не соответствует ожидаемому.

**Пример использования**:

```python
input_json = '{"components": [...]}'
result = analyze_components(input_json)
```
