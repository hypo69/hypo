## Обзор

Модуль `test_executor.py` содержит набор тестов для класса `ExecuteLocator`, проверяющего его функциональность при работе с веб-элементами. Тесты охватывают получение элементов по локатору, получение атрибутов элементов, отправку сообщений элементам и отправку сообщений с заданной скоростью печати.

## Фикстуры

### `driver_mock`

**Описание**:
Создает фиктивный объект веб-драйвера (MagicMock), который может быть использован для имитации взаимодействия с реальным веб-драйвером.

**Возвращает**:
- `MagicMock`: Фиктивный объект веб-драйвера.

### `execute_locator`

**Описание**:
Создает экземпляр класса `ExecuteLocator` с фиктивным веб-драйвером.

**Параметры**:
- `driver_mock` (`MagicMock`): Фиктивный объект веб-драйвера, созданный фикстурой `driver_mock`.

**Возвращает**:
- `ExecuteLocator`: Экземпляр класса `ExecuteLocator`, использующий фиктивный веб-драйвер.

## Тесты

### `test_get_webelement_by_locator_single_element`

**Описание**:
Проверяет корректное получение одного веб-элемента по заданному локатору.

**Параметры**:
- `execute_locator` (`ExecuteLocator`): Экземпляр класса `ExecuteLocator`, использующий фиктивный веб-драйвер.
- `driver_mock` (`MagicMock`): Фиктивный объект веб-драйвера.

**Проверяет**:
- Вызов метода `find_elements` фиктивного веб-драйвера с правильными аргументами.
- Возвращение веб-элемента, найденного по локатору.

### `test_get_webelement_by_locator_multiple_elements`

**Описание**:
Проверяет корректное получение нескольких веб-элементов по заданному локатору.

**Параметры**:
- `execute_locator` (`ExecuteLocator`): Экземпляр класса `ExecuteLocator`, использующий фиктивный веб-драйвер.
- `driver_mock` (`MagicMock`): Фиктивный объект веб-драйвера.

**Проверяет**:

- Возвращение списка веб-элементов, найденных по локатору.

### `test_get_webelement_by_locator_no_element`

**Описание**:
Проверяет поведение при попытке получить веб-элемент по локатору, для которого нет соответствующих элементов на странице.

**Параметры**:

- `driver_mock` (`MagicMock`): Фиктивный объект веб-драйвера.

**Проверяет**:

**Описание**:
Проверяет корректную отправку сообщения веб-элементу по заданному локатору с заданной скоростью печати.

**Параметры**:
- `execute_locator` (`ExecuteLocator`): Экземпляр класса `ExecuteLocator`, использующий фиктивный веб-драйвер.
- `driver_mock` (`MagicMock`): Фиктивный объект веб-драйвера.

**Проверяет**:

- Вызов метода `send_keys` у найденного веб-элемента для каждого символа сообщения.
- Вызов функции `time.sleep` с заданной скоростью печати.
- Возвращение `True` после успешной отправки сообщения.
