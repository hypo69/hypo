# Инструкция для бота

## Обзор

Этот файл содержит инструкции для взаимодействия с ботом, который используется в проекте `hypotez`. Инструкции представлены в виде команд, которые пользователь может отправлять боту для выполнения различных задач, таких как обучение модели, тестирование модели, архивирование файлов и т.д.

## Подробней

Этот файл предоставляет список команд, которые бот может распознать и выполнить. Каждая команда имеет свой синтаксис и назначение. Разработчики и пользователи могут использовать эти команды для управления ботом и выполнения различных задач, связанных с машинным обучением и обработкой данных.

## Команды

### `!hi`

**Назначение**: Приветствует пользователя.

**Как работает команда**:
1. Пользователь отправляет боту команду `!hi`.
2. Бот отправляет приветственное сообщение пользователю.

**Параметры**:
- Нет параметров.

**Возвращает**:
- Нет возвращаемого значения.

**Вызывает исключения**:
- Нет исключений.

**Примеры**:
```
!hi
```

### `!train <data> <data_dir> <positive> <attachment>`

**Назначение**: Обучает модель с предоставленными данными.

**Как работает команда**:
1. Пользователь отправляет боту команду `!train` с указанием данных для обучения.
2. Бот использует предоставленные данные для обучения модели. Данные могут быть переданы в виде файла (`data`), директории (`data_dir`) или прикрепленного файла (`attachment`). Параметр `positive` указывает на положительные примеры в данных.

**Параметры**:
- `<data>`: Путь к файлу с данными для обучения.
- `<data_dir>`: Путь к директории с данными для обучения.
- `<positive>`: Имя файла или директории, содержащей положительные примеры.
- `<attachment>`: Прикрепленный файл с данными для обучения.

**Возвращает**:
- Нет возвращаемого значения.

**Вызывает исключения**:
- Нет исключений.

**Примеры**:
```
!train data.txt data_dir positive.txt attachment.zip
```

2. Бот использует предоставленные данные для тестирования обученной модели.

**Параметры**:
- `<test_data>`: JSON-данные для тестирования модели.

**Возвращает**:
- Нет возвращаемого значения.

**Вызывает исключения**:
- Нет исключений.

**Примеры**:
```
!test test_data.json
```

### `!select_dataset <path_to_dir_positive> <positive>`

**Назначение**: Выбирает набор данных для обучения из указанной директории.

**Как работает команда**:
1. Пользователь отправляет боту команду `!select_dataset` с указанием пути к директории с положительными примерами и имени файла или директории с положительными примерами.
2. Бот выбирает указанный набор данных для обучения.

**Параметры**:
- `<path_to_dir_positive>`: Путь к директории с положительными примерами.
- `<positive>`: Имя файла или директории, содержащей положительные примеры.

**Возвращает**:
- Нет возвращаемого значения.

**Вызывает исключения**:
- Нет исключений.

**Примеры**:
```
!select_dataset positive_dir positive.txt
```
