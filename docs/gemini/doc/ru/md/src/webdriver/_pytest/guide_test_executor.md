## Введение

Класс `ExecuteLocator` предназначен для взаимодействия с веб-элементами через Selenium WebDriver. Он предоставляет методы для получения веб-элементов, извлечения атрибутов и отправки сообщений.  Это руководство поможет тестировщикам эффективно протестировать функциональность класса.


## Подготовка окружения

### Установка зависимостей

Убедитесь, что все необходимые библиотеки установлены.  Выполните команду:

```bash
pip install -r requirements.txt
```

Файл `requirements.txt` должен содержать следующие зависимости:

```
pytest==7.4.0
selenium==4.16.1
```

### Настройка WebDriver

Установите WebDriver для используемого браузера (например, ChromeDriver для Chrome).


## Написание тестов

### Структура тестов

Создайте файл `test_executor.py` в директории `tests`.  Этот файл будет содержать тесты для `ExecuteLocator`.

### Реализация тестов

#### `test_get_webelement_by_locator_single_element`

**Описание**: Тестирует случай, когда метод находит один элемент.

**Параметры**:
- `execute_locator`: Экземпляр класса `ExecuteLocator`.
- `driver_mock`: Мок-объект для драйвера.

**Возвращает**:
- `True` если тест пройден, `False` иначе.

**Вызывает исключения**:
- `ExecuteLocatorException` в случае ошибки.

#### `test_get_webelement_by_locator_multiple_elements`

**Описание**: Тестирует случай, когда метод находит несколько элементов.

**Параметры**:
- `execute_locator`: Экземпляр класса `ExecuteLocator`.
- `driver_mock`: Мок-объект для драйвера.


#### `test_get_webelement_by_locator_no_element`

**Описание**: Тестирует случай, когда элемент не найден.


#### `test_get_attribute_by_locator`

**Описание**: Тестирует получение атрибута элемента.

#### `test_send_message`

**Описание**: Тестирует отправку сообщения элементу.

#### `test_send_message_typing_speed`

**Описание**: Тестирует отправку сообщения с заданной скоростью печати.


## Запуск тестов

Для запуска тестов выполните следующую команду из корневой директории проекта:

```bash
pytest tests/test_executor.py
```


## Проверка результатов тестирования

`pytest` отобразит результаты тестов в терминале. Убедитесь, что все тесты прошли успешно.


## Обновление тестов

По мере изменений в коде `ExecuteLocator` тесты должны быть обновлены.
