# Порядок действий для создания новой роли

## Обзор

Этот документ описывает порядок действий для создания новой роли в системе.  Процесс включает в себя добавление роли в базовые данные, обновление переводов, создание системного промпта и инструкции для пользователя.

## Порядок действий

### Шаг 1: Добавить роль в `code_assistant.json`

* **Описание**: В файле `code_assistant.json` хранится список доступных ролей.  Необходимо добавить новую роль в этот список, указав ее уникальный идентификатор и другие необходимые атрибуты.

### Шаг 2: Добавить роль в файл переводов `translations/translations.json`

* **Описание**: Для корректного отображения имени роли пользователю необходимо добавить ее перевод в файл `translations/translations.json`.  Это позволит отображать название роли на разных языках.

### Шаг 3: Создать системный промпт в `ai/prompts/develpoper`

* **Описание**: Создать системный промпт, который будет инструктировать модель об использовании новой роли.  Промпт должен содержать информацию о специфике роли и о том, какие задачи она должна выполнять.

### Шаг 4: Создать командную инструкцию в `instructions/...`

* **Описание**:  Разработать командную инструкцию, которая объяснит пользователю, как использовать новую роль.  Эта инструкция должна быть понятна и доступна для широкого круга пользователей.
