# תיעוד קוד - פורמט reStructuredText (RST)

**הקדמה:**

המדריך זה מספק הנחיות לתיעוד קוד בפורמט reStructuredText (RST).  התיעוד יכלול תיאורי מודולים, מחלקות, פונקציות ושיטות, כולל דוגמאות לשימוש.  כל ההערות צריכות להיות ברורות, אינפורמטיביות, ולעמוד בסטנדרטים של RST.


**1. תיעוד מודול:**

מודולים צריכים להכיל תיאור מלא של מטרתו, דוגמאות לשימוש, פלטפורמות תמיכה, וסינופסיס.

**דוגמה:**

```rst
מודול לעבודה עם עוזר תוכנה
=========================================================================================

מודול זה מכיל את המחלקה :class:`CodeAssistant`, המשמשת לעבודה עם דגמי AI שונים,
כגון Google Gemini ו-OpenAI, לביצוע משימות עיבוד קוד.

שימוש לדוגמה
--------------------

שימוש במחלקה `CodeAssistant`:

.. code-block:: python

    assistant = CodeAssistant(role='code_checker', lang='ru', model=['gemini'])
    assistant.process_files()


פלטפורמות תמיכה:
--------------------
- Python 3.x

סינופסיס:
-----------
המודול מאפשר עיבוד קבצים באמצעות דגמי AI.

:class:`CodeAssistant`:
-----------------------
תיאור מלא של המחלקה :class:`CodeAssistant` יינתן בהמשך.
```


**2. תיעוד מחלקה:**

כל מחלקה צריכה להיות מתועדת בהתאם למטרתה, כולל תיאור, מאפיינים ושיטות.


**דוגמה:**

```rst
מחלקה לעבודה עם עוזר תוכנה
=========================================================================================

המחלקה :class:`CodeAssistant` משמשת לעבודה עם דגמי AI שונים כמו Google Gemini,
ומספקת שיטות לניתוח ויצירת תיעוד עבור קוד.

מאפיינים:
----------
- `role`: תפקיד העוזר (לדוגמה, 'code_checker').
- `lang`: השפה בה העוזר יעבוד (לדוגמה, 'ru').
- `model`: רשימה של דגמי AI בשימוש (לדוגמה, ['gemini']).

שיטות:
--------
- `process_files`: שיטה לעיבוד קבצי קוד.


שימוש לדוגמה:
--------------------

.. code-block:: python

    assistant = CodeAssistant(role='code_checker', lang='ru', model=['gemini'])
    assistant.process_files()

:meth:`process_files`:
----------------------

תיאור מלא של השיטה `process_files` יינתן בהמשך.
```


**3. תיעוד פונקציות ושיטות:**

כל פונקציה או שיטה צריכות להיות מתועדות עם תיאור, פרמטרים, ערכים מוחזרים, ודוגמאות.

**דוגמה:**

```rst
שיטה לעיבוד קבצים
=========================================================================================

שיטה זו משמשת לניתוח ועיבוד קבצי קוד.

פרמטרים:
-----------
- `files`: רשימה של קבצים לעיבוד.
- `options`: פרמטרים נוספים לשימוש בקביעת עיבוד הקבצים.

ערך מוחזר:
----------------------
- מחזיר את תוצאת העיבוד כרשימה של נתונים מעובדים.

שימוש לדוגמה:
--------------------

.. code-block:: python

    assistant = CodeAssistant(role='code_checker', lang='ru', model=['gemini'])
    result = assistant.process_files(files=['file1.py', 'file2.py'], options={})

```


**4. הערות בקוד:**

השתמש בלוקים של הערות RST כדי להסביר את הלוגיקה והחלטות בקוד.

**דוגמה:**

```python
# כאן מתבצע טיפול בשגיאות כדי להמשיך בביצוע אם הקובץ לא נמצא
try:
    process_file(file)
except FileNotFoundError as ex:
    handle_exception(ex)
```


**5. תיעוד שגיאות (Exceptions):**

תעד את כל השגיאות שעלולות להתעורר.

**דוגמה:**

```rst
שגיאת קובץ לא נמצא
=========================================================================================

שגיאה זו מתעוררת כאשר קובץ לא נמצא במהלך העיבוד.

פרמטרים:
-----------
- `file`: נתיב הקובץ שלא נמצא.

שימוש לדוגמה:
--------------------

.. code-block:: python

    try:
        open(file)
    except FileNotFoundError as ex:
        raise FileNotFoundError("הקובץ לא נמצא") from ex
```

**הערה חשובה:**  להמשך תהליך התיעוד, יש להוסיף תיעוד מפורט יותר עבור המחלקה `CodeAssistant` (כגון :meth:`process_files`),  בנוסף לתיעוד כל פונקציה או שיטה שאולי קיימת בקוד.
