## <алгоритм>
1. **Инициализация**:
   - Устанавливаются глобальные переменные `MODE`, `RESET` и словари для `TEXT_COLORS`, `BG_COLORS`, `FONT_STYLES`, которые содержат ANSI escape-коды для стилизации текста.
   
2. **Функция `_color_text`**:
   - Принимает текст, цвет текста, цвет фона и стиль шрифта в качестве параметров.
   - Комбинирует ANSI escape-коды для указанных стилей.
   - Возвращает стилизованную строку текста.
     - _Пример:_ `_color_text("Hello", text_color="red", font_style="bold")`  вернет `\033[1m\033[31mHello\033[0m`

3. **Функция `pprint`**:
   - Принимает данные для печати, цвет текста, цвет фона и стиль шрифта как параметры.
   - Приводит значения цветов и стилей к нижнему регистру и проверяет их наличие в соответствующих словарях, используя значения по умолчанию, если не найдены.
   - Обрабатывает различные типы данных:
     - Если `print_data` равен `None`, печатает сообщение об отсутствии данных красным цветом.
       - _Пример:_ `pprint(None)` напечатает "No data to print!" красным цветом.
     - Если `print_data` является словарем, то выводит его в формате JSON с отступами, используя заданный цвет текста.
       - _Пример:_ `pprint({"a": 1, "b": 2}, text_color="green")` напечатает отформатированный JSON.
     - Если `print_data` является списком, то выводит каждый элемент списка на отдельной строке, используя заданный цвет текста.
       - _Пример:_ `pprint(["item1", "item2"], text_color="blue")` напечатает каждый элемент списка на новой строке.
     - Если `print_data` является строкой или `Path`, то:
       - Проверяет, является ли путь файлом.
       - Если файл имеет расширение `.csv` или `.xls`, печатает сообщение о том, что чтение поддерживается только для этих форматов.
       - Для остальных случаев выводит сообщение о неподдерживаемом формате.
       - Если тип данных не подходит ни к одному из вышеперечисленных вариантов, преобразует `print_data` в строку и печатает с указанным цветом.
         - _Пример:_ `pprint(123, text_color="yellow")` напечатает "123" желтым цветом.
   - Обрабатывает исключения при печати, выводя сообщение об ошибке красным цветом.

4. **Пример использования**:
   - Если скрипт запущен напрямую (`if __name__ == '__main__':`), то вызывается функция `pprint` для вывода словаря с данными, при этом используется зеленый цвет.
     - _Пример:_ `pprint({"name": "Alice", "age": 30}, text_color="green")` напечатает отформатированный JSON с данными зеленым цветом.

## <объяснение>
### Импорты:
- **`json`**: Используется для преобразования Python-объектов (в частности, словарей) в JSON-строку и наоборот. Это необходимо для корректного отображения словарей в функции `pprint`.
- **`csv`**: Импортирован, но не используется в данном коде, что может быть потенциальной проблемой (неиспользуемый импорт).
- **`pandas`**: Импортирован, но не используется в данном коде, что также является потенциальной проблемой (неиспользуемый импорт).
- **`pathlib.Path`**: Используется для работы с файловыми путями, включая проверку, является ли путь файлом и для получения расширения файла.
- **`typing.Any`**:  Используется для определения типа `print_data` как любого типа данных в функции `pprint`.
- **`pprint` (из `pprint`)**: Импортируется как `pretty_print`, но не используется в коде, что является потенциальной проблемой (неиспользуемый импорт).

### Переменные:

-   **`MODE`**: Строковая переменная, устанавливающая режим работы (в данном случае, 'dev'). Напрямую не используется в текущем коде, но может использоваться в других частях проекта для определения условий работы.
-   **`RESET`**: Строка, содержащая ANSI escape-код для сброса стилей текста.
-   **`TEXT_COLORS`**: Словарь, где ключи — это строковые названия цветов текста (например, `"red"`, `"blue"`), а значения — это соответствующие ANSI escape-коды.
-   **`BG_COLORS`**: Словарь, аналогичный `TEXT_COLORS`, но для фоновых цветов.
-   **`FONT_STYLES`**: Словарь, где ключи — это названия стилей шрифта (например, `"bold"`, `"underline"`), а значения — это соответствующие ANSI escape-коды.

### Функции:

-   **`_color_text(text: str, text_color: str = "", bg_color: str = "", font_style: str = "") -> str`**:
    -   **Аргументы**:
        -   `text` (str): Текст, который нужно стилизовать.
        -   `text_color` (str, optional): Цвет текста. По умолчанию "".
        -   `bg_color` (str, optional): Цвет фона. По умолчанию "".
        -   `font_style` (str, optional): Стиль шрифта. По умолчанию "".
    -   **Возвращаемое значение**:
        -   `str`: Стилизованный текст, включая ANSI escape-коды.
    -   **Назначение**:
        -   Применяет указанные стили (цвет текста, цвет фона, стиль шрифта) к входному тексту с использованием ANSI escape-кодов и возвращает стилизованный текст.
    -   **Пример**:
        -  `_color_text("Test", text_color="green", font_style="bold")` вернет `\033[1m\033[32mTest\033[0m`

-   **`pprint(print_data: Any = None, text_color: str = "white", bg_color: str = "", font_style: str = "") -> None`**:
    -   **Аргументы**:
        -   `print_data` (Any, optional): Данные для печати. Может быть `None`, словарем, списком, строкой или объектом `Path`. По умолчанию `None`.
        -   `text_color` (str, optional): Цвет текста. По умолчанию "white".
        -   `bg_color` (str, optional): Цвет фона. По умолчанию "".
        -   `font_style` (str, optional): Стиль шрифта. По умолчанию "".
    -   **Возвращаемое значение**:
        -   `None`: Функция ничего не возвращает, а только печатает в консоль.
    -   **Назначение**:
        -   Выводит предоставленные данные в консоль в отформатированном виде. Поддерживает стилизацию текста (цвет, фон, шрифт) и форматирование словарей в JSON.  
    -   **Пример**:
        -   `pprint({"name": "John", "age": 30}, text_color="blue", font_style="underline")` напечатает JSON представление словаря синим подчеркнутым текстом.

### Потенциальные ошибки и области для улучшения:

1.  **Неиспользуемые импорты:**
    -   `csv`, `pandas`, `pprint` импортированы, но не используются в текущем коде. Это следует исправить, удалив лишние импорты.
2.  **Обработка файлов:**
    -   Функция `pprint` проверяет только расширения `.csv` и `.xls`, не обрабатывая сами файлы. Чтение файлов не реализовано.
3.  **Отсутствие обработки ошибок**:
    -   В блоке обработки файлов, если файл не является `.csv` или `.xls`, выводится только сообщение о неподдерживаемом формате. Следует добавить обработку ошибок чтения файлов (например, если файл не существует).
4.  **Жестко заданные значения цветов и стилей**:
    -   Цвета и стили заданы в виде словарей с фиксированными именами. Можно сделать настройку цветов более гибкой, например, с помощью конфигурационного файла или аргументов командной строки.
5.  **Расширение функциональности**:
    -   Можно добавить поддержку других типов данных, таких как кортежи, множества, или настраиваемое форматирование для разных типов данных.
6.  **Тесты**:
     -  Отсутствуют юнит-тесты, которые могли бы проверять корректность работы функций `_color_text` и `pprint`.
