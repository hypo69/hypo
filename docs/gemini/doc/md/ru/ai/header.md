# Модуль hypotez/src/logger/header.py

## Обзор

Этот модуль определяет корневой путь к проекту `hypotez`.  Все импорты строятся относительно этого пути.  В дальнейшем этот путь рекомендуется перенести в системную переменную.  Модуль содержит функцию `get_project_root`, которая находит корневой каталог проекта, начиная от текущего файла.  Он также загружает настройки из файла `settings.json` и документацию из `README.MD`, если такие файлы существуют.

## Содержание

### [Содержание](Содержание.md)

### Функции

#### `get_project_root`

**Описание**:  Находит корневой каталог проекта.

**Параметры**:

- `marker_files` (tuple): Кортеж имен файлов или каталогов, по которым определяется корневой каталог. По умолчанию `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:

- `Path`: Путь к корневому каталогу проекта. Если корневой каталог не найден, возвращает директорию, в которой находится текущий скрипт.

**Вызывает исключения**:

- Не вызывает исключений.


#### `get_project_root`

**Описание**:  Получение корневого пути проекта.  Ищет в родительских директориях, начиная с текущего файла, до тех пор, пока не найдёт каталог, содержащий один из файлов из списка `marker_files`.


**Параметры**:
- `marker_files`: Кортеж файлов, по которым ищется корневой каталог.


**Возвращает**:
- `Path`: Путь к корневому каталогу.


**Вызывает исключения**:
- Не вызывает исключений.


## Переменные

#### `__root__`

**Описание**: Путь к корневому каталогу проекта. Получается вызовом функции `get_project_root`.


**Тип**: `Path`


## Модули

#### `gs`

**Описание**: Модуль `gs` (вероятно, связан с настройками).


## Примечания

- Модуль `hypotez.src.logger.header`  является вспомогательным.
- Для получения настроек и документации используется стандартный метод обработки файлов.
