# Модуль hypotez/src/suppliers/bangood/graber.py

## Обзор

Этот модуль содержит класс `Graber`, предназначенный для сбора данных о товарах с сайта bangood.com. Класс наследуется от базового класса `Graber` из `src.suppliers`. Он предоставляет асинхронную функцию `grab_page` для извлечения данных, используя веб-драйвер и дополнительные поля.  В классе реализованы функции обработки различных полей товара, а также возможность добавления дополнительных обработчиков для нестандартных случаев.  Предварительные действия перед запросом к веб-драйверу могут быть настроены через декоратор `close_pop_up`, который по умолчанию реализован в базовом классе.

## Классы

### `Graber`

**Описание**: Класс для сбора данных о товарах с сайта bangood.com. Наследуется от класса `Graber` из родительского модуля.

**Методы**:

- `__init__(self, driver: Driver)`:
    **Описание**: Инициализация класса.
    **Параметры**:
        - `driver (Driver)`: Экземпляр веб-драйвера.
    **Возвращает**:
        -  None.

- `grab_page(self, driver: Driver) -> ProductFields`:
    **Описание**: Асинхронная функция для сбора полей продукта.
    **Параметры**:
        - `driver (Driver)`: Экземпляр веб-драйвера.
    **Возвращает**:
        - `ProductFields`:  Объект содержащий собранные поля продукта.


## Функции


### `fetch_all_data`


**Описание**: Функция для получения всех данных о продукте, вызывающая методы для конкретных полей.

**Параметры**:
  -  **kwargs**: Словарь, в котором ключи соответствуют именам функций обработки полей, а значения - соответствующие значения.

**Возвращает**:
 - None


**Обрабатываемые функции (не описаны подробно, так как код содержит множество одинаковых параметров):**

В коде присутствует множество асинхронных функций, обработчиков отдельных полей (`id_product`, `name`, `price`, ...), ожидающих параметры из `kwargs`.  Эти функции извлекают значения из соответствующих элементов на странице.  Поля описаны в параметрах каждой из них.


**Примечание:**  Полный список функций и их описаний не помещается из-за большого количества функций. Полное описание отдельных функций из `fetch_all_data` было бы слишком громоздким.
