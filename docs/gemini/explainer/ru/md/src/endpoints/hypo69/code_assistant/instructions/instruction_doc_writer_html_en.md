# Инструкции по генерации документации в формате HTML для файлов Python

Данные инструкции описывают процесс генерации документации в формате HTML для файлов Python, включая структуру, форматирование и содержимое документации.

**Требования к структуре документации:**

* **Формат документации:**
    * Используется стандартный формат HTML.
    * Каждый файл начинается с заголовка и краткого описания его содержимого.
    * Для всех классов и функций используется следующий формат комментариев:

```python
def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:
    """
    Args:
        param (str): Описание параметра `param`.
        param1 (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию `None`.

    Returns:
        dict | None: Описание возвращаемого значения. Возвращает словарь или `None`.

    Raises:
        SomeError: Описание ситуации, при которой возникает исключение `SomeError`.
    """
```
    * В блоках обработки исключений вместо `e` используется `ex`.


* **Содержание (TOC):**
    * В начале каждого файла документации должен быть раздел со списком содержимого (TOC).
    * Структура должна содержать ссылки на все основные разделы документации модуля.


* **Форматирование документации:**
    * Используйте правильный синтаксис HTML для всех заголовков, списков и ссылок.
    * При документировании классов, функций и методов включайте структурированные разделы с описаниями, деталями параметров, возвращаемыми значениями и возбуждаемыми исключениями.  Пример:

```html
<h2>Функции</h2>

<h3><code>function_name</code></h3>

<p><strong>Описание</strong>: Краткое описание функции.</p>

<p><strong>Параметры</strong>:</p>
<ul>
  <li><code>param</code> (str): Описание параметра `param`.</li>
  <li><code>param1</code> (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию `None`.</li>
</ul>

<p><strong>Возвращаемое значение</strong>:</p>
<ul>
  <li><code>dict | None</code>: Описание возвращаемого значения.</li>
</ul>

<p><strong>Возбуждаемые исключения</strong>:</p>
<ul>
  <li><code>SomeError</code>: Описание ситуации, при которой возникает исключение `SomeError`.</li>
</ul>
```


* **Заголовки разделов:**
    * Используйте заголовки уровня 1 (<code><h1></code>), уровня 2 (<code><h2></code>), уровня 3 (<code><h3></code>) и уровня 4 (<code><h4></code>) последовательно в файле.


* **Пример файла (фрагмент):**

```html
<h1>Имя модуля</h1>

<h2>Обзор</h2>
<p>Краткое описание цели модуля.</p>

<h2>Классы</h2>

<h3><code>ИмяКласс</code></h3>

<p><strong>Описание</strong>: Краткое описание класса.</p>

<p><strong>Методы</strong>:</p>
<ul>
  <li><code>метод_имени</code>: Краткое описание метода.</li>
</ul>

<h2>Функции</h2>

<h3><code>функция_имени</code></h3>

<p><strong>Описание</strong>: Краткое описание функции.</p>

<p><strong>Параметры</strong>:</p>
<ul>
  <li><code>параметр</code> (str): Описание параметра `параметр`.</li>
  <li><code>параметр1</code> (Optional[str | dict | str], optional): Описание параметра `параметр1`. По умолчанию `None`.</li>
</ul>

<p><strong>Возвращаемое значение</strong>:</p>
<ul>
  <li><code>dict | None</code>: Описание возвращаемого значения.</li>
</ul>

<p><strong>Возбуждаемые исключения</strong>:</p>
<ul>
  <li><code>SomeError</code>: Описание ситуации, при которой возникает исключение `SomeError`.</li>
</ul>
```


**Формат ответа:** HTML.

**Важно:**  Данная инструкция предполагает, что у вас есть исходные файлы Python, для которых необходимо сгенерировать HTML-документацию.  Для генерации самих HTML-файлов требуется соответствующий инструмент, например, script на Python, который будет парсить файлы с кодом Python и генерировать HTML-документацию согласно указанным правилам.
