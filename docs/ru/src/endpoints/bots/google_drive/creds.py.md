# Модуль для хранения учетных данных

## Обзор

Этот модуль содержит класс `Creds`, который предназначен для хранения учетных данных, необходимых для работы с Telegram ботом и Google Drive API.
Модуль используется для централизованного хранения и управления токенами и идентификаторами, необходимыми для аутентификации и авторизации при взаимодействии с внешними сервисами.

## Подробнее

Этот модуль необходим для хранения и предоставления доступа к различным идентификаторам и токенам, используемым в проекте. В частности, он содержит:
- Токен Telegram бота (`TG_TOKEN`).
- Идентификатор папки TeamDrive (`TEAMDRIVE_FOLDER_ID`).
- Идентификатор TeamDrive (`TEAMDRIVE_ID`).

Эти параметры необходимы для корректной работы функций, связанных с Telegram ботом и загрузкой файлов в Google Drive.

## Классы

### `Creds`

**Описание**: Класс `Creds` предназначен для хранения учетных данных, необходимых для работы с Telegram ботом и Google Drive API.

**Принцип работы**: Класс содержит статические переменные, которые хранят значения токенов и идентификаторов. Эти переменные используются другими модулями для аутентификации и авторизации при работе с внешними сервисами.

**Аттрибуты**:
- `TG_TOKEN` (str): Токен Telegram бота. По умолчанию пустая строка.
- `TEAMDRIVE_FOLDER_ID` (str): Идентификатор папки TeamDrive. По умолчанию пустая строка.
- `TEAMDRIVE_ID` (str): Идентификатор TeamDrive. По умолчанию пустая строка.
