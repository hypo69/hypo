## Документирование кода для Sphinx
    -  Данная инструкция предназначена для подготовки документации, компилируемой с помощью Sphinx.
    -  Убедитесь, что Sphinx установлен и настроен в вашем проекте.

### Структура index.rst:
    - Файл `index.rst` должен содержать дерево оглавлений (TOC) со ссылками на все модули.
    - Пример:
    ```rst
    .. toctree::
        :maxdepth: 2
        :caption: Содержание:

        module1
        module2
        module3
    ```
        -  Где `module1`, `module2` и `module3` — это имена файлов `.rst` с документацией по модулям.

### Примеры использования:
   - Для каждого модуля:
     ```rst
     Название модуля
     ==============
     .. automodule:: module_name
        :members:
        :undoc-members:
        :show-inheritance:

     Функции
     --------

     .. autofunction:: module_name.function_name
     ```
    -  Где `module_name` это имя модуля (файла Python)
    -  `function_name` имя конкретной функции в этом модуле.

### Документация исключений:
    -  В документации к функциям можно использовать раздел "Исключения" для описания ошибок:
    ```python
    def example_function(param: str) -> str:
        """
        Выполняет примерную задачу.

        :param param: Имя пользователя
        :return: Приветствие пользователя.

        :raises ValueError: Если имя пользователя не валидно
        """
        ...
    ```
