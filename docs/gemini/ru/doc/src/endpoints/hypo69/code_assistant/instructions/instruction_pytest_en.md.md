# Инструкция для написания `pytest` тестов

## Обзор

Эта инструкция предназначена для создания тестов с использованием библиотеки `pytest` для Python-кода. Тесты должны охватывать основные функции, методы или классы, проверяя их корректность. Необходимо также включить проверку граничных случаев и обработки исключений.

## Формат тестов

- Используйте `pytest` для создания тестов.
- Каждая тестовая функция должна иметь понятное имя, отражающее её цель.
- Убедитесь, что все тесты изолированы и независимы друг от друга.
- Комментируйте логику тестов для облегчения понимания.

## Содержание тестов

- **Валидные входные данные**: Тесты, проверяющие корректную работу с корректными входными данными.
- **Невалидные входные данные**: Тесты, проверяющие корректную обработку некорректных входных данных.
- **Граничные случаи**: Тесты, проверяющие поведение в граничных условиях.
- **Обработка исключений**: Использование `pytest.raises` для проверки генерации ожидаемых исключений.
- **Фикстуры**: Определение фикстур для подготовки данных, если это необходимо.

**Описание**: Проверяет корректную обработку функцией `function1` невалидных входных данных.

**Описание**: Проверяет поведение функции `function2` в граничных случаях.
