# Модуль `hypotez/src/templates/version.py`

## Обзор

Этот модуль отвечает за получение и хранение метаданных проекта, таких как имя проекта, версия, автор и другие сведения.  Данные берутся из файла `settings.json` в родительском каталоге.  В случае отсутствия файла или некорректного формата JSON, значения по умолчанию устанавливаются.

## Описание переменных

### `MODE`

**Описание**: Переменная, определяющая режим работы (например, `dev`, `prod`). Текущее значение – `'dev'`.

### `settings`

**Описание**: Словарь, хранящий настройки проекта, загруженный из файла `../settings.json`. Если файл не найден или содержит некорректный JSON, `settings` устанавливается в `None`.

### `__project_name__`

**Описание**: Строковая переменная, содержащая название проекта. Получается из `settings.json` или имеет значение по умолчанию `'hypotez'`, если `settings` не задано.

### `__version__`

**Описание**: Строковая переменная, содержащая версию проекта. Получается из `settings.json` или имеет значение по умолчанию пустую строку, если `settings` не задано.

### `__doc__`, `__details__`, `__author__`, `__copyright__`, `__cofee__`

**Описание**: Строковые переменные, содержащие документацию, детали, автора, копирайт и ссылку на донат. Значения взяты из `settings.json` или установлены на значения по умолчанию, если `settings` не задано.


## Обработка исключений

### Обработка файла `settings.json`

**Описание**: При попытке открыть `settings.json`, могут возникнуть следующие исключения:
    * `FileNotFoundError`: Если файл не найден в указанном пути.
    * `json.JSONDecodeError`: Если файл имеет некорректный формат JSON.

В блоке `try...except` эти исключения обрабатываются, и в случае возникновения ошибки переменная `settings` остаётся `None`. Это позволяет избежать сбоя программы при отсутствии или некорректном формате файла.
