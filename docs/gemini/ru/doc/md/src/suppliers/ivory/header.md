# Модуль `hypotez/src/suppliers/ivory/header.py`

## Обзор

Этот модуль содержит функции для работы с настройками проекта, получения пути к корневой директории проекта и загрузки данных из файла `settings.json`.  Он также предоставляет переменные, содержащие информацию о проекте, такие как имя, версия, описание и другие детали.


## Функции

### `set_project_root`

**Описание**:  Находит корневую директорию проекта, начиная от текущего файла и поднимаясь вверх по директориям, пока не найдет директорию, содержащую один из указанных маркеров файлов (pyproject.toml, requirements.txt, .git).

**Параметры**:

- `marker_files` (tuple): Список файлов или директорий, которые используются для определения корневой директории проекта. По умолчанию включает 'pyproject.toml', 'requirements.txt' и '.git'.

**Возвращает**:

- `Path`: Путь к корневой директории проекта. Если корневая директория не найдена, возвращается директория, в которой расположен текущий файл.

**Пример использования**:

```python
root_path = set_project_root()
print(root_path)
```

## Переменные

### `__root__`

**Описание**: Переменная, содержащая путь к корневой директории проекта. Инициализируется функцией `set_project_root()`.


## Обработка исключений

При попытке открыть и обработать файл `settings.json` и `README.MD` могут возникнуть исключения:

- `FileNotFoundError`: Если файлы `settings.json` или `README.MD` не найдены.
- `json.JSONDecodeError`: Если файл `settings.json` содержит некорректный JSON.

В таких случаях значения соответствующих переменных (settings, doc_str) будут установлены в `None` или пустую строку.  Этот код обрабатывает эти потенциальные исключения с помощью блоков `try...except`.

##  Примечания

- Модуль использует `Path` для работы с путями.
- Изменяет `sys.path`, добавляя корень проекта для импорта модулей из подпапок.
- Использует `settings.get()` для безопасного доступа к значениям в словаре настроек, предотвращая ошибки `KeyError`.
