# Модуль `header`

## Обзор

Модуль `header` определяет корневой путь к проекту и загружает базовые настройки. Все импорты в проекте строятся относительно этого пути.

## Функции

### `set_project_root`

**Описание**:
Находит корневой каталог проекта, начиная с каталога текущего файла,
просматривая вверх и останавливаясь на первом каталоге, содержащем любой из файлов-маркеров.

**Параметры**:
- `marker_files` (tuple): Имена файлов или каталогов, используемые для определения корня проекта.

**Возвращает**:
- `Path`: Путь к корневому каталогу, если он найден, иначе каталог, где расположен скрипт.

## Переменные

### `__root__`

**Описание**:
Путь к корневому каталогу проекта.

### `settings`

**Описание**:
Словарь, содержащий настройки проекта, загруженные из файла `settings.json`. Значение `None`, если файл не найден или произошла ошибка декодирования JSON.

### `doc_str`

**Описание**:
Строка, содержащая содержимое файла `README.MD`. Значение `None`, если файл не найден или произошла ошибка.

### `__project_name__`

**Описание**:
Имя проекта, полученное из настроек или значение по умолчанию `'hypotez'`.

### `__version__`

**Описание**:
Версия проекта, полученная из настроек, или пустая строка, если версия не задана.

### `__doc__`

**Описание**:
Описание проекта, взятое из файла `README.MD`, или пустая строка, если файл не найден.

### `__details__`

**Описание**:
Строка с дополнительной информацией о проекте. Изначально пустая строка.

### `__author__`

**Описание**:
Имя автора проекта, полученное из настроек, или пустая строка, если автор не задан.

### `__copyright__`

**Описание**:
Информация о копирайте проекта, полученная из настроек, или пустая строка, если копирайт не задан.

### `__cofee__`

**Описание**:
Строка с призывом угостить разработчика чашечкой кофе, полученная из настроек, или строка по умолчанию.
