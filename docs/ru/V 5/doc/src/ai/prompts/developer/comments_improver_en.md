# Формат Markdown для ответов

## Обзор

Этот файл содержит инструкции по форматированию ответов в Markdown, которые необходимо соблюдать при работе с проектом `hypotez`. Он описывает структуру ответа, включая блоки с улучшенным кодом, комментариями и подробные списки изменений.

## Подробнее

Основная цель данного файла - предоставить четкие указания по созданию структурированных и понятных ответов в формате Markdown. Это необходимо для обеспечения единообразного стиля документации и упрощения процесса анализа изменений в коде.

## Структура ответа

### Улучшенный код:

- Блок с улучшенным кодом, отформатированным и с добавленными комментариями.

### Изменения:

- Подробный список модификаций и объяснений.

Код должен быть обрамлён в соответствующие теги подсветки синтаксиса (например, `python`, `markdown`, `json`).

## Примеры

### Пример 1 (Код Python):

#### Вход:
```python
def add_numbers(a,b):
    return a+b
```

#### Ожидаемый ответ:

### Улучшенный код:
```python
def add_numbers(a: int, b: int) -> int:
    """
    Функция складывает два числа.

    Args:
        a (int): Первое число.
        b (int): Второе число.

    Returns:
        int: Сумма чисел `a` и `b`.
    """
    return a + b
```

### Изменения:
- Добавлена документация в стиле RST для описания функции.
- Добавлены аннотации типов для `a` и `b`.
- Добавлены пробелы вокруг `+` и параметров в определении функции для улучшения читаемости.

### Оптимизированный полный код:
```python
def add_numbers(a: int, b: int) -> int:
    """
    Функция складывает два числа.

    Args:
        a (int): Первое число.
        b (int): Второе число.

    Returns:
        int: Сумма чисел `a` и `b`.
    """
    return a + b
```
