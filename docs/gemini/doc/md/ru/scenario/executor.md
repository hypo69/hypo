# Модуль executor.py

## Обзор

Этот модуль содержит функции для выполнения сценариев сбора данных о продуктах с веб-сайтов и последующего сохранения их в базе данных PrestaShop.  Модуль предоставляет функции для обработки списка файлов сценариев, отдельных файлов и списка сценариев, позволяя гибко управлять процессом.

## Функции

### `run_scenario_files`

**Описание**: Функция выполняет сценарии, указанные в переданном списке файлов.  Каждый файл интерпретируется как набор сценариев, и каждый сценарий выполняется последовательно.

**Параметры**:
- `s` (Supplier): Экземпляр класса Supplier, содержащий необходимые настройки и модули для выполнения сценария.
- `scenario_files_list` (List[Path] | Path): Список путей к файлам сценариев в формате JSON.  Если передан один путь, то предполагается, что это путь к файлу, а не список путей.  Если параметр не передан, используется значение из настроек `s.scenario_files`.

**Возвращает**:
- `bool`: `True`, если все сценарии были успешно выполнены, иначе `False`.

**Вызывает исключения**:
- `FileNotFoundError`: Если файл сценария не найден.
- `Exception`: Если возникла любая другая ошибка во время выполнения сценариев.


### `run_scenario_file`

**Описание**: Функция загружает сценарии из заданного JSON-файла и выполняет их.

**Параметры**:
- `s` (Supplier): Экземпляр класса Supplier, содержащий необходимые настройки и модули для выполнения сценария.
- `scenario_file` (Path | str): Путь к файлу сценария в формате JSON.

**Возвращает**:
- `bool`: `True`, если все сценарии в файле были успешно выполнены, иначе `False`.

**Вызывает исключения**:
- `FileNotFoundError`: Если файл сценария не найден.
- `json.JSONDecodeError`: Если файл сценария не является валидным JSON.
- `Exception`: Если возникла любая другая ошибка во время выполнения сценариев.


### `run_scenarios`

**Описание**: Функция выполняет переданные сценарии.  Принимает список словарей сценариев или один словарь сценария.

**Параметры**:
- `s` (Supplier): Экземпляр класса Supplier.
- `scenarios` (List[dict] | dict, optional): Список сценариев или один сценарий в формате словаря.  Если параметр не передан, используется значение из `s.current_scenario`.

**Возвращает**:
- `List | dict | False`: Результат выполнения сценариев, или `False` в случае ошибки.

**Вызывает исключения**:
- `Exception`: Если возникла любая ошибка во время выполнения сценариев.


### `run_scenario`

**Описание**: Функция выполняет один сценарий.

**Параметры**:
- `supplier` (Supplier): Экземпляр класса Supplier.
- `scenario` (dict): Словарь, описывающий сценарий.
- `scenario_name` (str): Имя сценария.
- `_journal` (dict, optional): Словарь для записи журнала выполнения.

**Возвращает**:
- `List | dict | False`: Результат выполнения сценария.

**Вызывает исключения**:
- `Exception`: Если возникла ошибка во время выполнения сценария.


### `insert_grabbed_data`

**Описание**: Функция вставляет данные, собранные во время выполнения сценария, в базу данных PrestaShop.

**Параметры**:
- `product_fields` (ProductFields): Объект, содержащий данные о продукте.

**Вызывает исключения**:
- `Exception`: Если возникла ошибка при вставке данных в базу данных.

### `execute_PrestaShop_insert`

**Описание**: Функция для выполнения вставки данных о продукте в PrestaShop.

**Параметры**:
- `f` (ProductFields): Объект, содержащий данные о продукте.
- `coupon_code` (str, optional): Купонный код.
- `start_date` (str, optional): Дата начала акции.
- `end_date` (str, optional): Дата окончания акции.

**Возвращает**:
- `bool`: `True`, если вставка успешна, `False` иначе.

**Вызывает исключения**:
- `Exception`: Если возникла ошибка при вставке данных.


## Классы

### `Supplier`

**Описание**: Класс, представляющий поставщика данных.  Должен содержать поля `scenario_files`, `driver`, `related_modules`.
