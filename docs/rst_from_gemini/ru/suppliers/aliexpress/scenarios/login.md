# Модуль `hypotez/src/suppliers/aliexpress/scenarios/login.py`

## Описание

Модуль содержит сценарии для авторизации на сайте aliexpress.com.  Метод `login` осуществляет вход в систему через веб-драйвер, используя сохранённые куки.

## Функции

**`login(s) -> bool`**

* **Описание:**  Функция выполняет логин на AliExpress через веб-драйвер.
* **Параметры:**
    * `s`: Объект класса `Supplier`, содержащий необходимые данные для авторизации (например, веб-драйвер, локаторы).
* **Возвращаемое значение:**
    * `True`: Если авторизация успешна.
    * *Возможные значения при неудаче*:  Функция *должна* возвращать `False` при неудачном логине.  Сейчас возвращается `True` (в режиме отладки), что некорректно.
* **Подробное описание:**
    * Получает `WebDriver` и локаторы из объекта `s`.
    * Переходит на страницу aliexpress.com.
    * Принимает куки (если нужно).
    * Открывает форму входа.
    * Ищет поля для ввода электронной почты, пароля и кнопки входа. (Используется метод `execute_locator` — предполагается, что это часть API для взаимодействия с веб-драйвером.)
    * *Критически важно:* При отсутствии элементов (например, если страница загрузилась некорректно) функция должна обрабатывать это исключение и возвращать `False`.  В текущем коде присутствуют `...` — это placeholder'ы для будущей обработки ошибок.
    * *Отсутствует логика обработки ошибок!*  Если какие-либо из элементов не найдены, происходит `...` — это критичный баг. Код должен обрабатывать эти ошибки и возвращать `False`.
    * Метод `set_language_currency_shipto` не документирован, но предположительно отвечает за настройку языка, валюты и страны доставки.


## Локаторы (Пример)

Локаторы (`_l`) хранятся в объекте `s.locators['login']` и должны содержать пути к элементам страницы для заполнения данных.

Пример:

```python
s.locators['login'] = {
    'cookies_accept': '//button[@id="accept-all-cookies"]',  # Пример локета для принятия куки
    'open_login': '//a[@data-testid="login-link"]',       # Пример локета для открытия формы
    'email_locator': '//input[@type="email"]',              # Локатор поля для ввода email
    'password_locator': '//input[@type="password"]',         # Локатор поля для ввода пароля
    'loginbutton_locator': '//button[@type="submit"]'      # Локатор кнопки входа
}
```


## Важные замечания

* **Обработка ошибок:**  Код должен быть расширен для обработки случаев, когда ожидаемые элементы не найдены.
* **Возвращаемое значение:**  Функция должна чётко определять успех/неуспех авторизации.
* **Логирование:**  Включить логирование (в `logger`) для отслеживания успехов и ошибок.
* **Тестирование:**  Необходимо написать тесты для проверки корректности работы функции `login`.
* **Документация:** Документировать методы `set_language_currency_shipto` и уточнить назначение переменных `_d` и `_l`.
* **Структура:**  Улучшить структуру функции. Разделить на более мелкие, более управляемые части.
