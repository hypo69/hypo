# Модуль для работы ассистента программиста

## Обзор

Этот модуль предоставляет класс `CodeAssistant` для работы с различными моделями ИИ, такими как Google Gemini и OpenAI, для выполнения задач по обработке кода.  Модуль предназначен для создания и анализа документации по Python-коду, обработки файлов кода и вывода результатов. Поддерживает работу с разными языками и моделями.


## Классы

### `CodeAssistant`

**Описание**: Класс `CodeAssistant` предназначен для взаимодействия с различными моделями ИИ. Он предоставляет методы для анализа и создания документации для кода, обработки файлов кода и вывода результатов.

**Атрибуты**:

- `role`: (str) Роль ассистента (например, 'code_checker').
- `lang`: (str) Язык, на котором будет работать ассистент (например, 'ru').
- `model`: (list) Список используемых моделей ИИ (например, ['gemini']).


**Методы**:

#### `process_files`

**Описание**: Метод для обработки файлов с кодом. Анализирует, обрабатывает и возвращает результат анализа.

**Параметры**:
- `files` (list[str]): Список путей к файлам для обработки.
- `options` (dict, optional): Словарь дополнительных параметров для настройки обработки.  По умолчанию None.

**Возвращаемое значение**:
- (list[dict]): Список словарей, каждый из которых содержит результат анализа соответствующего файла. Возвращает None при ошибке.

**Пример использования**:

```python
assistant = CodeAssistant(role='code_checker', lang='ru', model=['gemini'])
result = assistant.process_files(files=['file1.py', 'file2.py'], options={'verbose': True})
if result:
  for item in result:
    print(item)
else:
  print("Ошибка обработки файлов.")
```

**Возможные исключения**:
- `ValueError`: Если параметры `files` или `options` некорректны.
- `FileNotFoundError`: Если один из файлов в списке `files` не найден.
- `Exception`: Любое другое исключение, возникающее при выполнении обработки.
