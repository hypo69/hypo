# Модуль `header`

## Обзор

Модуль `header.py` отвечает за настройку корневой директории проекта, загрузку настроек из файла `settings.json` и получение основных данных проекта (имя, версия, документация и т.д.).

## Функции

### `set_project_root`

**Описание**:
Находит корневой каталог проекта, начиная с каталога текущего файла,
поиск ведется вверх и останавливается на первом каталоге, содержащем любой из файлов-маркеров.

**Параметры**:
- `marker_files` (tuple, optional): Имена файлов или каталогов, идентифицирующие корень проекта. По умолчанию `('__root__', '.git')`.

**Возвращает**:
- `Path`: Путь к корневому каталогу, если найден, в противном случае - каталог, где находится скрипт.

## Глобальные переменные

### `__root__`
**Описание**:
Путь к корневому каталогу проекта.

### `settings`
**Описание**:
Словарь с настройками проекта, загруженный из файла `settings.json`.

### `doc_str`
**Описание**:
Строка документации, загруженная из файла `README.MD`.

### `__project_name__`
**Описание**:
Имя проекта, полученное из настроек, или "hypotez" по умолчанию.

### `__version__`
**Описание**:
Версия проекта, полученная из настроек, или пустая строка по умолчанию.

### `__doc__`
**Описание**:
Строка документации проекта, полученная из файла `README.MD`, или пустая строка по умолчанию.

### `__details__`
**Описание**:
Строка деталей проекта, по умолчанию пустая строка.

### `__author__`
**Описание**:
Автор проекта, полученный из настроек, или пустая строка по умолчанию.

### `__copyright__`
**Описание**:
Авторские права проекта, полученные из настроек, или пустая строка по умолчанию.

### `__cofee__`
**Описание**:
Сообщение с предложением угостить разработчика кофе, полученное из настроек, или стандартное сообщение со ссылкой на Boosty по умолчанию.
