# Инструкции по генерации документации для кода

## Обзор

Этот документ содержит инструкции по созданию документации в формате Markdown для Python-файлов.  Документация должна быть подробной, структурированной и легко читаемой, предоставляя полное описание функциональности каждого файла.

## Структура документации

Каждый документ Markdown должен содержать:

### Заголовок

Заголовок в формате `# Название модуля`, описывающий цель и функциональность документации.

### Оглавление

Список ссылок на основные разделы (Классы, Функции, Методы), например:

```markdown
[Оглавление](#оглавление)

* [Классы](#классы)
* [Функции](#функции)
* [Методы](#методы)
```

### Классы

Для каждого класса в файле создается раздел:

```markdown
## Классы

### `ClassName`

**Описание**: Краткое описание класса.

**Методы**:

- `methodName`: Краткое описание метода `methodName`.
```

Внутри раздела `Методы` следует использовать подробное описание, как это показано в примере ниже.

### Функции

Для каждой функции в файле создается раздел:

```markdown
## Функции

### `functionName`

**Описание**: Краткое описание функции.

**Параметры**:

- `param1` (Тип): Описание параметра `param1`.
- `param2` (Тип, опционально): Описание параметра `param2`. По умолчанию `None`.

**Возвращает**:

- `Возвращаемый тип`: Описание возвращаемого значения.

**Вызывает исключения**:

- `Исключение`: Описание ситуации, в которой возникает исключение.
```

### Методы

Для каждого метода в файле создается раздел (аналогично функциям):

```markdown
## Методы

### `methodName`

**Описание**: Краткое описание метода.

**Параметры**:

- `param1` (Тип): Описание параметра `param1`.
- `param2` (Тип, опционально): Описание параметра `param2`. По умолчанию `None`.

**Возвращает**:

- `Возвращаемый тип`: Описание возвращаемого значения.

**Вызывает исключения**:

- `Исключение`: Описание ситуации, в которой возникает исключение.
```

### Примеры использования

Включите примеры использования функций/методов в каждом разделе, где это возможно. Это поможет пользователям понять, как использовать код в практике.


## Пример

```markdown
# Модуль обработки данных

## Обзор

Этот модуль предоставляет функции для обработки данных.

## Классы

### `DataProcessor`

**Описание**: Класс для обработки данных.

**Методы**:

- `process_data(data: list) -> dict`: Обрабатывает список данных и возвращает обработанные данные в формате словаря.
```


## Важные замечания

- Используйте ясный и понятный язык.
- Указывайте типы данных для параметров и возвращаемых значений.
- Предоставляйте подробные описания для всех параметров, возвращаемых значений и исключений.
- Примеры кода должны быть краткими и демонстрировать типичное использование.
- Структурируйте документацию в соответствии с инструкциями.
- В случае сложных логических операций, разбейте описание на несколько шагов.
