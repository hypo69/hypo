## Обзор

Этот документ описывает инструкции для создания документации для Python-кода в формате Markdown. Инструкции предназначены для стандартизации процесса документирования и обеспечения единообразного формата.

## Содержание

### 1. Формат документации

- Документация должна быть представлена в формате `Markdown (.md)`.
- Каждый файл документации должен начинаться с заголовка и краткого описания содержимого.
- Комментарии для классов и функций должны соответствовать следующему формату:
```python
def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:
    """
    Args:
        param (str): Описание параметра `param`.
        param1 (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию `None`.

    Returns:
        dict | None: Описание возвращаемого значения. Возвращает словарь или `None`.

    Raises:
        SomeError: Описание ситуации, в которой возникает исключение `SomeError`.
    """
```
- Вместо `e` в блоках обработки исключений следует использовать `ex`.

### 2. Оглавление (TOC)
- В начале каждого файла документации должен быть раздел с оглавлением.
- Оглавление должно содержать ссылки на все основные разделы документации модуля.

### 3. Форматирование документации

- Необходимо использовать правильный синтаксис Markdown для заголовков, списков и ссылок.
- Документация классов, функций и методов должна включать структурированные разделы с описаниями, деталями параметров, возвращаемых значений и вызываемых исключений.

### 4. Заголовки разделов

- Использовать заголовки первого (`#`), второго (`##`), третьего (`###`) и четвёртого уровня (`####`) последовательно на протяжении всего файла.

## Пример

```markdown
# Название модуля

## Обзор

Краткое описание назначения модуля.

## Классы

### `ClassName`

**Описание**: Краткое описание класса.

**Методы**:
- `method_name`: Краткое описание метода.
- `method_name`: Краткое описание метода.
**Параметры**:
- `param` (str): Описание параметра `param`.
- `param1` (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию `None`.


## Функции

### `function_name`

**Описание**: Краткое описание функции.

**Методы**:
- `method_name`: Краткое описание метода.   
- `method_name`: Краткое описание метода.

**Параметры**:
- `param` (str): Описание параметра `param`.
- `param1` (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию `None`.

**Возвращает**:
- `dict | None`: Описание возвращаемого значения.

**Вызывает исключения**:
- `SomeError`: Описание ситуации, в которой возникает исключение `SomeError`.
```
