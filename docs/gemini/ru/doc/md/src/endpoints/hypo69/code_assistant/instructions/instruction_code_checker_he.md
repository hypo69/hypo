# מדריך לתיעוד קוד Python

## מבוא

המדריך מציג כיצד ליצור תיעוד Markdown עבור קבצי Python, בהתאם לדרישות הספציפיות.

## מבנה התיעוד

### פורמט RST

השתמשו בפורמט reStructuredText (RST) עבור כל הערות ו-Docstrings.  השתמשו בגרש בודד (`'`) בקוד Python במקום בגרש כפול (`"`).

### שמירת הערות קיימות

כל ההערות הקיימות אחרי `#` צריכות להישמר ללא שינוי. בלוקים של קוד הדורשים שינוי יתוייגו שורה אחר שורה עם `#`.

### שימוש ב-j_loads/j_loads_ns

השתמשו ב-`j_loads` או `j_loads_ns` מתוך `src.utils.jjson` במקום `json.load` לקריאת קבצים. השאירו כל `...` בקוד ללא שינוי.

### ניתוח מבנה הקוד

בדקו והוסיפו מחסרים (imports) לקוד, אם יש צורך. יישרו שמות פונקציות, משתנים ומחסרים בהתאם לקבצים קיימים.

### שיפור ושדרוג

הוסיפו הערות RST לכל הפונקציות, השיטות והמחלקות. השתמשו ב-`from src.logger import logger` לתיעוד שגיאות. הימנעו משימוש יתר ב-`try-except`, והעדיפו טיפול בשגיאות באמצעות `logger.error`.

### תבנית תגובה

התגובה צריכה לכלול שלושה חלקים:

* **Received Code** — הקוד המקורי ללא שינויים.
* **Improved Code** — הקוד לאחר הוספת הערות ותיקונים.
* **Changes Made** — רשימה מפורטת של כל השינויים שבוצעו.

### הערות RST

```python
# דוגמה לפונקציה עם תיעוד RST
def example_function(param1: str, param2: int) -> str:
    """
    פונקציה לדוגמה.

    :param param1: פרמטר 1 (מחרוזת).
    :param param2: פרמטר 2 (מספר שלם).
    :return: מחרוזת המייצגת את התוצאה.
    :raises ValueError: אם param2 אינו מספר שלם.
    """
    # בדיקה אם param2 הוא מספר שלם
    if not isinstance(param2, int):
        logger.error("param2 אינו מספר שלם")
        raise ValueError("param2 must be an integer")

    return f"תוצאה: {param1} {param2}"
