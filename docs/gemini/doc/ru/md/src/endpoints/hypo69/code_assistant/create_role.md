# Создание новой роли

## Обзор

Данный документ описывает порядок действий для создания новой роли в системе.  Процесс включает в себя добавление новой роли в базу данных, обновление файлов переводов, создание системных промтов и командных инструкций.

## Порядок действий

### Шаг 1: Файл `code_assistant.json`

**Описание**:  Этот файл содержит список доступных ролей.  Для создания новой роли необходимо добавить её в этот список.

**Действие**:  Добавить информацию о новой роли в `code_assistant.json`.


### Шаг 2: Добавление роли в `translations/translations.json`

**Описание**:  Необходимо добавить перевод названия роли на все поддерживаемые языки в файле переводов.

**Действие**: Добавить в `translations/translations.json` запись, содержащую имя роли и её переводы.


### Шаг 3: Создание системного промпта в `ai/prompts/develpoper`

**Описание**:  Создаётся промпт (инструкция для модели), который будет использовать система для выполнения задач, связанных с новой ролью.

**Действие**:  Добавить новый промпт в `ai/prompts/develpoper` для соответствующей новой роли.


### Шаг 4: Создание командной инструкции в `instructions/...`

**Описание**:  Создаётся инструкция для пользователя, описывающая, как использовать новую роль.

**Действие**:  Добавить новую командную инструкцию в `instructions/...`, которая описывает использование новой роли.
