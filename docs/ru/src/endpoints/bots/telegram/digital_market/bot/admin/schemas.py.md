# Модуль для работы со схемами продуктов

## Обзор

Модуль содержит Pydantic-схемы для валидации данных, связанных с продуктами. Он включает схемы для идентификации продукта (`ProductIDModel`) и для представления информации о продукте (`ProductModel`).

## Подробнее

Этот модуль предоставляет структуры данных, которые используются для определения формата и ограничений данных продуктов в системе. Схемы Pydantic обеспечивают автоматическую валидацию данных, что помогает гарантировать целостность и корректность информации о продуктах.
`ProductIDModel` используется для валидации ID продукта, а `ProductModel` используется для валидации таких полей, как имя, описание, цена и категория продукта.

## Классы

### `ProductIDModel`

**Описание**: Модель Pydantic для представления идентификатора продукта.

**Как работает класс**:
Класс `ProductIDModel` наследуется от `BaseModel` и определяет структуру для представления ID продукта. Он содержит одно поле `id`, которое должно быть целым числом.

**Параметры**:
- `id` (int): Идентификатор продукта.

**Примеры**:

```python
from pydantic import BaseModel

class ProductIDModel(BaseModel):
    id: int

# Пример использования
product_id = ProductIDModel(id=123)
print(product_id)
```

### `ProductModel`

**Описание**: Модель Pydantic для представления информации о продукте.

**Как работает класс**:
Класс `ProductModel` наследуется от `BaseModel` и определяет структуру для представления информации о продукте, включая имя, описание, цену, ID категории, ID файла и скрытое содержимое. Он использует `Field` для определения ограничений на поля, таких как минимальная длина имени и описания, а также минимальное значение цены и ID категории.

**Параметры**:
- `name` (str): Название продукта (минимальная длина - 5 символов).
- `description` (str): Описание продукта (минимальная длина - 5 символов).
- `price` (int): Цена продукта (должна быть больше 0).
- `category_id` (int): ID категории продукта (должен быть больше 0).
- `file_id` (str | None): ID файла продукта (может быть `None`).
- `hidden_content` (str): Скрытое содержимое продукта (минимальная длина - 5 символов).

**Примеры**:

```python
from pydantic import BaseModel, Field

class ProductModel(BaseModel):
    name: str = Field(..., min_length=5)
    description: str = Field(..., min_length=5)
    price: int = Field(..., gt=0)
    category_id: int = Field(..., gt=0)
    file_id: str | None = None
    hidden_content: str = Field(..., min_length=5)

# Пример использования
product = ProductModel(
    name="Test Product",
    description="This is a test product",
    price=100,
    category_id=1,
    hidden_content="Hidden content"
)
print(product)
```
