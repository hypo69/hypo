# Модуль `hypotez/src/suppliers/aliexpress/gapi/header.py`

## Обзор

Данный модуль определяет корневой путь к проекту `hypotez`. Все импорты строятся относительно этого пути.  В дальнейшем планируется перенести эту логику в системные переменные.

## Функции

### `set_project_root`

**Описание**:  Находит корневой каталог проекта, начиная с текущего каталога файла, перемещаясь вверх по каталогам и останавливаясь на первом, содержащем указанные маркерные файлы.

**Параметры**:

- `marker_files` (tuple): Список имен файлов или каталогов, которые используются в качестве маркеров корневого каталога проекта. По умолчанию (`pyproject.toml`, `requirements.txt`, `.git`).

**Возвращает**:

- `Path`: Путь к корневому каталогу проекта, если найден, иначе возвращается каталог, в котором находится текущий файл.

**Вызывает исключения**:

- Нет


## Переменные

### `__root__`

**Описание**: Содержит путь к корневому каталогу проекта, полученный с помощью функции `set_project_root`.

**Тип**: `Path`

### `settings`

**Описание**: Словарь со настройками, загруженный из файла `settings.json`.

**Тип**: `dict`

### `doc_str`

**Описание**: Содержит строку документации, загруженную из файла `README.MD`.

**Тип**: `str`


### `__project_name__`

**Описание**: Название проекта. Получается из настроек или принимает значение по умолчанию.

**Тип**: `str`

### `__version__`

**Описание**: Версия проекта. Получается из настроек или принимает значение по умолчанию.

**Тип**: `str`

### `__doc__`

**Описание**: Документация проекта. Получается из настроек или принимает значение по умолчанию.

**Тип**: `str`

### `__details__`

**Описание**: Подробности о проекте. Получается из настроек или принимает значение по умолчанию.

**Тип**: `str`

### `__author__`

**Описание**: Автор проекта. Получается из настроек или принимает значение по умолчанию.

**Тип**: `str`

### `__copyright__`

**Описание**: Авторские права. Получается из настроек или принимает значение по умолчанию.

**Тип**: `str`

### `__cofee__`

**Описание**: Ссылка на бонусную систему поддержки разработчика.

**Тип**: `str`


## Обработка исключений

При чтении файлов `settings.json` и `README.MD` используются блоки `try...except`, обрабатывающие следующие исключения:

- `FileNotFoundError`: Обрабатывается случай, когда файл не найден.
- `json.JSONDecodeError`: Обрабатывается случай, когда содержимое файла не является валидным JSON.

В блоках `except` используется пустое `...` для обозначения того, что обработка исключения не приведет к изменению поведения программы.
