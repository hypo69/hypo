# Модуль defaults.py

## Обзор

Модуль `defaults.py` содержит определения стандартных HTTP-заголовков, используемых в запросах к серверам. Он также включает проверку наличия библиотеки `brotli` для поддержки сжатия `br`.

## Подробнее

Этот модуль определяет стандартные заголовки для веб-запросов, обеспечивая совместимость и корректное взаимодействие с различными серверами. Заголовки включают информацию о поддерживаемых типах контента, кодировках, языке, источнике запроса и пользовательском агенте.

## Функции

### `None`

В этом модуле нет функций.

## Переменные

### `has_brotli`

```python
has_brotli = True | False
```

Определяет, установлена ли библиотека `brotli`. Если библиотека установлена, переменная имеет значение `True`, иначе `False`.

**Как работает переменная:**

1.  Попытка импорта библиотеки `brotli`.
2.  Если импорт успешен, `has_brotli` устанавливается в `True`.
3.  Если возникает `ImportError`, `has_brotli` устанавливается в `False`.

```ascii
Попытка импорта brotli
    |
    Успешно?
    /   \
   Да   Нет
   |    |
   V    V
has_brotli = True  has_brotli = False
```

**Примеры:**

```python
import brotli
has_brotli = True # если brotli установлена
```

или

```python
has_brotli = False # если brotli не установлена
```

### `DEFAULT_HEADERS`

```python
DEFAULT_HEADERS: dict
```

Содержит стандартные HTTP-заголовки, используемые по умолчанию для запросов.

**Описание полей словаря:**

-   `"accept"`: Типы контента, которые клиент может обработать (`"*/*"` означает все типы).
-   `"accept-encoding"`: Поддерживаемые методы сжатия (`"gzip, deflate"` и, возможно, `"br"` если `has_brotli` истинно).
-   `"accept-language"`: Предпочитаемый язык (`"en-US"`).
-   `"referer"`: URL страницы, с которой был сделан запрос (пустая строка по умолчанию).
-   `"sec-ch-ua"`: Информация о браузере (Chrome).
-   `"sec-ch-ua-mobile"`: Указывает, является ли устройство мобильным (`"?0"` означает нет).
-   `"sec-ch-ua-platform"`: Операционная система (`"Windows"`).
-   `"sec-fetch-dest"`: Место назначения запроса (`"empty"`).
-   `"sec-fetch-mode"`: Режим запроса (`"cors"`).
-   `"sec-fetch-site"`: Сайт, с которого был сделан запрос (`"same-origin"`).
-   `"user-agent"`: Строка, идентифицирующая браузер (`"Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/123.0.0.0 Safari/537.36"`).

**Как работает переменная:**

Переменная `DEFAULT_HEADERS` содержит словарь, который определяет набор HTTP-заголовков, отправляемых с каждым запросом. Эти заголовки сообщают серверу о возможностях клиента, его предпочтениях и контексте запроса.

**Примеры:**

```python
DEFAULT_HEADERS = {
    "accept": "*/*",
    "accept-encoding": "gzip, deflate, br",
    "accept-language": "en-US",
    "referer": "",
    "sec-ch-ua": "\\"Not(A:Brand\\";v=\\"99\\", \\"Google Chrome\\";v=\\"133\\", \\"Chromium\\";v=\\"133\\"",
    "sec-ch-ua-mobile": "?0",
    "sec-ch-ua-platform": "\\"Windows\\"",
    "sec-fetch-dest": "empty",
    "sec-fetch-mode": "cors",
    "sec-fetch-site": "same-origin",
    "user-agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/123.0.0.0 Safari/537.36",
}
```

### `WEBVIEW_HAEDERS`

```python
WEBVIEW_HAEDERS: dict
```

Содержит HTTP-заголовки, используемые для запросов, выполняемых в контексте WebView.

**Описание полей словаря:**

-   `"Accept"`: Типы контента, которые клиент может обработать (`"*/*"` означает все типы).
-   `"Accept-Encoding"`: Поддерживаемые методы сжатия (`"gzip, deflate, br"`).
-   `"Accept-Language"`: Предпочитаемые языки (пустая строка по умолчанию).
-   `"Referer"`: URL страницы, с которой был сделан запрос (пустая строка по умолчанию).
-   `"Sec-Fetch-Dest"`: Место назначения запроса (`"empty"`).
-   `"Sec-Fetch-Mode"`: Режим запроса (`"cors"`).
-   `"Sec-Fetch-Site"`: Сайт, с которого был сделан запрос (`"same-origin"`).
-   `"User-Agent"`: Строка, идентифицирующая браузер (пустая строка по умолчанию).

**Как работает переменная:**

Переменная `WEBVIEW_HAEDERS` содержит словарь, определяющий набор HTTP-заголовков, специфичных для запросов, выполняемых внутри WebView. WebView - это компонент, который позволяет отображать веб-контент внутри нативного приложения.

**Примеры:**

```python
WEBVIEW_HAEDERS = {
    "Accept": "*/*",
    "Accept-Encoding": "gzip, deflate, br",
    "Accept-Language": "",
    "Referer": "",
    "Sec-Fetch-Dest": "empty",
    "Sec-Fetch-Mode": "cors",
    "Sec-Fetch-Site": "same-origin",
    "User-Agent": "",
}
```
