**Описание:**

Модуль `check_relise.py` отвечает за проверку последней версии релиза репозитория на GitHub. Он использует API GitHub для получения данных о релизах и возвращает версию последнего релиза в виде строки.  Если доступ к API не удаётся или релизов нет, функция возвращает `None`.

**Функции:**

* **`check_latest_release(owner: str, repo: str)`:**
    * **Описание:** Получает последнюю версию релиза репозитория на GitHub.
    * **Аргументы:**
        * `owner` (строка): Имя владельца репозитория.
        * `repo` (строка): Название репозитория.
    * **Возвращаемое значение:**
        * `str`: Версия последнего релиза, если доступна.
        * `None`: Если доступ к данным не удался или релизы отсутствуют.
    * **Обработка ошибок:** Функция проверяет статус ответа от API GitHub. Если статус не 200 (Успешно), то генерируется сообщение об ошибке (сейчас выключено, но необходимо реализовать). В будущем необходимо добавить более подробную обработку ошибок (например, вывод сообщения с причиной ошибки).
    * **Использование `logger`:**  Использует модуль `src.logger` для записи сообщений об ошибках.

**Пример использования (в другом модуле):**

```python
from check_relise import check_latest_release

owner = "your_github_username"
repo = "your_repository_name"

latest_version = check_latest_release(owner, repo)

if latest_version:
    print(f"Latest release version: {latest_version}")
else:
    print("Could not retrieve the latest release.")
```
