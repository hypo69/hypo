# Модуль `hypotez/src/suppliers/aliexpress/api/errors/exceptions.py`

## Обзор

Данный модуль содержит пользовательские исключения, предназначенные для обработки ошибок, возникающих при взаимодействии с API AliExpress.  Он определяет иерархию исключений, начиная от общего базового класса `AliexpressException`, и специфических исключений для различных ситуаций.

## Классы

### `AliexpressException`

**Описание**: Базовый класс для всех исключений, связанных с API AliExpress.  Содержит причину ошибки.

**Методы**:

- `__init__(self, reason: str)`: Инициализирует исключение с указанной причиной.

- `__str__(self) -> str`: Возвращает строковое представление исключения, содержащее причину.


### `InvalidArgumentException`

**Описание**: Исключение, которое поднимается, когда переданные аргументы некорректны.  Наследуется от `AliexpressException`.


### `ProductIdNotFoundException`

**Описание**: Исключение, которое поднимается, если идентификатор продукта не найден. Наследуется от `AliexpressException`.


### `ApiRequestException`

**Описание**: Исключение, которое поднимается, если запрос к API AliExpress завершился неудачно. Наследуется от `AliexpressException`.


### `ApiRequestResponseException`

**Описание**: Исключение, которое поднимается, если ответ от API AliExpress некорректен. Наследуется от `AliexpressException`.


### `ProductsNotFoudException`

**Описание**: Исключение, которое поднимается, если не найдено ни одного продукта. Наследуется от `AliexpressException`.


### `CategoriesNotFoudException`

**Описание**: Исключение, которое поднимается, если не найдено ни одной категории. Наследуется от `AliexpressException`.


### `InvalidTrackingIdException`

**Описание**: Исключение, которое поднимается, если идентификатор отслеживания не предоставлен или некорректен. Наследуется от `AliexpressException`.
