# Инструкция по установке Google Chrome для тестирования

## Введение
   
   Эта инструкция описывает процесс установки Google Chrome, который необходим для запуска автоматизированных тестов в веб-браузере. 
   Следующие шаги подойдут для большинства Linux систем.

## Предварительные шаги

   Перед началом установки убедитесь, что у вас есть права администратора (root) или sudo.
   Также рекомендуется обновить список пакетов:

   .. code-block:: bash

      sudo apt update

## Шаги по установке Google Chrome

1.  **Загрузка ключа Google:**

    .. code-block:: bash
    
        wget -q -O - https://dl-ssl.google.com/linux/linux_signing_key.pub | sudo apt-key add -
        
    *   Код выполняет загрузку публичного ключа Google для проверки подлинности пакетов.

2.  **Добавление репозитория Google Chrome:**

    .. code-block:: bash

        sudo sh -c 'echo "deb [arch=amd64] http://dl.google.com/linux/chrome/deb/ stable main" >> /etc/apt/sources.list.d/google-chrome.list'
        
    *   Код добавляет репозиторий Google Chrome в список источников пакетов.

3.  **Обновление списка пакетов:**
 
    .. code-block:: bash

       sudo apt update
      
    *   Код обновляет список пакетов из добавленного репозитория.

4.  **Установка Google Chrome:**

     .. code-block:: bash

        sudo apt install google-chrome-stable

    * Код устанавливает стабильную версию Google Chrome из добавленного репозитория.

5.  **Проверка установки:**
     
     .. code-block:: bash

        google-chrome --version

     *   Код проверяет установленную версию браузера. Вывод должен содержать информацию о версии.

## Альтернативные методы установки

   В зависимости от вашей операционной системы и требований, могут быть доступны и другие методы установки:

   *  **Установка через .deb пакет:**
      Можно скачать .deb пакет с официального сайта Google Chrome и установить его через `dpkg`:
         
      .. code-block:: bash

         sudo dpkg -i google-chrome-stable_current_amd64.deb
         sudo apt-get install -f # исправление зависимостей, если необходимо
         
   * **Использование менеджера пакетов, отличного от apt:**
      Для других дистрибутивов Linux, например, RedHat или CentOS, могут потребоваться другие менеджеры пакетов (`yum`, `dnf`).

## Завершение

   После выполнения этих шагов у вас должен быть установлен Google Chrome. Теперь вы можете использовать его для автоматизированного тестирования.
   
   Если у вас возникли проблемы, обратитесь к официальной документации Google Chrome или задайте вопрос в сообществе.
