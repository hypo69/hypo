# Модуль `hypotez/src/ai/openai/bully.py`

## Обзор

Модуль `bully.py` предоставляет функцию для взаимодействия с моделью GPT-3.5-turbo или GPT-4, чтобы получить примеры из литературного произведения о том, как хулиганы запугивают своих жертв.

## Переменные

### `MODE`

**Описание**: Строковая переменная, содержащая значение `'dev'`.

**Значение**: `'dev'`


## Функции

### `bully`

**Описание**: Функция, генерирующая примеры из литературных произведений о запугивании с помощью модели GPT-3.5-turbo или GPT-4.

**Параметры**:
- `user_message` (str): Входящее сообщение от пользователя (по умолчанию `"Hello!"`).
- `messages` (list): Список сообщений для модели (по умолчанию список с системным запросом).

**Возвращает**:
- list: Список сообщений, включая ответ модели.

**Вызывает исключения**:
- `Exception`: Возникает при ошибке в запросе к API OpenAI.
```
```python
def bully(user_message="Hello!", messages=[{"system": "user", "content": system_prompt}]):
	messages.append({"role": "user", "content": user_message})
	completion = openai.ChatCompletion.create(
		model="gpt-3.5-turbo",
		messages=messages
	)
	messages.append({"role": "user", "content": completion.choices[0].message})
	return messages
```
**Примечания**:

* Функция `bully` использует библиотеку `openai`. Необходимо установить ее и задать ключ API.
* Входной параметр `messages` предполагает, что он инициализирован системным запросом, который будет передаваться в модель.  Это важная часть для корректной работы функции.
* Функция возвращает список сообщений.  Обратите внимание на то, что она модифицирует `messages` в ходе выполнения.  Это потенциально может привести к непредсказуемому поведению, если `messages` используется вне функции.


## Замечания

Модуль предназначен для использования с API OpenAI. Необходимо установить библиотеку `openai` и правильно настроить переменную `openai.API_KEY`.
