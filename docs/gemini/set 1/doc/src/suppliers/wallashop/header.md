# Модуль hypotez/src/suppliers/wallashop/header.py

## Обзор

Данный модуль содержит функции для определения корневой директории проекта, загрузки настроек из файла `settings.json` и получения документации из файла `README.MD`. Он использует библиотеки `pathlib`, `json` и `packaging.version`.

## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная с текущей директории, и ищет файлы-маркеры (pyproject.toml, requirements.txt, .git).

**Параметры**:
- `marker_files` (tuple): Кортеж имен файлов или директорий, по которым определяется корень проекта.

**Возвращает**:
- `Path`: Путь к корневой директории проекта, если найдена, иначе путь к директории, где расположен скрипт.

**Вызывает исключения**:
- Нет

## Глобальные переменные

### `__root__`

**Описание**: Путь к корневой директории проекта, полученный из функции `set_project_root`.


### `settings`

**Описание**: Словарь с настройками проекта, загруженный из файла `settings.json`. Значение может быть `None`, если файл не найден или содержит невалидный JSON.


### `doc_str`

**Описание**: Строка с содержимым файла `README.MD`. Значение может быть `None`, если файл не найден или содержит ошибки.


### `__project_name__`

**Описание**: Имя проекта. Получается из настроек, по умолчанию `hypotez`.


### `__version__`

**Описание**: Версия проекта. Получается из настроек, по умолчанию пустая строка.


### `__doc__`

**Описание**: Документация проекта. Получается из файла `README.MD`. По умолчанию пустая строка.


### `__details__`

**Описание**: Дополнительные детали о проекте. По умолчанию пустая строка.


### `__author__`

**Описание**: Автор проекта. Получается из настроек, по умолчанию пустая строка.


### `__copyright__`

**Описание**: Авторские права. Получается из настроек, по умолчанию пустая строка.


### `__cofee__`

**Описание**: Ссылка на платежную систему для поддержания разработчика. Получается из настроек, по умолчанию ссылка на Boosty.


## Обработка исключений

Этот модуль использует блоки `try...excep` для обработки `FileNotFoundError` и `json.JSONDecodeError`, обеспечивая устойчивость к потенциальным ошибкам при чтении файлов.
