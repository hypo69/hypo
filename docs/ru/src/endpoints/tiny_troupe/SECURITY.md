# Документация для разработчика: SECURITY.md

## Обзор

Этот файл представляет собой стандартный файл `SECURITY.md`, используемый Microsoft для информирования о политиках безопасности, отчетности об уязвимостях и предпочтительных языках общения. Он предназначен для проектов, размещенных в репозиториях GitHub, принадлежащих Microsoft.

## Подробнее

Файл `SECURITY.md` предоставляет информацию о том, как сообщать об уязвимостях безопасности в продуктах и сервисах Microsoft. Он содержит ссылки на ресурсы Microsoft Security Response Center (MSRC) и предоставляет рекомендации по предоставлению информации, которая поможет Microsoft лучше понять характер и масштабы проблемы.

## Security

Раздел посвящен безопасности программных продуктов и сервисов Microsoft, включая репозитории исходного кода, управляемые через GitHub.

## Reporting Security Issues

### `Reporting Security Issues`

**Назначение**: Предоставляет инструкции о том, как сообщать об уязвимостях безопасности в Microsoft.

**Как работает функция**:

1. **Запрет на публичные отчеты**: Уязвимости не должны сообщаться через публичные issue в GitHub.
2. **Перенаправление в MSRC**: Направляйте отчеты в Microsoft Security Response Center (MSRC).
3. **Альтернативные методы**: Предоставляет альтернативные методы отправки отчетов, включая адрес электронной почты и PGP-ключ для шифрования.
4. **Ожидание ответа**: Ожидание ответа в течение 24 часов.

```
A: Обнаружение уязвимости
|
-- B: Сообщение в MSRC (не через GitHub)
|
C: Ожидание ответа от MSRC
```

**Параметры**: Нет параметров.

**Возвращает**: Нет возвращаемого значения.

**Вызывает исключения**: Нет информации об исключениях.

**Примеры**:
