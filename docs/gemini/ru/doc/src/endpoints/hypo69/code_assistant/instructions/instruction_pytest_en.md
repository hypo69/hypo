# Инструкция по написанию тестов `pytest`

## Обзор

Данная инструкция описывает, как написать тесты для Python-кода с использованием библиотеки `pytest`.  Она фокусируется на создании полных и надежных тестов, охватывая важные случаи, такие как корректные входные данные, некорректные входные данные, граничные случаи и обработку исключений.

## Требования

1. **Имена тестов:** Используйте ясные и описательные имена функций тестов, отражающие их цель.

2. **Изолированность:** Убедитесь, что все тесты изолированы и не зависят друг от друга.

3. **Разнообразные сценарии:** Рассмотрите различные сценарии, включая:
    - Валидные входные данные.
    - Невалидные или неожиданные входные данные, где это уместно.
    - Граничные случаи.

4. **Тестирование исключений:** Используйте `pytest.raises` для тестирования обработки исключений.

5. **Фикстуры:** Если для функций требуются фикстуры, определите их отдельно.

6. **Комментарии:** Добавьте комментарии, объясняющие логику тестовых случаев.

##  Советы по написанию тестов


*   **Покрытие кода:**  Стремитесь к максимальному покрытию кода тестами.
*   **Чёткость и краткость:** Пишите тесты с чёткими и краткими описаниями.
*   **Конкретность:** Тесты должны быть конкретными и проверять определенные аспекты поведения кода.
*   **Декомпозиция:** Разбивайте сложные задачи на более мелкие, более легко тестируемые модули.
*   **Документация:**  Включайте полную документацию к тестовым функциям.

##  Примеры использования `pytest.raises`

```python
import pytest

def my_function(x):
    if x < 0:
        raise ValueError("Значение должно быть неотрицательным")
    return x * 2

def test_my_function_raises_value_error():
    with pytest.raises(ValueError) as excinfo:
        my_function(-1)
    assert str(excinfo.value) == "Значение должно быть неотрицательным"

def test_my_function_valid_input():
    assert my_function(5) == 10
```
