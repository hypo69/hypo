# Модуль `tinytroupe.control`

## Обзор

Данный модуль предоставляет классы и функции для управления симуляциями, включая инициализацию, сохранение состояния, добавление агентов, сред и фабрик. Модуль реализует механизм кэширования состояний симуляции для повышения производительности и возможности отката. Поддерживается транзакционная обработка, обеспечивающая целостность симуляции.

## Классы

### `Simulation`

**Описание**: Класс `Simulation` отвечает за управление состоянием симуляции. Он хранит информацию об агентах, средах, фабриках и текущем состоянии симуляции. Реализует механизм кэширования и транзакционной обработки.

**Атрибуты**:

- `id`: Идентификатор симуляции (строка). По умолчанию "default".
- `agents`: Список агентов в симуляции.
- `name_to_agent`: Словарь, связывающий имя агента с объектом агента.
- `environments`: Список сред в симуляции.
- `factories`: Список фабрик в симуляции.
- `name_to_factory`: Словарь, связывающий имя фабрики с объектом фабрики.
- `name_to_environment`: Словарь, связывающий имя среды с объектом среды.
- `status`: Состояние симуляции (строка). Может быть "stopped" или "started".
- `cache_path`: Путь к файлу кэша (строка).
- `auto_checkpoint`: Автоматически сохранять состояние после каждой транзакции (булево значение).
- `has_unsaved_cache_changes`: Флаг, указывающий на наличие несохраненных изменений в кэше.
- `_under_transaction`: Флаг, указывающий на то, что симуляция находится в транзакции.
- `cached_trace`: Список состояний симуляции, кэшированных для отката.
- `execution_trace`: Список состояний симуляции во время выполнения.


**Методы**:

- `__init__(self, id="default", cached_trace:list=None)`: Инициализирует объект `Simulation`.
- `begin(self, cache_path:str=None, auto_checkpoint:bool=False)`: Начинает управление симуляцией.
- `end(self)`: Завершает управление симуляцией.
- `checkpoint(self)`: Сохраняет текущее состояние симуляции в кэш.
- `add_agent(self, agent)`: Добавляет агента в симуляцию.
- `add_environment(self, environment)`: Добавляет среду в симуляцию.
- `add_factory(self, factory)`: Добавляет фабрику в симуляцию.
- `_execution_trace_position(self) -> int`: Возвращает текущую позицию в трейсе выполнения.
- `_function_call_hash(self, function_name, *args, **kwargs) -> int`: Вычисляет хеш заданного вызова функции.
- `_skip_execution_with_cache(self)`: Пропускает текущее выполнение, если соответствующее состояние уже кэшировано.
- `_is_transaction_event_cached(self, event_hash) -> bool`: Проверяет, есть ли заданный хеш события в кэше.
- `_drop_cached_trace_suffix(self)`: Сбрасывает часть кэшированного трейса, начиная с текущей позиции.
- `_add_to_execution_trace(self, state: dict, event_hash: int, event_output)`: Добавляет состояние в список `execution_trace`.
- `_add_to_cache_trace(self, state: dict, event_hash: int, event_output)`: Добавляет состояние в список `cached_trace`.
- `_load_cache_file(self, cache_path:str)`: Загружает состояние из файла кэша.
- `_save_cache_file(self, cache_path:str)`: Сохраняет состояние в файл кэша.
- `begin_transaction(self)`: Начинает транзакцию.
- `end_transaction(self)`: Завершает транзакцию.
- `is_under_transaction(self)`: Проверяет, находится ли симуляция в транзакции.
- `_clear_communications_buffers(self)`: Очищает буферы связи всех агентов и сред.
- `_encode_simulation_state(self) -> dict`: Кодирует текущее состояние симуляции в словарь.
- `_decode_simulation_state(self, state: dict)`: Декодирует состояние симуляции из словаря.


### `Transaction`

**Описание**: Класс `Transaction` обрабатывает транзакции с объектами в симуляции.

**Методы**:

- `__init__(self, obj_under_transaction, simulation, function, *args, **kwargs)`: Инициализирует объект `Transaction`.
- `execute(self)`: Выполняет заданную функцию в рамках транзакции, используя механизм кэширования.
- `_encode_function_output(self, output) -> dict`: Кодирует выходное значение функции.
- `_decode_function_output(self, encoded_output: dict)`: Декодирует выходное значение функции.



## Функции

### `reset()`

**Описание**: Сбрасывает состояние управления симуляцией.

### `_simulation(id="default")`

**Описание**: Возвращает объект `Simulation` по заданному ID.

### `begin(cache_path=None, id="default", auto_checkpoint=False)`

**Описание**: Начинает симуляцию.

### `end(id="default")`

**Описание**: Завершает симуляцию.

### `checkpoint(id="default")`

**Описание**: Сохраняет текущее состояние симуляции.

### `current_simulation()`

**Описание**: Возвращает текущую симуляцию.


### `transactional(func)`

**Описание**: Декоратор, который делает функцию транзакционной.

## Исключения

### `SkipTransaction`

### `CacheOutOfSync`

### `ExecutionCached`

**Описание**: Исключение, возникающее при несоответствии кэшированных и текущих данных симуляции.

### `ValueError`

**Описание**: Возникает при ошибках валидации, например, при попытке начать симуляцию, которая уже запущена.
