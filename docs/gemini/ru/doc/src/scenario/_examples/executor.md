# Модуль `executor`

## Обзор

Модуль `executor` предоставляет функции для выполнения сценариев, обработки файлов сценариев, работы с продуктами и взаимодействия с API PrestaShop.  Он содержит примеры использования для различных задач, включая запуск списка файлов сценариев, одиночного файла, одиночного сценария, работы с продуктами и добавлением купонов.

## Функции

### `run_scenario_files`

**Описание**: Функция для запуска списка файлов сценариев.

**Параметры**:
- `supplier` (объект `Supplier`): Объект, представляющий поставщика данных и контекст сценариев.
- `scenario_files` (список `Path`): Список путей к файлам сценариев.

**Возвращает**:
- `bool`: `True`, если все сценарии выполнены успешно, `False` в противном случае.


### `run_scenario_file`

**Описание**: Функция для запуска одного файла сценария.

**Параметры**:
- `supplier` (объект `Supplier`): Объект, представляющий поставщика данных и контекст сценариев.
- `scenario_file` (`Path`): Путь к файлу сценария.

**Возвращает**:
- `bool`: `True`, если сценарий выполнен успешно, `False` в противном случае.


### `run_scenarios`

**Описание**:  Функция для запуска списка сценариев. (Подробности реализации не указаны в примере)

**Параметры**:
- `supplier` (объект `Supplier`): Объект, представляющий поставщика данных и контекст сценариев.
- `scenarios` (список): Список сценариев.

**Возвращает**:
- `bool`: `True`, если все сценарии выполнены успешно, `False` в противном случае.


### `run_scenario`

**Описание**: Функция для запуска одного сценария.

**Параметры**:
- `supplier` (объект `Supplier`): Объект, представляющий поставщика данных и контекст сценариев.
- `scenario` (словарь): Словарь, содержащий сценарий.

**Возвращает**:
- `bool`: `True`, если сценарий выполнен успешно, `False` в противном случае.


### `insert_grabbed_data`

**Описание**: Функция для вставки полученных данных о продукте в PrestaShop.

**Параметры**:
- `product_fields` (объект `ProductFields`): Объект, содержащий данные о продукте.

**Возвращает**:
- Нет возвращаемого значения.


### `execute_PrestaShop_insert`

**Описание**: Функция для синхронной вставки данных о продукте в PrestaShop.

**Параметры**:
- `product_fields` (объект `ProductFields`): Объект, содержащий данные о продукте.

**Возвращает**:
- `bool`: `True`, если вставка выполнена успешно, `False` в противном случае.


### `execute_PrestaShop_insert_async`

**Описание**: Функция для асинхронной вставки данных о продукте в PrestaShop.

**Параметры**:
- `product_fields` (объект `ProductFields`): Объект, содержащий данные о продукте.

**Возвращает**:
- Нет возвращаемого значения.  (Асинхронная функция, результат доступен в callback)


### `add_coupon`

**Описание**: Функция для добавления купона в PrestaShop.

**Параметры**:
- `credentials` (словарь): Словарь с данными авторизации для API PrestaShop.
- `reference` (строка): Идентификатор продукта, для которого добавляется купон.
- `coupon_code` (строка): Код купона.
- `start_date` (строка): Дата начала действия купона (в формате YYYY-MM-DD).
- `end_date` (строка): Дата окончания действия купона (в формате YYYY-MM-DD).

**Возвращает**:
- Нет возвращаемого значения.


## Классы

### `MockSupplier`

**Описание**:  Моковая реализация класса `Supplier`, необходимая для примеров.

### `MockRelatedModules`

**Описание**: Моковая реализация класса `RelatedModules`, используемая для примеров.  Содержит методы для получения списка продуктов в категории и получения страницы продукта.

### `MockDriver`

**Описание**: Моковая реализация класса `Driver`, используемая для примеров. Имеет метод `get_url`.


### `ProductFields`

**Описание**: Класс, представляющий данные о продукте. (Подробности реализации не указаны в примере).



**Примечания:** Документация предполагает существование классов `Supplier`, `RelatedModules`, `Driver` и `ProductFields` с соответствующими методами и атрибутами, которые не представлены в предоставленном коде.  В реальной документации необходимо будет описать эти классы подробно.  Также неясен точный функционал класса `PrestaShop` и его взаимодействие с модулем.
