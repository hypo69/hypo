# Amazin (מחולל מבוכים)

## סקירה כללית

משחק זה הוא מחולל מבוכים, היוצר בכל פעם מבוך ייחודי עם נתיב נכון יחיד. השחקן מגדיר את מידות המבוך, והתוכנית בונה אותו בהתאם לפרמטרים שצוינו.

- **מאפיינים:**
  - השחקן מזין את רוחב וגובה המבוך.
  - התוכנית מבטיחה קיום נתיב יחיד דרך המבוך.
  - המבוך מוצג על המסך.

## הוראות שלב אחר שלב ליישום:

### 1. אתחול המשחק
   - הצע לשחקן להזין את מידות המבוך (רוחב וגובה).
   - בדוק כי המידות שהוזנו תקינות (למשל, גדולות מאחת).

### 2. הלוגיקה הבסיסית של יצירת המבוך
   - צור מטריצה במידה שצוינה, המייצגת רשת של תאים.
   - השתמש באלגוריתם ליצירת מבוכים, לדוגמה, *אלגוריתם חיפוש עומק רקורסיבי (DFS)*:
     - התחל עם תא התחלתי אקראי.
     - התקדם לתאים סמוכים, תוך הסרת קירות בין התא הנוכחי והתא הבא.
     - אם כל השכנים כבר ביקרו, חזור לתא הקודם והמשך.
     - סיים את התהליך כאשר כל התאים יבקרו.
   - ודא קיום נתיב יחיד דרך המבוך.

### 3. הצגת המבוך
   - השתמש בתווים להצגה:
     - `+`, `-`, `|` לקירות.
     - רווחים למעברים.
   - הצג את המבוך המוגמר בטקסט.

### 4. פונקציות נוספות
   - אפשרות להגדיר גודל מוגדר מראש (למשל, 10x10), אם המשתמש הזין נתונים שגויים.
   - אזהרה לגבי מידות גדולות מדי של המבוך כדי למנוע עומס יתר על הזיכרון.

## דוגמה לפעולת התוכנית

1. **תחילת עבודה**:
   ```
   הזן את רוחב וגובה המבוך:
   > 10 8
   ```

2. **פלט של המבוך**:
   ```
   +--+--+--+--+--+--+--+--+--+--+
   |        |        |           |
   +  +--+  +  +--+  +  +--+--+  +
   |     |     |     |        |  |
   +--+  +  +--+  +  +  +--+  +  +
   |     |        |     |     |  |
   +--+--+--+--+--+--+--+--+--+--+
   ```

3. **יציאה מהתוכנית**:
   ```

   להתראות!
   ```
