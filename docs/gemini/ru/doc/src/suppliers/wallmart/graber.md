# Модуль hypotez/src/suppliers/wallmart/graber.py

## Обзор

Модуль `hypotez/src/suppliers/wallmart/graber.py` предоставляет класс `Graber`, предназначенный для сбора данных с веб-страниц товаров на сайте `wallmart.com`. Класс наследуется от базового класса `Grbr` (из модуля `src.suppliers.graber`).  Он обеспечивает обработку различных полей страницы товара, предлагая возможность переопределения стандартных функций обработки для нестандартных случаев.  Модуль использует веб-драйвер и логирование.  Перед запросом к веб-драйверу можно выполнить предварительные действия с помощью декоратора, который по умолчанию находится в родительском классе.

## Классы

### `Graber`

**Описание**: Класс `Graber` отвечает за сбор данных с страниц товаров на `wallmart.com`. Он расширяет функциональность базового класса `Grbr`, позволяя настраивать и переопределять обработку полей.

**Атрибуты**:

- `supplier_prefix`: Строка, представляющая префикс поставщика (`wallmart`).

**Методы**:

- `__init__(self, driver: Driver)`:
    **Описание**: Инициализирует объект `Graber`.
    **Параметры**:
      - `driver (Driver)`: Экземпляр класса `Driver` для управления веб-драйвером.
    **Возвращает**:  Не имеет значения возвращаемого типа.

## Функции


### `close_pop_up` (комментировано)

**Описание**: Декоратор для закрытия всплывающих окон.  По умолчанию этот декоратор не используется, т.к. он закомментирован. При необходимости реализовать свой декоратор для закрытия окон -  раскомментируйте этот блок кода и реализуйте логику обработки.

**Параметры**:
- `value (Any, optional)`: Дополнительные данные для декоратора. По умолчанию `None`.

**Возвращает**:
- `Callable`: Декоратор, который оборачивает функцию.

**Вызывает исключения**:
- `ExecuteLocatorException`:  В случае ошибок при выполнении локатора. (Описание ошибки в логе.)
