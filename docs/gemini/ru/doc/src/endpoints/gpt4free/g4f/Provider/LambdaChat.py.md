# Документация модуля `LambdaChat.py`

## Обзор

Модуль `LambdaChat.py` предоставляет класс `LambdaChat`, который является подклассом `HuggingChat` и предназначен для взаимодействия с сервисом Lambda Chat. Он определяет специфические параметры и настройки для этого сервиса, такие как домен, URL, модели, используемые для чата, и их альтернативы.

## Подробнее

Модуль определяет класс `LambdaChat`, который наследует функциональность от `HuggingChat` и настраивает его для работы с сервисом `lambda.chat`. В частности, здесь задаются параметры подключения, модели по умолчанию, модели для логических рассуждений (reasoning model), и запасные варианты моделей. Также определены псевдонимы для моделей, что позволяет использовать более короткие и понятные имена.

## Классы

### `LambdaChat`

**Описание**: Класс `LambdaChat` предназначен для взаимодействия с сервисом Lambda Chat. Он наследует функциональность от `HuggingChat` и предоставляет специфичные настройки для работы с этим сервисом.

**Наследует**:
- `HuggingChat`: Класс, предоставляющий базовую функциональность для взаимодействия с чат-сервисами на основе моделей Hugging Face.

**Атрибуты**:
- `label` (str): Метка для данного провайдера, значение `"Lambda Chat"`.
- `domain` (str): Доменное имя сервиса, значение `"lambda.chat"`.
- `origin` (str): Полный URL сервиса, формируется как `"https://{domain}"`.
- `url` (str): URL для доступа к сервису, идентичен `origin`.
- `working` (bool): Флаг, указывающий, что сервис находится в рабочем состоянии, значение `True`.
- `use_nodriver` (bool): Флаг, указывающий на необходимость использования драйвера, значение `False`.
- `needs_auth` (bool): Флаг, указывающий на необходимость аутентификации, значение `False`.
- `default_model` (str): Модель, используемая по умолчанию, значение `"deepseek-llama3.3-70b"`.
- `reasoning_model` (str): Модель, используемая для логических рассуждений, значение `"deepseek-r1"`.
- `image_models` (list): Список моделей, поддерживающих обработку изображений, пустой список `[]`.
- `fallback_models` (list): Список запасных моделей, используемых в случае недоступности основной модели.
- `models` (list): Список доступных моделей, изначально копия `fallback_models`.
- `model_aliases` (dict): Словарь псевдонимов для моделей, позволяющий использовать более короткие имена.

## Переменные класса

- `label`: Строка, представляющая метку для класса, в данном случае "Lambda Chat".
- `domain`: Строка, представляющая доменное имя, в данном случае "lambda.chat".
- `origin`: Строка, представляющая URL, сгенерированный из домена.
- `url`: Строка, представляющая URL для доступа к сервису.
- `working`: Булево значение, указывающее, работает ли класс.
- `use_nodriver`: Булево значение, указывающее, следует ли использовать драйвер.
- `needs_auth`: Булево значение, указывающее, требуется ли аутентификация.
- `default_model`: Строка, представляющая модель по умолчанию, в данном случае "deepseek-llama3.3-70b".
- `reasoning_model`: Строка, представляющая модель рассуждений, в данном случае "deepseek-r1".
- `image_models`: Список строковых моделей, поддерживающих обработку изображений.
- `fallback_models`: Список строковых моделей, которые будут использоваться, если модель по умолчанию недоступна.
- `models`: Список строковых моделей, изначально являющийся копией `fallback_models`.
- `model_aliases`: Словарь, сопоставляющий псевдонимы строковых моделей с их полными именами.
