# Модуль `src.logger`

## Обзор

Модуль `src.logger` предоставляет гибкую систему логирования, поддерживающую логирование в консоль, файлы и формате JSON. Он использует паттерн Singleton для обеспечения того, что используется только один экземпляр логгера во всем приложении. Логгер поддерживает различные уровни логирования (например, `INFO`, `ERROR`, `DEBUG`) и включает цветной вывод для логов в консоли. Вы также можете настраивать форматы вывода логов и управлять логированием в разные файлы.

## Классы

### `SingletonMeta`

**Описание**: Метакласс, реализующий паттерн Singleton для логгера.

### `JsonFormatter`

**Описание**: Специальный форматер, который выводит логи в формате JSON.

### `Logger`

**Описание**: Основной класс логгера, поддерживающий логирование в консоль, файлы и формате JSON.


## Функции

### `__init__`

**Описание**: Инициализирует экземпляр `Logger` с плацехолдерами для различных типов логгеров (консоль, файл, JSON).

### `_configure_logger(name: str, log_path: str, level: Optional[int] = logging.DEBUG, formatter: Optional[logging.Formatter] = None, mode: Optional[str] = 'a') -> logging.Logger`

**Описание**: Настраивает и возвращает экземпляр логгера.

**Параметры:**

- `name` (str): Имя логгера.
- `log_path` (str): Путь к лог-файлу.
- `level` (Optional[int], optional): Уровень логирования, например, `logging.DEBUG`. По умолчанию `logging.DEBUG`.
- `formatter` (Optional[logging.Formatter], optional): Настраиваемый форматер (необязательно).
- `mode` (Optional[str], optional): Режим файла, например, `'a'` для добавления (по умолчанию).

**Возвращает**: Настроенный экземпляр `logging.Logger`.


### `initialize_loggers(info_log_path: Optional[str] = '', debug_log_path: Optional[str] = '', errors_log_path: Optional[str] = '', json_log_path: Optional[str] = '')`

**Описание**: Инициализирует логгеры для консольного и файлового логирования (информация, отладка, ошибка и JSON).

**Параметры:**

- `info_log_path` (Optional[str], optional): Путь к файлу логов уровня информации.
- `debug_log_path` (Optional[str], optional): Путь к файлу логов уровня отладки.
- `errors_log_path` (Optional[str], optional): Путь к файлу логов уровня ошибок.
- `json_log_path` (Optional[str], optional): Путь к файлу логов в формате JSON.


### `log(level, message, ex=None, exc_info=False, color=None)`

**Описание**: Логирует сообщение на указанном уровне (например, `INFO`, `DEBUG`, `ERROR`) с необязательными исключениями и форматированием цвета.

**Параметры:**

- `level`: Уровень логирования (например, `logging.INFO`, `logging.DEBUG`).
- `message`: Сообщение для логирования.
- `ex` (Optional[Exception], optional): Необязательное исключение для логирования.
- `exc_info` (bool, optional): Включать ли информацию об исключении (по умолчанию `False`).
- `color` (Optional[tuple], optional): Кортеж с текстовым и фоновым цветами для вывода в консоль (необязательно).


### `info(message, ex=None, exc_info=False, colors: Optional[tuple] = None)` и т.д.

(Аналогичные функции для `success`, `warning`, `debug`, `error`, `critical` с одинаковыми параметрами)

## Параметры логгера

Класс `Logger` принимает несколько необязательных параметров для настройки поведения логирования:

- **Уровень (Level)**: Управляет уровнем важности логов. Распространенные уровни:
    - `logging.DEBUG`: Подробная информация, полезная для диагностики проблем.
    - `logging.INFO`: Общая информация, например, успешные операции.
    - `logging.WARNING`: Предупреждения, которые не требуют немедленного вмешательства.
    - `logging.ERROR`: Сообщения об ошибках.
    - `logging.CRITICAL`: Критические ошибки, требующие немедленного внимания.

- **Форматировщик (Formatter)**: Определяет, как форматируются сообщения логов. По умолчанию сообщения форматируются как `'%(asctime)s - %(levelname)s - %(message)s'`. Вы можете указать пользовательский форматер для различных форматов, например, JSON.

- **Цвет (Color)**: Цвета для сообщений логов в консоли. Цвет задается кортежем из двух элементов:
    - **Цвет текста (Text color)**: Указывает цвет текста (например, `colorama.Fore.RED`).
    - **Цвет фона (Background color)**: Указывает цвет фона (например, `colorama.Back.WHITE`).

Цвет можно настраивать для разных уровней логирования (например, зелёный для информации, красный для ошибок и т.д.).


## Настройка файлового логирования (`config`)

Для логирования сообщений в файл, вы можете указать пути к файлам в конфигурации:

```python
config = {
    'info_log_path': 'logs/info.log',
    'debug_log_path': 'logs/debug.log',
    'errors_log_path': 'logs/errors.log',
    'json_log_path': 'logs/log.json'
}
```

Пути к файлам, предоставленные в `config`, используются для записи логов в соответствующие файлы для каждого уровня логирования.

## Заключение

Этот модуль предоставляет всестороннюю и гибкую систему логирования для Python-приложений. Вы можете настроить консольное и файловое логирование с разными форматами и цветами, управлять уровнями логирования и эффективно обрабатывать исключения.  Конфигурация файлового логирования хранится в словаре `config`, что позволяет легко настраивать её.
