# ИНСТРУКЦИЯ ПО ОБРАБОТКЕ PYTHON-ФAЙЛОВ

## Обзор

Этот документ описывает инструкцию по обработке Python-файлов для генерации Markdown-документации.

## Инструкции

Для каждого входящего Python-файла необходимо создать Markdown-документацию, которая включает в себя:

* **Заголовок:** Название файла с кратким описанием.
* **Оглавление (TOC):** Список ссылок на все разделы документации модуля.
* **Классы:**  Описание каждого класса с его методами.
* **Функции:** Описание каждой функции с ее параметрами, возвращаемыми значениями и возможными исключениями.

## Формат комментариев для Python-кода

Для функций и методов необходимо использовать docstrings следующего формата:

```python
def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:
    """
    Args:
        param (str): Описание параметра `param`.
        param1 (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию `None`.

    Returns:
        dict | None: Описание возвращаемого значения. Возвращает словарь или `None`.

    Raises:
        SomeError: Описание ситуации, в которой возникает исключение `SomeError`.
    """
    # Тело функции
```

Обратите внимание на использование `ex` вместо `e` в блоках обработки исключений.

## Пример Markdown-документации

```markdown
# Модуль обработки данных

## Оглавление

* [Обзор](#обзор)
* [Классы](#классы)
* [Функции](#функции)

## Обзор

Модуль `data_processing` предоставляет инструменты для обработки данных.

## Классы

### `DataProcessor`

**Описание**: Класс для обработки данных.

**Методы**:

* `process_data(data: list):` Обрабатывает список данных.
* `validate_data(data: list):` Проверяет данные на корректность.

## Функции

### `calculate_average(numbers: list) -> float`

**Описание**: Вычисляет среднее значение списка чисел.

**Параметры**:
* `numbers` (list): Список чисел.

**Возвращает**:
* `float`: Среднее значение.

**Вызывает исключения**:
* `ValueError`: Если входной список пустой.


```

## Обработка кода

Модуль должен проходить через следующий цикл:

1. **Парсинг**: Выделить код.
2. **Анализ**: Найти функции, классы, методы, атрибуты.
3. **Генерация**: Сгенерировать markdown-документацию.
4. **Форматирование**: При необходимости отформатировать сгенерированный текст.

Последовательно обрабатывайте каждый Python-файл и создавайте соответствующую Markdown-документацию.
