# Модуль `header.py`

## Обзор

Модуль `header.py` предназначен для определения основных констант и настроек проекта, включая корневой путь, версию, имя проекта, описание, автора и т.д. Он также загружает конфигурацию из файла `settings.json` и README.md.

## Функции

### `set_project_root`

**Описание**:
Функция `set_project_root` находит корневую директорию проекта, начиная с директории текущего файла. Поиск ведётся вверх по дереву каталогов, пока не будет найдена директория, содержащая один из файлов-маркеров (например, `pyproject.toml`, `requirements.txt` или `.git`).

**Параметры**:
- `marker_files` (tuple): Кортеж имен файлов или директорий, которые идентифицируют корневую директорию проекта. По умолчанию `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:
- `Path`: Путь к корневой директории проекта. Если корневая директория не найдена, возвращается путь к директории, где находится скрипт.

## Переменные

### `MODE`

**Описание**:
Режим работы приложения. По умолчанию установлен в `dev`.

### `__root__`

**Описание**:
Путь к корневой директории проекта, вычисленный с помощью функции `set_project_root`.

### `settings`

**Описание**:
Словарь настроек, загруженный из файла `settings.json`. Значение равно `None`, если файл не найден или не удалось декодировать JSON.

### `doc_str`

**Описание**:
Содержимое файла `README.md` в виде строки. Значение равно `None`, если файл не найден или не удалось прочитать.

### `__project_name__`

**Описание**:
Имя проекта, полученное из `settings.json` (ключ `project_name`). Если в `settings.json` нет такого ключа или `settings` равно `None`, то используется значение `'hypotez'`.

### `__version__`

**Описание**:
Версия проекта, полученная из `settings.json` (ключ `version`). Если в `settings.json` нет такого ключа или `settings` равно `None`, то используется пустая строка `''`.

### `__doc__`

**Описание**:
Строка с документацией проекта, считанная из файла `README.md`. Если `doc_str` равно `None`, то используется пустая строка `''`.

### `__details__`

**Описание**:
Строка с дополнительной информацией о проекте. В данном случае всегда имеет значение пустой строки `''`.

### `__author__`

**Описание**:
Автор проекта, полученный из `settings.json` (ключ `author`). Если в `settings.json` нет такого ключа или `settings` равно `None`, то используется пустая строка `''`.

### `__copyright__`

**Описание**:
Информация об авторских правах, полученная из `settings.json` (ключ `copyrihgnt`). Если в `settings.json` нет такого ключа или `settings` равно `None`, то используется пустая строка `''`.

### `__cofee__`

**Описание**:
Сообщение с предложением поддержать разработчика. Значение по умолчанию "Treat the developer to a cup of coffee for boosting enthusiasm in development: https://boosty.to/hypo69". Если в `settings.json` есть ключ `cofee`, то используется его значение.
