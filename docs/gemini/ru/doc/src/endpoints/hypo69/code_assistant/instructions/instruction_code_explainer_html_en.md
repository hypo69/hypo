## Требования к документации

### Формат документации

Документация должна быть в формате Markdown (.md). Каждый файл должен начинаться с заголовка и краткого описания его содержимого.  Для всех классов и функций необходимо использовать комментарии в стиле docstrings.  Обратите внимание на использование `ex` вместо `e` в блоках обработки исключений.


### Содержание (TOC)

В начале документации должен быть раздел с оглавлением, включающим ссылки на все основные разделы.


### Форматирование документации

Используйте правильный синтаксис Markdown для заголовков, списков и ссылок. Для описания классов, функций и методов используйте структурированные разделы с описаниями, параметрами, возвращаемыми значениями и исключениями.


### Заголовки разделов

Используйте заголовки первого уровня (#), второго уровня (##), третьего уровня (###) и четвёртого уровня (####).

## Инструкция

Эта часть предоставляет подробные инструкции для генерации документации.  Она описывает алгоритм, который должен быть использован при анализе кода для создания подробной документации.


### Описание алгоритма

1. **Анализ кода:** Прочитать предоставленный Python-код.
2. **Структурирование документации:** Разбить код на логические части (классы, функции) и создать соответствующие разделы в документации Markdown.
3. **Описание классов:** Для каждого класса описать его назначение, атрибуты и методы.
4. **Описание функций:** Для каждой функции описать ее назначение, параметры, возвращаемые значения и возможные исключения.
5. **Форматирование:**  Использовать правильный синтаксис Markdown для создания заголовков, списков, таблиц.
6. **Построение TOC:** Создать оглавление, содержащее ссылки на все разделы документации.


### Примечания
