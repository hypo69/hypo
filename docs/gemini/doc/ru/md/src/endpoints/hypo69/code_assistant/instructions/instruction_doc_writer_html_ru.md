<h1>Инструкция по созданию документации для Python-файлов в формате HTML</h1>

<p>Данная инструкция описывает процесс создания документации для Python-файлов в формате HTML, следуя заданным требованиям.</p>

<h2>Формат документации</h2>

<ul>
  <li>Используется стандарт HTML.</li>
  <li>Каждый файл начинается с заголовка и краткого описания.</li>
  <li>Комментарии для функций и классов имеют строго определенный формат:</li>
</ul>

<pre><code>python
def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:
    """
    Args:
        param (str): Описание параметра `param`.
        param1 (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию значение равно `None`.

    Returns:
        dict | None: Описание возвращаемого значения. Возвращает словарь или `None`.

    Raises:
        SomeError: Описание ситуации, в которой возникает исключение `SomeError`.
    """
</code></pre>
<ul>
  <li>Используется <code>ex</code> вместо <code>e</code> в блоках обработки исключений.</li>
</ul>


<h2>Оглавление (TOC)</h2>

<ul>
  <li>В начале каждого файла документации размещается раздел оглавления.</li>
  <li>Оглавление содержит ссылки на основные разделы.</li>
</ul>


<h2>Форматирование документации</h2>

<ul>
  <li>Используется правильный синтаксис HTML для заголовков, списков и ссылок.</li>
  <li>Должны быть структурированные разделы для описания классов, функций и методов, включающие:</li>
</ul>

<ul>
  <li>Описание.</li>
  <li>Параметры (с типами и описаниями).</li>
  <li>Возвращаемые значения (с типами и описаниями).</li>
  <li>Вызываемые исключения (с типами и описаниями).</li>
</ul>
