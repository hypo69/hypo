# Модуль header.py

## Обзор

Данный модуль содержит функцию `get_project_root`, которая находит корневую директорию проекта, начиная от текущего файла. Модуль также инициализирует переменные, такие как `__root__`, `settings`, `doc_str`, `__project_name__`, `__version__`, `__doc__`, `__details__`, `__author__`, `__copyright__`, и `__cofee__`, считывая значения из файла `settings.json` и `README.MD`.

## Функции

### `get_project_root`

**Описание**: Находит корневую директорию проекта, начиная от текущего файла.

**Параметры**:
- `marker_files` (tuple): Кортеж файлов или директорий, по которым будет определяться корневая директория проекта.

**Возвращает**:
- `Path`: Путь к корневой директории проекта. Если корневая директория не найдена, возвращает директорию, в которой расположен текущий файл.


**Вызывает исключения**:
- Нет


###

## Переменные

### `__root__`

**Описание**: Путь к корневой директории проекта.


**Тип**: `Path`


### `settings`

**Описание**: Словарь настроек проекта, полученный из файла `settings.json`.


**Тип**: `dict` или `None`


### `doc_str`

**Описание**: Строка с содержанием файла `README.MD`.


**Тип**: `str` или `None`


### `__project_name__`

**Описание**: Название проекта, полученное из файла `settings.json`.


**Тип**: `str`


### `__version__`

**Описание**: Версия проекта, полученная из файла `settings.json`.


**Тип**: `str`


### `__doc__`

**Описание**: Документация проекта, полученная из файла `README.MD`.


**Тип**: `str`


### `__details__`

**Описание**: Подробная информация о проекте.


**Тип**: `str`


### `__author__`

**Описание**: Автор проекта, полученный из файла `settings.json`.


**Тип**: `str`


### `__copyright__`

**Описание**: Авторские права проекта, полученные из файла `settings.json`.


**Тип**: `str`


### `__cofee__`

**Описание**: Ссылка для поддержки разработчика.


**Тип**: `str`


## Обработка исключений

Модуль обрабатывает исключения `FileNotFoundError` и `json.JSONDecodeError` при чтении файлов `settings.json` и `README.MD`. Если файл не найден или не удалось декодировать JSON, то соответствующие переменные устанавливаются в `None` или сохраняют текущее значение.
