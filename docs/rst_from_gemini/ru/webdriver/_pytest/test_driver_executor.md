# Тесты для модуля `src.webdriver._pytest`

Файл: `hypotez/src/webdriver/_pytest/test_driver_executor.py`

Этот файл содержит тесты для модуля `src.webdriver._pytest`, проверяющие корректную работу класса `ExecuteLocator` и взаимодействия с WebDriver. Тесты охватывают различные сценарии, включая навигацию по страницам, поиск элементов, отправку сообщений, получение атрибутов и обработку исключений.

## Обзор тестов

* **`test_navigate_to_page`**: Проверяет, что WebDriver успешно переходит на указанную страницу.
* **`test_get_webelement_by_locator_single_element`**: Проверяет получение элемента по валидному локатору.
* **`test_get_webelement_by_locator_no_element`**: Проверяет поведение при отсутствии элемента по локатору.  Ожидается, что метод вернёт `False`.
* **`test_send_message`**: Проверяет отправку текста в элемент (например, в поле поиска).  Важно указать корректный локатор для поля ввода.
* **`test_get_attribute_by_locator`**: Проверяет получение значения атрибута элемента по локатору.  Необходимо проверить корректное возвращаемое значение атрибута.
* **`test_execute_locator_event`**: Проверяет выполнение события (например, клик) на элементе.
* **`test_get_locator_keys`**: Проверяет корректность списка ключей локатора.
* **`test_navigate_and_interact`**: Проверяет последовательность навигации по страницам и взаимодействия с элементами на другой странице. Тест навигации к `https://www.wikipedia.org/` и проверку поиска.
* **`test_invalid_locator`**: Проверяет обработку исключения `ExecuteLocatorException` при использовании некорректного локатора.

## Важные замечания

* **Путь к ChromeDriver**: В фикстуре `driver` необходимо заменить `/path/to/chromedriver` на фактический путь к исполняемому файлу ChromeDriver на вашей системе.
* **Локаторы**:  При написании тестов убедитесь, что локаторы (XPath, CSS и т.д.) верны и соответствуют структуре целевой веб-страницы.  Тесты `test_send_message`, `test_get_attribute_by_locator`, `test_execute_locator_event` и другие нуждаются в корректировке локаторов, чтобы они соответствовали реальным элементам на сайте `example.com`.  Предположительно, локатор для поля поиска (в `test_send_message`) и ссылки `more-information` (в `test_get_attribute_by_locator`) должны быть пересмотрены, чтобы соответствовать структуре реальной страницы.
* **`http://example.com`**: Тесты предположительно проверяют работу на странице `example.com`.  Проверьте, что эта страница доступна.
* **`https://www.wikipedia.org/`**: Тест `test_navigate_and_interact` навигации переходит на страницу Wikipedia.
* **Утверждения (`assert`)**: Проверьте корректность используемых утверждений (`assert`) в каждом тесте.  Убедитесь, что ожидаемые результаты соответствуют действительности на целевой веб-странице.  Например, в `test_navigate_and_interact`, проверьте, что содержимое результата поиска соответствует ожидаемому.
* **Fixture `driver`**: Fixture `driver` инициализирует WebDriver и устанавливает начальную страницу.  Важной частью является `driver.quit()` в конце fixture.  Это гарантирует закрытие браузера после завершения каждого набора тестов.
* **Фикстура `execute_locator`**: Создает экземпляр класса `ExecuteLocator`, используя установленный WebDriver.
