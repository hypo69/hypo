# Модуль header.py

## Обзор

Этот модуль определяет корневой путь к проекту `hypotez`.  Он используется для корректной импортации модулей.  В нём также хранятся настройки проекта, загруженные из файла `settings.json`.

## Функции

### `get_project_root`

**Описание**:  Ищет корневую директорию проекта, начиная с текущего файла и поднимаясь вверх по иерархии директорий.  Останавливается на первой директории, содержащей один из файлов-маркеров (pyproject.toml, requirements.txt, .git).

**Параметры**:
- `marker_files` (tuple): Кортеж имен файлов или каталогов, используемых для определения корневой директории проекта. По умолчанию ('pyproject.toml', 'requirements.txt', '.git').

**Возвращает**:
- `Path`: Путь к корневой директории проекта, если найдена. Иначе - директория, где расположен данный скрипт.

**Вызывает исключения**:
  Нет.


## Переменные

### `__root__`

**Описание**: Путь к корневой директории проекта.  Получается с помощью функции `get_project_root`.

**Тип**: `Path`

### `settings`

**Описание**: Словарь настроек проекта, загруженный из файла `settings.json`.

**Тип**: `dict` или `None`

### `doc_str`

**Описание**: Содержимое файла README.md (строка).


**Тип**: `str` или `None`

### `__project_name__`

**Описание**: Название проекта.  Получается из настроек в `settings.json`.

**Тип**: `str`


### `__version__`

**Описание**: Версия проекта.  Получается из настроек в `settings.json`.

**Тип**: `str`

### `__doc__`


**Описание**: Документация проекта.

**Тип**: `str`

### `__details__`

**Описание**:  Подробная информация о проекте.

**Тип**: `str`

### `__author__`

**Описание**: Автор проекта.

**Тип**: `str`


### `__copyright__`

**Описание**: Авторские права.

**Тип**: `str`

### `__cofee__`

**Описание**: Ссылка на платёж разработчику.

**Тип**: `str`


## Обработка исключений

Используются `try...except` блоки для обработки потенциальных ошибок при чтении файла `settings.json` и `README.MD`.  Если файлы не найдены или содержат некорректный JSON, то соответствующие переменные устанавливаются в `None`.
