# Command Instruction for Model: Analyzing Computer Components from JSON

## Обзор

Этот документ содержит инструкции по созданию документации для разработчика в формате `Markdown` для Python-скрипта, который обрабатывает JSON-данные, описывающие компьютерные компоненты. Скрипт анализирует эти компоненты, классифицирует тип сборки (например, игровой, рабочая станция), предоставляет заголовки и описания на иврите, переводит детали компонентов и возвращает структурированный JSON-вывод. Документация также охватывает требования к формату, включая кодировку UTF-8, структуру вывода и проверку целостности данных.

## Кодировка текста ответа

### `UTF-8`

**Требование**: В выходных данных запрещено использовать последовательности Unicode Escape Sequence (например, `\\uXXXX`). Все тексты должны возвращаться в формате, читаемом в UTF-8 (например, `מחשב גיימינג`).

## Требования к формату

### Общие требования

-   **Соответствие шаблону**: Формат ответа должен полностью соответствовать указанному шаблону.
-   **Проверка целостности**: Перед отправкой вывода необходимо выполнить проверку целостности каждого словаря вывода, чтобы убедиться в отсутствии недостающих полей или структурных ошибок.
-   **Перевод**: Все входные данные, представленные на иврите, должны быть переведены на стандартный иврит в выводе.
-   **Кодировка**: Весь текст в ответе должен быть в формате UTF-8 без использования Unicode Escape Sequence.
-   **Синонимы**: Слова `product` и `component` являются синонимами и относятся к компонентам для сборки компьютера.
