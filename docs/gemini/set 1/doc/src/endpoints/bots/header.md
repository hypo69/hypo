# Модуль hypotez/src/endpoints/bots/header.py

## Обзор

Этот модуль содержит функции для определения корневой директории проекта, загрузки настроек из файла `settings.json` и получения документации из файла `README.MD`. Он также определяет переменные, содержащие информацию о проекте, такие как имя, версия, описание и автор.

## Функции

### `set_project_root`

**Описание**:  Находит корневую директорию проекта, начиная с текущей директории файла и поднимаясь вверх по иерархии директорий, пока не найдет директорию содержащую один из файлов-маркеров (pyproject.toml, requirements.txt, .git).

**Параметры**:

- `marker_files` (tuple): Кортеж имен файлов или директорий, используемых для определения корневой директории проекта. По умолчанию заданы ('pyproject.toml', 'requirements.txt', '.git').

**Возвращает**:

- `Path`: Путь к корневой директории проекта, если найдена, иначе - директория, в которой расположен текущий файл.


**Пример использования**:

```python
root_path = set_project_root()
print(root_path)
```

## Переменные

### `__root__`

**Описание**: Переменная, содержащая путь к корневой директории проекта, полученная вызовом функции `set_project_root()`. Тип: `Path`.

### `settings`

**Описание**: Словарь настроек проекта, загруженный из файла `settings.json` в корневой директории проекта.  Тип: `dict` или `None`, если файл не найден или некорректен.

### `doc_str`

**Описание**: Строка с содержимым файла `README.MD`. Тип: `str` или `None`, если файл не найден или некорректен.


### `__project_name__`

**Описание**: Имя проекта. Тип: `str`

### `__version__`

**Описание**: Версия проекта. Тип: `str`

### `__doc__`

**Описание**: Документация проекта. Тип: `str`


### `__details__`

**Описание**: Дополнительные детали проекта. Тип: `str`


### `__author__`

**Описание**: Автор проекта. Тип: `str`


### `__copyright__`

**Описание**: Авторские права. Тип: `str`

### `__cofee__`

**Описание**: Ссылка для поддержки разработчика. Тип: `str`



## Обработка исключений

В модуле используется обработка исключений `try...except` для безопасного обращения с файлами и данными:

- `FileNotFoundError`: Возникает, если файл `settings.json` или `README.MD` не найден.
- `json.JSONDecodeError`: Возникает, если файл `settings.json` содержит некорректный JSON.

В блоках `except` используется `...` для обозначения того, что никаких действий в случае возникновения исключения не производится (например, программа просто продолжает выполнение).
