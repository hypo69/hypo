# Документация для модуля `src.logger`

## Обзор

Модуль `src.logger` предоставляет гибкую систему логирования для Python-приложений. Он поддерживает логирование в консоль, файлы и в формате JSON. Модуль использует шаблон проектирования Singleton, чтобы обеспечить использование единственного экземпляра логгера во всем приложении. Логгер поддерживает различные уровни логирования (`INFO`, `ERROR`, `DEBUG`) и включает цветное отображение для вывода в консоль. Также доступны настройки форматов вывода и управление логированием в различные файлы.

## Подробнее

Модуль `src.logger` предоставляет централизованный механизм для записи информации о работе приложения. Это помогает отслеживать ошибки, предупреждения и общую информацию о функционировании. Использование Singleton гарантирует, что все части приложения используют один и тот же экземпляр логгера, что упрощает управление и настройку логирования.

## Классы

### `SingletonMeta`

**Описание**: Метакласс, реализующий шаблон Singleton. Это гарантирует, что у класса будет только один экземпляр.

**Принцип работы**:
Метакласс `SingletonMeta` переопределяет метод `__call__`, чтобы убедиться, что при попытке создать новый экземпляр класса возвращается уже существующий экземпляр. Если экземпляр еще не существует, он создается и сохраняется для последующих вызовов.

### `JsonFormatter`

**Описание**: Кастомный форматтер для вывода логов в формате JSON.

**Принцип работы**:
Класс `JsonFormatter` форматирует записи логов в формате JSON. Он наследуется от `logging.Formatter` и переопределяет метод `format`, чтобы преобразовывать записи логов в JSON-строки.

### `Logger`

**Описание**: Основной класс логгера, поддерживающий логирование в консоль, файлы и в формате JSON.

**Принцип работы**:
Класс `Logger` предоставляет методы для инициализации и настройки логгеров для различных целей, таких как вывод в консоль, запись в файлы (информация, отладка, ошибки и JSON). Он использует `logging.Logger` для фактической записи логов и предоставляет удобные методы для логирования сообщений разных уровней.

**Методы**:

- `__init__`: Инициализирует экземпляр класса Logger с плейсхолдерами для различных типов логгеров (консоль, файлы и JSON).
- `_configure_logger`: Настраивает и возвращает экземпляр логгера.
- `initialize_loggers`: Инициализирует логгеры для логирования в консоль и файлы (информация, отладка, ошибки и JSON).
- `log`: Логирует сообщение на указанном уровне (например, `INFO`, `DEBUG`, `ERROR`) с возможным исключением и цветовым форматированием.
- `info`: Логирует информационное сообщение.
- `success`: Логирует сообщение об успешной операции.
- `warning`: Логирует предупреждение.
- `debug`: Логирует сообщение для отладки.
- `error`: Логирует сообщение об ошибке.
- `critical`: Логирует критическое сообщение.

## Функции

### `_configure_logger`

```python
        level (Optional[int], optional): Уровень логирования, например, `logging.DEBUG`. По умолчанию `logging.DEBUG`.
        formatter (Optional[logging.Formatter], optional): Кастомный форматтер (опционально). По умолчанию `None`.
        mode (Optional[str], optional): Режим работы с файлом, например, `'a'` для добавления. По умолчанию `'a'`.

    Returns:
        logging.Logger: Настроенный экземпляр `logging.Logger`.
    """
    ...
```

**Назначение**: Настройка и возвращение экземпляра логгера с заданными параметрами.

**Параметры**:
- `name` (str): Имя логгера.
- `log_path` (str): Путь к файлу логов.
- `level` (Optional[int], optional): Уровень логирования, например, `logging.DEBUG`. По умолчанию `logging.DEBUG`.
- `formatter` (Optional[logging.Formatter], optional): Кастомный форматтер (опционально). По умолчанию `None`.
- `mode` (Optional[str], optional): Режим работы с файлом, например, `'a'` для добавления. По умолчанию `'a'`.

**Возвращает**:
- `logging.Logger`: Настроенный экземпляр `logging.Logger`.

**Как работает функция**:

1. Функция создает экземпляр `logging.Logger` с заданным именем.
2. Устанавливает уровень логирования для логгера.
3. Создает обработчик файлов (`logging.FileHandler`) для записи логов в указанный файл.
4. Устанавливает форматтер для обработчика.
5. Добавляет обработчик к логгеру.
6. Возвращает настроенный экземпляр `logging.Logger`.

```
Начало
↓
Создание логгера с именем
↓
Установка уровня логирования
↓
Создание обработчика файлов
↓
Установка форматтера для обработчика
↓
Добавление обработчика к логгеру
↓
Возврат настроенного логгера
Конец
```

**Примеры**:

```python
import logging
from src.logger import Logger  # Предположим, что Logger находится в src/logger.py

logger_instance = Logger()
log = logger_instance._configure_logger(name='my_logger', log_path='my_log_file.log', level=logging.INFO)
log.info('Это информационное сообщение')
```

```python
    """
    ...
```

**Назначение**: Инициализация логгеров для логирования в консоль и файлы.

**Параметры**:
- `info_log_path` (Optional[str], optional): Путь к файлу логов информации. По умолчанию пустая строка.
- `debug_log_path` (Optional[str], optional): Путь к файлу логов отладки. По умолчанию пустая строка.
- `errors_log_path` (Optional[str], optional): Путь к файлу логов ошибок. По умолчанию пустая строка.
- `json_log_path` (Optional[str], optional): Путь к файлу логов в формате JSON. По умолчанию пустая строка.

1. Функция настраивает логгер консоли с уровнями `INFO`, `DEBUG` и `ERROR`.
2. Если указаны пути к файлам логов, то для каждого пути настраивается отдельный логгер, который записывает логи в соответствующий файл.
3. Используется кастомный форматтер `JsonFormatter` для логирования в формате JSON.

```
Начало
↓
Настройка логгера консоли
↓
Проверка наличия путей к файлам логов
↓
Если есть пути к файлам:
    Создание и настройка логгеров для каждого файла
↓
Конец
```

**Примеры**:

```python
from src.logger import Logger

logger = Logger()
config = {
    'info_log_path': 'logs/info.log',
    'debug_log_path': 'logs/debug.log',
    'errors_log_path': 'logs/errors.log',
    'json_log_path': 'logs/log.json'

logger.initialize_loggers(**config)
```

1. Функция проверяет, нужно ли использовать цветовое форматирование.

3. Логирует сообщение с указанным уровнем и информацией об исключении (если указано).

```
Начало
↓
Проверка необходимости цветового форматирования

↓

```

**Примеры**:

```python
import logging
from src.logger import Logger

logger = Logger()
```

## Параметры логгера

Класс `Logger` принимает несколько опциональных параметров для настройки поведения логирования.

- **Уровень**: Контролирует, какие сообщения будут записаны. Основные уровни:
  - `logging.DEBUG`: Подробная информация для диагностики.
  - `logging.INFO`: Общая информация, например, успешные операции.
  - `logging.WARNING`: Предупреждения, не требующие немедленного действия.
  - `logging.ERROR`: Сообщения об ошибках.
  - `logging.CRITICAL`: Критические ошибки, требующие немедленного внимания.

- **Форматтер**: Определяет формат сообщений. По умолчанию используется `'%(asctime)s - %(levelname)s - %(message)s'`. Можно задать кастомный форматтер, например для JSON.

- **Цвета**: Задают цвет текста и фона в консоли. Цвета указываются кортежем:
  - **Цвет текста**: Например, `colorama.Fore.RED`.
  - **Цвет фона**: Например, `colorama.Back.WHITE`.

## Конфигурация для логирования в файл

Для записи сообщений в файл можно указать пути в конфигурации.

```python
config = {
    'info_log_path': 'logs/info.log',
    'debug_log_path': 'logs/debug.log',
    'errors_log_path': 'logs/errors.log',
    'json_log_path': 'logs/log.json'
}
```

## Примеры использования

#### 1. Инициализация логгера:

```python
logger: Logger = Logger()
config = {
    'info_log_path': 'logs/info.log',
    'debug_log_path': 'logs/debug.log',
    'errors_log_path': 'logs/errors.log',
    'json_log_path': 'logs/log.json'
}
logger.initialize_loggers(**config)
```

#### 2. Логирование сообщений:

```python
logger.info('Это информационное сообщение')
logger.success('Это сообщение об успешной операции')
logger.warning('Это предупреждение')
logger.debug('Это сообщение для отладки')
logger.error('Это сообщение об ошибке')
logger.critical('Это критическое сообщение')
```

#### 3. Настройка цветов:

```python
logger.info('Это сообщение будет зеленым', colors=(colorama.Fore.GREEN, colorama.Back.BLACK))
logger.error('Это сообщение с красным фоном', colors=(colorama.Fore.WHITE, colorama.Back.RED))
```

## Заключение

Модуль `src.logger` предоставляет полноценную систему логирования для Python-приложений. Вы можете настроить логирование в консоль и файлы с различными форматами и цветами, управлять уровнями логирования и обрабатывать исключения. Конфигурация логирования в файлы задается через словарь `config`, что позволяет легко изменять настройки.
