## Раздел `[OpenAI]`

### Описание
Этот раздел содержит конфигурации для работы с OpenAI API, включая параметры модели, типа API (OpenAI или Azure), таймауты и прочие настройки.

### Параметры

-   `API_TYPE` (str): Тип API для использования: `openai` или `azure`. По умолчанию `openai`.
-   `AZURE_API_VERSION` (str): Версия API для Azure OpenAI Service. По умолчанию `2023-05-15`.
-   `MODEL` (str): Модель OpenAI для использования. По умолчанию `gpt-4o`.
-   `MAX_TOKENS` (int): Максимальное количество токенов, которое может быть сгенерировано моделью. По умолчанию `4000`.
-   `TEMPERATURE` (float): Параметр температуры для контроля случайности вывода. По умолчанию `0.3`.
-   `FREQ_PENALTY` (float): Штраф за частоту токенов. По умолчанию `0.0`.
-  `PRESENCE_PENALTY` (float): Штраф за наличие токена. По умолчанию `0.0`.
-  `TIMEOUT` (int): Таймаут в секундах для запросов к API. По умолчанию `60`.
- `MAX_ATTEMPTS` (int): Максимальное количество попыток запросов к API в случае сбоя. По умолчанию `5`.
- `WAITING_TIME` (int): Время ожидания между попытками запросов к API в секундах. По умолчанию `1`.
-  `EXPONENTIAL_BACKOFF_FACTOR` (int): Коэффициент экспоненциального увеличения времени ожидания между попытками. По умолчанию `5`.
-   `EMBEDDING_MODEL` (str): Модель OpenAI для создания эмбеддингов текста. По умолчанию `text-embedding-3-small`.
-   `CACHE_API_CALLS` (bool): Флаг для включения/выключения кэширования вызовов API. По умолчанию `False`.
-   `CACHE_FILE_NAME` (str): Имя файла для кэширования вызовов API. По умолчанию `openai_api_cache.pickle`.
- `MAX_CONTENT_DISPLAY_LENGTH` (int): Максимальная длина отображаемого текста. По умолчанию `1024`.

## Раздел `[Simulation]`

### Описание
Этот раздел содержит настройки симуляции, включая предотвращение вредоносного контента и нарушений авторских прав.

### Параметры
-   `RAI_HARMFUL_CONTENT_PREVENTION` (bool): Флаг для предотвращения генерации вредоносного контента. По умолчанию `True`.
-   `RAI_COPYRIGHT_INFRINGEMENT_PREVENTION` (bool): Флаг для предотвращения нарушений авторских прав. По умолчанию `True`.

## Раздел `[Logging]`

### Описание
Этот раздел содержит настройки для логирования, определяя уровень детализации и формат сообщений.

### Параметры
-   `LOGLEVEL` (str): Уровень детализации логирования (`ERROR`, `WARNING`, `INFO`, `DEBUG`). По умолчанию `ERROR`.
