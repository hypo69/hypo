# Модуль `Supplier`

## Обзор

Этот модуль содержит базовый класс `Supplier`, который служит основой для создания классов, взаимодействующих с различными поставщиками данных (например, Amazon, AliExpress, Walmart и т.д.). Он предоставляет общие методы и атрибуты для работы с поставщиками, которые могут быть переопределены в конкретных реализациях.

## Подробнее

Класс `Supplier` предназначен для унификации процесса взаимодействия с различными поставщиками данных. Он предоставляет базовую функциональность для загрузки настроек, инициализации веб-драйвера, выполнения входа на сайт поставщика (если требуется) и запуска сценариев для сбора данных. Этот класс упрощает создание новых поставщиков, так как общая логика уже реализована в базовом классе.

## Классы

### `Supplier`

**Описание**: Базовый класс для работы с поставщиками данных.

**Как работает класс**:
1.  **Инициализация**: При создании объекта `Supplier`, конструктор `__init__` загружает настройки поставщика и инициализирует необходимые компоненты, такие как префикс поставщика, локаль и веб-драйвер.
2.  **Загрузка конфигурации**: Метод `_payload` загружает конфигурации для данного поставщика, включая локаторы для страниц и сценарии выполнения.
3.  **Вход на сайт**: Метод `login` используется для выполнения процесса входа на сайт поставщика, если это требуется.
4.  **Выполнение сценариев**: Методы `run_scenario_files` и `run_scenarios` запускают сценарии, которые определяют, какие действия нужно выполнить (например, сбор данных).

**Методы**:

**Примеры**:

```python
# Создаем объект для поставщика 'aliexpress'
supplier = Supplier(supplier_prefix='aliexpress', locale='en', webdriver='chrome')

# Выполняем вход на сайт поставщика
supplier.login()

# Запускаем сценарии из файлов
supplier.run_scenario_files(['example_scenario.json'])

# Или запускаем сценарии по определенным условиям
supplier.run_scenarios([{'action': 'scrape', 'target': 'product_list'}])
```
