# instruction_code_checker

## Обзор

Этот модуль содержит инструкции для проверки кода Python и улучшения его качества, включая добавление документации в формате reStructuredText (RST), обработку исключений с использованием логгера, а также изменение методов обработки данных.

## Основные требования

- Использование reStructuredText для всех комментариев и docstrings.
- Сохранение всех существующих комментариев после символа `#`.
- Использование `j_loads` или `j_loads_ns` из `src.utils.jjson` вместо `json.load`.
- Сохранение символов `...` в коде.
- Проверка и добавление недостающих импортов.
- Соответствие именованию функций, переменных и импортов ранее обработанным файлам.
- Добавление комментариев в формате RST к функциям, методам и классам.
- Использование `from src.logger.logger import logger` для логгирования ошибок.
- Избегать чрезмерного использования стандартных блоков `try-except`, отдавать предпочтение обработке ошибок с использованием `logger.error`.
- В комментариях избегать нечетких слов, таких как 'get' или 'do', использовать конкретные термины, такие как 'validation', 'execution' или 'sending'.
- Соответствие шаблону ответа (полученный код, улучшенный код, внесенные изменения, полный код).
- Представление полного кода (исходный код с улучшениями) в одном блоке, готовом к копированию и вставке.
- Комментирование изменённых частей кода с помощью `#`.
- Примеры RST документации и возможные улучшения в формате `TODO`.
- Переписывание всех комментариев к модулям, функциям, методам и переменным в формате RST (описание модуля, документация для каждой функции, метода и переменной, соответствие стандартам Python docstring, например, в стиле Sphinx).


## Функции

### `process_files`

**Описание**: Обрабатывает список файлов Python.

**Параметры**:

- `files` (list): Список путей к файлам.
- `model_type` (str): Тип модели.


**Возвращает**:
- `None` : Ничего не возвращает.


**Вызывает исключения**:
- `FileNotFoundError`: Возникает, если файл не найден.
- `Exception`: Возникает при других ошибках.
