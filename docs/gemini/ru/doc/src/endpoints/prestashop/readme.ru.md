# Документация для модуля управления сайтами на PrestaShop

## Обзор

Документ описывает структуру и способ работы с сайтами на платформе PrestaShop, а также методы хранения и использования ключей API для взаимодействия с этими сайтами. В документе рассматриваются основные принципы работы с API PrestaShop, включая примеры запросов и рекомендации по безопасности.

## Подробнее

Этот модуль предназначен для управления сайтами, работающими на платформе PrestaShop, через API. Он включает в себя информацию о том, как безопасно хранить ключи API и как использовать их для доступа к различным функциям PrestaShop. Документ предназначен для разработчиков, работающих с PrestaShop API.

## Хранение ключей API

### Описание

Ключи API для каждого сайта хранятся в файле `credentials.kdbx`. Этот файл представляет собой защищенную базу данных паролей.

**Атрибуты:**
- URL сайта
- Ключ API
- Дополнительные метаданные (если необходимо)

Для работы с ключами рекомендуется использовать менеджер паролей, поддерживающий формат `.kdbx`, например, [KeePass](https://keepass.info/) или [KeePassXC](https://keepassxc.org/).

## Пример использования API

### Описание

Этот раздел описывает, как подключиться к API одного из сайтов PrestaShop и выполнить запрос данных через API.

#### Запрос данных через API

**Шаблон API-запроса:**
```bash
curl -X GET 'https://<URL_сайта>/api/<endpoint>' \
-H 'Authorization: Basic <base64(API_KEY)>'
```

**Параметры:**
- `<URL_сайта>` — адрес сайта, например, `e-cat.co.il`.
- `<endpoint>` — конечная точка API (например, `products`, `customers`).
- `<API_KEY>` — ключ API, закодированный в формате Base64.

#### Пример вызова API

Для получения списка продуктов на сайте `e-cat.co.il`:
```bash
curl -X GET 'https://e-cat.co.il/api/products' \
-H 'Authorization: Basic <base64(API_KEY)>'
```

## Рекомендации по безопасности

### Описание

Этот раздел содержит важные рекомендации по обеспечению безопасности при работе с ключами API и файлом `credentials.kdbx`.

**Рекомендации:**
- Никогда не передавайте файл `credentials.kdbx` третьим лицам. ❗
- Убедитесь, что файл находится в защищенном месте, доступном только вам. (папка `secrets` в корне проекта исключена из `git`)
- Регулярно обновляйте ключи API и пароли для базы данных.
