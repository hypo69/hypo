# Модуль `header.py`

## Обзор

Модуль `header.py` предназначен для определения и настройки основных параметров проекта, таких как корневая директория, имя проекта, версия, автор и т.д. Он также загружает настройки из файла `settings.json` и документацию из файла `README.MD`.

## Функции

### `set_project_root`

**Описание**:
Находит корневую директорию проекта, начиная с директории текущего файла,
поиск идет вверх по дереву каталогов до первого каталога, содержащего один из маркерных файлов.

**Параметры**:
- `marker_files` (tuple): Кортеж имен файлов или директорий, определяющих корень проекта. По умолчанию `('__root__', '.git')`.

**Возвращает**:
- `Path`: Путь к корневой директории, если она найдена. В противном случае возвращается путь к директории, где находится скрипт.

## Глобальные переменные

### `__root__`
**Описание**:
Путь к корневой директории проекта.

### `settings`
**Описание**:
Словарь с настройками проекта, загруженный из файла `settings.json`. Если файл не найден или не может быть декодирован, значение равно `None`.

### `doc_str`
**Описание**:
Строка с документацией проекта, загруженная из файла `README.MD`. Если файл не найден или не может быть прочитан, значение равно `None`.

### `__project_name__`
**Описание**:
Имя проекта. По умолчанию `hypotez`. Если в `settings.json` есть параметр `project_name`, то используется его значение.

### `__version__`
**Описание**:
Версия проекта. По умолчанию `''`. Если в `settings.json` есть параметр `version`, то используется его значение.

### `__doc__`
**Описание**:
Документация проекта. Значение берется из переменной `doc_str`, если она установлена, иначе `''`.

### `__details__`
**Описание**:
Дополнительная информация о проекте. На данный момент всегда пустая строка `''`.

### `__author__`
**Описание**:
Автор проекта. По умолчанию `''`. Если в `settings.json` есть параметр `author`, то используется его значение.

### `__copyright__`
**Описание**:
Авторские права проекта. По умолчанию `''`. Если в `settings.json` есть параметр `copyrihgnt`, то используется его значение.

### `__cofee__`
**Описание**:
Строка, содержащая информацию о возможности угостить разработчика кофе. Если в `settings.json` есть параметр `cofee`, то используется его значение. В противном случае используется строка по умолчанию "Treat the developer to a cup of coffee for boosting enthusiasm in development: https://boosty.to/hypo69".
