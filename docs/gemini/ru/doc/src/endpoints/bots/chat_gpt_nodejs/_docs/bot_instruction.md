# Инструкция для бота

## Обзор

Этот файл содержит инструкции для бота, определяющие доступные команды и их параметры. Инструкции представлены в виде списка команд с их синтаксисом и кратким описанием.

## Подробней

Этот файл используется для отображения справочной информации о командах, которые может выполнять бот. Он предоставляет пользователям понимание того, как правильно использовать команды бота и какие параметры необходимо указывать.

## Функции

### `!hi`

**Назначение**: Приветствует пользователя.

**Как работает функция**:
1. Функция обрабатывает команду `!hi`.
2. Бот отправляет приветственное сообщение пользователю.

**Параметры**:
- Отсутствуют.

**Возвращает**:
- Ничего.

**Вызывает исключения**:
- Отсутствуют.

**Примеры**:
- `!hi` - Бот отвечает приветствием.

### `!train <data> <data_dir> <positive> <attachment>`

**Назначение**: Обучает модель с предоставленными данными.

**Как работает функция**:
1. Функция обрабатывает команду `!train` с параметрами `<data>`, `<data_dir>`, `<positive>` и `<attachment>`.
2. Функция определяет, какой тип данных предоставлен для обучения: файл (`<data>`), директория (`<data_dir>`) или прикрепленный файл (`<attachment>`).
3. Функция вызывает соответствующие методы для обучения модели на основе предоставленных данных.
4. Параметр `<positive>` используется для указания положительных примеров для обучения.

**Параметры**:
- `<data>`: Путь к файлу с данными для обучения.
- `<data_dir>`: Путь к директории с данными для обучения.
- `<positive>`: Указание положительных примеров для обучения.
- `<attachment>`: Прикрепленный файл с данными для обучения.

**Возвращает**:
- Ничего.

**Вызывает исключения**:
- Отсутствуют.

**Примеры**:
- `!train data.txt ./data true` - Обучение модели с использованием файла `data.txt` и положительных примеров из директории `./data`.
- `!train ./data_dir ./data true` - Обучение модели с использованием данных из директории `./data_dir`.
- `!train attachment ./data true` - Обучение модели с использованием прикрепленного файла.

### `!test <test_data>`

**Назначение**: Тестирует модель с предоставленными тестовыми данными в формате JSON.

**Как работает функция**:
1. Функция обрабатывает команду `!test` с параметром `<test_data>`.
2. Функция загружает тестовые данные из JSON.
3. Функция тестирует модель с использованием загруженных данных.
4. Функция возвращает результаты тестирования.

**Параметры**:
- `<test_data>`: Путь к файлу с тестовыми данными в формате JSON.

**Возвращает**:
- Результаты тестирования.

**Вызывает исключения**:
- Отсутствуют.

**Примеры**:
- `!test test_data.json` - Тестирование модели с использованием данных из файла `test_data.json`.

### `!archive <directory>`

**Назначение**: Архивирует файлы в указанной директории.

**Как работает функция**:
1. Функция обрабатывает команду `!archive` с параметром `<directory>`.
2. Функция архивирует файлы, находящиеся в указанной директории.
3. Функция сохраняет архив в указанном месте.

**Параметры**:
- `<directory>`: Путь к директории, которую необходимо заархивировать.

**Возвращает**:
- Ничего.

**Вызывает исключения**:
- Отсутствуют.

**Примеры**:
- `!archive ./data` - Архивирование файлов из директории `./data`.

### `!select_dataset <path_to_dir_positive> <positive>`

**Назначение**: Выбирает набор данных для обучения из указанной директории.

**Как работает функция**:
1. Функция обрабатывает команду `!select_dataset` с параметрами `<path_to_dir_positive>` и `<positive>`.
2. Функция выбирает набор данных из указанной директории для обучения.
3. Параметр `<positive>` используется для указания положительных примеров.

**Параметры**:
- `<path_to_dir_positive>`: Путь к директории с набором данных.
- `<positive>`: Указание положительных примеров.

**Возвращает**:
- Ничего.

**Вызывает исключения**:
- Отсутствуют.

**Примеры**:
- `!select_dataset ./positive_data true` - Выбор набора данных из директории `./positive_data` с указанием положительных примеров.
