# check_relise.py - Модуль для проверки последней версии релиза

Файл: `C:\Users\user\Documents\repos\hypotez\src\check_relise.py`
Расположение в проекте: `hypotez/src`

**Описание:**

Данный модуль (`check_relise.py`) предназначен для получения информации о последней версии релиза репозитория GitHub. Он использует API GitHub для получения данных и возвращает строковое представление тега последней версии.

**Использование:**

Функция `check_latest_release` принимает имя владельца репозитория (`owner`) и имя репозитория (`repo`) в качестве аргументов. Она формирует URL-запрос к API GitHub, получает ответ и, если ответ успешный (код статуса 200), извлекает значение `tag_name` из ответа в формате JSON.  В противном случае возвращает `None`.

**Пример использования:**

```python
from hypotez.src.check_relise import check_latest_release

owner = "your_github_username"
repo = "your_repository_name"

latest_version = check_latest_release(owner, repo)

if latest_version:
    print(f"Последняя версия: {latest_version}")
else:
    print("Не удалось получить информацию о последней версии.")
```

**Функции:**

* **`check_latest_release(owner: str, repo: str) -> str or None`:**
    * Получает информацию о последней версии релиза репозитория GitHub.
    * Возвращает строку, содержащую значение `tag_name` последнего релиза, или `None`, если произошла ошибка.

**Внутренняя логика:**

* Использует библиотеку `requests` для отправки запросов к API GitHub.
* Использует модуль `logger` для вывода сообщений об ошибках.

**Примечания:**

* Требуется установка библиотеки `requests`:
  ```bash
  pip install requests
  ```
* Должен быть доступен модуль `logger`, определенный в файле `src/logger.py`.
