# Документация модуля DeepSeek

## Обзор

Модуль `DeepSeek` предназначен для взаимодействия с API DeepSeek, предоставляя функциональность для работы с моделью deepseek-chat. Этот модуль является частью проекта `hypotez` и расширяет возможности класса `OpenaiAPI`, адаптируя его для работы с конкретным провайдером DeepSeek.

## Подробней

Модуль содержит класс `DeepSeek`, который определяет основные параметры для подключения к API DeepSeek, такие как URL, необходимость аутентификации, поддержку потоковой передачи и историю сообщений. Он также устанавливает модель по умолчанию и список запасных моделей.

## Классы

### `DeepSeek`

**Описание**: Класс `DeepSeek` предназначен для взаимодействия с API DeepSeek. Он наследует функциональность от класса `OpenaiAPI` и адаптирует ее для работы с сервисом DeepSeek.

**Наследует**:
- `OpenaiAPI`: Обеспечивает базовую функциональность для взаимодействия с API на основе OpenAI.

**Атрибуты**:
- `label` (str): Метка, идентифицирующая провайдера как "DeepSeek".
- `url` (str): URL платформы DeepSeek.
- `login_url` (str): URL страницы для управления ключами API DeepSeek.
- `working` (bool): Флаг, указывающий на работоспособность провайдера (True).
- `api_base` (str): Базовый URL для API DeepSeek.
- `needs_auth` (bool): Флаг, указывающий на необходимость аутентификации (True).
- `supports_stream` (bool): Флаг, указывающий на поддержку потоковой передачи (True).
- `supports_message_history` (bool): Флаг, указывающий на поддержку истории сообщений (True).
- `default_model` (str): Модель, используемая по умолчанию ("deepseek-chat").
- `fallback_models` (list[str]): Список запасных моделей, используемых в случае недоступности основной модели.
