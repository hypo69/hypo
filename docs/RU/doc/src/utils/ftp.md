# Модуль `hypotez/src/utils/ftp.py`

## Обзор

Модуль `hypotez/src/utils/ftp.py` предоставляет интерфейс для взаимодействия с FTP-серверами. Он включает функции для отправки, получения и удаления файлов с FTP-сервера.  Этот модуль предназначен для работы с медиафайлами (изображения, видео), электронными таблицами и другими файлами.

## Функции

### `write`

**Описание**: Отправляет файл на FTP-сервер.

**Параметры**:
- `source_file_path` (str): Путь к файлу, который нужно отправить.
- `dest_dir` (str):  Целевой каталог на FTP-сервере.
- `dest_file_name` (str): Имя файла на FTP-сервере.

**Возвращает**:
- `bool`: `True`, если файл успешно отправлен, `False` в противном случае.

**Обрабатывает исключения**:
- Любые исключения, возникающие при установлении соединения с FTP-сервером или при отправке файла, будут обработаны и записаны в лог.


### `read`

**Описание**: Получает файл с FTP-сервера.

**Параметры**:
- `source_file_path` (str): Путь, куда будет сохранен файл локально.
- `dest_dir` (str): Каталог на FTP-сервере, где находится файл.
- `dest_file_name` (str): Имя файла на FTP-сервере.

**Возвращает**:
- `Union[str, bytes, None]`: Содержимое файла, если оно было успешно получено, `None` в противном случае.

**Обрабатывает исключения**:
- Любые исключения, возникающие при установлении соединения с FTP-сервером или при получении файла, будут обработаны и записаны в лог.


### `delete`

**Описание**: Удаляет файл с FTP-сервера.

**Параметры**:
- `source_file_path` (str): Путь к файлу на локальной машине (не используется).
- `dest_dir` (str):  Каталог на FTP-сервере, где находится файл.
- `dest_file_name` (str): Имя файла на FTP-сервере.

**Возвращает**:
- `bool`: `True`, если файл успешно удален, `False` в противном случае.

**Обрабатывает исключения**:
- Любые исключения, возникающие при установлении соединения с FTP-сервером или при удалении файла, будут обработаны и записаны в лог.

##  Конфигурация подключения

Конфигурация подключения к FTP-серверу (`_connection`) предполагается определенной вне этого модуля. В примере, показанном в коде, значения прописаны в словаре, но в реальном применении они должны быть взяты из файла конфигурации или другого источника.

## Примечания

- Обратите внимание на использование блоков `try...except` для обработки потенциальных ошибок при взаимодействии с FTP-сервером, а также при закрытии соединения.  Это крайне важно для надежности работы модуля.
- Для корректной работы модуля убедитесь, что переменные `_connection` содержат правильные значения сервера, пользователя и пароля.
- Модуль использует логирование (через `logger`), чтобы записывать информацию о успехе/неудаче операций. Не забудьте импортировать и настроить модуль `src.logger`.
- Типизация функций (`typing`)  делает код более понятным и позволяет использовать проверки типов.
