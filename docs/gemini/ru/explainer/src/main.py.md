## <алгоритм>

1. **Начало программы:**
   - Программа начинается с вызова `main()` при запуске скрипта (`if __name__ == "__main__": main()`).

2. **Обработка аргументов командной строки (функция `main()`):**
   - Создается `argparse.ArgumentParser` для разбора аргументов командной строки.
   - Добавляется аргумент `--help`, который при наличии отображает справочную информацию.
   - Разбираются аргументы командой `parser.parse_args()`.
   - **Пример:**
     - Если скрипт запущен с аргументом `--help` (`python main.py --help`), `args.help` будет `True`, и программа выведет справочную информацию, вызвав `show_help()`, затем программа завершается.
     - Если аргумент `--help` отсутствует (`python main.py`), `args.help` будет `False`, и программа перейдет к интерактивному меню.

3. **Интерактивное меню (функция `interactive_menu()`):**
    - Выводится приветственное сообщение.
    - Запускается бесконечный цикл `while True`, который отображает доступные команды.
    - Пользователю предлагается ввести номер команды.
    - Полученный ввод (`choice`) обрабатывается:
        - **Пример:** Если пользователь вводит `1`, вызывается `script1()`.
        - **Пример:** Если пользователь вводит `2`, вызывается `script2()`.
        - **Пример:** Если пользователь вводит `3` или `--help`, вызывается `show_help()`.
        - **Пример:** Если пользователь вводит `exit`, программа выводит прощальное сообщение и выходит из цикла `break`.
        - Если ввод некорректен, выводится сообщение об ошибке в консоль (logger) и пользователь остается в цикле для повторного ввода.

4. **Выполнение скриптов (функции `script1()` и `script2()`):**
   - Функции `script1()` и `script2()` выводят сообщения о своем запуске и могут содержать произвольный код.
   - **Пример:**  При выборе команды `1` или `2`, программа выводит сообщение "`Script 1 started`" или "`Script 2 started`" и ждет дальнейший ввод.

5. **Отображение справки (функция `show_help()`):**
   - Функция `show_help()` выводит информацию о доступных командах в консоль.
   - **Пример:** Выводит список команд: "Run script 1", "Run script 2", "--help", "exit".

6. **Завершение программы:**
   - Программа завершается после выхода из цикла `interactive_menu()` при вводе "exit" или после вывода справки если запущен с аргументом --help

**Объяснение зависимостей:**

-   **`Start`**: Начальная точка выполнения программы.
-   **`MainFunc`**: Функция `main()` отвечает за обработку аргументов командной строки. Принимает решение о запуске справки или интерактивного меню в зависимости от наличия аргумента `--help`.
-   **`ShowHelpFunc`**: Функция `show_help()` отвечает за отображение списка доступных команд и их описания. Вызывается либо из `main()` если есть аргумент `--help`, либо из `interactive_menu()` при вводе 3 или `--help`.
-   **`InteractiveMenuFunc`**: Функция `interactive_menu()` реализует интерактивное меню.  Она запрашивает у пользователя ввод и вызывает соответствующий функционал.
-   **`Script1Func`**: Функция `script1()` представляет собой заглушку для кода первого скрипта, вызывается из `interactive_menu()`.
-   **`Script2Func`**: Функция `script2()` представляет собой заглушку для кода второго скрипта, вызывается из `interactive_menu()`.
-    **`ErrorLog`**:  В случае некорректного ввода выводится сообщение об ошибке с помощью `logger.error()` и возвращает пользователя обратно в интерактивное меню.
-   **`End`**: Конечная точка выполнения программы.

## <объяснение>

**Импорты:**

-   `import argparse`: Используется для обработки аргументов командной строки. Позволяет задавать различные параметры запуска скрипта, в данном случае проверяется наличие `--help`.
-   `from src.utils.jjson import j_loads, j_loads_ns`: Импортирует функции для работы с JSON из модуля `jjson` в пакете `src.utils`. В текущем коде эти импорты не используются и могут быть удалены, они являются частью шаблона.
-   `from src.logger.logger import logger`: Импортирует объект `logger` для логирования ошибок и других событий, используется для вывода сообщения об ошибке при некорректном вводе в интерактивном меню.

**Функции:**

-   `script1()`:
    -   **Аргументы:** Нет.
    -   **Возвращаемое значение:** Нет.
    -   **Назначение:** Выводит сообщение о начале выполнения скрипта 1. Представляет собой заглушку для фактического кода скрипта 1.
    -   **Пример:** `script1()` при выборе команды "1" в интерактивном меню.
-   `script2()`:
    -   **Аргументы:** Нет.
    -   **Возвращаемое значение:** Нет.
    -   **Назначение:** Выводит сообщение о начале выполнения скрипта 2. Представляет собой заглушку для фактического кода скрипта 2.
    -   **Пример:** `script2()` при выборе команды "2" в интерактивном меню.
-   `show_help()`:
    -   **Аргументы:** Нет.
    -   **Возвращаемое значение:** Нет.
    -   **Назначение:** Выводит на экран список доступных команд и их описание.
    -   **Пример:** `show_help()` вызывается при запуске скрипта с аргументом `--help` или при выборе команды "3" или "--help" в интерактивном меню.
-   `interactive_menu()`:
    -   **Аргументы:** Нет.
    -   **Возвращаемое значение:** Нет.
    -   **Назначение:** Обеспечивает интерактивный интерфейс для выбора и запуска скриптов. Отображает меню команд, обрабатывает пользовательский ввод, вызывает соответствующие функции.
    -   **Пример:**  `interactive_menu()` работает в бесконечном цикле, пока пользователь не выберет команду "exit", при каждом вводе вызывается соотвествующая команда
-   `main()`:
    -   **Аргументы:** Нет.
    -   **Возвращаемое значение:** Нет.
    -   **Назначение:** Основная функция, которая отвечает за разбор аргументов командной строки и запуск либо интерактивного меню, либо отображение справки.
    -   **Пример:** `main()` запускается при старте скрипта.

**Переменные:**

-   `parser`: Объект `argparse.ArgumentParser`, используемый для определения и разбора аргументов командной строки.
-   `args`: Объект, содержащий разобранные аргументы командной строки, полученные при помощи `parser.parse_args()`.
-   `choice`: Строка, хранящая ввод пользователя в интерактивном меню.

**Потенциальные ошибки и области для улучшения:**

1.  **Обработка ошибок ввода:**
    -   В функции `interactive_menu()` есть обработка неверного ввода, но она может быть расширена, например проверка ввода на числовой формат при выборе пункта меню.
2.  **Реализация скриптов:**
    -   Функции `script1()` и `script2()` являются заглушками. Необходимо реализовать фактическую логику этих скриптов.
3.  **Использование logger**
    -   Используется только для вывода сообщения об ошибке, в будущем можно расширить его функциональность для логирования всех действий программы.
4. **Удаление неиспользуемого кода**
   - Есть импорт `j_loads, j_loads_ns`, но данные импорты не используются и их можно удалить.
   - В коде присутсвует закомментированный вариант функций, его можно удалить.
5.  **Код может быть расширен**
   -  Можно добавить возможность выбора и запуска большего количества скриптов, сделать динамическое меню.

**Цепочка взаимосвязей:**

1.  `main()` является точкой входа и контролирует запуск либо справки, либо интерактивного меню.
2.  `interactive_menu()` вызывает `script1()`, `script2()` или `show_help()`.
3. `logger` используется в `interactive_menu()` для логирования ошибок.
4.  `argparse` используется для парсинга аргументов командной строки в `main()`.
