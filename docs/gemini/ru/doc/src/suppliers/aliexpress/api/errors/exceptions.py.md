# Модуль исключений API AliExpress

## Обзор

Модуль содержит пользовательские исключения, используемые для обработки ошибок, возникающих при взаимодействии с API AliExpress. Он определяет базовый класс `AliexpressException`, от которого наследуются все остальные исключения, специфичные для различных ситуаций ошибок, таких как неверные аргументы, отсутствие идентификатора продукта, сбои запросов к API и т. д.

## Подробнее

Этот модуль предоставляет набор исключений, которые могут быть использованы для более точной обработки ошибок при работе с API AliExpress. Использование пользовательских исключений позволяет улучшить читаемость и поддерживаемость кода, а также упрощает отладку.

**Описание**: Базовый класс для всех исключений API AliExpress.

Метод `__str__` переопределен для возврата этой причины в строковом представлении исключения.

- `reason` (str): Причина возникновения исключения.

### `InvalidArgumentException`

**Описание**: Исключение, выбрасываемое при некорректных аргументах.

**Принцип работы**:
Класс `InvalidArgumentException` наследуется от `AliexpressException` и представляет собой исключение, которое выбрасывается, когда аргументы, переданные в функцию или метод, являются некорректными.

### `ProductIdNotFoundException`

**Описание**: Исключение, выбрасываемое, если идентификатор продукта не найден.

**Принцип работы**:
Класс `ProductIdNotFoundException` наследуется от `AliexpressException` и представляет собой исключение, которое выбрасывается, когда не удается найти продукт с указанным идентификатором.

### `ApiRequestException`

**Описание**: Исключение, выбрасываемое, если запрос к API AliExpress завершается неудачей.

**Принцип работы**:
Класс `ApiRequestException` наследуется от `AliexpressException` и представляет собой исключение, которое выбрасывается, когда происходит ошибка при выполнении запроса к API AliExpress.

### `ApiRequestResponseException`

**Описание**: Исключение, выбрасываемое, если ответ на запрос не является допустимым.

**Принцип работы**:
Класс `ApiRequestResponseException` наследуется от `AliexpressException` и представляет собой исключение, которое выбрасывается, когда полученный ответ от API AliExpress не соответствует ожидаемому формату или содержит некорректные данные.

### `ProductsNotFoudException`

**Описание**: Исключение, выбрасываемое, если продукты не найдены.

**Принцип работы**:
Класс `ProductsNotFoudException` наследуется от `AliexpressException` и представляет собой исключение, которое выбрасывается, когда не удается найти ни одного продукта по заданным критериям.

### `CategoriesNotFoudException`

**Описание**: Исключение, выбрасываемое, если категории не найдены.

**Принцип работы**:
Класс `CategoriesNotFoudException` наследуется от `AliexpressException` и представляет собой исключение, которое выбрасывается, когда не удается найти ни одной категории по заданным критериям.

### `InvalidTrackingIdException`

**Описание**: Исключение, выбрасываемое, если идентификатор отслеживания отсутствует или недействителен.

**Принцип работы**:
Класс `InvalidTrackingIdException` наследуется от `AliexpressException` и представляет собой исключение, которое выбрасывается, когда идентификатор отслеживания отсутствует или имеет неверный формат.
