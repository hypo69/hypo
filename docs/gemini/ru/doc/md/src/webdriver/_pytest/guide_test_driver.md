# Руководство по тестированию драйвера (Guide_test_driver)

## Обзор

Данное руководство предоставляет инструкции по тестированию класса `DriverBase` с использованием фреймворка `pytest`. Оно описывает необходимые компоненты, шаги по настройке окружения, выполнению тестов и интерпретации результатов.

## Предварительные условия

Перед началом тестирования убедитесь, что на вашей системе установлены следующие компоненты:

### Python 3.12

Убедитесь, что установлена версия Python 3.12. Проверьте версию с помощью команды:

```bash
python --version
```

### pytest

Установите `pytest`, если он еще не установлен:

```bash
pip install pytest
```

### unittest.mock

Библиотека `unittest.mock` включена в стандартную библиотеку Python, начиная с версии 3.3.

## Настройка окружения

### Клонирование репозитория

Склонируйте репозиторий на локальную машину:

```bash
git clone <URL_вашего_репозитория>
cd <имя_проекта>
```

### Создание и активация виртуального окружения

Создайте виртуальное окружение:

```bash
python -m venv venv
source venv/bin/activate  # На Windows используйте: venv\Scripts\activate
```

### Установка зависимостей

Установите необходимые зависимости, перечисленные в `requirements.txt` (если таковой имеется):

```bash
pip install -r requirements.txt
```


## Написание и запуск тестов

### Написание тестов

Тесты для класса `DriverBase` находятся в файле `tests/test_driver.py`. Примеры тестов включают тестирование методов `driver_payload`, `scroll`, `locale` и других.

### Запуск тестов

Для запуска тестов выполните команду в корневой директории проекта:

```bash
pytest tests/test_driver.py
```

### Интерпретация результатов

После запуска тестов `pytest` предоставит отчет о результатах. Пример:

```
============================== test session starts ===============================\
platform linux -- Python 3.12.0, pytest-7.0.0, pluggy-0.13.1
rootdir: /path/to/your/project
collected 9 items

tests/test_driver.py ........                                        [100%]

=============================== 9 passed in 1.23s ================================
```

Отчет показывает количество пройденных тестов и время выполнения. Если тест не пройден, `pytest` предоставит детали ошибки.


## Полезные команды

### Запуск всех тестов

```bash
pytest
```

### Запуск тестов с подробным выводом

```bash
pytest -v
```

### Запуск тестов с отчетом о покрытии кода

Установите `pytest-cov`:

```bash
pip install pytest-cov
```

Затем запустите тесты с отчетом:

```bash
pytest --cov=src tests/
```

## Заключение

Тестирование является важной частью разработки. Используя `pytest` и это руководство, вы можете эффективно тестировать функциональность `DriverBase`, обеспечивая корректную работу и предотвращая регрессии.
