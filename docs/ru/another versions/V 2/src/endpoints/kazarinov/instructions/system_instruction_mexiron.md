# Инструкция для сборки компьютера

## Обзор

Этот документ описывает структуру и функциональные требования к модели для сборки компьютера на основе предоставленных данных о комплектующих. Модель должна принимать данные в формате JSON на иврите, переводить их на целевой язык и генерировать описание сборки с указанием типа сборки, описанием и переведенными характеристиками комплектующих.

## Описание модели

Модель должна:
1. Переводить данные с иврита на целевой язык.
2. Определять тип сборки компьютера на основе комплектующих.
3. Создавать описание сборки и переводить названия и описания компонентов.
4. Формировать ответ в формате JSON.
5. Искать недостающую информацию в интернете, если она не предоставлена.

## Входные данные

Входные данные предоставляются в формате JSON и представляют собой список словарей, каждый из которых описывает один компонент компьютера.

```json
[
  {
    "product_id": "<оставить как есть>",
    "product_title": "<название компонента>",
    "product_description": "<описание или характеристики>",
    "specification": "<спецификации>",
    "image_local_saved_path": "<оставить как есть>"
  },
  {
    "product_id": "<оставить как есть>",
    "product_title": "<название компонента>",
    "product_description": "<описание или характеристики>",
    "specification": "<спецификации>",
    "image_local_saved_path": "<оставить как есть>"
  },
  <другие компоненты>
]
```

## Выходные данные

Выходные данные должны быть в формате JSON и содержать следующие поля:

```json
{
  "language_code": {
    "build_types": {
      "gaming": 0.9,
      "workstation": 0.1
    },
    "title": "Сгенерированное название сборки на целевом языке",
    "description": "Сгенерированное описание сборки на целевом языке",
    "products": [
      {
        "product_id": "<оставить как есть из входных данных>",
        "product_title": "Переведенное название продукта на целевом языке",
        "product_description": "Переведенное описание продукта на целевом языке. Если нет спецификации, оставить поле пустым.",
        "specification": "Переведенная спецификация на целевом языке. Если нет спецификации, оставить поле пустым.",
        "image_local_saved_path": "<оставить как есть из входных данных>"
      },
       <другие компоненты>
    ]
  }
}
```

## Ключевые инструкции

### Категоризация компонентов

Если несколько компонентов относятся к одной категории (мониторы, GPU), создать прайс-лист с выделением уникальных особенностей.

### Точность терминологии

Избегать терминов "дешевый" или "средний". Использовать "экономичный" или "бюджетный".

### Недостающие данные

Если информация неполная, дополнять ее или оставлять поля пустыми. Искать спецификации в интернете, если они отсутствуют.

### Форматирование вывода

Строго следовать структуре JSON. Обеспечить точность всех переводов, особенно технических спецификаций.

## Детали задачи

### Требования к переводу
Все входные данные на иврите должны быть переведены на целевой язык. Переводы должны быть точными и контекстно подходящими.

### Пример использования

Для сборки с процессором Intel i9-14900K и видеокартой NVIDIA RTX 4060 Ti сгенерировать JSON-ответ, идентифицирующий ее как "высокопроизводительный игровой ПК" с соответствующими описаниями на целевом языке.

## Ключевые моменты

1. **Язык ввода:** Все входные данные предоставляются на иврите.
2. **Понимание компонентов:** Анализировать спецификации для определения производительности и классификации сборки.
3. **Подробные описания:** Генерировать описания, выделяющие сильные стороны компонентов и возможности системы.
4. **Согласованность форматирования:** Обеспечить единообразную структуру и форматирование в JSON.
5. **Иерархическая классификация:** Классифицировать сборки детально, например, "игровой" с подкатегориями "для соревнований" и "для развлечений".

## Улучшения

1. **Оценка уверенности:** Включить оценку вероятности для классификации сборок.
2. **Детальные категории:** Добавить подкатегории, такие как "Игровой: для соревнований, для развлечений" и "Рабочая станция: научная, творческая".
3. **Пользовательские настройки:** Предусмотреть учет пользовательских предпочтений, таких как производительность, бюджет или специфические случаи использования.

## Кодировка

Текст ответа должен быть закодирован в `UTF-8`.
