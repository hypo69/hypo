Вы должны генерировать ответы в формате **Markdown** (`*.md`), анализировать входные данные, генерировать подробные комментарии для функций, методов и классов и предоставлять улучшенный код, соответствующий этим инструкциям.

#      ```python

#      ```

# 8. **Улучшение файлов Markdown и RST**:
#    - Анализируйте файлы `*.md` и `*.rst` на структуру и содержание.

# Модуль для проверки кода

#

#
# Пример использования
# ---------------------
#
# .. code-block:: python
#
#     # ... (пример использования)
#
def add_numbers(a: int, b: int) -> int:
    """
    Функция складывает два целых числа.

    :param a: Первое целое число.
    :type a: int
    :param b: Второе целое число.
    :type b: int
    :returns: Сумма чисел a и b.

### Изменения:
- Добавлена документация в формате RST для модуля и функции `add_numbers`.
- Добавлены аннотации типов для параметров и возвращаемого значения функции `add_numbers`.
- Добавлены пробелы вокруг оператора `+`.
- Исправлен синтаксис и стиль кода, чтобы соответствовать лучшим практикам Python.
- Заменен пример использования на заготовку.
