# Модуль `src.endpoints.bots`

## Обзор

Этот модуль содержит ботов для взаимодействия с различными сервисами.  Он предоставляет интерфейсы для работы с различными типами сообщений, обработкой запросов и ответов.


## Функции

### `create_bot`

**Описание**: Создает нового бота.

**Параметры**:
- `config (dict)`: Конфигурация бота.  Необходимы ключи для инициализации бота.
- `service (str)`: Идентификатор сервиса, с которым взаимодействует бот.

**Возвращает**:
- `Bot`: Объект бота.

**Вызывает исключения**:
- `ValueError`: Если входные данные некорректны или отсутствуют необходимые параметры в конфигурации.
- `TypeError`: Если типы входных параметров не соответствуют ожидаемым.

### `process_message`

**Описание**: Обрабатывает входное сообщение от пользователя.

**Параметры**:
- `message (dict)`: Входящее сообщение в формате словаря.
- `bot (Bot)`: Объект бота, для которого обрабатывается сообщение.

**Возвращает**:
- `dict | None`: Результат обработки сообщения в формате словаря или `None`, если обработка не выполнена.

**Вызывает исключения**:
- `BotError`:  Если бот не может обработать сообщение по каким-либо причинам.
- `TypeError`: Если тип входных данных не соответствует ожидаемому.


## Классы

### `Bot`

**Описание**: Базовый класс для ботов.

**Методы**:
- `send_message(message: str, channel: str) -> bool`: Отправляет сообщение в указанный канал.
- `handle_request(request: dict) -> dict`: Обрабатывает входящий запрос.

**Параметры**:
- `config(dict)`: Конфигурация бота.

**Возвращает**:
- `bool`: `True` в случае успешной отправки, `False` - при ошибке.
- `dict`: Ответ на входящий запрос.

**Вызывает исключения**:
- `ConnectionError`:  Ошибка соединения с целевым сервисом.
- `TypeError`:  Неверный тип входных данных.
