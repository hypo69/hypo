# Модуль `src.suppliers.aliexpress.graber`

## Обзор

Модуль предназначен для сбора данных о товарах с веб-сайта AliExpress. Он содержит класс `Graber`, который наследуется от базового класса `src.suppliers.graber.Graber` и предоставляет методы для обработки различных полей товара на странице.

## Подробней

Модуль `src.suppliers.aliexpress.graber` является частью системы сбора данных о товарах с различных онлайн-платформ. Он специализируется на AliExpress и использует веб-драйвер для взаимодействия с сайтом и извлечения информации о товарах.

## Классы

### `Graber`

**Описание**: Класс для сбора данных о товарах с AliExpress.

**Наследует**:
- `src.suppliers.graber.Graber`: Базовый класс для граберов, определяющий основную логику сбора данных.

**Атрибуты**:
- `supplier_prefix` (str): Префикс поставщика, устанавливается в значение `'aliexpress'`.

**Методы**:
- `__init__`: Инициализирует экземпляр класса `Graber`, вызывая конструктор родительского класса и устанавливая префикс поставщика.

### `close_pop_up`

**Описание**: Декоратор для закрытия всплывающих окон перед выполнением основной логики функции.

**Принцип работы**:
Этот декоратор предназначен для обработки всплывающих окон, которые могут появляться на сайте AliExpress и мешать сбору данных. Он пытается закрыть всплывающее окно, используя локатор, определенный в `Context.locator_for_decorator.close_pop_up`. Если локатор определен и выполнение локатора успешно, всплывающее окно закрывается, и выполнение основной функции продолжается. Если выполнение локатора завершается неудачей, ошибка логируется, но выполнение основной функции все равно продолжается.


**Назначение**: Инициализация экземпляра класса `Graber`.

**Параметры**:
- `driver` (Driver): Экземпляр веб-драйвера, используемый для взаимодействия с браузером и выполнения запросов к сайту AliExpress.
- `lang_index` (int): Индекс языка.

**Как работает функция**:

1. Устанавливает атрибут `supplier_prefix` равным `'aliexpress'`, что идентифицирует AliExpress как поставщика данных.
2. Вызывает конструктор родительского класса `Graber` (из модуля `src.suppliers.graber`), передавая `supplier_prefix`, `driver` и `lang_index`. Это позволяет инициализировать общие параметры и настройки, необходимые для сбора данных.
3. Устанавливает значение `Context.locator_for_decorator` в `None`. Этот атрибут используется декоратором `@close_pop_up` для закрытия всплывающих окон. Установка в `None` означает, что по умолчанию декоратор не будет выполнять никаких действий.
