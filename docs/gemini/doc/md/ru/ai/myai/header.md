# Модуль `header.py`

## Обзор

Этот модуль содержит функции для определения корневого каталога проекта, загрузки настроек из файла `settings.json` и получения информации о проекте, такой как название, версия, описание и другие метаданные.

## Оглавление

- [Функции](#функции)
  - [`get_project_root`](#get_project_root)


## Функции

### `get_project_root`

**Описание**: Находит корневой каталог проекта, начиная от текущего каталога, ищет вверх по иерархии каталогов, пока не найдёт каталог, содержащий один из файлов-маркеров.

**Параметры**:
- `marker_files` (tuple): Кортеж имён файлов или каталогов, по которым определяется корневой каталог проекта. По умолчанию: `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:
- `Path`: Путь к корневому каталогу проекта, если найден, иначе путь к каталогу, где расположен скрипт.

**Обрабатываемые исключения**:
- Нет.


## Постоянные переменные

### `MODE`

**Описание**: Переменная, определяющая режим работы проекта (разработка, тестирование и т.д.). Имеет строковое значение.

```python
MODE = 'development'
```


### `__root__`

**Описание**: Путь к корневому каталогу проекта. Задается функцией `get_project_root()`.


## Инициализация проекта

- Функция `get_project_root` определяет корень проекта и добавляет его в sys.path для корректного импорта модулей.
- Загружает настройки из файла `settings.json`, если он существует, используя `gs.path.root / 'src' / 'settings.json'`. Если файл не найден или некорректный, переменная `settings` остается None.
- Загружает строку документации из файла `README.MD` в переменную `doc_str`.


## Доступные переменные

- `__project_name__`: Имя проекта, полученное из настроек. По умолчанию 'hypotez'.
- `__version__`: Версия проекта, полученная из настроек. По умолчанию ''.
- `__doc__`: Документация проекта, полученная из файла `README.MD` (если он существует).
- `__details__`: Подробная информация о проекте.
- `__author__`: Автор проекта.
- `__copyright__`: Авторские права.
- `__cofee__`: Ссылка для поддержки разработчика.
