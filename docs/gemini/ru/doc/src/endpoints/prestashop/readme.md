# Документация для модуля PrestaShop

## Обзор

Этот файл предоставляет информацию о структуре и использовании веб-сайтов PrestaShop, а также о хранении и использовании API-ключей.

## Веб-сайты

Список веб-сайтов PrestaShop:

1. [e-cat.co.il](https://e-cat.co.il)
2. [emil-design.com](https://emil-design.com)
3. [sergey.mymaster.co.il](https://sergey.mymaster.co.il)

Каждый веб-сайт использует API для взаимодействия с различными параметрами и функциями.

## Хранение API-ключей

API-ключи для каждого веб-сайта хранятся в файле `credentials.kdbx`.  Этот файл — защищенная база данных паролей, содержащая следующую информацию для каждого сайта:

- URL веб-сайта
- API-ключ
- Дополнительные метаданные (при необходимости)

Для работы с ключами из файла используйте менеджер паролей, поддерживающий формат `.kdbx`, например [KeePass](https://keepass.info/) или [KeePassXC](https://keepassxc.org/).

## Пример использования API

Для подключения к API одного из ваших веб-сайтов используйте шаблон ниже:

### Пример запроса к API

**Шаблон запроса к API:**

```bash
curl -X GET 'https://<SITE_URL>/api/<endpoint>' \
-H 'Authorization: Basic <base64(API_KEY)>'
```

**Описание параметров:**

- `<SITE_URL>` — адрес веб-сайта, например, `e-cat.co.il`.
- `<endpoint>` — конечная точка API (например, `products`, `customers`).
- `<API_KEY>` — API-ключ, закодированный в Base64.

### Пример вызова API

Для получения списка продуктов с сайта `e-cat.co.il`:

```bash
curl -X GET 'https://e-cat.co.il/api/products' \
-H 'Authorization: Basic <base64(API_KEY)>'
```

## Рекомендации по безопасности

- Никогда не делитесь файлом `credentials.kdbx` с другими.
- Обеспечьте хранение файла в защищенном месте, доступном только вам.
- Регулярно обновляйте API-ключи и пароли базы данных.

## Дополнительные ресурсы

Если у вас возникнут проблемы или вопросы по подключению к API, обратитесь к [официальной документации PrestaShop API](https://devdocs.prestashop.com/), которая содержит информацию о доступных конечных точках и способах взаимодействия с ними.
