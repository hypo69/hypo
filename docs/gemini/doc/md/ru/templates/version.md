# Модуль version.py

## Обзор

Этот модуль отвечает за получение информации о версии проекта из файла `../settings.json`. Если файл `../settings.json` отсутствует или содержит некорректные данные, то используются значения по умолчанию.  Модуль определяет переменные, содержащие имя проекта, версию, документацию, дополнительные детали, имя автора, копирайт и ссылку на пожертвования.

## Переменные

### `MODE`

**Описание**:  Переменная, содержащая режим работы приложения. В данном случае, значение установлено как 'development'.

### `settings`

**Описание**: Словарь, содержащий настройки проекта. Считывается из файла `../settings.json`.

### `__project_name__`

**Описание**: Имя проекта. Получается из `settings['project_name']`, по умолчанию 'hypotez'.

### `__version__`

**Описание**: Версия проекта. Получается из `settings['version']`, по умолчанию ''.

### `__doc__`

**Описание**: Документация проекта. Получается из `settings['doc']`, по умолчанию ''.

### `__details__`

**Описание**: Дополнительные детали проекта. Получается из `settings['details']`, по умолчанию ''.

### `__author__`

**Описание**: Автор проекта. Получается из `settings['author']`, по умолчанию ''.

### `__copyright__`

**Описание**: Копирайт проекта. Получается из `settings['copyrihgnt']`, по умолчанию ''.

### `__cofee__`

**Описание**: Ссылка на пожертвования разработчику. Получается из `settings['cofee']`, по умолчанию ссылка на Boosty.


## Обработка исключений

### `FileNotFoundError` и `json.JSONDecodeError`

**Описание**:  Если файл `../settings.json` не найден или содержит некорректный JSON, исключения `FileNotFoundError` и `json.JSONDecodeError` будут обработаны, и соответствующие переменные примут значения по умолчанию.
