# Модуль `hypotez/src/endpoints/emil/header.py`

## Обзор

Этот модуль содержит функции для получения корневого каталога проекта, а также для загрузки настроек из файла `settings.json` и документации из файла `README.MD`.

## Функции

### `get_project_root`

**Описание**: Находит корневую директорию проекта, начиная с текущей директории и двигаясь вверх по древу директорий.

**Параметры**:
- `marker_files` (tuple): Кортеж имен файлов или директорий, которые используются для определения корневой директории проекта. По умолчанию это `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:
- `Path`: Путь к корневой директории проекта, если она найдена. В противном случае возвращает директорию, в которой находится текущий скрипт.

**Вызывает исключения**:
- Не вызывает никаких исключений.


### `get_settings`

**Описание**: Возвращает настройки из файла `settings.json`.

**Параметры**:
-  Не принимает параметров.


**Возвращает**:
- `dict`: Словарь с настройками, загруженными из файла `settings.json`. Возвращает `None`, если файл не найден или содержит некорректный JSON.

**Вызывает исключения**:
- `FileNotFoundError`: Если файл `settings.json` не найден.
- `json.JSONDecodeError`: Если файл `settings.json` содержит некорректный JSON.

### `get_doc_string`

**Описание**: Возвращает содержимое файла `README.MD`.

**Параметры**:
- Не принимает параметров.


**Возвращает**:
- `str`: Содержимое файла `README.MD`. Возвращает `None`, если файл не найден или содержит некорректный JSON.

**Вызывает исключения**:
- `FileNotFoundError`: Если файл `README.MD` не найден.
- `json.JSONDecodeError`: Если файл `README.MD` содержит некорректный JSON.


## Переменные

### `__root__`

**Описание**: Путь к корневой директории проекта.

**Тип**: `Path`

**Значение**: Результат вызова функции `get_project_root()`.

### `settings`

**Описание**: Словарь настроек проекта, полученный из файла `settings.json`.

**Тип**: `dict`

**Значение**: Результат вызова функции `get_settings()`.

### `doc_str`

**Описание**: Строка с содержимым файла `README.MD`.

**Тип**: `str`

**Значение**: Результат вызова функции `get_doc_string()`.

### `__project_name__`

**Описание**: Название проекта, полученное из настроек.

**Тип**: `str`

**Значение**: Значение поля `project_name` в настройках, по умолчанию `'hypotez'`.

### `__version__`

**Описание**: Версия проекта, полученная из настроек.

**Тип**: `str`

**Значение**: Значение поля `version` в настройках, по умолчанию `''`.

### `__doc__`

**Описание**: Строка с содержимым документации, полученная из файла `README.MD`.

**Тип**: `str`

**Значение**: Значение `doc_str`, по умолчанию `''`.


### `__details__`, `__author__`, `__copyright__`, `__cofee__`

**Описание**: Дополнительные поля из настроек (Подробности, Автор, Авторские права, Ссылка на поддержку).

**Тип**: `str`

**Значение**: Значения полей `details`, `author`, `copyright`, `cofee` в настройках соответственно. Если поле не найдено, используется значение по умолчанию (пустая строка для `__details__`, `__author__`, `__copyright__` и значение из `settings.json`, по умолчанию "Treat the developer to a cup of coffee for boosting enthusiasm in development: https://boosty.to/hypo69" для `__cofee__`.)
