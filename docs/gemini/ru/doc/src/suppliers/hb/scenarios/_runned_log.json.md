# Модуль _runned_log.json

## Обзор

Этот модуль предназначен для хранения и управления данными о выполненных сценариях. Данные включают в себя информацию о времени запуска, параметрах и результатах выполнения каждого сценария.

## Функции

### `add_log_data`

**Описание**: Добавляет информацию о выполненном сценарии в лог.

**Параметры**:
- `data` (dict): Данные о выполненном сценарии, включая время запуска, параметры и результат.

**Возвращает**:
- `None`

**Вызывает исключения**:
- `FileNotFoundError`: Возникает, если файл лога не найден.
- `json.JSONDecodeError`: Возникает, если не удается декодировать JSON из файла.
- `Exception`: Возникает при других ошибках записи в лог.

### `get_log_data`

**Описание**: Получает данные из лога.

**Параметры**:
- `limit` (Optional[int], optional): Ограничение на количество возвращаемых записей. По умолчанию `None`, что возвращает все записи.

**Возвращает**:
- `list[dict]`: Список словарей, где каждый словарь представляет информацию о выполненном сценарии.

**Вызывает исключения**:
- `FileNotFoundError`: Возникает, если файл лога не найден.
- `json.JSONDecodeError`: Возникает, если не удается декодировать JSON из файла.
- `Exception`: Возникает при других ошибках чтения из лога.

### `_write_to_file`

**Описание**: Записывает данные в файл.

**Параметры**:
- `data` (list[dict]): Список словарей с данными для записи.

**Возвращает**:
- `None`

**Вызывает исключения**:
- `Exception`: Возникает при ошибке записи в файл.

### `_read_from_file`

**Описание**: Читает данные из файла.

**Параметры**:
- `None`

**Возвращает**:
- `list[dict]`: Список словарей, где каждый словарь представляет собой запись из файла.

**Вызывает исключения**:
- `FileNotFoundError`: Возникает, если файл лога не найден.
- `json.JSONDecodeError`: Возникает, если не удается декодировать JSON из файла.
- `Exception`: Возникает при других ошибках чтения из файла.
