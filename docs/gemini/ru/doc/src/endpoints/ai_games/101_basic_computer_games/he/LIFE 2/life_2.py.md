# LIFE 2

## Обзор

Модуль реализует симуляцию клеточного автомата "Игра Жизнь" Джона Конвея. Игра происходит на двумерной сетке, где каждая ячейка может быть либо "живой" (1), либо "мертвой" (0). На каждом шаге состояние ячеек изменяется в соответствии с количеством их живых соседей. Это игра без реального игрока, и эволюция клеток отображается как последовательность поколений.

## Функции

### `create_grid`

**Описание**: Создает случайную сетку с живыми и мертвыми клетками.

**Возвращает**:
- `list`: Двумерный список (сетка) с клетками (0 или 1).

### `display_grid`

**Описание**: Выводит сетку на экран, где живые клетки отмечены символом `*`, а мертвые - пробелом.

**Параметры**:
- `grid` (list): Сетка для отображения.

### `count_live_neighbors`

**Описание**: Подсчитывает количество живых соседей для заданной ячейки.

**Параметры**:
- `grid` (list): Сетка.
- `row` (int): Строка ячейки.
- `col` (int): Столбец ячейки.

**Возвращает**:
- `int`: Количество живых соседей.

### `update_cell_state`

**Описание**: Обновляет состояние клетки в соответствии с правилами игры.

**Параметры**:
- `cell` (int): Текущее состояние клетки (0 или 1).
- `neighbors` (int): Количество живых соседей.

**Возвращает**:
- `int`: Новое состояние клетки (0 или 1).

### `create_new_generation`

**Описание**: Создает новое поколение сетки в соответствии с правилами игры.

**Параметры**:
- `grid` (list): Текущая сетка.

**Возвращает**:
- `list`: Новая сетка (следующее поколение).

### `play_life`

**Описание**: Запускает игру "Жизнь".

## Описание кода

1.  **Импорт модулей**:
    - `import random`: Импортирует модуль `random` для создания случайных значений для начальной сетки.
    - `import time`: Импортирует модуль `time` для управления скоростью обновления сетки.
    - `import os`: Импортирует модуль `os` для очистки экрана.

2.  **Константы**:
    - `ROWS = 20`: Количество строк в сетке.
    - `COLS = 60`: Количество столбцов в сетке.

3.  **Функция `create_grid()`**:
    - Создает начальную случайную сетку.
    - Возвращает двумерный список, где каждая ячейка получает случайное значение 0 или 1 (мертвая или живая клетка).

4.  **Функция `display_grid(grid)`**:
    - Принимает сетку в качестве входных данных и выводит ее в консоль.
    - Очищает экран перед выводом.
    - Живые клетки отображаются как `*`, а мертвые как пробелы.

5.  **Функция `count_live_neighbors(grid, row, col)`**:
    - Принимает сетку, строку ячейки и столбец ячейки.
    - Подсчитывает количество живых соседей данной ячейки.
    - Возвращает количество живых соседей.

6.  **Функция `update_cell_state(cell, neighbors)`**:
    - Принимает текущее состояние клетки (живая или мертвая) и количество живых соседей.
    - Обновляет состояние клетки в соответствии с правилами игры:
        - Живая клетка с менее чем двумя живыми соседями умирает.
        - Живая клетка с двумя или тремя живыми соседями остается живой.
        - Живая клетка с более чем тремя живыми соседями умирает.
        - Мертвая клетка с тремя живыми соседями оживает.
    - Возвращает новое состояние клетки.

7.  **Функция `create_new_generation(grid)`**:
    - Принимает текущую сетку и создает следующее поколение.
    - Создает новую сетку и, используя цикл, для каждой клетки в текущей сетке, обновляет состояние новой клетки в соответствии с правилами игры, добавляя ее в новую сетку.
    - Возвращает новую сетку (следующее поколение).

8.  **Функция `play_life()`**:
    - Создает начальную сетку, вызывая функцию `create_grid()`.
    - Начинает бесконечный цикл.
    - В каждой итерации цикла:
        - Выводит сетку с помощью `display_grid()`.
        - Создает следующее поколение с помощью `create_new_generation()`.
        - Ждет короткое время с помощью `time.sleep(0.1)`, чтобы замедлить скорость игры.

9.  **Основной запуск кода**:
    - Код `if __name__ == "__main__":` гарантирует, что функция `play_life()` будет запущена только при прямом выполнении файла.
