# Модуль `header`

## Обзор

Модуль `header` предназначен для определения корневой директории проекта, загрузки настроек из `settings.json` и информации из `README.MD`, а также для хранения основных метаданных проекта.

## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная с директории текущего файла, просматривая вверх и останавливаясь на первой директории, содержащей один из маркеров.

**Параметры**:
- `marker_files` (tuple): Кортеж с именами файлов или директорий, которые идентифицируют корневую директорию проекта. По умолчанию `('__root__', '.git')`.

**Возвращает**:
- `Path`: Путь к корневой директории, если она найдена, иначе - путь к директории, где расположен скрипт.

## Переменные модуля

### `__root__`

**Описание**: `Path` к корневой директории проекта.

### `settings`

**Описание**: Словарь с настройками проекта, загруженный из `settings.json`.

### `doc_str`

**Описание**: Содержимое файла `README.MD` в виде строки.

### `__project_name__`

**Описание**: Название проекта, полученное из настроек или `'hypotez'` по умолчанию.

### `__version__`

**Описание**: Версия проекта, полученная из настроек или пустая строка по умолчанию.

### `__doc__`

**Описание**: Строка с документацией проекта, взятая из файла `README.MD`.

### `__details__`

**Описание**: Строка с детальной информацией о проекте. По умолчанию пустая строка.

### `__author__`

**Описание**: Автор проекта, полученный из настроек или пустая строка по умолчанию.

### `__copyright__`

**Описание**: Информация об авторских правах, полученная из настроек или пустая строка по умолчанию.

### `__cofee__`

**Описание**: Сообщение с призывом поддержать разработчика чашкой кофе, полученное из настроек или стандартное сообщение с ссылкой на boosty.
