# Модуль `hypotez/src/logger/header.py`

## Обзор

Модуль `header.py` определяет корневой путь к проекту `hypotez`. Все импорты строятся относительно этого пути. В дальнейшем, для большей гибкости, рекомендуется перенести логику определения корневого пути в системные переменные.

## Функции

### `set_project_root`

**Описание**:  Находит корневую директорию проекта, начиная с текущей директории и двигаясь вверх по дереву директорий. Останавливается на первой директории, содержащей один из указанных файлов-маркеров (например, `pyproject.toml`, `requirements.txt`, `.git`).

**Параметры**:

- `marker_files` (tuple): Кортеж имён файлов или директорий, используемых для определения корневой директории проекта.  По умолчанию: `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:

- `Path`: Путь к корневой директории проекта, если найдена. В противном случае, возвращает путь к директории, в которой находится текущий скрипт.

**Пример использования**:

```python
root_path = set_project_root()
print(root_path)
```

**Примечания**:
Добавляет корневой путь к `sys.path`, что позволяет импортировать модули из корня проекта.

### `__init__`

**Описание**:
Инициализирует модуль, находит корневой путь, загружает настройки из `settings.json`, `README.MD` и сохраняет их в глобальных переменных (project_name, version, doc, details, author, copyright, cofee).


**Возвращает**:
(Не применимо, т.к. это метод инициализации модуля)


## Глобальные переменные

### `__root__`

**Описание**:  Путь к корневой директории проекта.

**Тип**: `Path`

### `settings`

**Описание**: Словарь с настройками проекта, загруженный из `src/settings.json`.

**Тип**: `dict`

### `doc_str`

**Описание**: Содержимое файла `README.MD`.

**Тип**: `str`

### `__project_name__`

**Описание**:  Имя проекта, полученное из настроек.

**Тип**: `str`

### `__version__`

**Описание**: Версия проекта, полученная из настроек.

**Тип**: `str`

### `__doc__`

**Описание**: Документация проекта, полученная из файла `README.MD`.

**Тип**: `str`

### `__details__`

**Описание**: Подробная информация о проекте.

**Тип**: `str`

### `__author__`

**Описание**: Автор проекта, полученный из настроек.

**Тип**: `str`

### `__copyright__`

**Описание**: Авторские права на проект, полученные из настроек.

**Тип**: `str`


### `__cofee__`

**Описание**: Ссылка для поддержки разработчика.

**Тип**: `str`

**Обработка исключений**:

- `FileNotFoundError`:  Исключение, которое возникает, если файл `settings.json` или `README.MD` не найден.
- `json.JSONDecodeError`: Исключение, которое возникает, если файл `settings.json` имеет неправильный формат JSON. В обоих случаях используется `...` для указания, что обработка исключений просто пропускается.
