# Анализ кода модуля `version.py`

**Качество кода**
7
-  Плюсы
    - Присутствует описание модуля в виде docstring.
    - Определены переменные `__version__`, `__doc__`, `__details__`, `__annotations__`, `__author__`.
-  Минусы
    - Отсутствуют необходимые импорты.
    - Присутствуют избыточные комментарии, не несущие смысловой нагрузки.
    - Некорректное использование docstring (множество пустых строк и повторений).
    - Отсутствуют docstring для переменных.
    - Нарушение PEP8 (пробелы вокруг знака равенства при объявлении переменных, лишние пустые строки,  отсутствие пробелов после `:` в аннотациях типов ).
    - Значение `__version__` выглядит нелогичным.

"""
Модуль для хранения информации о версии и авторе.
=========================================================================================

Этот модуль содержит информацию о версии и авторе текущего проекта.

Пример использования:
--------------------

.. code-block:: python

    from src.webdriver.chrome.extentions import version

    print(f"Версия: {version.__version__}")
    print(f"Автор: {version.__author__}")
"""
__name__: str = __name__
"""Имя модуля."""
__version__: str = "1.0.0"
"""Текущая версия модуля."""
__doc__: str = __doc__
"""Документация модуля."""
__details__: str = "Details about version for module or class"
"""Детали о версии для модуля или класса."""
__annotations__ = {}
"""Словарь аннотаций типов."""
__author__: str = 'hypotez'
"""Автор модуля."""
