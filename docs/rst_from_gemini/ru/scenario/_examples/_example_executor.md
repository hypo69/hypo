# hypotez/src/scenario/_examples/_example_executor.py

Этот файл содержит примеры использования модуля `executor` из пакета `src.scenario.executor`. Он демонстрирует, как запускать сценарии, обрабатывать файлы сценариев и взаимодействовать с API PrestaShop.

* **Пример 1:** Запуск списка файлов сценариев.
* **Пример 2:** Запуск одного файла сценария.
* **Пример 3:** Запуск одного сценария.
* **Пример 4:** Пример выполнения сценария для страницы продукта.
* **Пример 5:** Добавление купона с использованием API PrestaShop.
* **Пример 6:** Асинхронное выполнение вставки данных в PrestaShop.

## Подробное описание примеров

**Пример 1 (Запуск списка файлов сценариев):**

Этот пример демонстрирует запуск нескольких сценариев, определенных в файлах JSON.  Он использует функцию `run_scenario_files` и проверяет результат.

**Пример 2 (Запуск одного файла сценария):**

Аналогично примеру 1, но выполняется только один файл сценария.

**Пример 3 (Запуск одного сценария):**

Этот пример показывает запуск сценария, представленного как словарь Python.  Это позволяет гибко управлять настройками сценария в коде.

**Пример 4 (Выполнение сценария для страницы продукта):**

Пример иллюстрирует получение данных с веб-страницы продукта и последующую обработку этих данных с использованием класса `ProductFields`.

**Пример 5 (Добавление купона):**

Пример демонстрирует, как использовать функцию `add_coupon` для добавления купона в PrestaShop.  Необходимо указать учетные данные API.

**Пример 6 (Асинхронная вставка данных):**

Использует `execute_prestashop_insert_async` для асинхронной вставки данных в PrestaShop.

**Пример 7 (Синхронная вставка данных):**

Использует `execute_prestashop_insert` для синхронной вставки данных в PrestaShop.

В примерах используются классы-заглушки (`MockSupplier`, `MockRelatedModules`, `MockDriver`). Это позволяет протестировать код без реального взаимодействия с внешними системами (PrestaShop, веб-сайтами).  Замените их на ваши реальные классы в рабочей версии.
