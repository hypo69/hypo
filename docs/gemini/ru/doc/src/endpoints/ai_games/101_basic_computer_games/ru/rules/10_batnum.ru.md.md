## Описание

Игра "Butnum" - это стратегическая игра, где игрок и компьютер по очереди убирают объекты из кучки. Игроки устанавливают минимальное и максимальное количество объектов для хода, а также правило выигрыша: забрать последний объект или не забрать его. Игра требует стратегического мышления и арифметики по модулю для определения оптимальных ходов.

## Пошаговая инструкция для реализации

### 1. Инициализация игры

- Начать игру с заданным количеством объектов.
- Установить минимальное и максимальное количество объектов для забора за один ход.
- Выбрать, кто ходит первым (игрок или компьютер).
- Определить правило выигрыша:
  - Побеждает тот, кто забирает последний объект.
  - Побеждает тот, кто не забирает последний объект.

### 2. Основной цикл игры

- Игрок и компьютер по очереди забирают от минимального до максимального числа объектов.
- В конце каждого хода проверить:
    - Если остался 1 объект и выбрано правило "не забрать последний", то игрок проигрывает.
    - Если объектов не осталось и выбрано правило "забрать последний", то игрок выигрывает.

### 3. Алгоритм для компьютера

- Компьютер рассчитывает ход, стараясь оставить игроку максимально невыгодное положение.
- Используется арифметика по модулю для определения оптимального хода.
- Если количество объектов кратно (M+1), компьютер стремится оставить это число.

### 4. Подсчёт победителя

- Игра заканчивается, когда остается 1 объект.
- Выводится результат в зависимости от правила выигрыша.

### 5. Завершение игры

- После завершения игры программа предлагает сыграть ещё раз.

## Пример работы программы

1. **Начало игры**:

```
Игра начинается с 20 объектов.
Введите минимальное количество объектов для забора (например, 1):
> 1
Введите максимальное количество объектов для забора (например, 3):
> 3
Кто ходит первым? (1 - игрок, 2 - компьютер): 1
```

2. **Ход игрока**:

```
Ваш ход: Сколько объектов вы хотите забрать? (1, 2 или 3)
> 2
Оставшиеся объекты: 18
Компьютер забрал 1 объект.
Оставшиеся объекты: 17
```

3. **Завершение игры**:

```
Оставшиеся объекты: 1
Вы забрали последний объект, вы проиграли.
Хотите сыграть снова? (да/нет)
> да
```

## Возможные ограничения

- Необходимо предусмотреть защиту от ввода неверных данных (например, количества объектов больше, чем осталось).
- Для предотвращения затягивания игры можно ввести лимит ходов или другие режимы игры.

## Стратегия победы

- Оставлять сопернику количество объектов, кратное (M+1), или на один меньше, чтобы он был вынужден забрать последний объект.
