## Классы

### `MockSupplier`

**Описание**: Мок-класс, имитирующий поставщика данных. Используется для тестирования функций без реального поставщика.

**Атрибуты**:
- `supplier_abs_path` (Path): Абсолютный путь к директории со сценариями.
- `scenario_files` (list[Path]): Список путей к файлам сценариев.
- `current_scenario` (None): Текущий выполняемый сценарий (изначально `None`).
- `supplier_settings` (dict): Настройки поставщика, включая список выполненных сценариев.
- `related_modules` (MockRelatedModules): Мок-объект связанных модулей.
- `driver` (MockDriver): Мок-объект драйвера.

### `MockRelatedModules`

**Описание**: Мок-класс, имитирующий связанные модули.

**Методы**:

#### `get_list_products_in_category`

**Описание**: Имитирует получение списка продуктов из категории.

**Параметры**:
- `s` (any): Параметр, представляющий категорию.

**Возвращает**:
- `list[str]`: Список URL-адресов продуктов.

#### `grab_product_page`

**Описание**: Имитирует получение данных со страницы продукта.

**Параметры**:
- `s` (any): Параметр, представляющий страницу продукта.

**Возвращает**:
- `ProductFields`: Объект с данными о продукте.

#### `grab_page`

**Описание**: Асинхронно имитирует получение данных со страницы.

**Параметры**:
- `s` (any): Параметр, представляющий страницу.

**Возвращает**:
- `ProductFields`: Объект с данными о продукте.

### `MockDriver`

**Описание**: Мок-класс, имитирующий драйвер.

**Методы**:

#### `get_url`

**Описание**: Имитирует переход по URL.

**Параметры**:
- `url` (str): URL-адрес для перехода.

**Возвращает**:
- `bool`: Возвращает `True`.
