# Модуль `hypotez/src/endpoints/kazarinov/header.py`

## Обзор

Этот модуль содержит функции для определения корневой директории проекта, загрузки настроек из файла `settings.json` и получения документации из `README.MD`.  Он также предоставляет глобальные переменные с информацией о проекте, такие как имя, версия, описание и автор.

## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная с текущей директории и поднимаясь вверх по дереву директорий, пока не обнаружит директорию, содержащую файлы-маркеры (`pyproject.toml`, `requirements.txt`, `.git`).

**Параметры**:

- `marker_files` (tuple): Кортеж имён файлов или директорий, используемых для определения корневой директории проекта. По умолчанию содержит (`'pyproject.toml'`, `'requirements.txt'`, `.git`).

**Возвращает**:

- `Path`: Путь к корневой директории проекта, если найдена, иначе - директория, в которой расположен скрипт.

**Вызывает исключения**:

- Нет


### Глобальные переменные

**Описание**: Этот модуль определяет следующие глобальные переменные, содержащие информацию о проекте:

- `__root__`: Путь к корневой директории проекта.
- `settings`: Словарь настроек, загруженный из `settings.json`.
- `doc_str`: Строка с документацией, загруженная из `README.MD`.
- `__project_name__`: Имя проекта, полученное из настроек или установленное по умолчанию.
- `__version__`: Версия проекта, полученная из настроек или установленная по умолчанию.
- `__doc__`: Документация проекта, полученная из `README.MD` или установленная по умолчанию.
- `__details__`: Дополнительная информация о проекте (по умолчанию пустая строка).
- `__author__`: Автор проекта, полученный из настроек или установленный по умолчанию.
- `__copyright__`: Авторские права на проект, полученный из настроек или установленный по умолчанию.
- `__cofee__`: Ссылка на способ поддержать разработчика (по умолчанию ссылка на Boosty).

**Примечания**:

Переменные `__root__`, `settings`, `doc_str` инициализируются при выполнении кода. Другие глобальные переменные являются результатами обработки `settings`.  Если `settings.json` или `README.MD` отсутствуют, соответствующие переменные будут установлены в значения по умолчанию. Обработка ошибок для `FileNotFoundError` и `json.JSONDecodeError` предотвращает аварийное завершение программы при отсутствии или некорректном формате файлов.
