# hypotez/src/goog/quickstart.py

**Расположение:** `C:\Users\user\Documents\repos\hypotez\src\goog\quickstart.py`

**Роль:** `doc_creator` - скрипт для создания и инициализации проекта Google Apps Script.

**Описание:**

Этот скрипт предоставляет простой способ создания нового проекта Google Apps Script, загрузки кода (helloWorld, manifest) и вывода URL проекта.

**Ключевые особенности:**

* **Авторизация:** Использует `google-auth-oauthlib` для безопасной авторизации и хранения токенов доступа.  `token.json` хранит токен.
* **Обработка ошибок:** `try...except` блок обрабатывает потенциальные ошибки при взаимодействии с API Google Apps Script.  Важный момент - вывод содержимого ошибки (`error.content`) позволяет диагностировать причину сбоя.
* **Использование констант:** `SCOPES`, `SAMPLE_CODE`, `SAMPLE_MANIFEST` улучшают читаемость и позволяют легко менять конфигурацию.
* **Обработка истечения срока действия токена:** Проверяет срок действия токена и обновляет его при необходимости.
* **Вызов API:**  Использует `googleapiclient` для взаимодействия с API.
* **Создание проекта:**  Создает новый проект Apps Script с заданным названием.
* **Загрузка файлов:** Загружает JavaScript код и manifest в созданный проект.
* **Вывод URL:**  Выводит URL проекта в консоль, позволяя пользователю получить доступ к созданному проекту.


**Использование:**

1.  **Установка зависимостей:** Убедитесь, что установлены необходимые библиотеки:
    ```bash
    pip install google-api-python-client google-auth-httplib2 google-auth-oauthlib
    ```
2.  **Ключи API:**  Создайте `credentials.json` файл, содержащий данные для доступа к API Google Apps Script (следуйте инструкциям Google Cloud Platform).  **ВАЖНО**: Не забудьте указать необходимые `SCOPES`.
3.  **Запуск:**
    ```bash
    python quickstart.py
    ```
4.  **Авторизация:**  Скрипт попросит пользователя авторизоваться в Google Accounts.

**Возможные проблемы и решения:**

* **Ошибка `HttpError`:** Проверьте правильность настроек, наличие `credentials.json` и корректность `SCOPES`.  Проанализируйте сообщение `error.content` для уточнения причины ошибки.
* **Отсутствие `token.json`:** Скрипт автоматически создаст `token.json`. Если это не происходит, обратите внимание на корректность `credentials.json` и `SCOPES`.
* **Отсутствующие зависимости:** Проверьте, что все необходимые библиотеки установлены.


**Рекомендации:**

* **Использование пакета `google-auth`: Обратите внимание, что в современной документации Google  рекомендуется использование пакета `google-auth` вместо `googleapiclient`. Это может привести к более современному и надежному коду.**

Этот документ предоставляет более подробную информацию о работе скрипта и помогает потенциальным пользователям в его использовании.
