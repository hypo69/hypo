# Модуль определения пользовательских исключений для проекта `hypotez`

## Обзор

Модуль `errors.py` содержит определения пользовательских исключений, используемых в проекте `hypotez`. Эти исключения позволяют обрабатывать специфические ошибки, возникающие в различных частях проекта, такие как ошибки, связанные с провайдерами, моделями, сетевыми запросами и другими аспектами работы.

## Подробнее

Этот модуль предоставляет набор классов исключений, которые наследуются от базового класса `Exception`. Каждое исключение предназначено для сигнализации об определенном типе ошибки, что позволяет более точно обрабатывать и регистрировать возникшие проблемы. Использование пользовательских исключений помогает улучшить читаемость и поддерживаемость кода, а также упрощает отладку.

## Классы

### `ProviderNotFoundError`

**Описание**: Исключение, которое выбрасывается, когда не удается найти указанного провайдера.

### `ProviderNotWorkingError`

**Описание**: Исключение, которое выбрасывается, когда провайдер не работает или недоступен.

### `StreamNotSupportedError`

**Описание**: Исключение, которое выбрасывается, когда потоковая передача не поддерживается.

### `ModelNotFoundError`

**Описание**: Исключение, которое выбрасывается, когда указанная модель не найдена.

### `ModelNotAllowedError`

**Описание**: Исключение, которое выбрасывается, когда модель не разрешена к использованию.

### `RetryProviderError`

**Описание**: Исключение, которое выбрасывается, когда необходимо повторить попытку использования провайдера.

### `RetryNoProviderError`

**Описание**: Исключение, которое выбрасывается, когда не осталось доступных провайдеров для повторной попытки.

### `VersionNotFoundError`

**Описание**: Исключение, которое выбрасывается, когда версия не найдена.

### `ModelNotSupportedError`

**Описание**: Исключение, которое выбрасывается, когда модель не поддерживается.

### `MissingRequirementsError`

**Описание**: Исключение, которое выбрасывается, когда отсутствуют необходимые зависимости или требования.

### `NestAsyncioError`

**Описание**: Исключение, которое выбрасывается при ошибках, связанных с `nest_asyncio`.
**Наследует**: `MissingRequirementsError`

### `MissingAuthError`

**Описание**: Исключение, которое выбрасывается, когда отсутствует аутентификация.

### `PaymentRequiredError`

**Описание**: Исключение, которое выбрасывается, когда требуется оплата.

### `NoMediaResponseError`

**Описание**: Исключение, которое выбрасывается, когда отсутствует ответ с медиа-данными.

### `ResponseError`

**Описание**: Исключение, которое выбрасывается при общих ошибках ответа.

### `ResponseStatusError`

**Описание**: Исключение, которое выбрасывается при ошибках статуса ответа.

### `RateLimitError`

**Описание**: Исключение, которое выбрасывается при превышении лимита запросов (ограничение скорости).
**Наследует**: `ResponseStatusError`

### `NoValidHarFileError`

**Описание**: Исключение, которое выбрасывается, когда файл HAR не является валидным.

### `TimeoutError`

**Описание**: Исключение, которое выбрасывается при истечении времени ожидания во время запросов к API.

### `ConversationLimitError`

**Описание**: Исключение, которое выбрасывается при достижении лимита разговоров во время запросов к API.
