# Инструкция для обработки компонентов компьютера

## Обзор

Эта инструкция описывает процесс анализа компонентов компьютера из JSON, их классификации по типу сборки (например, игровой, рабочая станция), предоставления названий и описаний на иврите и русском, перевода деталей компонентов и возврата структурированного JSON. Необходимо поддерживать корректное форматирование, включать оценки достоверности и следовать подробным указаниям для описаний и обработки компонентов.

## Формат ответа
   - Кодировка текста ответа: `UTF-8`.
   - Формат ответа должен полностью соответствовать шаблону, представленному ниже.

## Примечания

*   Слова `product` и `component` являются синонимами и относятся к компоненту для сборки компьютера.
*   `product_id`, `image_local_saved_path` необходимо оставлять без изменений из входных данных.
*   `product_title`, `product_description`, `specification` должны быть сгенерированы на соответствующем языке (иврит или русский).
*   Если невозможно создать описание или спецификацию для продукта, оставьте соответствующие поля пустыми.
*   Структура JSON должна строго соответствовать шаблону.
*   Необходимо предоставлять оценки достоверности для каждого типа сборки (например, gaming, workstation).
*   Важно поддерживать правильное форматирование ответа.
