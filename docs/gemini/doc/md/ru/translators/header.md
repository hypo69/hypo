# Модуль header.py

## Обзор

Данный модуль содержит функции для получения корневого каталога проекта и доступа к настройкам проекта, а также инициализирует переменные, содержащие информацию о проекте, такие как имя, версия, описание и автор.

## Функции

### `get_project_root`

**Описание**: Находит корневой каталог проекта, начиная с текущего каталога и проходя вверх по иерархии каталогов.

**Параметры**:
- `marker_files` (tuple): Кортеж из имён файлов или каталогов, которые используются для определения корневого каталога проекта.

**Возвращает**:
- `Path`: Путь к корневому каталогу проекта, если он найден. В противном случае возвращает путь к каталогу текущего файла.

**Вызывает исключения**:
- Нет.


## Переменные

### `__root__`

**Описание**: Путь к корневому каталогу проекта.


### `settings`

**Описание**: Словарь с настройками проекта.


### `doc_str`

**Описание**: Текст документации из файла README.


### `__project_name__`

**Описание**: Имя проекта.


### `__version__`

**Описание**: Версия проекта.


### `__doc__`

**Описание**: Описание проекта.


### `__details__`

**Описание**: Дополнительные детали о проекте.


### `__author__`

**Описание**: Автор проекта.


### `__copyright__`

**Описание**: Авторские права на проект.


### `__cofee__`

**Описание**: Ссылка на страницу сбора средств на кофе для разработчика.


## Обработка исключений

### `FileNotFoundError`, `json.JSONDecodeError`

**Описание**: Обрабатывает исключения `FileNotFoundError` и `json.JSONDecodeError`, которые могут возникнуть при чтении файла настроек (`settings.json`) или файла `README.MD`. В случае возникновения исключения переменные `settings` и `doc_str` соответственно остаются `None`
