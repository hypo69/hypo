# Модуль `product`

## Обзор

Данный модуль предоставляет инструменты для работы с продуктовой информацией. Он включает в себя классы для управления продуктами, их атрибутами (поля), локейтерами для веб-элементов и средства для управления версиями.  Модуль имеет структурированный подход к обработке данных, используя отдельный модуль `product_fields` для управления полями продуктов, обеспечивая гибкость и масштабируемость. Включает примеры использования.

## Локейтеры

### Модуль `locator`

**Описание**: Содержит определения локейтеров для веб-элементов, связанных с продуктами. Используется для взаимодействия с веб-страницами с помощью Selenium WebDriver.

## Класс `Product`

### `Product`

**Описание**: Класс для управления продуктовой информацией.

**Методы**:

* `get_product_data(product_id: str) -> dict | None`:
    **Описание**: Получает данные продукта по его идентификатору.
    **Параметры**:
        - `product_id` (str): Идентификатор продукта.
    **Возвращает**:
        - `dict | None`: Словарь с данными продукта или `None`, если продукт не найден.
    **Вызывает исключения**:
        - `ValueError`: Если передан неверный тип данных для `product_id`.
        - `ProductNotFoundError`: Если продукт не найден.


## Модуль `product_fields`

**Описание**: Модуль для управления полями и атрибутами продуктов.

**Компоненты**:
- `product_fields.py`: Определяет поля и операции над ними.
- `product_fields_default_values.json`: Файл JSON с предопределёнными значениями полей.
- `product_fields_translator.py`: Обрабатывает перевод названий и значений полей.


## Класс `ProductFields`

### `ProductFields`

**Описание**: Класс для управления полями продуктов.

**Методы**:

* `update_field(field_name: str, field_value: any) -> None`:
    **Описание**: Обновляет значение поля продукта.
    **Параметры**:
        - `field_name` (str): Название поля.
        - `field_value` (any): Новое значение поля.
    **Возвращает**:
        - `None`
    **Вызывает исключения**:
        - `ValueError`: Если передан некорректный тип данных для `field_name` или `field_value`.
        - `FieldNotFoundEx`: Если поле не найдено.


## Управление версиями

### Модуль `version`

**Описание**: Модуль для управления информацией о версии.  Содержит сведения о текущей версии модуля.

## Примеры использования

### Пример 1

```python
from product.product import Product
from product.product_fields import ProductFields

# Инициализация классов
product = Product()
product_fields = ProductFields()

# Пример получения данных продукта
product_data = product.get_product_data(product_id="12345")
if product_data:
    print(product_data)

# Пример обновления значения поля
product_fields.update_field("price", 19.99)
```

**Примечание**:  В данном примере предполагается, что классы `Product` и `ProductFields` уже определены в соответствующих модулях.  Также необходимо определить классы исключений `ProductNotFoundError` и `FieldNotFoundEx`.
```
