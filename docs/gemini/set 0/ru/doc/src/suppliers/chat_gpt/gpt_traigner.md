# Модуль `hypotez/src/suppliers/chat_gpt/gpt_traigner.py`

## Обзор

Этот модуль предоставляет классы и функции для обработки и обучения модели на основе данных из чата ChatGPT.  Он предназначен для сбора данных из файлов, определения их эмоциональной окраски и сохранения результатов в различных форматах (JSONL, CSV, текстовый).

## Классы

### `GPT_Traigner`

**Описание**: Класс `GPT_Traigner` отвечает за сбор, обработку и сохранение данных из файлов чата ChatGPT.

**Методы**:

#### `__init__`

**Описание**: Инициализирует экземпляр класса.

#### `determine_sentiment`

**Описание**: Определяет эмоциональную окраску пары диалога.

**Параметры**:

- `conversation_pair` (dict[str, str]): Пара диалога в формате словарь.
- `sentiment` (str, optional): Желаемый тип эмоции (по умолчанию 'positive').

**Возвращает**:

- str:  'positive' или 'negative' в зависимости от определения.

#### `save_conversations_to_jsonl`

**Описание**: Сохраняет пары диалогов в файл JSONL.

**Параметры**:

- `data` (list[dict]): Список словарей, каждый из которых представляет пару диалога.
- `output_file` (str): Путь к файлу для сохранения.

**Вызывает исключения**:

- `FileNotFoundError`: Если файл не найден.


#### `dump_downloaded_conversations`

**Описание**: Собрать диалоги из локальных HTML-файлов и сохранить их в CSV и JSONL.

**Возвращает**:

- None

## Модульные переменные

### `MODE`

**Описание**: Переменная `MODE` определяет режим работы. В данном случае `'dev'` (разработка).


## Использование

Модуль `gpt_traigner.py` предназначен для обработки и сохранения данных из HTML-файлов, содержащих пары диалогов ChatGPT. Он позволяет сохранять данные в CSV и JSONL формате, а также в сыром текстовом формате.  После запуска скрипта данные будут сохранены в папке, указанной в переменных среды (или в заранее заданном месте, например, google_drive). Далее, они могут быть использованы для обучения модели машинного обучения.
