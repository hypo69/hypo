# Документация для модуля `app.py`

## Обзор

Модуль `app.py` предназначен для создания и конфигурации Flask-приложения, используемого в проекте `hypotez`. Он определяет функцию `create_app`, которая настраивает приложение, указывая пути к шаблонам и статическим файлам, а также включает автоматическую перезагрузку шаблонов в режиме отладки.

## Подробней

Этот модуль важен для запуска веб-сервера и отображения пользовательского интерфейса. Расположение шаблонов и статических файлов определяется в зависимости от того, запущен ли код как "замороженное" приложение (например, с использованием PyInstaller) или в обычном режиме разработки. Это позволяет приложению находить необходимые ресурсы независимо от способа запуска.

## Функции

### `create_app`

```python
def create_app() -> Flask:
    """
    Создает и конфигурирует Flask-приложение.

    Args:
        None

    Returns:
        Flask: Объект Flask-приложения.

    Raises:
        None

    Как работает функция:
    1. Определяет, запущено ли приложение в "замороженном" состоянии (например, как исполняемый файл, созданный с помощью PyInstaller).
    2. В зависимости от этого определяет путь к папке с шаблонами: если приложение "заморожено", путь берется из атрибута `sys._MEIPASS`, иначе используется относительный путь `"../client"`.
    3. Создает экземпляр Flask-приложения с указанной папкой для шаблонов и статических файлов.
    4. Включает автоматическую перезагрузку шаблонов, установив параметр `app.config["TEMPLATES_AUTO_RELOAD"]` в `True`.
    5. Возвращает созданное Flask-приложение.

    Внутри функции происходят следующие действия и преобразования:
    A - Проверка статуса заморозки приложения
    |
    -- B - Определение пути к шаблонам в зависимости от статуса заморозки
    |
    C - Создание экземпляра Flask-приложения
    |
    D - Включение автоматической перезагрузки шаблонов
    |
    E - Возврат Flask-приложения

    Примеры:
    >>> app = create_app()
    >>> isinstance(app, Flask)
    True
    """
