# Модуль `TypeGPT`

## Обзор

Модуль `TypeGPT` предоставляет класс для взаимодействия с сервисом `chat.typegpt.net` через API OpenAI. Он наследует базовый класс `OpenaiTemplate` и предназначен для использования моделей GPT. Модуль содержит настройки для доступа к API, выбора моделей и обработки запросов.

## Подробнее

Модуль предназначен для упрощения взаимодействия с API `chat.typegpt.net`. Он определяет URL, заголовки запросов, список поддерживаемых моделей и псевдонимы моделей. Класс `TypeGPT` позволяет получать список доступных моделей и использовать их для генерации текста.

## Классы

### `TypeGPT`

**Описание**: Класс `TypeGPT` является наследником `OpenaiTemplate` и предоставляет методы для работы с API `chat.typegpt.net`.

**Принцип работы**:
Класс определяет статические переменные, такие как `label`, `url`, `api_base`, `working`, `headers`, `default_model`, `default_vision_model`, `vision_models`, `fallback_models`, `image_models` и `model_aliases`, которые используются для настройки взаимодействия с API. Метод `get_models` позволяет получить список доступных моделей.

**Методы**:

- `get_models`: Получает список доступных моделей из API `chat.typegpt.net`.

#### `get_models`

```python
@classmethod
def get_models(cls, **kwargs):
    """Получает список доступных моделей из API chat.typegpt.net.

    Args:
        **kwargs: Дополнительные аргументы (не используются).

    Returns:
        list[str]: Список доступных моделей.

    Как работает функция:
    1. Проверяет, загружены ли уже модели (cls.models).
    2. Если модели не загружены, выполняет GET-запрос к `cls.url}/api/config` для получения конфигурации.
    3. Извлекает строку с моделями из JSON-ответа (`["customModels"].split(",")`).
    4. Фильтрует и преобразует список моделей, удаляя префиксы "+" и "@", а также исключая модели изображений.
    5. Возвращает список доступных моделей.

    Примеры:
        >>> TypeGPT.get_models()
        ['gpt-4o-mini-2024-07-18', 'gpt-3.5-turbo', ...]
    """
