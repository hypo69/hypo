# Модуль `executor.py` (Selenium WebDriver)

## Описание

Модуль `executor.py` содержит класс `ExecuteLocator`, предназначенный для выполнения действий с элементами веб-страницы с помощью Selenium WebDriver. Класс работает с конфигурационными данными в формате словарей локаторов, обеспечивая навигацию и взаимодействие с веб-страницей.

## Класс `ExecuteLocator`

Класс `ExecuteLocator` предоставляет методы для работы с веб-элементами, обработки локаторов и выполнения различных действий.

### Атрибуты:

*   `driver`: Экземпляр `WebDriver`, используемый для взаимодействия с браузером.
*   `actions`: Экземпляр `ActionChains`, для сложных действий с элементами.
*   `by_mapping`: Словарь для преобразования строковых локаторов в объекты `By` из Selenium.

### Методы:

*   `__init__(self, driver, *args, **kwargs)`: Конструктор класса. Инициализирует `driver` и `actions`.
*   `execute_locator(self, locator: dict, message: str = None, typing_speed: float = 0, continue_on_error: bool = True)`: Основной метод. Выполняет действие, указанное в `locator`.  
    *   `locator`: Словарь с параметрами для выполнения.
    *   `message`: Сообщение для отправки (например, в текстовое поле).
    *   `typing_speed`: Скорость ввода текста.
    *   `continue_on_error`: Флаг, указывающий, нужно ли продолжать выполнение после ошибки.
*   `get_webelement_by_locator(self, locator: dict | SimpleNamespace, message: str = None)`: Возвращает веб-элемент(ы) по заданному локатору.
*   `get_attribute_by_locator(self, locator: dict | SimpleNamespace, message: str = None)`: Возвращает значение атрибута веб-элемента по заданному локатору.
*   `_get_element_attribute(self, element: WebElement, attribute: str)`: Вспомогательный метод для получения значения атрибута заданного элемента.
*   `send_message(self, locator: dict | SimpleNamespace, message: str, typing_speed: float, continue_on_error:bool)`: Отправляет сообщение в веб-элемент.
*   `evaluate_locator(self, attribute: str | list | dict)`: Оценивает атрибут локатора, вероятно, для определения типа поиска.
*   `_evaluate(self, attribute: str)`: Вспомогательный метод для оценки одного атрибута локатора.
*   `get_locator_keys()`: Возвращает список доступных ключей в словаре локатора.

### Обработка Локаторов (`locator`):

Локаторы передаются в виде словарей, содержащих информацию для поиска и взаимодействия с элементами.  Важно, что словарь `locator` должен соответствовать определённому формату, который определяет как искать элемент. Примеры таких локаторов предоставлены в документе.  Ключевые параметры локаторов:

* `attribute`: Атрибут, по которому будет производиться поиск.
* `by`: Тип селектора (например, `xpath`, `id`, `name`).
* `selector`:  Селектор, используемый для поиска элемента (xpath, css-селектор).


## Использование

Класс `ExecuteLocator` используется для управления взаимодействием с веб-сайтом.  Он предоставляет методы для поиска элементов, получения их атрибутов и отправки данных.  Применение класса предполагает подготовку локаторов (`locator`) в виде словарей.

## Важные Модули и Библиотеки

Класс использует Selenium WebDriver для взаимодействия с браузером, а также собственные модули для логирования, обработки ошибок и работы с конфигурацией (например, `gs`, `logger`).


**Важно:**  Документация содержит примеры локаторов.  Для корректной работы необходимо убедиться, что данные локаторов соответствуют требуемому формату, чтобы код мог их корректно обработать.
