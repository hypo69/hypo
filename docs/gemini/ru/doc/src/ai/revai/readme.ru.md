# Модуль revai

## Обзор

Этот модуль предоставляет инструменты для работы с API сервиса rev.com, специализирующегося на обработке аудиофайлов (переговоры, совещания, звонки и т.п.). Он позволяет взаимодействовать с API для различных задач, таких как распознавание речи, извлечение информации и т.д.  Модуль основан на документации API rev.ai.

## Функции

### `get_transcript`

**Описание**: Функция для получения транскрипта аудиофайла.

**Параметры**:
- `audio_file_path` (str): Путь к аудиофайлу.
- `rev_api_key` (str): Ключ API сервиса rev.com.
- `language_code` (str, optional): Код языка аудиофайла (по умолчанию - автоматическое определение).
- `channel_name` (str, optional): Имя канала или источника аудиофайла.


**Возвращает**:
- `dict | None`: Словарь с данными транскрипта (текст, временные метки, ошибки) или None в случае ошибки.

**Вызывает исключения**:
- `APIError`: В случае ошибки при обращении к API rev.com.
- `FileNotFoundError`: Если указанный аудиофайл не найден.
- `ValueError`: Если передан некорректный формат данных.


### `get_insights`

**Описание**: Функция для извлечения инсайтов из аудиофайла (например, тегирование ключевых слов, распознавание эмоций и т.д.).

**Параметры**:
- `transcript` (str): Текст транскрипта аудиофайла (полученный с помощью функции `get_transcript`).
- `insights_types` (list[str], optional): Список типов инсайтов, которые нужно извлечь (например, `["keywords", "emotions"]`). По умолчанию извлекаются все доступные типы.


**Возвращает**:
- `dict | None`: Словарь с результатами анализа (например, словарь с ключевыми словами или эмоциями) или None в случае ошибки.


**Вызывает исключения**:
- `ValueError`: Если передан некорректный формат данных.
