# Модуль hypotez/src/logger/header.py

## Обзор

Этот модуль отвечает за определение корневого пути к проекту `hypotez`.  Он ищет директорию, содержащую файлы `pyproject.toml`, `requirements.txt` или `.git`, начиная от текущего файла и поднимаясь вверх по иерархии директорий.  Найденный корневой путь добавляется в системный путь `sys.path`, что позволяет импортировать модули из других директорий проекта. Модуль также загружает настройки из файла `settings.json` и документ из `README.MD` в переменные для дальнейшего использования.

## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная от текущего файла и двигаясь вверх по дереву директорий.

**Параметры**:
- `marker_files` (tuple): Кортеж строк, представляющих имена файлов или директорий, используемых для определения корневой директории проекта (по умолчанию: `('pyproject.toml', 'requirements.txt', '.git')`).

**Возвращает**:
- `Path`: Путь к корневой директории проекта, если она найдена, иначе — путь к директории, где расположен текущий скрипт.

**Обрабатывает исключения**:
- Нет.

###  `set_project_root`

**Описание**: Эта функция ищет корневую папку проекта, начиная от текущего файла. Она последовательно проверяет родительские директории, пока не найдет директорию, содержащую один из указанных маркеров (файлов или папок). Если такая директория найдена, функция возвращает её путь. В противном случае, она возвращает путь к текущей директории.

**Параметры**:
- `marker_files`: Кортеж строк, представляющих имена файлов или директорий, которые указывают на корень проекта. По умолчанию (`('pyproject.toml', 'requirements.txt', '.git')`)

**Возвращает**:
- `Path`: Путь к корневой директории проекта.

**Обрабатывает исключения**:
- Нет.


## Переменные

### `__root__`

**Описание**: Содержит путь к корневой директории проекта, полученный с помощью функции `set_project_root()`.


## Дополнительные сведения

### Загрузка настроек

Модуль загружает настройки из файла `settings.json` в переменную `settings`.

### Чтение README

Модуль пытается прочитать файл `README.MD` и сохранить его содержимое в переменную `doc_str`.

### Переменные проекта

- `__project_name__`: Название проекта, полученное из `settings.json`. По умолчанию `hypotez`.
- `__version__`: Версия проекта, полученная из `settings.json`. По умолчанию пустая строка.
- `__doc__`: Содержимое файла `README.MD` (если он существует). По умолчанию пустая строка.
- `__details__`: Пустая строка (без использования).
- `__author__`: Автор проекта, полученный из `settings.json`. По умолчанию пустая строка.
- `__copyright__`: Авторские права, полученные из `settings.json`. По умолчанию пустая строка.
- `__cofee__`: Ссылка на платформа для сбора средств на развитие проекта. По умолчанию ссылка на boosty.to/hypo69.

## Обработка ошибок

Модуль содержит блоки `try...except` для обработки `FileNotFoundError` и `json.JSONDecodeError` при чтении файлов `settings.json` и `README.MD`. В случае ошибки, соответствующие переменные остаются `None` или пустыми.
