## Предисловие

В предисловии указывается, что это не первая коллекция компьютерных игр, но она уникальна тем, что содержит полные листинги, примеры запусков и описания каждой игры. 

Также обсуждается образовательная ценность игр, мотивация, обучение через открытия и вопросы, связанные с использованием компьютерного времени. Подчеркивается важность не только игры в игры, но и их создания. Упоминаются популярные категории игр и их вариации. Книга включает 108 игр, хотя в названии указано 101, так как некоторые игры являются разными версиями других.

## Описание игр

### ACEYDU

**Описание**: Симуляция карточной игры Acey Ducey. Компьютер сдает две карты, и игрок решает, ставить ли деньги, в зависимости от того, будет ли следующая карта между значениями первых двух. Игра продолжается, пока у игрока есть деньги или пока он не прервет программу.

**Параметры**:

-   `M`: Ставка игрока.
-   `A`, `B`: Значения первых двух карт.
-   `C`: Значение третьей карты.

### AMAZIN

**Описание**: Программа генерирует лабиринт с единственным путем, позволяя игроку выбирать его размеры (ширину и длину). Максимальный размер лабиринта зависит от доступной памяти.

**Параметры**:

-   `H`: Ширина лабиринта.
-   `Y`: Длина лабиринта.
-   `R`, `S`: Координаты текущей позиции в лабиринте.
-   `W`: Массив для представления лабиринта.

### ANIMAL

**Описание**: Компьютер пытается угадать загаданное игроком животное, задавая вопросы. При неправильном угадывании компьютер запрашивает новый вопрос для обучения, "запоминая" новые животные. Дополнительные животные и вопросы сохраняются в файле "ANIMAL.GME."

**Параметры**:

-   `Z9$`: Ответ игрока (YES или NO).
-   `Z7$`: Ответ на вопрос компьютера, отличающий животное.
-   `V29$`: Название загаданного животного.
-   `Z8$`: Вопрос, позволяющий отличить загаданное животное.
-   `AS`: Массив для хранения вопросов и ответов.

### AWARI

**Описание**: Компьютерная версия древней настольной игры Авари, где игроки собирают бобы из ячеек на доске. Основная цель – собрать больше бобов в своем "доме". Компьютер улучшает свою игру, обучаясь на прошлых играх.

**Параметры**:

-   `B`: Массив, представляющий ячейки на доске.
-   `M`: Выбранная игроком ячейка для хода.
-   `C`: Счетчик ходов.
-   `K`: Переменная для подсчета бобов.

### BAGLES

**Описание**: Классическая игра, где компьютер загадывает трехзначное число, а игрок пытается его угадать. Компьютер даёт подсказки: "PICO" (верная цифра не на своём месте), "FERMI" (верная цифра на своём месте) или "BAGLES" (нет верных цифр).

**Параметры**:

-   `A`: Массив, представляющий загаданное число.
-   `B`: Массив, представляющий число, предложенное игроком.
-   `C`: Счетчик "FERMI".
-   `D`: Счетчик "PICO".

### BANNER

**Описание**: Программа для создания текстовых баннеров, которые могут выводиться на печать с заданными параметрами размера и отступа. Использует строку, введенную пользователем, для формирования баннера.

**Параметры**:

-   `H`: Высота баннера в дюймах.
-   `W`: Ширина баннера в дюймах.
-   `S`: Отступ от левого края в дюймах.
-   `A`: Массив символов для печати.
-   `GS`: Массив графических символов.

### BASBAL

**Описание**: Симуляция бейсбольного матча между двумя командами, где игрок управляет питчером и отбивающим в разных игровых ситуациях. Программа состоит из двух частей: одна для вывода правил, вторая - для самой игры.

**Параметры**:

-   `T`, `C`, `F`, `Z`: Массивы данных для статистики игроков.
-   `A`, `B`: Массивы для хранения введенных значений.
-   `I`, `X`, `D`: Переменные для учета раундов.

### BASKET

**Описание**: Симуляция баскетбольного матча между командой Dartmouth College и командой, выбранной игроком. Игрок контролирует тип броска и защиты. Игра немного предвзята в пользу Dartmouth.

**Параметры**:

-   `S`: Массив для хранения очков команд.
-   `D`: Выбранная игроком защита (PRESS, MAN-TO-MAN, ZONE, NONE).
-   `Z`: Выбранный игроком бросок (LONG JUMP SHOT, SHORT JUMP SHOT, LAY UP, SET SHOT).
-   `T`: Время игры.

### BATNUM

**Описание**: Игра, где игрок и компьютер поочередно убирают предметы из кучи. Игроки заранее устанавливают минимальное и максимальное количество убираемых предметов и условие победы: взятие последнего предмета или его избежание.

**Параметры**:

-   `N`: Начальное количество объектов в куче.
-   `M`: Условие победы (1 - взять последний, 2 - не брать последний).
-   `A`, `B`: Минимальное и максимальное количество предметов для хода.
-   `S`: Определение первого хода (1 - компьютер, 2 - игрок).
-   `P`: Количество убираемых предметов.

### BATTLE

**Описание**: Симуляция морского сражения, где игрок должен потопить флот противника, расположенный на игровом поле размером 6x6. Игрок вводит координаты для обстрела, а компьютер показывает результаты. Задача игрока заключается в том, чтобы понять принцип шифрования расположения флота компьютера.

**Параметры**:

-   `F`: Массив, представляющий расположение кораблей противника.
-   `H`: Массив, используемый для хранения информации о собственных ударах.
-   `A`, `B`, `C`, `L`: Переменные для хранения данных о координатах и типе кораблей.

### BINGO

**Описание**: Симуляция игры в бинго между пользователем и компьютером. Программа генерирует карты бинго, а затем случайным образом вытягивает номера.

**Параметры**:

-   `B`: Массив для хранения чисел на карте бинго.
-   `L`: Массив для проверки вытянутых чисел.
-   `M`: Множитель для генерации случайного числа.
-   `W`, `V`: Флаги для проверки выигрыша (вертикаль и диагональ).

### BLK J A C

**Описание**: Симуляция игры в блэкджек (21) по правилам Лас-Вегаса. Игрок может разделять руку, делать страховку при тузе дилера и удваивать ставку. Игра заканчивается, когда у игрока заканчиваются деньги. Программа может перезагрузить колоду в любой момент.

**Параметры**:

-   `D`: Массив, представляющий колоду карт.
-   `E`, `V`, `T`: Массивы для хранения карт, их значений и сумм.
-   `W`: Текущая ставка.
-   `K`:  Условный флаг для запроса пересдачи карт.
-  `A`, `B`: Параметры для обработки действий игрока.

### BOAT

**Описание**: Игра, в которой игрок управляет подводной лодкой и должен потопить канонерскую лодку, управляемую компьютером. Игрок должен нанести два точных попадания, либо один точный и два неточных, после чего начинается перестрелка до уничтожения одной из лодок. Компьютер улучшает игру, обучаясь на прошлых играх.

**Параметры**:

-   `B`: Массив, используемый для хранения данных о положении и перемещении кораблей.
-   `G`: Угол для выстрела торпеды.
-   `C`: Количество попаданий в лодку компьютера.
-   `S`: Расстояние от лодки компьютера до подлодки игрока.
-   `W`: Расстояние от выпущенной торпеды до цели.

### BOMBER

**Описание**: Симуляция полета бомбардировщика во время Второй Мировой войны. Игрок выбирает сторону, тип самолета и цель. В зависимости от характеристик самолета, опыта пилота и типа противника, игрок может успешно выполнить миссию, потерпеть крушение или вернуться домой.

**Параметры**:

-   `A`: Сторона, за которую играет игрок.
-   `B`: Цель миссии.
-   `G`: Тип самолета.
-   `D`: Опыт игрока.
-   `R`: Тип обороны противника.
-   `S`: Сила ПВО противника.

### BOUNCE

**Описание**: Программа для построения графика движения подпрыгивающего мяча, показывающая траекторию его движения. Игрок задает начальную скорость мяча, коэффициент упругости и шаг времени для вывода графика.

**Параметры**:

-   `V`: Начальная скорость мяча.
-   `C`: Коэффициент упругости.
-   `S`: Шаг времени для отслеживания движения.
-   `T`: Массив для хранения промежуточных значений.
-   `H`: Высота траектории.
-   `L`: Позиция мяча на оси X.

### BOWL

**Описание**: Симуляция боулинга, позволяющая от одного до четырех игроков играть 10 фреймов. Игрок вводит "ROLL", а компьютер отображает состояние кеглей и анализирует результаты броска.

**Параметры**:

-   `B`: Массив для представления кеглей.
-   `F`: Массив для записи результатов игры.
-   `P`: Количество игроков.
-   `M`: Флаг для следующего ролла.
-   `Q`: Флаг для определения типа броска.

### BOXING

**Описание**: Симуляция трех-раундового боксерского поединка. Игрок контролирует действия своего боксера (удар и защита), в то время как компьютер управляет действиями противника. Игрок определяет сильные стороны своего боксера и его уязвимость.

**Параметры**:

-   `B`: Лучший удар игрока.
-   `D`: Уязвимость игрока.
-   `X`: Флаг нанесенного удара.
-   `P`: Код удара.

### BUG

**Описание**: Игрок и компьютер поочередно "собирают жука", бросая виртуальный кубик. Каждое выпавшее число соответствует определённой части жука. Побеждает тот, кто первым соберёт все части.

**Параметры**:

-   `A`, `B`, `H`, `N`, `P`, `Q`, `R`, `S`, `T`, `L`, `V`, `Y`: Переменные, представляющие наличие частей тела жука.
-  `Z`: Результат броска кубика.

### BULCOW

**Описание**: Эта игра является расширенной версией игры BAGLES, где игроки пытаются угадать 5-значное число. Игрок получает подсказки: "БЫК" за правильную цифру на правильной позиции и "КОВА" за правильную цифру, но не на своей позиции.

**Параметры**:

-   `A`: Массив, представляющий загаданное число.
-   `B`: Массив, представляющий число, предложенное игроком.
-   `C`: Счетчик "БЫК".
-   `G`: Счетчик "КОВА".

### BULEYE

**Описание**: Игра, где игрок бросает дротики в мишень с четырьмя зонами (10, 20, 30 и 40 очков). Задача – набрать 200 очков. Игрок выбирает один из трех способов бросания, каждый из которых имеет свои шансы на успех.

**Параметры**:

-   `S`: Массив для хранения очков игроков.
-   `T`: Выбор способа броска (1, 2 или 3).

### BULL

**Описание**: Симуляция корриды, где игрок выступает в роли матадора и должен убить быка, применяя различные движения с плащом и шпагой. Результат зависит от храбрости матадора и мастерства его помощников.

**Параметры**:

-   `A`: Тип быка.
-   `M`: Определение движения матадора с плащом.
-   `F`: Оценка удачности хода матадора.
-   `D`: Состояние помощников матадора.

### BUNNY

**Описание**: Программа для отображения рисунка зайчика из журнала Playboy.

### BUZZWD

**Описание**: Генератор «модных слов», составляющий фразы из трех слов, которые часто используются в деловых и технологических разговорах. Слова выбираются случайным образом из предопределенных списков.

**Параметры**:

- `N`, `M`, `P`: Три числа, которые управляют выбором слов.

### CALNDR

**Описание**: Программа для вывода календаря на любой год. Пользователь задает день недели начала года, а также необходимо внести исправления в программу для високосного года.

**Параметры**:

-   `D`: День недели начала года (0 - воскресенье, -1 - понедельник и т.д.).
-   `T`: Массив для хранения количества дней в каждом месяце.
-   `Y`: Год.
-   `J`: Номер месяца в цикле.

### CAN-AM

**Описание**: Симуляция автомобильной гонки серии Can-Am на трассе длиной 5.3 мили. Игрок участвует в гонке, выбирая скорость на разных участках трассы.

### CHANGE

**Описание**: Программа-имитация кассира, рассчитывающая сдачу в долларах и центах. Игрок вводит стоимость товара и сумму оплаты, а программа выдает сдачу.

**Параметры**:

-   `A`: Стоимость товара.
-   `P`: Сумма, данная покупателем.
-  `C`: Сдача.
-   `D`, `E`, `F`, `N`, `G`, `H`, `I`, `J`, `K`: Количество купюр и монет разного достоинства.

### CHECKR

**Описание**: Игра в шашки, где игрок играет против компьютера. Игрок перемещает шашки, указывая начальные и конечные координаты шашки. Компьютер делает свои ходы автоматически.

**Параметры**:

-   `S`: Массив, представляющий доску шашек.
-   `L`: Переменная, определяющая чей ход.
-   `X`, `Y`: Координаты начальной позиции.
-   `U`, `V`: Координаты конечной позиции.

### CHEMST

**Описание**: Игра, где игрок должен правильно разбавить криптоциановую кислоту, используя соотношение 7 частей воды к 3 частям кислоты. При несоблюдении пропорции кислота взрывается, и игрок теряет одну из 9 жизней.

**Параметры**:

-   `A`: Количество кислоты.
-   `W`: Количество воды.
-   `R`: Ввод игрока количества воды.
-   `D`: Разница между требуемым количеством воды и количеством, введенным игроком.
-   `T`: Количество потерянных жизней.

### CHIEF

**Описание**: Программа для тренировки навыков устного счета (арифметических действий) в увлекательной игровой форме. Компьютер предлагает арифметические задачи, а игрок должен их решить. Если игрок ошибается, то программа показывает правильное решение.

**Параметры**:

-   `B`: Произвольное число игрока.
-   `C`: Результат вычислений компьютера.
-   `F`, `G`, `H`, `I`, `J`: Промежуточные переменные для вычислений.
