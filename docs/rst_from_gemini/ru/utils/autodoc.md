# autodoc.py: Модуль для автоматического обновления docstring функций

Этот модуль предоставляет декоратор `autodoc`, который автоматически обновляет docstring функций с указанием времени последнего вызова.

## Модуль `src.utils`

Константа `MODE` установлена в `'debug'`.


## Функция `autodoc(func)`

Декоратор, который автоматически обновляет docstring функции перед ее вызовом.  Он использует декоратор `functools.wraps`, чтобы сохранить оригинальные метаданные функции (например, имя, аргументы).

**Аргументы:**

* `func`: Функция, к которой применяется декоратор.

**Возвращаемое значение:**

Функция с обновленным docstring.


## Функция `update_docstring(func)`

Функция, обновляющая docstring функции.

**Аргументы:**

* `func`: Функция, docstring которой необходимо обновить.

**Действия:**

1. Получает текущую дату и время.
2. Проверяет, существует ли у функции docstring.
3. Если docstring существует, добавляет информацию о последнем вызове в конец docstring.
4. Если docstring не существует, устанавливает его с информацией о последнем вызове.


## Пример использования

```python
@autodoc
def example_function(param1: int, param2: str) -> None:
    """Пример функции.

    Args:
        param1 (int): Первое значение.
        param2 (str): Второе значение.
    """
    print(f"Processing {param1} and {param2}")

# Тестирование функции
example_function(1, "test")
print(example_function.__doc__)  # Вывод обновленного docstring
example_function(2, "another test")
print(example_function.__doc__)  # Вывод обновленного docstring
```

Этот пример демонстрирует, как использовать декоратор `autodoc`. После каждого вызова функции `example_function` ее docstring будет обновляться с указанием времени последнего вызова.

**Важно:**

* Модуль `src.utils` не имеет описания в этом модуле.
* При использовании в проекте убедитесь, что у функций, к которым вы применяете декоратор, есть соответствующие типы возвращаемых значений и аргументов, как в примере.
* Декоратор `autodoc` предназначен для добавления дополнительной информации в docstring;  не полагайтесь на него для корректности функциональности вашей программы.

**Рекомендации:**

* Рассмотрите возможность добавления обработки исключений в `update_docstring`, чтобы предотвратить ошибки при отсутствии docstring.
* Для лучшей читаемости, можно добавить комментарии к отдельным блокам кода, поясняя их логику.
* Принимая во внимание возможность многочисленных вызовов, в будущем можно добавить логику, чтобы предотвратить дублирование сообщений о времени последнего вызова в docstring, или чтобы хранить время последнего вызова в переменной для обновления.
