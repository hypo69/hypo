# Модуль schemas.py

## Обзор

Модуль `schemas.py` содержит Pydantic модели, используемые для валидации и сериализации данных, связанных с пользователями Telegram, продуктами и платежами в контексте цифрового рынка. Эти схемы определяют структуру данных, ожидаемую при взаимодействии с Telegram ботом.

## Подробней

Модуль содержит схемы для идентификации пользователей Telegram, продуктов и категорий продуктов, а также для хранения данных об оплате. Схемы используются для обеспечения типизации данных и упрощения обработки информации, передаваемой между ботом и другими компонентами системы. Pydantic используется для автоматической валидации данных и преобразования типов.

## Классы

### `TelegramIDModel`

**Описание**: Базовая модель для идентификации пользователя Telegram по его уникальному ID.

**Принцип работы**:
Класс `TelegramIDModel` наследуется от `BaseModel` из библиотеки `pydantic` и определяет поле `telegram_id` как целое число. `model_config = ConfigDict(from_attributes=True)` позволяет создавать экземпляры модели из атрибутов объекта.

**Аттрибуты**:
- `telegram_id` (int): Уникальный идентификатор пользователя в Telegram.

### `UserModel`

**Описание**: Модель пользователя Telegram, расширяющая `TelegramIDModel` дополнительными данными, такими как имя пользователя, имя и фамилия.

**Принцип работы**:
Класс `UserModel` наследуется от `TelegramIDModel` и добавляет поля для хранения имени пользователя, имени и фамилии. `model_config = ConfigDict(from_attributes=True)` позволяет создавать экземпляры модели из атрибутов объекта.

**Аттрибуты**:
- `username` (str | None): Имя пользователя в Telegram (может отсутствовать).
- `first_name` (str | None): Имя пользователя (может отсутствовать).
- `last_name` (str | None): Фамилия пользователя (может отсутствовать).

### `ProductIDModel`

**Описание**: Модель для идентификации продукта по его уникальному ID.

**Принцип работы**:
Класс `ProductIDModel` наследуется от `BaseModel` из библиотеки `pydantic` и определяет поле `id` как целое число.

**Аттрибуты**:
- `id` (int): Уникальный идентификатор продукта.

### `ProductCategoryIDModel`

**Описание**: Модель для идентификации категории продукта по её уникальному ID.

**Принцип работы**:
Класс `ProductCategoryIDModel` наследуется от `BaseModel` из библиотеки `pydantic` и определяет поле `category_id` как целое число.

**Аттрибуты**:
- `category_id` (int): Уникальный идентификатор категории продукта.

### `PaymentData`

**Описание**: Модель данных об оплате, содержащая информацию о пользователе, платеже, товаре и типе оплаты.

**Принцип работы**:
Класс `PaymentData` наследуется от `BaseModel` из библиотеки `pydantic` и определяет поля для хранения информации об оплате, включая ID пользователя, ID платежа, цену, ID товара и тип оплаты. Использование `Field` позволяет задавать дополнительные параметры для каждого поля, такие как описание и максимальная длина.

**Аттрибуты**:
- `user_id` (int): ID пользователя Telegram.
- `payment_id` (str): Уникальный ID платежа (максимальная длина 255 символов).
- `price` (int): Сумма платежа в рублях.
- `product_id` (int): ID товара.
- `payment_type` (str): Тип оплаты.
