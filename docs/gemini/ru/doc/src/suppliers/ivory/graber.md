# Модуль `hypotez/src/suppliers/ivory/graber.py`

## Обзор

Модуль `hypotez/src/suppliers/ivory/graber.py` содержит класс `Graber`, предназначенный для сбора данных о товарах с сайта `ivory.co.il`.  Этот класс наследуется от `Grbr` (класс `Graber` из родительского модуля). Класс предоставляет асинхронную функцию `grab_page` для извлечения данных о товаре,  поля которого извлекаются через отдельные функции, реализованные в классе.  Для каждого поля товара есть соответствующая функция, которая перегружается в этом классе, если требуется нестандартная обработка.  Возможность предварительной обработки данных перед запросом к веб-драйверу реализована с помощью декоратора `@close_pop_up`.

## Классы

### `Graber`

**Описание**: Класс для сбора данных о товарах с сайта `ivory.co.il`. Наследуется от `Graber` из родительского модуля.

**Методы**:

- `__init__(self, driver: Driver)`:
    **Описание**: Инициализирует экземпляр класса.
    **Параметры**:
        - `driver` (Driver): Экземпляр драйвера для взаимодействия с веб-страницей.
    **Возвращает**:  None

- `grab_page(self, driver: Driver) -> ProductFields`:
    **Описание**: Асинхронная функция для сбора данных о товаре.
    **Параметры**:
        - `driver` (Driver): Экземпляр драйвера.
    **Возвращает**:
        - `ProductFields`: Объект с собранными данными о товаре.
    **Вызывает исключения**:
        - Возможны исключения, связанные с взаимодействием с веб-драйвером.

**Возвращает**:
    - `Callable`: Декоратор, оборачивающий функцию.

### `fetch_all_data(**kwards)`
**Описание**: Функция для вызова других функций обработки данных. 
**Параметры**:
    - `**kwards` (dict): Словарь аргументов.  Ключи соответствуют именам функций, например, `id_product`, `description`,  и значениями, переданными в качестве аргументов этих функций.
