# Руководство для Тестера класса DriverBase

## Обзор

Данное руководство описывает процесс тестирования класса `DriverBase` с использованием фреймворка `pytest`. Оно содержит информацию о необходимых предварительных условиях, структуре проекта, настройке окружения, написании и запуске тестов, а также интерпретации результатов.

## Предварительные условия

Перед началом тестирования необходимо убедиться в наличии следующих компонентов:

* **Python 3.12:**  Убедитесь, что установлена версия Python 3.12.  Проверить версию можно командой `python --version`.
* **pytest:** Установите `pytest`, если он еще не установлен:
    ```bash
    pip install pytest
    ```
* **unittest.mock:** Библиотека `unittest.mock` входит в стандартную библиотеку Python, начиная с версии 3.3. Она не требует дополнительной установки.

## Написание и запуск тестов

### Написание тестов

Тесты для класса `DriverBase` находятся в файле `tests/test_driver.py`. Тесты должны соответствовать принципам написания тестов с использованием `pytest`. Примеры тестов покрывают методы `driver_payload`, `scroll`, `locale` и другие.

### Запуск тестов

Для запуска тестов выполните следующую команду в корневой директории проекта:

```bash
pytest tests/test_driver.py
```

### Интерпретация результатов

После запуска тестов `pytest` выведет отчет о результатах. Пример вывода:

```
============================== test session starts ==============================
platform linux -- Python 3.12.0, pytest-7.0.0, pluggy-0.13.1
rootdir: /path/to/your/project
collected 9 items

tests/test_driver.py ........                                        [100%]

=============================== 9 passed in 1.23s ================================
```

Этот вывод показывает количество пройденных тестов и общее время выполнения. Если какой-либо тест не прошел, `pytest` предоставит детали ошибки (трейсбэк).


## Полезные команды

* **Запуск всех тестов:**
    ```bash
    pytest
    ```
* **Запуск тестов с подробным выводом:**
    ```bash
    pytest -v
    ```
* **Запуск тестов с отчетом о покрытии кода:**
    1. Установите `pytest-cov`:
    ```bash
    pip install pytest-cov
    ```
    2. Запустите тесты с отчетом о покрытии:
    ```bash
    pytest --cov=src tests/
    ```
