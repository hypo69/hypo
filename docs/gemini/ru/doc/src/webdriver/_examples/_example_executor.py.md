# Модуль `_example_executor.py`

## Обзор

Данный модуль представляет собой пример использования класса `ExecuteLocator` для взаимодействия с веб-страницей с помощью Selenium WebDriver. Он демонстрирует различные способы использования локаторов, включая простые и сложные, отправку сообщений, обработку ошибок, работу со списками локаторов и извлечение значений атрибутов.

## Импорты

Модуль импортирует следующие библиотеки:

- `selenium.webdriver`: для управления браузером.
- `src.webdriver.executor.ExecuteLocator`: для выполнения локаторов на веб-странице.
- `src.settings.gs`: для доступа к глобальным настройкам.
- `src.logger.exceptions.ExecuteLocatorException`: для обработки исключений, возникающих при выполнении локаторов.

## Функции

### `main`

**Описание**: Главная функция, которая демонстрирует использование класса `ExecuteLocator` для выполнения различных операций на веб-странице.

**Параметры**:
- Нет.

**Возвращает**:
- Нет.

**Вызывает исключения**:
- `ExecuteLocatorException`: Вызывается при ошибке во время выполнения локатора.

```python
def main():
    """
    Args:
        None

    Returns:
        None

    Raises:
        ExecuteLocatorException: При ошибке во время выполнения локатора.
    """
```
   
**Описание**:

1. **Создание экземпляра WebDriver**:
   - Инициализируется `webdriver.Chrome` с указанием пути к исполняемому файлу драйвера Chrome, который берется из глобальных настроек `gs`.
   - Выполняется переход на страницу `https://example.com`.
   
2. **Создание экземпляра ExecuteLocator**:
   - Создается экземпляр класса `ExecuteLocator`, которому передается драйвер `driver` для взаимодействия с веб-страницей.
  
3. **Пример простого локатора**:
   - Демонстрируется создание и использование простого локатора для получения текста из элемента `<h1>` на странице.
   - Локатор содержит:
     - `by`: "XPATH" (метод поиска элемента).
     - `selector`: "//h1" (XPath для поиска элемента `<h1>`).
     - `attribute`: "textContent" (атрибут для извлечения).
     - `timeout`: 0 (таймаут ожидания элемента).
     - `timeout_for_event`: "presence_of_element_located" (событие для ожидания).
     - `event`: None (событие для выполнения).
     - `if_list`: "first" (выбор первого элемента из списка).
     - `use_mouse`: False (использование мыши не требуется).
     - `mandatory`: True (обязательный локатор).
     - `locator_description`: "Getting the page title" (описание локатора).
   - Вызывается метод `execute_locator` для получения результата.
 
4. **Пример сложного локатора**:
    - Демонстрируется создание и использование сложного локатора, включающего несколько шагов:
      - Получение ссылок на товары.
      - Клик на пагинацию.
      - Клик на кнопку "Следующая страница".
   -  Вызывается метод `execute_locator` для выполнения серии действий с локаторами.
    
5. **Пример обработки ошибок**:
    - Показывается, как можно обработать ошибки, которые могут возникнуть при выполнении локаторов.
    - `execute_locator` вызывается с параметром `continue_on_error=True`, что позволяет продолжить выполнение скрипта, даже если какой-либо из локаторов не будет найден.
    - Ошибки, если возникнут, перехватываются и выводятся в консоль.

6. **Пример использования `send_message`**:
   - Демонстрируется использование метода `send_message` для отправки сообщения в поле ввода.
   - Локатор для поля поиска на странице (`//input[@name=\'search\']`) и текст сообщения `message` передаются в метод `send_message`,  который имитирует ввод текста в поле поиска.
   - Опционально задается скорость печати `typing_speed=0.05` и `continue_on_error=True`, позволяя продолжить выполнение скрипта, если элемент не будет найден.
  
7. **Пример использования списка локаторов**:
   - Демонстрируется выполнение нескольких локаторов, каждый из которых имеет свои параметры.
   - Создается список локаторов с разными параметрами (метод поиска, селектор, атрибут, событие и т.д.).
   - Вызывается метод `execute_locator`, чтобы выполнить все локаторы последовательно.
   - Результаты выполнения всех локаторов выводятся в консоль.
   
8.  **Пример использования `evaluate_locator`**:
   - Демонстрируется извлечение значения атрибута из веб-элемента с помощью метода `evaluate_locator`.
   - Создается локатор для мета-описания страницы.
   - Вызывается метод `evaluate_locator`, для получения значения атрибута "content" мета-тега.
   
9. **Пример обработки исключений**:
   - Демонстрируется, как можно обрабатывать исключения, которые могут возникнуть при выполнении локаторов, используя блок `try-except`.
    - При возникновении ошибки в процессе выполнения локатора, исключение `ExecuteLocatorException` перехватывается и выводится в консоль.
  
10. **Пример полного теста**:
    - Демонстрируется выполнение локатора в рамках полного тестового сценария.
    - Локатор для заголовка страницы (`//h1`) и вызывается метод `execute_locator`, для получения значения заголовка.
    
11. **Завершение работы драйвера**:
    - В конце работы драйвер закрывается с помощью метода `driver.quit()`.

12. **Запуск главной функции**:
    - Проверяется условие `if __name__ == "__main__":`, которое запускает функцию `main`, если скрипт запускается напрямую, а не импортируется как модуль.
