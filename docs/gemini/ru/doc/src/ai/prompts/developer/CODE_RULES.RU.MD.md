# CODE_RULES.RU.MD

## Обзор

Этот файл содержит системные инструкции для обработки кода проекта `hypotez`. Он определяет функциональность и правила для генерации, анализа и улучшения кода, направленные на обеспечение последовательного и читаемого стиля кодирования, соответствующего установленным требованиям. Инструкции охватывают принципы кодирования, правила комментирования, форматирование кода, использование логирования и основные требования к документации.

## Подробней

Файл `CODE_RULES.RU.MD` играет важную роль в обеспечении единообразия и качества кода в проекте `hypotez`. Он служит руководством для разработчиков, определяя стандарты кодирования, документирования и логирования. Следование этим инструкциям помогает поддерживать читаемость, понятность и поддерживаемость кода, что особенно важно при работе в команде. Анализ местоположения файла показывает, что он является частью подсистемы `ai`, в частности, модуля `prompts` и подмодуля `developer`, что указывает на его роль в определении правил для разработчиков, использующих AI-инструменты.

## Основные принципы

### Общие указания:
- Соблюдение четкого и понятного стиля кодирования.
- Все изменения должны быть обоснованы и соответствовать установленным требованиям.

### Комментарии:
- Использование `#` для внутренних комментариев.
- Документация всех функций, методов и классов должна соответствовать определенному формату.

### Форматирование кода:
- Использование одинарных кавычек.
- Добавление пробелов вокруг операторов.
- Все параметры должны быть аннотированы типами.
- Использование `|` вместо `Union`.

### Логирование:
- Использование модуля `logger` из `src.logger.logger`.
- Логирование ошибок с использованием `logger.error`.

## Основные требования

### Формат ответов в Markdown:
- Все ответы должны быть выполнены в формате **Markdown**.

### Формат комментариев:
- Использование указанного стиля для комментариев и документации в коде.

### Пробелы вокруг операторов присваивания:
- Всегда добавляйте пробелы вокруг оператора `=`, чтобы повысить читаемость.

### Использование `j_loads` или `j_loads_ns`:
- Замена стандартного использования `open` и `json.load` на `j_loads` или `j_loads_ns` для чтения JSON или конфигурационных файлов.

### Сохранение комментариев:
- Все существующие комментарии, начинающиеся с `#`, должны быть сохранены без изменений.

### Обработка `...` в коде:
- Оставляйте `...` как указатели в коде без изменений.
- Не документируйте строки с `...`.
