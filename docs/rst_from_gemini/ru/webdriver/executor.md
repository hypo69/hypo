# executor.py - Модуль для взаимодействия с веб-элементами

Файл: `hypotez/src/webdriver/executor.py`

Этот модуль предоставляет инструменты для взаимодействия с веб-элементами с помощью Selenium. Он обрабатывает различные действия (клики, отправка сообщений, выполнение событий, получение атрибутов) на основе заданных локейторов (конфигураций).  Модуль фокусируется на гибкости, обработке ошибок и поддержке различных типов локейторов.

**Ключевые особенности:**

* **Обработка локейторов:** Преобразует словари локейторов в объекты `SimpleNamespace` для удобного доступа к информации о локейторах. Поддержка различных типов локейторов (XPATH, ID, CSS-селектор и др.).

* **Взаимодействие с веб-элементами:** Выполняет различные действия над веб-элементами, включая клики, отправку текста (send_keys), выполнение JavaScript-кода, получение атрибутов.

* **Обработка ошибок:**  Поддерживает продолжение выполнения при возникновении ошибок (`ElementClickInterceptedException`, `NoSuchElementException` и др.), что позволяет автоматизировать работу с нестабильными веб-страницами.  

* **Управление таймаутами:**  Позволяет задавать таймаут ожидания элемента и типы ожидания (presence_of_element_located, element_to_be_clickable) для повышения устойчивости сценариев.

* **Обработка нескольких локейторов:**  Поддерживает работу как с одиночными, так и с множественными локейторами, что дает возможность взаимодействовать с несколькими веб-элементами одновременно.

* **Управление скоростью набора текста:** Опция `typing_speed` позволяет регулировать скорость отправки текста при использовании `send_keys`.

* **Запись скриншотов:** Возможность записи скриншотов найденных элементов.

* **Логирование:** Использование `logger` для записи сообщений об ошибках и отладочной информации.

**Класс `ExecuteLocator`:**

* `execute_locator()`: Основной метод для выполнения действия над элементом, принимающий локейтор и параметры.
* `evaluate_locator()`: Обрабатывает атрибуты локейтора, позволяя использовать динамические выражения.
* `get_attribute_by_locator()`: Получает атрибуты заданного элемента.
* `get_webelement_by_locator()`: Находит веб-элемент по локейтору.
* `get_webelement_as_screenshot()`:  Функция для получения скриншота найденного элемента.
* `execute_event()`:  Метод для выполнения заданных событий (например, клик, написание текста) над элементом.
* `send_message()`: Метод для отправки сообщения в текстовое поле веб-страницы.


**Примеры использования (предполагается импортирование класса и объекта `gs`):**

```python
# Пример выполнения клика
await executor.execute_locator(locator={"by": "XPATH", "selector": "//button[@id='myButton']", "event": "click()"}, timeout=10)

# Пример получения атрибута
attribute_value = await executor.execute_locator(locator={"by": "ID", "selector": "myInput", "attribute": "value"}, timeout=5)

# Пример отправки сообщения
await executor.send_message(locator={"by": "ID", "selector": "messageBox"}, message="Hello world!", typing_speed=0.1)
```


**Важные моменты:**

*  Код требует импорта необходимых библиотек Selenium, асинхронных операций и других вспомогательных модулей.
*  Логирование (`logger`) играет важную роль в отладке и анализе работы модуля.
*  Используйте `gs.mode` для выбора режима (debug/production), влияющего на уровень логирования.
*  Проверьте корректность локейторов, чтобы избежать ошибок поиска элементов.
*  Обратите внимание на обработку возможных исключений, таких как `NoSuchElementException` и `ElementClickInterceptedException`.
