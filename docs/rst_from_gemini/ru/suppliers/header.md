Этот файл содержит функцию `get_project_root`, которая находит корневую директорию проекта.

## Краткое описание

Модуль `src.suppliers` предоставляет вспомогательные функции для работы с поставщиками данных.  Главная функция `get_project_root` находит корень проекта, начиная с текущей директории и поднимаясь по иерархии каталогов.

## Функция `get_project_root`

```python
def get_project_root(marker_files=('pyproject.toml', 'requirements.txt', '.git')) -> Path:
    """!
    Находит корневую директорию проекта, начиная с текущей директории и поднимаясь по иерархии каталогов,
    пока не найдет один из указанных маркеров файлов или папок.

    Args:
        marker_files (tuple): Имена файлов или папок, которые указывают на корневую директорию проекта.
            По умолчанию используются `pyproject.toml`, `requirements.txt` и `.git`.

    Returns:
        Path: Путь к корневой директории проекта, если найдена, иначе текущая директория.
    """
    __root__: Path 
    current_path = Path(__file__).resolve().parent
    __root__ = current_path
    for parent in [current_path] + list(current_path.parents):
        if any((parent / marker).exists() for marker in marker_files):
            __root__ = parent
            break
    if str(__root__) not in sys.path:
        sys.path.insert(0, str(__root__))
        
    return __root__
```

**Описание аргумента `marker_files`:**

Аргумент `marker_files` позволяет указать файлы или директории, которые будут использоваться для определения корневой директории проекта. Это может быть полезно, если проект имеет необычную структуру директорий.


**Описание возвращаемого значения:**

Функция возвращает объект `Path` содержащий путь к корневой директории проекта.  Если корень проекта не найден, возвращается путь к текущей директории.

**Пример использования:**

```python
root_dir = get_project_root()
print(root_dir)
```

**Важные замечания:**

* Функция `get_project_root` добавляет найденную директорию в `sys.path`, что позволяет импортировать модули из корневой директории проекта.
* `__root__` используется как внутрифункциональная переменная для хранения пути к корню. Важно соблюдать соглашения о именовании переменных.
* Документация по использованию `Path` и `sys.path` может быть полезна для понимания кода.

**Возможные улучшения:**

* Добавить обработку исключений, например, если какой-либо из файлов или папок в `marker_files` не существует.
* Предостережение о том, что если файл, указанный в `marker_files`, находится не в том же каталоге, что и этот скрипт, то функция может не сработать.
