# Модуль создания Flask приложения для GUI

## Обзор

Этот модуль содержит функцию `create_app`, которая создает и конфигурирует Flask приложение для графического интерфейса (GUI). Он определяет местоположение шаблонов в зависимости от того, запущен ли код в замороженном (скомпилированном) состоянии или в режиме разработки.

## Подробнее

Этот модуль важен для инициализации Flask приложения, используемого для графического интерфейса. Он обрабатывает различные пути к шаблонам в зависимости от среды выполнения, что позволяет приложению работать как в режиме разработки, так и после компиляции в исполняемый файл.

## Функции

### `create_app`

```python
def create_app() -> Flask:
    """
    Создает и конфигурирует Flask приложение для GUI.

    Args:
        None

    Returns:
        Flask: Сконфигурированное Flask приложение.

    Raises:
        None

    Example:
        >>> app = create_app()
        >>> type(app)
        <class 'flask.app.Flask'>
    """
```

**Как работает функция**:

1. **Определение состояния**: Функция начинает с проверки атрибута `sys.frozen`, чтобы определить, запущено ли приложение в замороженном состоянии (например, как скомпилированный исполняемый файл) или в режиме разработки.
2. **Установка путей к шаблонам**:
   - Если приложение заморожено, путь к шаблонам устанавливается как подкаталог `client` внутри временной директории, созданной PyInstaller (`sys._MEIPASS`).
   - Если приложение не заморожено (т.е. запущено в режиме разработки), путь к шаблонам устанавливается как `../client`, что предполагает, что директория `client` находится на один уровень выше текущей директории.
3. **Создание Flask приложения**: Создается экземпляр Flask приложения с указанием директории шаблонов и статических файлов.
4. **Включение автоматической перезагрузки шаблонов**: Конфигурируется параметр `TEMPLATES_AUTO_RELOAD` для автоматической перезагрузки шаблонов в режиме отладки.
5. **Возврат приложения**: Возвращает сконфигурированное Flask приложение.

```
Проверка состояния приложения (заморожено или разработка)
↓
Установка пути к шаблонам в зависимости от состояния
↓
Создание экземпляра Flask приложения с указанием шаблонов и статических файлов
↓
Включение автоматической перезагрузки шаблонов в режиме отладки
↓
Возврат сконфигурированного Flask приложения
```

**Примеры**:

```python
from flask import Flask

app = create_app()
print(type(app))  # <class 'flask.app.Flask'>
