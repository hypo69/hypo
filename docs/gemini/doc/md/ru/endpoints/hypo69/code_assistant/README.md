# Документация к модулю Code Assistant

## Обзор

Модуль `Code Assistant` предназначен для взаимодействия с моделями Gemini и OpenAI для обработки исходного кода. Он выполняет различные задачи, включая создание документации, проверку кода и генерацию тестов.

## Функции

### `process_files`

**Описание**: Функция обрабатывает файлы в заданных директориях, используя указанную модель и роль.

**Параметры**:
- `start_dirs` (list[str]): Список путей к директориям для обработки.
- `lang` (str): Язык обработки (например, "ru" или "en").
- `role` (str): Роль модели (например, "doc_writer", "code_checker").
- `models` (list[str]): Список моделей для использования (например, ["gemini", "openai"]).
- `exclude_file_patterns` (list[str], optional): Список регулярных выражений для исключения файлов. По умолчанию `None`.
- `exclude_dirs` (list[str], optional): Список директорий для исключения. По умолчанию `None`.
- `exclude_files` (list[str], optional): Список файлов для исключения. По умолчанию `None`.


**Возвращает**:
- `None`: Функция не возвращает значение.

**Вызывает исключения**:
- `ValueError`: Если передан неверный тип данных для параметров.
- `FileNotFoundError`: Если указанные директории или файлы не найдены.
- `Exception`: Если возникает любая другая ошибка при работе с файлами или моделями.

### `load_prompts`

**Описание**: Загружает файлы с промптами для заданной роли и языка.

**Параметры**:
- `role` (str): Роль модели.
- `lang` (str): Язык.

**Возвращает**:
- `dict`: Словарь с промптами, где ключ — имя файла, значение — содержимое файла. Возвращает пустой словарь, если файлы не найдены.

**Вызывает исключения**:
- `FileNotFoundError`: Если файлы с промптами не найдены.
- `Exception`: Если возникает любая другая ошибка при чтении файлов.


## Классы

### `CodeAssistant`

**Описание**: Класс, представляющий обработчик кода.

**Методы**:
- `__init__`: Инициализирует объект `CodeAssistant` с заданными параметрами.
- `process_file`: Обрабатывает один файл с кодом.
- `generate_output`: Генерирует выходные данные на основе полученных ответов от моделей.

## Настройки

### settings.json

**Описание**: Файл с настройками, содержащий параметры для обработки кода.

**Пример**:

```json
{
  "role": "doc_writer",
  "lang": "en",
  "models": ["gemini", "openai"],
  "start_dirs": ["/path/to/dir1", "/path/to/dir2"],
  "exclude_file_patterns": ["test*.py"],
  "exclude_dirs": ["/path/to/dir3"]
}
```
