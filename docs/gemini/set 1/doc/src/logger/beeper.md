# Модуль hypotez/src/logger/beeper.py

## Обзор

Модуль `beeper` предоставляет функциональность для воспроизведения звуковых сигналов (бипов) в зависимости от уровня события, используя библиотеку `winsound`.  Модуль определяет уровни событий (SUCCESS, INFO, ATTENTION, WARNING, DEBUG, ERROR, LONG_ERROR, CRITICAL, BELL) с соответствующими мелодиями.  Также реализованы средства для управления режимом "беззвучия".

## Классы

### `BeepLevel`

**Описание**: Перечислитель уровней событий, каждому уровню соответствует определённая мелодия.

**Методы**:

* `value`: Возвращает список нот и длительностей для мелодии соответствующей уровню события.


### `BeepHandler`

**Описание**: Обработчик бипов, который воспроизводит звуки, соответствующие уровню события из логгирования.

**Методы**:

* `emit(self, record)`: Обрабатывает запись лога и воспроизводит звук, соответствующий уровню события.


## Функции

### `silent_mode(func)`

**Описание**: Функция-декоратор, которая управляет режимом "беззвучия" для функции.

**Параметры**:

* `func` (функция): Функция, которую нужно декорировать.

**Возвращает**:

* (функция): Декорированная функция, которая проверяет режим "беззвучия" перед воспроизведением звука.


### `Beeper`

**Описание**: Класс для воспроизведения звуковых сигналов.

**Атрибуты**:

* `silent` (bool): Флаг, определяющий режим "беззвучия". По умолчанию `False`.

**Методы**:

* `beep(level: BeepLevel | str = BeepLevel.INFO, frequency: int = 400, duration: int = 1000) -> None`:  Вызывает звуковой сигнал оповещения с указанным уровнем, частотой и длительностью.

**Параметры**:

* `level` (`BeepLevel | str`, опционально, по умолчанию `BeepLevel.INFO`): Уровень события (например, `BeepLevel.INFO`, `'warning'` или `'success'`).
* `frequency` (int, опционально, по умолчанию 400): Частота звука (от 37 до 32000).
* `duration` (int, опционально, по умолчанию 1000): Длительность звука в миллисекундах.

**Возвращает**:
* `None`

**Вызывает исключения**:
* Возможна ошибка `Exception` во время воспроизведения звука `winsound.Beep`. В случае ошибки выводится сообщение об ошибке и функция возвращает `None`.


## Константы

### `MODE`

**Описание**: Переменная, определяющая режим работы модуля. В текущем примере имеет значение `'dev'`.
