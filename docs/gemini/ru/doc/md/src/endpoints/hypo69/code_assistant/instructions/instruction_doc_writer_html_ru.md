<h1>Инструкция по созданию документации для Python-файлов в формате HTML</h1>

<h2>Обзор</h2>
<p>Данная инструкция описывает, как генерировать HTML-документацию для Python-файлов, следуя заданным форматам и требованиям.</p>

<h2>Формат документации</h2>
<ul>
  <li>Используется стандарт HTML.</li>
  <li>Каждый файл начинается с заголовка и краткого описания содержимого.</li>
  <li>Все классы и функции должны быть задокументированы с использованием следующего формата комментариев:</li>
</ul>
<pre><code>python
def function(param: str, param1: Optional[str | dict | str] = None) -> dict | None:
    """
    Args:
        param (str): Описание параметра `param`.
        param1 (Optional[str | dict | str], optional): Описание параметра `param1`. По умолчанию значение равно `None`.

    Returns:
        dict | None: Описание возвращаемого значения. Возвращает словарь или `None`.

    Raises:
        SomeError: Описание ситуации, в которой возникает исключение `SomeError`.
    """
</code></pre>
<ul>
  <li>Используйте <code>ex</code> вместо <code>e</code> в блоках обработки исключений.</li>
</ul>

<h2>Оглавление (TOC)</h2>
<ul>
  <li>Включите раздел оглавления в начале каждого документа.</li>
  <li>Структура оглавления должна включать ссылки на все основные разделы документации модуля.</li>
</ul>

  <li>Для документирования классов, функций и методов используйте структурированные разделы с описаниями, деталями параметров, возвращаемых значений и вызываемых исключений. Пример:</li>
</ul>
