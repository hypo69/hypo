# Модуль `graber.py`

## Обзор

Модуль `graber.py` предназначен для сбора данных о товарах с сайта `morlevi.co.il`. Он расширяет базовый класс `Graber` и содержит логику для обработки полей товаров, специфичных для данного поставщика. Модуль также включает функциональность для предварительной обработки запросов к веб-драйверу с использованием декораторов.

## Подробней

Этот модуль является частью системы для автоматического сбора данных о товарах с сайтов поставщиков. Он специализируется на сайте `morlevi.co.il`. Для каждого поля товара на странице создана функция обработки. Если требуется нестандартная обработка, функция переопределяется в этом классе.

Перед отправкой запроса к веб-драйверу можно выполнить предварительные действия через декоратор. Декоратор по умолчанию находится в родительском классе. Чтобы декоратор сработал, нужно передать значение в `Context.locator`. Если необходимо реализовать свой декоратор, следует раскомментировать строки с декоратором и переопределить его поведение.

## Классы

### `Graber`

**Описание**: Класс `Graber` предназначен для захвата и обработки данных с сайта `morlevi.co.il`. Он наследуется от базового класса `Graber` (`Grbr`) и переопределяет некоторые методы для адаптации к специфике поставщика.

**Как работает класс**:

1.  **Инициализация**: При инициализации класса устанавливается префикс поставщика (`supplier_prefix`) и инициализируется родительский класс `Graber`.
2.  **Декораторы**: Класс использует декораторы для предварительной обработки запросов к веб-драйверу. Декоратор по умолчанию находится в родительском классе. Для активации декоратора необходимо передать значение в `Context.locator`.
3.  **Обработка полей товара**: Для каждого поля товара на странице создана функция обработки. Если требуется нестандартная обработка, функция переопределяется в этом классе.

**Методы**:

*   `__init__`: Инициализирует класс `Graber` и устанавливает необходимые параметры.

    **Параметры**:
    *   `driver` (`Driver`): Экземпляр веб-драйвера для управления браузером.
    *   `lang_index` (`int`): Индекс языка, используемый для локализации.

    **Как работает**:

    1.  Вызывает конструктор родительского класса `Graber` (`Grbr`) с указанием префикса поставщика, драйвера и индекса языка.
    2.  Устанавливает `Context.locator_for_decorator` равным `self.locator.close_pop_up`, что позволяет использовать декоратор для закрытия всплывающих окон.
