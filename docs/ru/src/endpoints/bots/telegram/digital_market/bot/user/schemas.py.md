# Модуль schemas.py

## Обзор

Модуль `schemas.py` содержит Pydantic-схемы для валидации и представления данных, связанных с пользователями Telegram, продуктами и платежами в контексте цифрового рынка. Эти схемы используются для обеспечения соответствия данных ожидаемым типам и структурам, а также для документирования форматов данных, используемых в приложении.

## Подробнее

Модуль содержит классы, определенные с использованием `pydantic.BaseModel`, которые служат для описания структуры данных, ожидаемых приложением. Каждый класс включает аннотации типов и, при необходимости, метаданные для валидации данных, такие как `max_length` и `description`.  Схемы используются для сериализации и десериализации данных, поступающих из Telegram или базы данных, что обеспечивает надежную передачу данных между различными компонентами приложения.

## Классы

### `TelegramIDModel`

**Описание**: Базовая модель для идентификации пользователя Telegram по его уникальному ID.

**Атрибуты**:
- `telegram_id` (int): Уникальный идентификатор пользователя в Telegram.

### `UserModel`

**Описание**: Модель данных, представляющая информацию о пользователе Telegram, включая его имя пользователя и имя.

**Наследует**:
- `TelegramIDModel`: Наследует атрибут `telegram_id` для идентификации пользователя.

**Атрибуты**:
- `username` (str | None): Имя пользователя в Telegram (может отсутствовать).
- `first_name` (str | None): Имя пользователя (может отсутствовать).
- `last_name` (str | None): Фамилия пользователя (может отсутствовать).

### `ProductIDModel`

**Описание**: Модель для идентификации продукта по его уникальному ID.

**Атрибуты**:
- `id` (int): Уникальный идентификатор продукта.

### `ProductCategoryIDModel`

**Описание**: Модель для идентификации категории продукта по её уникальному ID.

**Атрибуты**:
- `category_id` (int): Уникальный идентификатор категории продукта.

### `PaymentData`

**Описание**: Модель данных, представляющая информацию о платеже, включая ID пользователя, ID платежа, сумму, ID продукта и тип оплаты.

**Атрибуты**:
- `user_id` (int): ID пользователя Telegram, совершившего платеж. Описание: "ID пользователя Telegram"
- `payment_id` (str): Уникальный ID платежа. Описание: "Уникальный ID платежа", максимальная длина 255 символов.
- `price` (int): Сумма платежа в рублях. Описание: "Сумма платежа в рублях"
- `product_id` (int): ID товара, за который произведен платеж. Описание: "ID товара"
- `payment_type` (str): Тип оплаты. Описание: "Тип оплаты"
