## ИНСТРУКЦИЯ

Инструкция описывает шаги по созданию Markdown-документации для Python-файлов. Она включает в себя требования к структуре, формату комментариев и содержанию документации.

## Формат документации

Документация должна соответствовать указанному в инструкции формату Markdown. Каждый файл должен начинаться с заголовка и краткого описания его содержимого. Для функций и классов должны быть использованы соответствующие комментарии.

## Содержание (TOC)

В начале каждого файла документации должен быть добавлен раздел с оглавлением (TOC), содержащий ссылки на все основные разделы документации модуля.

## Форматирование документации

Необходимо использовать правильный синтаксис Markdown для заголовков, списков и ссылок. Документация должна быть структурированной и содержать подробные описания классов, функций и методов.

## Заголовки разделов

Для разделов должны быть использованы заголовки первого, второго, третьего и четвертого уровней.
