# Модуль hypotez/src/endpoints/advertisement/header.py

## Обзор

Этот модуль отвечает за загрузку настроек проекта, версии и документации. Он находит корневую директорию проекта, загружает данные из файла `settings.json` и `README.MD`, и инициализирует переменные, содержащие информацию о проекте.

## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная с директории текущего файла. Ищет директорию, содержащую файлы `pyproject.toml`, `requirements.txt` или `.git`.

**Параметры**:

- `marker_files` (tuple): Кортеж имен файлов или директорий, по которым определяется корневая директория проекта. По умолчанию содержит `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:

- `Path`: Путь к корневой директории проекта, если найдена. В противном случае возвращает директорию, где расположен текущий скрипт.

**Вызывает исключения**:

Нет.


## Глобальные переменные

### `__root__`

**Описание**: Путь к корневой директории проекта, полученный функцией `set_project_root()`.

**Тип**: `Path`

### `settings`

**Описание**: Словарь, содержащий настройки проекта, загруженный из файла `settings.json`.

**Тип**: `dict`

### `doc_str`

**Описание**: Строка, содержащая содержимое файла `README.MD`.

**Тип**: `str`


### `__project_name__`

**Описание**: Название проекта, полученное из настроек. По умолчанию `hypotez`.

**Тип**: `str`

### `__version__`

**Описание**: Версия проекта, полученная из настроек. По умолчанию пустая строка.

**Тип**: `str`

### `__doc__`

**Описание**: Документация проекта, полученная из файла `README.MD`. По умолчанию пустая строка.

**Тип**: `str`

### `__details__`

**Описание**: Подробная информация о проекте. По умолчанию пустая строка.

**Тип**: `str`

### `__author__`

**Описание**: Автор проекта, полученный из настроек. По умолчанию пустая строка.

**Тип**: `str`

### `__copyright__`

**Описание**: Авторские права, полученные из настроек. По умолчанию пустая строка.

**Тип**: `str`

### `__cofee__`

**Описание**: Ссылка на донаты для поддержания проекта.

**Тип**: `str`



## Обработка исключений

В модуле используются `try...except` блоки для обработки потенциальных ошибок при чтении файлов `settings.json` и `README.MD`.  Исключения `FileNotFoundError` и `json.JSONDecodeError` обрабатываются с помощью пустого блока `...`, что означает, что если файлы не найдены или содержат невалидный JSON, переменные `settings` и `doc_str` остаются `None`.  Это позволяет программе продолжить выполнение без аварийного завершения.
