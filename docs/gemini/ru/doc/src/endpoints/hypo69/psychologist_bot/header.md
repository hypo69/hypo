# Модуль `hypotez/src/endpoints/hypo69/psychologist_bot/header.py`

## Обзор

Этот модуль содержит вспомогательные функции для работы с проектом `hypotez`, включая определение корневой директории проекта, загрузку настроек из файла `settings.json` и чтение документации из файла `README.MD`.  Он также определяет константы, такие как `MODE` и переменные для хранения метаданных проекта (название, версия, автор и т.д.).

## Переменные

### `MODE`

**Описание**: Переменная, определяющая режим работы приложения (например, `dev`, `prod`).

**Значение**: `"dev"` по умолчанию.

### `__root__`

**Описание**: Путь к корневой директории проекта, полученный функцией `set_project_root`.

**Тип**: `Path`

### `settings`

**Описание**: Словарь с настройками проекта, загруженными из файла `settings.json`.

**Тип**: `dict`

### `doc_str`

**Описание**: Строка с содержимым файла `README.MD`.

**Тип**: `str`


### `__project_name__`

**Описание**: Название проекта, полученное из настроек.

**Тип**: `str`

### `__version__`

**Описание**: Версия проекта, полученная из настроек.

**Тип**: `str`

### `__doc__`

**Описание**: Документация проекта, полученная из файла `README.MD`.

**Тип**: `str`

### `__details__`

**Описание**: Дополнительные сведения о проекте.

**Тип**: `str`

### `__author__`

**Описание**: Автор проекта, полученный из настроек.

**Тип**: `str`

### `__copyright__`

**Описание**: Авторские права, полученные из настроек.

**Тип**: `str`


### `__cofee__`

**Описание**: Ссылка на страницу для поддержки разработчика чашкой кофе.

**Тип**: `str`

## Функции

### `set_project_root`

**Описание**: Находит корневую директорию проекта, начиная с директории текущего файла.

**Параметры**:

- `marker_files` (tuple): Список файлов/директорий, по наличию которых определяется корень проекта.  По умолчанию `('pyproject.toml', 'requirements.txt', '.git')`.

**Возвращает**:

- `Path`: Путь к корневой директории проекта.  Возвращает директорию текущего файла, если корневая директория не найдена.

**Вызывает исключения**:

-  Нет указанных исключений.


## Примечания

Этот модуль использует модуль `pathlib` для работы с путями к файлам и директориям. Он ищет корень проекта, начиная с текущей директории и двигаясь вверх по дереву директорий.  Использует файл `settings.json` для хранения настроек проекта и `README.MD` для получения документации.  При отсутствии указанных файлов переменные `settings` и `doc_str` устанавливаются в `None`.
