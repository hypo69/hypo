# Модуль LambdaChat

## Обзор

Модуль `LambdaChat` предоставляет класс `LambdaChat`, который является наследником класса `HuggingChat` и предназначен для взаимодействия с чат-моделями, размещенными на платформе lambda.chat. Он определяет специфические параметры и настройки, такие как домен, URL, модели и их псевдонимы, используемые для взаимодействия с API lambda.chat.

## Подробней

Этот модуль расширяет возможности класса `HuggingChat`, адаптируя его для работы с конкретными моделями и параметрами, предоставляемыми платформой lambda.chat. Модуль содержит информацию о доступных моделях, псевдонимах моделей и настройках для работы с API.

## Классы

### `LambdaChat`

**Описание**: Класс `LambdaChat` предназначен для взаимодействия с чат-моделями, размещенными на платформе `lambda.chat`. Он наследуется от класса `HuggingChat` и переопределяет некоторые атрибуты для адаптации к особенностям платформы `lambda.chat`.

**Наследует**:

- `HuggingChat`: Класс, предоставляющий базовую функциональность для взаимодействия с чат-моделями Hugging Face.

**Атрибуты**:

- `label` (str): Метка провайдера, значение `"Lambda Chat"`.
- `domain` (str): Доменное имя платформы, значение `"lambda.chat"`.
- `origin` (str): Базовый URL платформы, формируется как `"https://{domain}"`.
- `url` (str): Полный URL платформы, совпадающий с `origin`.
- `working` (bool): Указывает, является ли провайдер рабочим, значение `True`.
- `use_nodriver` (bool): Указывает, требуется ли веб-драйвер для работы, значение `False`.
- `needs_auth` (bool): Указывает, требуется ли аутентификация, значение `False`.
- `default_model` (str): Модель, используемая по умолчанию, значение `"deepseek-llama3.3-70b"`.
- `reasoning_model` (str): Модель, используемая для рассуждений, значение `"deepseek-r1"`.
- `image_models` (list): Список моделей, поддерживающих обработку изображений, пустой список `[]`.
- `fallback_models` (list): Список моделей, используемых в качестве запасных вариантов, включает `default_model`, `reasoning_model` и другие.
- `models` (list): Список доступных моделей, инициализируется копией `fallback_models`.
- `model_aliases` (dict): Словарь псевдонимов моделей, где ключи - псевдонимы, а значения - фактические имена моделей.

**Принцип работы**:

1.  **Инициализация**: При создании экземпляра класса `LambdaChat` устанавливаются основные параметры и настройки для работы с платформой lambda.chat.
2.  **Выбор модели**: Класс предоставляет возможность выбора модели для взаимодействия. Если конкретная модель не указана, используется `default_model`.
3.  **Взаимодействие с API**: Класс использует унаследованные методы от `HuggingChat` для отправки запросов к API lambda.chat и получения ответов.

**Методы**:

-   В данном классе не представлены методы, так как он в основном предназначен для определения атрибутов и настроек, специфичных для платформы lambda.chat.

**Примеры**:

```python
from src.endpoints.gpt4free.g4f.Provider.LambdaChat import LambdaChat

# Создание экземпляра класса LambdaChat
lambda_chat = LambdaChat()

# Вывод информации о провайдере
print(f"Провайдер: {lambda_chat.label}")
print(f"Домен: {lambda_chat.domain}")
print(f"URL: {lambda_chat.url}")
print(f"Модель по умолчанию: {lambda_chat.default_model}")
print(f"Список запасных моделей: {lambda_chat.fallback_models}")

# Получение псевдонима для модели
alias = lambda_chat.model_aliases.get("deepseek-v3")
print(f"Псевдоним для deepseek-v3: {alias}")
