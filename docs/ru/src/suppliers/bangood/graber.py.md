# Модуль для сбора данных с сайта Banggood
## Обзор
Модуль `graber.py` предназначен для сбора данных о товарах с сайта bangood.com. Он содержит класс `Graber`, который наследует функциональность сбора данных от родительского класса `Graber` (из модуля `src.suppliers.graber`). Класс `Graber` специализируется на обработке специфичных для сайта Banggood полей товаров.

## Подробней
Этот модуль является частью системы для автоматизированного сбора информации о товарах с различных онлайн-площадок. Он использует веб-драйвер для взаимодействия с сайтом Banggood и извлекает необходимые данные, такие как названия, описания, цены и характеристики товаров. Модуль предоставляет гибкий механизм для переопределения стандартных методов обработки полей, если это необходимо для конкретного сайта.

## Классы

### `Graber`
**Описание**: Класс `Graber` предназначен для сбора данных о товарах с сайта Banggood. Он наследует функциональность сбора данных от базового класса `Graber` и адаптирует её для специфических особенностей сайта Banggood.

**Принцип работы**:
1.  При инициализации класса устанавливается префикс поставщика (`bangood`).
2.  Вызывается конструктор родительского класса `Graber` для инициализации общих параметров и настроек.
3.  Устанавливаются глобальные настройки через класс `Context`, включая локатор для декоратора `@close_pop_up` (в данном случае, он устанавливается в `None`, что означает, что декоратор не будет выполняться по умолчанию).

**Методы**:
- `__init__(self, driver: Driver, lang_index: int)`: Инициализация класса `Graber`.

## Функции

### `__init__(self, driver: Driver, lang_index: int)`

**Назначение**: Инициализирует экземпляр класса `Graber`, устанавливая префикс поставщика и вызывая конструктор родительского класса.

**Параметры**:
- `driver` (Driver): Экземпляр веб-драйвера, используемый для взаимодействия с сайтом Banggood.
- `lang_index` (int): Индекс языка, используемого на сайте Banggood.

**Возвращает**:
- `None`

**Вызывает исключения**:
- `Exception`: Может вызывать исключения, возникающие в процессе инициализации родительского класса или при установке глобальных настроек через `Context`.

**Как работает функция**:
1.  Устанавливает атрибут `supplier_prefix` равным `'bangood'`.
2.  Вызывает конструктор родительского класса `Graber` (`super().__init__(supplier_prefix=self.supplier_prefix, driver=driver, lang_index=lang_index)`) для выполнения базовой инициализации.
3.  Устанавливает `Context.locator_for_decorator = None`, что отключает выполнение декоратора `@close_pop_up` по умолчанию.

**ASCII flowchart**:

```
Инициализация Graber
│
├── Установка supplier_prefix = 'bangood'
│
├── Вызов Graber.__init__()
│
└── Установка Context.locator_for_decorator = None
│
Конец
```

**Примеры**:

```python
from src.webdriver.driver import Driver, Chrome
from src.suppliers.bangood.graber import Graber

# Пример создания экземпляра класса Graber
driver = Driver(Chrome)
graber = Graber(driver=driver, lang_index=0)
```
```python
# Пример использования Context
from src.webdriver.driver import Driver, Chrome
from src.suppliers.bangood.graber import Graber
from src.suppliers.graber import Context

driver = Driver(Chrome)
graber = Graber(driver=driver, lang_index=0)
